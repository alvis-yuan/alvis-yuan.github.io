Chapter 1
引言 Introduction
USB 2.0 的动机、目标、范围与文档组织方式
USB（通用串行总线）诞生于1996年，由 Intel、Compaq、Microsoft、NEC 等公司联合制定，旨在解决 PC 外设连接繁杂、标准不统一的问题。USB 2.0 于2000年4月发布，将速率从 12 Mbps 提升至 480 Mbps。

📌 1.1 动机

解决计算机外设接口混乱问题：串口、并口、PS/2、游戏端口各自为政，难以统一管理和使用。

🎯 1.2 规范目标

为 PC 外设提供单一、标准化的连接方式；支持热插拔；提供总线供电；简化驱动程序开发。

📋 1.3 文档范围

涵盖电气、机械、协议及软件接口规范，适用于主机、集线器和设备端实现者。

📅 USB 版本演进时间线

USB 1.0 — 1996年1月
首个正式版本。低速 1.5 Mbps / 全速 12 Mbps，键盘、鼠标等低带宽设备。
USB 1.1 — 1998年9月
修复 1.0 中若干问题，全面普及，大量外设采用。
USB 2.0 — 2000年4月 ★
高速模式 480 Mbps，向下兼容 USB 1.x，本文档即基于此版本。
USB 3.x / 4 — 2008年后
SuperSpeed 5 Gbps → 40 Gbps，引入 Type-C 连接器。

🏗 5.1 实现者视角 — USB 分层模型

📦 5.4–5.8 四种传输类型详解

🔧 控制传输 Control Transfer

双向保证送达三阶段

用于设备配置和控制命令。每个设备必须实现端点0（EP0）用于控制传输。分为 Setup → Data（可选）→ Status 三个阶段。

FS最大数据包：64字节 · HS：64字节

🔄 同步传输 Isochronous Transfer

固定带宽无重传实时性

用于音视频等对时延敏感的应用。保证带宽但不保证数据完整性（无 ACK/NAK）。每帧/微帧传输一次。

FS最大：1023字节/帧 · HS：1024字节/微帧×3

⏰ 中断传输 Interrupt Transfer

有保证延迟小数据量轮询

用于需要定期轮询的设备，如键盘、鼠标。主机周期性查询设备，保证最大响应延迟（端点描述符中指定轮询间隔）。

LS：8字节 · FS：64字节 · HS：1024字节

📦 批量传输 Bulk Transfer

大数据量无错误尽力而为

用于大量数据传输，如 U 盘、打印机。保证数据完整性（CRC + 重传），但不保证带宽和延迟，利用空闲带宽传输。

FS最大：64字节 · HS：512字节

🤝 7.1.7.5 高速握手（Chirp Sequence）

设备连接（Connect）
设备在 D+ 上拉（Full Speed）。主机检测到设备连接。
主机复位（Reset）
主机发送 SE0 持续 ≥10ms 的复位信号。
设备发 Chirp-K
设备检测到复位，发出 Chirp-K（D-拉低），持续 1~7ms，表示"我支持高速"。
主机响应 Chirp-K/J 序列
主机回应 3对 Chirp-KJ（KJKJKJ），双方确认高速能力。
切换至高速模式 480 Mbps
握手完成，双方切换到高速信号电平（400mV 差分），开始高速通信。

🔧 速度识别方式

USB 通过上拉电阻位置识别设备速度：

🔢 CRC 校验机制

CRC5（5位）：用于令牌包（Token）中的地址和端点字段保护。生成多项式：x⁵ + x² + 1
CRC16（16位）：用于数据包（Data）内容保护。生成多项式：x¹⁶ + x¹⁵ + x² + 1

检错能力：所有1位、2位错误，所有奇数位错误，以及所有长度≤16的突发错误。

🔍 9.1.2 枚举（Enumeration）完整流程

① 设备连接 → Hub 检测
设备接入，Hub 通过 D+/D- 上拉电阻检测到连接，向主机报告状态变化。
② 主机复位设备
主机向 Hub 发送 Port Reset 命令，Hub 对该端口发送 SE0 复位信号（≥10ms）。设备进入默认状态（Default State），监听地址0。
③ 主机读取设备描述符（前8字节）
使用地址0、EP0 发送 GET_DESCRIPTOR 请求，读取设备描述符前8字节，获取 bMaxPacketSize0。
④ 再次复位 → 分配地址
主机发送 SET_ADDRESS 请求，为设备分配唯一 7位地址（1~127）。设备进入地址状态（Address State）。
⑤ 读取完整描述符
主机使用新地址读取完整的设备描述符、配置描述符（含接口+端点）、字符串描述符。
⑥ 设置配置 → 枚举完成
主机选择合适的配置，发送 SET_CONFIGURATION 请求。设备进入配置状态（Configured State），可以正常传输数据。

📅 10.2 帧调度（Frame Scheduling）

主机控制器每 1ms（全速）或 125µs（高速）产生一次 SOF 包，并在该帧内按优先级调度各类传输：

优先级传输类型带宽保证
1（最高）同步传输预留固定带宽
2中断传输保证最大延迟
3控制传输保留≥10%
4（最低）批量传输剩余带宽
