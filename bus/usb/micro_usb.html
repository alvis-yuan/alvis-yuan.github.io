Chapter 1
引言 Introduction
规范的背景、目的、适用范围及相关文档说明
规范定位：本文档是对 USB 2.0 规范的补充说明（Supplement），而非独立标准。凡本规范未明确修改的内容，一律以 USB 2.0 Specification 及 USB On-The-Go Supplement 为准。本规范专门针对手机与便携设备市场设计的 Micro 系列连接器进行规定。
📋 1.1 总体说明 General

USB 已成为手机与便携设备之间交换数据的主流接口。然而，这些设备体积越来越薄，传统 Mini-USB 连接器在尺寸与耐久性上已无法满足需求。

核心痛点：Mini-USB 高度约 3.0mm，而 Micro-USB 高度仅约 1.8mm，大幅减少了设备厚度占用；同时耐久次数从 Mini-USB 的 5,000 次提升至 10,000 次。
🎯 1.2 规范目标 Objective

本规范的目的是定义一种 Micro-USB 连接器的要求与特性，使其：

  • 满足手机与便携设备市场的当前和未来需求
  • 符合 USB 2.0 规范的所有电气性能要求
  • 在物理尺寸与形状上比 Mini-USB 更小巧
  • 通过 ID 引脚支持 USB On-The-Go（OTG）功能
  • 提供比现有互连器更高的耐久性（≥10,000 次插拔循环）
👥 1.3 适用对象 / 范围

本规范适用于：

  • Micro-USB 连接器制造商
  • 使用 Micro-USB 的设备开发者
  • 线缆组件设计者
  • USB 合规测试机构

🔌 设备角色术语（Device Role Terms）
术语英文全称定义
A-Device A-Device 插入 Micro-A 插头的设备。会话开始时默认为主机，负责向 VBUS 供电。若为 OTG 设备，可在特定条件下让出主机角色。
B-Device B-Device 插入 Micro-B 插头的设备。会话开始时默认为外设（Peripheral）。若为 OTG 设备，可获得来自 A-Device 授予的主机角色。
Host Host 当前以 USB 主机角色运作的物理实体。发起所有数据事务并提供帧起始（SOF）信号。
Peripheral Peripheral 当前以 USB 从机角色运作的物理实体，响应主机的总线请求。
OTG Device On-The-Go Device 同时具备主机与外设能力的设备，可通过 ID 引脚与 HNP/SRP 协议切换角色。
Application Application 运行在设备上、能够控制 USB 端口行为或动作的任意软件的通用称谓。

🔄 OTG 协议术语
HNP — Host Negotiation Protocol（主机协商协议）
允许 A 设备与 B 设备在会话期间动态交换主机角色的机制。B 设备向 A 设备请求，A 设备同意后挂起总线，B 设备发出连接信号成为新主机。
SRP — Session Request Protocol（会话请求协议）
允许 B 设备在 VBUS 掉电后请求 A 设备重新上电并开始新会话的机制，节省 A 设备电量。
ID 引脚（Identification Pin）：Micro 连接器第 5 引脚，用于区分 Micro-A 插头（ID 接地，阻抗 <10Ω）与 Micro-B 插头（ID 浮空或阻抗 >100kΩ）。OTG 设备通过检测 ID 引脚阻抗来确定自身初始角色。

⚡ 3.1 USB 2.0 规范合规性

Micro-USB 完整继承并遵循 USB 2.0 规范的所有电气性能要求，支持：

Low Speed 1.5 Mb/s Full Speed 12 Mb/s High Speed 480 Mb/s 差分信号 D+/D- NRZI 编码 CRC 错误检测
本规范不改变 USB 2.0 的协议层、传输层和软件接口，仅在物理连接器尺寸与电气接触参数上进行特定补充规定。
🔄 3.2 On-The-Go 设备支持

Micro-USB 连接器通过第 5 引脚（ID 引脚）原生支持 USB OTG 功能，使两台便携设备无需 PC 主机即可直接通信。

  • Micro-A 插头：ID 引脚接地（<10Ω）→ 设备成为主机
  • Micro-B 插头：ID 引脚浮空（>100kΩ）→ 设备成为外设
  • Micro-AB 插座：同时接受 Micro-A 或 Micro-B 插头
🔌 3.3 连接器类型

Micro-USB 规范定义三种连接器形态：

Micro-A 插头：用于 A 端设备（OTG 主机），ID 引脚接地，不对称键控，只能插入 Micro-AB 插座
Micro-B 插头：用于 B 端设备（外设），ID 浮空，可插入 Micro-B 或 Micro-AB 插座
Micro-AB 插座：同时接受 A 或 B 插头，专用于 OTG 设备，区别于只接受 B 的普通 Micro-B 插座
🔧 3.4 & 3.5 线缆组件与包覆模具

规范对合规线缆和插头包覆模具的约束：

  • 传播延迟 ≤ 10 ns
  • 物理长度 ≤ 2.0 m
  • 线缆需包含 VBUS、D-、D+、ID（Micro-A 端）、GND 共 5 线
  • 插头包覆模具尺寸须符合 Figure 4-4/4-5 的约束形状
  • 包覆模具须防止误插（键控设计）
  • 直角（侧/下）插头同样需满足相同形状约束

⚡ 5.3 接触电流额定值（Contact Current Rating）
5.3.1 信号接触点（Pin 2、3、4）
在 25°C 环境温度下，最小额定电流为 1 A。仅用于数据信号传输的引脚（D-、D+、ID），须能承受此电流而不发生过热或损坏。
5.3.2 电源接触点（Pin 1 和 5）
VBUS（Pin 1）和 GND（Pin 5）须支持 USB 2.0 规范所要求的完整供电电流，在标准总线供电条件下为最大 500 mA（高功耗设备配置）。

🔬 6.4 镀层规格（Plating）

镀层规格决定接触可靠性和耐久性，分两种选项：

选项接触区非接触区
Option I金（Au）+ 镍（Ni）底层锡（Sn）可接受
Option II金（Au）厚层 + Ni 底层更高耐久镀层
原因：金镀层具有优异的抗氧化性和低接触电阻，是高可靠连接器的首选，配合镍底层可防金属扩散。
6.5 可焊性（Solderability）：接触端和屏蔽端须满足相关可焊性标准，确保 PCB 焊接可靠。
6.6 剥离强度（Peel Strength）（仅参考）：规定连接器固定到 PCB 的剥离力参考值，不作强制要求。
6.7 抗扭强度（Wrenching Strength）（仅参考）：规定插拔时连接器抗扭矩的参考能力。
6.8 引线共面性（Lead Co-Planarity）：SMT 贴装时，接触尾部须在 PCB 焊盘平面内保持良好共面性，确保回流焊接可靠。
♻️ 6.9 RoHS 合规性

Micro-USB 连接器必须符合欧盟 RoHS 指令（限制有害物质使用指令），禁止使用以下六种有害物质：

铅 Pb 汞 Hg 镉 Cd 六价铬 Cr(VI) 多溴联苯 PBB 多溴二苯醚 PBDE
RoHS 合规是进入欧盟市场的前提，也是现代消费电子产品的基本环保要求。
🛡️ 6.10 外壳与锁扣材料

连接器外壳（Shell）与锁扣（Latch）的材料须满足：

  • 足够的机械强度以承受 10,000 次插拔循环
  • 良好的弹性恢复性（锁扣需持续保持弹性）
  • 耐工作温度范围内的热变形
  • 通常采用不锈钢冲压件（厚度约 0.3mm）
  • 屏蔽壳体还需提供 EMI 防护
Micro-USB 的不锈钢壳体加锁扣设计（被动锁扣）是实现 10,000 次耐久性的关键机械结构。

🔋 SRP（会话请求协议）省电机制

SRP 允许 B 设备在 VBUS 掉电后主动请求 A 设备重新为总线供电，避免 A 设备持续维持 VBUS 浪费电量。

A 设备关闭 VBUS
会话结束或超时后，A 设备停止为 VBUS 供电，进入低功耗待机状态。
B 设备发起 SRP
B 设备通过在 D+ 上产生短暂脉冲（数据线信号方式）或 VBUS 脉冲，向 A 设备请求重新启动会话。
A 设备响应，恢复 VBUS
A 设备检测到 SRP 请求后，决定是否为 VBUS 重新供电，并启动新的连接会话。
会话建立，正常通信
VBUS 恢复，双方重新完成枚举，进入正常数据传输状态。
节能效果：SRP 使 A 设备（如手机）无需一直维持 VBUS 供电等待 B 设备，大幅延长电池寿命，尤其适合电池供电的 OTG 应用场景。
实际应用：当手机作为 OTG 主机连接 U 盘，数据传输完成后主机可关闭 VBUS；下次 U 盘需要传输时通过 SRP 唤醒手机重新供电，而无需用户干预拔插线缆。
