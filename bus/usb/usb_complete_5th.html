🌐USB的用途与优势

USB（通用串行总线）是连接计算机与外部设备的标准接口。其设计目标是取代多种慢速串行和并行接口，提供统一的、用户友好的即插即用体验。

USB 典型应用：鼠标、键盘、打印机、摄像头、外置存储、数据采集设备、工业控制系统等几乎一切外设。

📊四种传输类型总览

图3-1: 四种USB传输类型特性对比

⏱️带宽分配规则

USB 2.0 每帧（1ms）带宽有限，主机按规则分配：

  • 同步传输最高可占 90% 带宽（含中断传输）
  • 批量和控制传输共用剩余带宽
  • 控制传输始终保留 10% 最低带宽
  • SuperSpeed 使用异步通知（ERDY TP）取代轮询
同步传输一旦分配，若设备无数据也会消耗带宽；批量传输在空闲时才使用剩余带宽。

🗂️流管道 vs 消息管道

USB定义了两种管道类型：

  • 消息管道 (Message Pipe)：双向，带格式化请求，仅用于控制传输（EP0）
  • 流管道 (Stream Pipe)：单向，无格式约束，用于批量、中断、同步传输
管道是逻辑概念，连接主机软件与设备端点。每个非零端点有一个对应的流管道。

🛠️芯片选型考量

  • 所需 USB 速度（LS/FS/HS/SS）
  • 端点数量和最大包大小
  • 是否内置 CPU/MCU
  • 与现有系统总线的接口（SPI/并口/PCIe）
  • 所支持的设备类（HID/CDC/MSC）
  • 开发文档和示例代码质量
  • 功耗要求与电源管理能力

图7-1: USB设备类体系（USB-IF标准类规范）

✅使用标准类的优势

  • 操作系统内置驱动，无需安装
  • 跨平台兼容性好（Windows/macOS/Linux）
  • 经USB-IF认证，互操作性有保证
  • 减少驱动开发工作量
  • 符合USB合规性要求

⚙️厂商自定义类策略

  • 使用 WinUSB 驱动（Windows内置）
  • 使用 libusb（跨平台）
  • 转换RS-232/并口旧接口
  • 实现两台PC之间的直连
  • 特殊功能无法归入标准类时
自定义类需要用户安装驱动，增加了部署复杂度。建议优先考虑HID类（零驱动安装）。

🖱️HID类概述

HID（Human Interface Device）类是最广泛使用的USB设备类之一。所有主流操作系统都内置HID驱动，无需安装额外驱动即可使用。

HID 类不仅限于传统人机输入设备，也常用于通用数据传输（如传感器、控制面板），因为其无驱动安装的优势。

📝无需INF的自动匹配

从 Windows 8 起，设备固件可通过 Microsoft OS Descriptors 告知系统自动使用 WinUSB 驱动，无需用户安装INF文件。

  • 在 BOS 描述符中包含 MS OS 2.0 描述符集
  • 设置 CompatibleID = "WINUSB"
  • Windows 8+ 自动加载 WinUSB.sys
  • 第15章详细介绍 System INF File 方式
这大幅简化了自定义USB设备的部署流程，用户插入即用。
Chapter 15
Using WinUSB's System INF File
使用Windows系统内置INF文件配置WinUSB设备

📄系统INF文件方式

Windows 提供了系统级 INF 文件（winusb.inf），通过设备管理器手动指定，或通过设备描述符中的 Microsoft OS Descriptors 自动触发。

使用系统INF文件的优势：无需发布自定义驱动包，只需固件配置正确即可。这对于快速原型开发非常有用。

🔧设备端配置要求

  • 在接口描述符中设置正确的类/子类/协议
  • 实现 Microsoft OS String Descriptor（字符串索引0xEE）
  • 响应 ExtendedCompatID OS Descriptor 请求
  • CompatibleID 字段值设为 "WINUSB\0\0"
  • 可选：SubCompatibleID 指定子功能

💡两种WinUSB使用方式对比

方式适用版本复杂度
厂商INF文件所有版本需要打包签名
系统INF文件Vista+中等，需设备支持
MS OS 2.0描述符Win 8.1+低，纯固件配置
Chapter 16
Using Hubs to Extend and Expand the Bus
USB集线器工作原理、级联规则与事务转换器

🔀Hub内部结构

图16-1: USB Hub内部功能模块（含TT事务转换器）

📏Hub级联限制

  • USB 2.0 最多 5层Hub（总线深度7层含设备）
  • USB 3.x 最多 5层Hub
  • 全系统最多 127个设备（含Hub）
  • 每个Hub最多 15个下行口
  • 级联增加延迟，影响时间敏感传输

⚡Hub供电管理

  • 总线供电Hub：从主机USB口取电，受500mA限制
  • 自供电Hub：独立电源，可为每口提供500mA
  • Hub 必须实现端口过流保护
  • 支持选择性挂起单个端口
  • USB 3.0+ 支持更精细的功率管理

Managing Power

⚡USB电源规格

图17-1: USB各规范最大供电能力对比

💤挂起与恢复机制

  • Global Suspend：总线3ms无活动后设备进入挂起
  • 挂起状态下设备耗电 ≤ 2.5mA
  • 选择性挂起：主机挂起指定Hub端口
  • 远程唤醒：设备可请求主机结束挂起
  • USB 2.0 LPM：L0(活跃) L1(低功耗) L2(挂起) L3(断电)
  • USB 3.0 U状态：U0~U3 四级功耗状态

🔋USB Type-C 与 Power Delivery

  • CC引脚用于检测方向和功率协商
  • PD 3.0 支持可编程电源供应（PPS）
  • 快充协议：5V/9V/15V/20V 多档电压
  • 最高支持 100W（20V × 5A）
  • 数据+供电可同时工作
  • 电缆主动/被动，速度和电流标注

🐧Linux USB 驱动架构

  • OTG控制器可同时作为Host和Device
  • 嵌入式Linux推荐使用libusb进行设备访问
  • RTOS上可使用 TinyUSB / USB Host Library
