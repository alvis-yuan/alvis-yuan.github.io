HONOR 50 Pro 与 CloudPrint_0159 经典蓝牙流程分析

输入抓包：honor50_sunmi_bt_ota.csv。本文按 Bluetooth Core v4.2 BR/EDR Controller 的典型链路过程，对两台设备之间的 inquiry、paging、connect、SSP 配对、加密和业务通道建立进行对照分析。

结论概览

116 CSV 解码事件总数
27.541s - 186.126s 抓包可见时间窗口
84 / 25 / 7 LMP / L2CAP / SDP 事件数量
Central -> Peripheral 推定 HONOR 50 Pro 主动连接 CloudPrint_0159
CSV 从 LMP 层事件开始，未包含 Inquiry ID 包、FHS 包、Page ID train 等更底层的基带原始包。因此 inquiry 和 paging 的空口前导过程只能按协议推断；抓包中可直接验证的是 page 成功后的 LMP、L2CAP、SDP 和加密流程。
CloudPrint_0159：被发现/被连接设备 HONOR 50 Pro：推定发起扫描、建链和 SDP 查询 SSP Host + SC Host 均被双方支持 最终启用 16 字节链路加密

抓包主线

27.541 - 27.560s短连接：能力交换、读取 CloudPrint_0159 名称后由远端用户终止连接。
28.372 - 28.524s第二次连接：版本/特性/扩展特性交换，Host Connection 接受，完成 packet type、slot、AFH、监督超时等 LMP 协商。
28.524 - 31.426sSSP 配对与安全：IO Capability、Simple Pairing Number、DH Key Check、安全认证、加密模式、密钥长度和 Start Encryption。
31.436 - 31.744s上层业务：L2CAP 配置，SDP 查询 PnP/HID/SPP 服务，随后建立 HID Control 和 HID Interrupt 通道。
31.747 - 186.126s链路维持：QoS、周期性 AFH map 更新、功率控制、Sniff/Sniff Subrating、Ping。

角色与协议对照

从 LMP Version Exchange (Central: 5.2 . Peripheral: 6.0)、后续 SDP 查询返回 CloudPrint 侧的 HID/SPP/PnP 服务，以及手机名 HONOR 50 Pro 在配对后才被读取来看，本次主流程可按下述角色理解：

角色设备依据
Central / Master / InitiatorHONOR 50 Pro主动发现外设、发起 BR/EDR page、发起 SDP 查询；IO Capability 为 DisplayYesNo 且要求 MITM。
Peripheral / Slave / AcceptorCloudPrint_0159被查询名称和服务，提供 HID、SPP、PnP 等 SDP 记录；IO Capability 为 NoInputNoOutput。
协议层级关系：Inquiry/Page 属于 BR/EDR Baseband；连接后的能力、时钟偏移、AFH、认证与加密主要由 LMP 完成；服务发现和 HID/RFCOMM 通道建立由 L2CAP/SDP 承载。
安全含义：一端 DisplayYesNo 且要求 MITM，另一端 NoInputNoOutput。按 SSP 关联模型，最终很可能退化为 Just Works，能建立加密但不能提供用户确认级别的 MITM 防护。

1Inquiry / 发现与名称识别

协议上，HONOR 50 Pro 若要发现 CloudPrint_0159，会在 inquiry hop train 上发送 GIAC/DIAC ID 包；CloudPrint_0159 处于 inquiry scan 时响应 FHS，FHS 中携带 BD_ADDR、class of device、clock 等用于后续 page 的信息。CSV 没有记录这些基带包，但 27.548s 的 LMP Name Transaction ("CloudPrint_0159") 是发现阶段之后读取远端名称的直接证据。

抓包时间事件解读
27.541400LMP Features Exchange已经进入 ACL/LMP 层，说明 inquiry/page 前导已完成或被过滤。
27.548900LMP Name Transaction ("CloudPrint_0159")HONOR 读取 CloudPrint 的 friendly name，常见于设备列表展示或重新确认目标。
27.560150LMP Detach (Remote User Terminated Connection)短连接完成名称读取后断开，未进入完整配对。

因此，本抓包对 inquiry 的结论是：发现动作本身未直接呈现，但名称读取和短连接断开符合“扫描到目标后读取名称”的行为。

2Paging / 建立物理 ACL 链路

BR/EDR page 阶段使用 inquiry 得到的 BD_ADDR 与 clock estimate 去命中对端 page scan。命中后，双方交换 page response/FHS，决定 master/slave 时钟和访问码，进入 connection state。CSV 第二段从 28.372s 的 LMP Version Exchange 开始，说明 page 已成功完成。

抓包时间事件解读
28.372662LMP Version Exchange (Central: 5.2 . Peripheral: 6.0)page 成功后的首批 LMP 控制面交换。
28.398911LMP Host Connection (Accepted)控制器通知/请求主机接受连接，主机侧放行。
28.403912LMP Setup CompleteLMP 连接建立阶段完成，可以继续安全和上层通道流程。

28.372s 前看不到 page train，不代表未发生，而是 CSV 的导出粒度已从 LMP 层开始；LMP 版本和特性交换是 page 成功后的可靠边界点。

3Connect / LMP 能力协商与链路参数建立

连接建立后，双方先对齐控制器能力、EDR packet type、slot、AFH、时钟精度、clock offset 和 supervision timeout。抓包显示双方均支持 SSP Host、LE Host、SC Host，控制器还支持 Secure Connections Controller 与 Ping。

阶段抓包证据协议意义
版本和基础特性28.372662、28.376411、28.391412、28.395162确认 LMP 版本、Feature Page 0/扩展页，后续是否可用 SSP/SC/EDR/AFH 取决于此。
主机连接确认28.398911 Host Connection AcceptedHost 允许 ACL 连接继续存在，避免控制器层面建链后被主机拒绝。
EDR 与时隙能力28.403287 Packet Type Table，28.409539 Max Slot Request，28.518913 Preferred Rate允许 2/3 Mbps EDR，使用 5-slot 包提高吞吐。
AFH 与抗干扰28.446412 Set AFH，28.447663 Channel Classification Request启用自适应跳频，并允许报告信道质量。
连接监督28.522663 Supervision Timeout 5.000s连续丢包或失联超过 5 秒可触发链路超时。

这一段没有异常拒绝。链路从建立后约 150 ms 内完成基本能力协商，随后立即进入安全配对。

4Pairing / SSP 认证与加密

28.523s 起进入 Secure Simple Pairing。HONOR 50 Pro 报告 DisplayYesNo、无 OOB、要求 MITM、Dedicated Bonding；CloudPrint_0159 报告 NoInputNoOutput、无 OOB、不要求 MITM、General Bonding。由于打印设备无输入输出能力，按 SSP 关联模型更接近 Just Works，而不是 Numeric Comparison。

抓包时间事件解读
28.523913LMP IO Capability Transaction交换 IO/OOB/MITM/Bonding 参数，决定 SSP 关联模型。
29.050170、29.053297LMP Simple Pairing NumberSSP 认证过程中的随机数/确认值相关交换，双方均接受。
31.373952、31.399579LMP DH Key Check双方基于 ECDH 派生结果做一致性校验。
31.402702LMP Secure Authentication Transaction认证完成前后的安全事务。
31.412702 - 31.426453Key Size Mask、Encryption Mode、Key Size、Start Encryption协商加密能力与 16 字节密钥长度，并开始链路加密。
配对结果：DH Key Check 双向 Accepted，随后启用 16 bytes encryption key，说明认证和链路加密建立成功。
风险点：虽然 HONOR 端要求 MITM，但对端 NoInputNoOutput 使用户确认无法完成。若产品安全需求要求强 MITM，应让 CloudPrint 侧具备显示/确认/OOB 能力，或通过应用层二维码/云端绑定补足身份校验。

5Service Discovery / SDP 与业务通道

加密开始后，HONOR 50 Pro 立刻通过 L2CAP/SDP 查询 CloudPrint_0159 的服务记录。返回的服务包括 PnP Information、HID、多个 Serial Port Profile RFCOMM 通道。随后建立 HID Control 和 HID Interrupt 通道，这说明配对后的业务连接目标不是仅停留在发现层，而是进入 HID 类设备通道。

抓包时间事件解读
31.435830 - 31.443953L2CAP Configure + SDP Service Search Attribute PnP查询设备身份信息。
31.456453 - 31.605206SDP 查询 HID、SPP、PnP、Phonebook Access一次综合服务枚举，CloudPrint 返回 HID 和多个 SPP server 通道。
31.693956 - 31.718957单独查询 HID 记录读取 HID descriptor、language、profile version 等属性。
31.723957 - 31.743957L2CAP Connection: HID Control / HID Interrupt建立 HID 控制和中断数据通道。
31.747083LMP Quality of Service (18 slots = 11.250 ms)为后续 HID 交互设置 QoS。

6链路维持 / AFH、功控与 Sniff

业务通道建立后，抓包长时间出现 LMP Set AFH、LMP Power Control、LMP Sniff Mode、LMP Ping。这属于连接稳定性和功耗优化流程，不是新的配对。

抓包时间事件解读
32.650s - 186.126s多次 LMP Set AFH，信道数从 57、48、46、38 逐步变化，后期最低约 20环境干扰或共存策略导致可用跳频信道不断重算。
35.105s、42.598s、54.641sLMP Power Control逐步提升发射功率，后续达到最大功率。
61.774sLMP Sniff Mode，T=48 slots = 30 ms进入低功耗 sniff，保持周期性监听。
61.866sSniff Subrating Not Accepted, LMP PDU Not Allowed首次尝试子速率被拒绝，后续 93.889s 重新协商成功但参数受限。
148.880s、184.371sLMP Ping安全连接场景下的链路存活/防超时确认。

关键判断与建议

问题判断建议
Inquiry 是否完整可见？不可见。CSV 首行已经是 LMP，缺少 ID/FHS 等基带层证据。若需要完整空口佐证，导出/保留 baseband packet、inquiry result、page train 相关解码。
Paging 是否成功？成功。28.372s 后连续 LMP 版本、特性、Host Connection Accepted、Setup Complete 可证明 ACL link 已建立。以 28.372s 作为第二次完整连接的可见起点。
配对是否成功？成功。DH Key Check 双向 Accepted，后续 Start Encryption，密钥长度 16 字节。若遇到兼容性问题，优先检查 IO Capability 组合和 Host bonding 策略。
MITM 是否满足？大概率不满足。NoInputNoOutput 对端无法做 Numeric Comparison 或 passkey。对高安全绑定场景，增加 OOB、二维码、应用层账号绑定或设备侧确认能力。
后续连接是否稳定？整体稳定，但 AFH 可用信道后期降到约 20，且功率到最大。关注 2.4 GHz 干扰、天线距离、Wi-Fi 共存与打印机射频性能。
