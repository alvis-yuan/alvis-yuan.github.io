阶段 A：第一次断开 → 第二次建链 46.902 – 47.622 s

LLCP Termination · CSV 第 5728 行

LLCP Termination (Remote User Terminated Connection) · 原因 0x13

第一次会话在 WiFi 扫描通知等业务结束后由对端/应用层关闭。

外设恢复广播 · 46.965 s 起

约 63 ms 后 CloudPrint 再次发送 Connectable Undirected Adv（名称、0x180A 与首次相同）。

间隔窗口内约 42 条 广播、2 条 Scan，HONOR 未长时间扫描即发起连接。

短间隔重连典型场景：应用层「断开后立即 reconnect」或协议栈 fast reconnect；外设持续可连接广播供 Central 快速二次建链。

阶段 D：ATT / GATT — 抓包观察 无可见 PDU

重要结论：第二次连接区间（CSV 第 5787–8207 行）内，Wireshark 未解出任何 ATT / L2CAP-ATT 报文（第一次连接共约 391 条 ATT，末条在 46.842 s）。
与第一次的差异（推断）
第一次有、第二次无可能原因
Read By Group/Type、Find InformationHost 使用 GATT 缓存（句柄 0x993F 等已在首次发现），跳过服务发现
Write CCC / Write Command / Notification① 业务已在首次连接完成；② 二次连接仅保持链路；③ 或存在未解密的加密载荷（全文件无 SMP PDU，本抓包无法证实加密）

第一次会话末条 ATT（对照）：

46.842 ATT Notification (0x993F: …04 04 00 00 00) WiFi 列表分片尾声 46.902 LLCP Termination — 约 0.78 s 无 ATT — 47.622 第二次 Connection Indication
若应用层在二次连接继续 OTA，按第一次模式应出现 AA 55 写命令与 Notification；当前导出中完全缺失，更符合「重连后仅维持 ACL、尚未下发 GATT」或「抓包在业务恢复前结束」。
