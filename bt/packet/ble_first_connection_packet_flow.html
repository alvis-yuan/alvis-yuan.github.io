必要性：可连接广播让 Central 获知设备地址、名称与部分服务 UUID，才能发起连接。Device Information (0x180A) 在广播中预告，便于扫描 UI 展示。
Scan Request / Scan Response

自约 0.915s 起，HONOR 对 CloudPrint 发送 Scan Request，外设回 Scan Response（内容与广播数据相同：名称、Flags、0x180A）。

可扫描广播（Scannable）允许 Central 在建立连接前拉取 Scan Response，减少广播 PDU 长度、同时保证连接前信息完整。

阶段 E：ATT 业务交互 36.229 – 46.852s

发现完成后，业务集中在私有特征 0x993F（Handle 0x002A）上的 Write / Notify，帧格式以 AA 55 为魔数，符合 Sunmi OTA/配网类协议。

启用 Notification · 36.229 – 36.312s

写 CCC：先关后开

36.229 Write CCC → Notifications Disabled · 36.282 Write CCC → Enabled
标准流程：先清 CCC 再置 Notification 位，避免重复订阅状态混乱。

Write Command PDU 示例 — 第 4011 行 Data 52 2A 00 AA 55 00 06 64 05 52 = ATT Write Command · Handle 0x002A (0x993F) AA55 = 帧头 · 64 05 = 命令字（业务层） Notification 示例 — 第 4018 行 1B 2A 00 AA 55 14 00 64 00 4E 35 30 37 44 35 33 53 31 30 31 35 39 00 → "N507D53S10159" 等设备型号信息
Write Command 无响应 ATT 包，适合 OTA 高频下发；Notification 用于异步上报扫描结果、状态，WiFi 列表因 MTU 251 被拆成多条 Notification 分片传输。

阶段 F：断开 46.901962s

LLCP Termination · CSV 第 5728 行

LLCP Termination (Remote User Terminated Connection)

原因码 0x13：远端用户终止 — 通常表示对端（CloudPrint 或 HONOR 协议栈）主动断开，例如 OTA 阶段结束、应用层 close、或超时策略。

Termination 后链路回到非连接态；CloudPrint 继续发送 Connectable Adv（抓包后续仍有广播）。本次会话未再复用同一连接上的加密重连流程。

与经典蓝牙抓包的关系

同一次 OTA 场景中，honor50_sunmi_bt_ota.csv 覆盖 BR/EDR（HID/SDP），本文件为 BLE GATT 通道（私有 UUID E7810A71 / 0x993F）。 二者可并行：手机经 BLE 做配网/OTA 控制，经经典蓝牙做 HID 或其他 profile。
