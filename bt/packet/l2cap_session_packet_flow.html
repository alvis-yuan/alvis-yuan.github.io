L2CAP 多次建链 · 配置 · 业务 · 断开

基于 honor50_sunmi_bt_ota.csv（HONOR 50 Pro ↔ CloudPrint_0159）。说明加密后 SDP/HID 阶段为何出现多轮 L2CAP 会话，以及报文字段如何对应各阶段。

L2CAP 在 ACL 上的位置 总览

一条 BR/EDR ACL 上可并行多条 L2CAP 通道。每条动态通道由本端 CID标识，通过 PSM 区分上层服务（SDP=0x0001、HID Control=0x0011 等）。建立流程统一为：Connection → Configure → 数据 → Disconnection。

抓包中的 CID 分配（HONOR 为 Initiator）

0x0051
SDP · 第 1 轮（PnP）
0x0052
SDP · 大范围枚举
0x0053
SDP · PnP handle
0x0054
SDP · HID handle
0x0055
HID Control → CloudPrint 0x0040
0x0056
HID Interrupt → CloudPrint 0x0041
对端 CloudPrint 在 Configure 里出现的 Src=0x0040/0x0041 是其分配的本地 CID；HONOR 的 Dst 即指向这些 CID。

阶段 0：ACL 级 L2CAP 信令（未建业务通道前）28.412s

在加密完成前，双方已在默认信令通道上交换 L2CAP Information，协商增强重传（ERTM）与固定信道能力。这不属于某次 SDP 短连接，而是整条 ACL 的 L2CAP 能力发现。

时间报文含义
28.412039L2CAP Information (Extended Features Supported · ERTM | FCS)支持增强重传与帧校验
28.530790L2CAP Information (Fixed Channels Supported · Signal)支持固定信令信道

后续所有 Connection Request、Configure Request 都走 CID 0x0001 上的 Signaling PDU 完成，业务 SDP/HID 数据才走各自动态 CID。

单次 L2CAP 会话的标准生命周期 四步

1. Connection指定 PSM，分配 CIDSig: 0x02 / 0x03
2. Configure协商 MTU、FlushSig: 0x04 / 0x05
3. Data上层 PDU（SDP/HID）L2CAP I-frame
4. Disconnection释放 CIDSig: 0x06 / 0x07

会话示例 ①：SDP 大范围查询 CID 0x0052 · PSM=0x0001

最完整的一轮：建链 → 双向 Configure（MTU 1024）→ 发送超大 Service Search Attribute → 再查 Phonebook → 断开。

Configure 报文解读

抓包描述字段含义
Configure (Dst=0x0052, MTU=256 . Src=0x0040)CloudPrint 配置 HONOR 刚建好的通道 0x0052，提议有效载荷 MTU=256（常见首包保守值）。
Configure (Dst=0x0040, MTU=1'024 . Src=0x0052, MTU=1'024)HONOR 回配 CloudPrint 的 0x0040，双方最终按 min(1024,1024)=1024 传 SDP；大 Attribute 可一次装下更多内容。

Wireshark 解码为 “HID + 4×SPP + PnP …” 是因为响应/请求参数里带了多组服务记录属性；该 PDU 走 L2CAP CID 0x0052 的 I-frame，外层已由 ACL 加密保护。

同一 CID 上 31.598955 还有一条较小的 SSA（Phonebook Access），仍属同一 L2CAP 会话内的第二次 SDP 事务，之后才 Disconnection。

业务通道：HID Control / Interrupt 建链后保持

SDP 全部断开后，HONOR 根据 SDP 中 ProtocolDescriptorList 的 PSM 0x0011 / 0x0013 建立 HID 通道。Configure 参数与 SDP 不同：MTU 更小、Flush Timeout 为无限重传。

L2CAP 层在做什么：每轮分配新 Src CID → 信令建链/配 MTU → 承载 SDP 或 HID 的 I-frame → 发 Disconnection 回收 CID（HID 除外）。
报文层在做什么：Configure 决定单包最大载荷；SDP 的 07/03/05 等 PDU 是 L2CAP 载荷；Disconnect 后同 PSM 可再次 Connect，故出现 0x0052→0x0053→0x0054 递增 CID。
