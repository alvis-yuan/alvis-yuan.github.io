SDP 服务发现过程

HONOR 50 Pro 在链路加密后（约 31.436s → 31.719s）对 CloudPrint_0159 的 SDP 查询。 数据源：honor50_sunmi_bt_ota.csv 第 45–66 行。

总览 SDP PDU 会话 1 会话 2 会话 3 会话 4 发现结果

发现过程总览

SDP 跑在 L2CAP PSM = 0x0001 上。HONOR 采用四轮短连接：每轮 Connect → Configure（MTU 1024）→ 一至多个 SDP 事务 → Disconnect。 前两轮用 Service Search Attribute Request (0x07) 按 UUID 批量拉属性；后两轮用 Search (0x03) + Attribute (0x05) 按 Handle 精查。

会话CID时间SDP 策略目的
10x005131.436–31.451SSA · 仅 PnP快速取设备身份 UUID 0x1200
20x005231.456–31.605SSA · 广搜 + Phonebook 探测枚举 HID、4×SPP、PnP 及属性
30x005331.643–31.688Search + Attribute确认 PnP Handle 0x00010008 全记录
40x005431.694–31.719Search + Attribute拉取 HID 0x00010001 含 Report Descriptor
CSV 第 45 行起直接为 L2CAP Configure (Dst=0x0051)，Connection 可能紧挨上一事件未单独导出。CloudPrint 侧固定 Src/Dst=0x0040 为 SDP 服务器 CID。

SDP PDU 与 Data 列解读

PDU ID名称本抓包出现作用
0x03Service Search Request会话 3、4按 Service Class UUID 模式搜索，返回 Handle 列表
0x05Service Attribute Request会话 3、4按 Handle + Attribute ID 范围 取一条服务记录
0x07Service Search Attribute Request会话 1、2搜索 + 直接返回指定 Attribute，一轮完成多服务发现

公共头格式（大端）：

[0] PDU ID [1-2] Transaction ID（匹配请求/响应） [3-4] Parameter Length（后续参数字节数） [5…] 参数：UUID 序列、Attribute ID 列表、Handle 等（Data Element 编码）

会话 1：PnP 快速查询 CID 0x0051

31.435829 Configure → 31.443953 SSA → 31.451453 Disconnect

L2CAP Configure

31.435829 / 31.437703 · MTU 256 → 双向 1024
必要性：SDP 响应可能较长，1024 MTU 减少分片。首轮 Configure 常先提议 256 再协商到 1024。

SDP Service Search Attribute — PnP Information

CSV 第 47 行 · 31.443953
07 00 00 00 4D 00 4A 35 48 36 00 45 ... 07 = Service Search Attribute Request TID = 0x0000 · ParamLen = 0x004D (77) 参数中含 UUID 0x1200 (PnP) 及要返回的 Attribute ID 列表

响应解码：PnP Information，L2CAP/SDP 版本等（Wireshark 摘要）。

发现目的：读取厂商/产品 ID（Attribute 0x0200–0x0205 族），用于识别打印机型号，无需先知道 Handle。

L2CAP Disconnection

31.451453 · Src=0x0051
必要性：Android 栈常见「单任务单通道」；PnP 查完即释放 CID。

会话 2：广谱服务枚举 CID 0x0052

31.456453 Connect → 31.522704 大 SSA → 31.598955 Phonebook → 31.605206 Disconnect

L2CAP Connection + Configure

31.456453 / 31.460830 / 31.485204 · PSM=SDP · MTU=1024

SDP Service Search Attribute — 多服务批量

CSV 第 52 行 · 31.522704
07 00 00 00 02 F0 02 ED 36 02 EA ... ParamLen = 0x02F0 (752 字节参数) — 本抓包最大 SDP 请求

响应摘要（解码器）：

  • HID 0x1124 — 「Ifx HID Device」、Keyboard/Infineon、HIDP PSM 0x11/0x13、HID 描述符二进制
  • SPP ×4 — RFCOMM Channel 2/3/4/5，名 SPP SERVER16/26/36/46
  • PnP Information 0x1200
发现目的：一次 SSA 完成「设备能做什么」的主扫描——手机据此决定连 HID 还是某路 SPP。 参数里包含多个 UUID（19 11 24=HID、19 11 01=Serial Port 等）及大段 Attribute ID 列表。

SDP Service Search Attribute — Phonebook Access

CSV 第 53 行 · 31.598955
07 00 00 00 05 00 02 35 00 00 ParamLen = 5 · 查询 Phonebook/PCE UUID

响应极短（35 00 00 空序列），表示未提供电话本服务或匹配为空。

发现目的：探测可选 Profile；打印机通常无 PBAP，属兼容性探测。

L2CAP Disconnection

31.605206 · Src=0x0052

会话 4：HID 按 Handle 精查 CID 0x0054

Search → Attribute（大）→ Disconnect → 建 HID 通道

L2CAP Connect + Configure

31.693956 – 31.698957

SDP Service Search — HID

CSV 第 64 行 · 31.703957
03 00 00 00 00 09 ... 搜 UUID 0x1124 (HID) 响应：HID: Handle 0x00010001

SDP Service Attribute — Handle 0x00010001

CSV 第 65 行 · 31.708956 · 响应约 0x0181 字节
05 00 01 01 81 ... ParamLen = 0x0181 (385) — 含完整 HID Descriptor (0x0206)

关键属性：ProtocolDescriptorList → L2CAP PSM 0x0011（Control）、0x0013（Interrupt）；ServiceName「Ifx HID Device」。

发现目的：HID 连接前必须拿到 PSM 与 Report Descriptor。会话 2 的 SSA 已含 HID，本轮按 Handle 再取一遍确保描述符完整（MTU 1024 下单响应承载）。

Disconnect + 转入业务连接

31.718957 Disconnect 0x0054 · 31.723957 HID Control 0x0055 · 31.738957 HID Interrupt 0x0056
发现完成标志：客户端已从 SDP 得知 HID Control/Interrupt 的 PSM，随即建立非 SDP 的 L2CAP 通道（不再经过 SDP）。

探测性查询
发现阶段典型手段本抓包对应得到的信息
广搜SSA + 多 UUID会话 2HID + SPP + PnP 属性全集
定向SSA 单 UUID会话 1仅 PnP
Handle 化Search → Attribute会话 3、4稳定 Handle + 完整单条记录
连接读 0x0004 ProtocolDescriptorList会话 4 后HID L2CAP 建链
策略解读：先 SSA 再 Search+Attribute 看似重复，实为手机栈分层——SSA 快速列表展示/策略决策，Search+Attribute 按 Handle 刷新单条记录（尤其 HID Descriptor）。打印机被动响应即可。
