Core 6.0 Part B — 数据信道 LL Control PDU

本索引覆盖已连接状态下 Data Physical Channel 上的 LL Control PDU（Header.LLID = 0b11），与 Link Layer Specification Part B 中 Table 2.22（Opcode）及 §2.4.2（CtrData）对齐，共 61 条现行 Opcode（0x00–0x3C）。过程分类依据 §5.1 Link Layer ACL control procedures。

相关文档：BLE 链路层 (Part B)

离线：单文件自包含，浏览器直接打开即可；仅规范外链需联网。

PDU 结构

Payload = Opcode (1 byte) + CtrData (固定长度)。与 LL Data PDU（LLID 0b01/0b10，承载 L2CAP）区分。

异常应答

不支持 Opcode → LL_UNKNOWN_RSP；可拒绝的过程 → LL_REJECT_IND / LL_REJECT_EXT_IND。
