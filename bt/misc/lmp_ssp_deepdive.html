LMP (链路管理协议) 核心职责与报文剖析

蓝牙底层的“外交官”：负责谈判、安全、状态切换与策略控制

🔗 链路配置与建立

基于物理信道建立逻辑层面的控制与数据链路。

  • 特性交换： 询问对方是否支持 EDR、AFH 等。
  • 时钟偏移请求： 获取时钟差以加速后续连接。
  • 版本交互： 确认双方的蓝牙协议版本。

🛡️ 安全与鉴权管理

防止设备被监听或非法接入，保障链路隐私。

  • 配对 (Pairing)： 生成并交换链路密钥 (Link Key)。
  • 鉴权 (Auth)： 校验对方设备的合法性。
  • 加密 (Encryption)： 协商加密算法并控制基带开启流加密。

🎭 角色与状态切换

在底层改变设备的物理拓扑属性。

  • 角色切换 (Role Switch)： 主(Master) 从(Slave) 身份互换。
  • 低功耗模式： 协商进入 Sniff (呼吸)、Hold、Park 等节能模式。

📶 AFH 与功耗控制

实时维护链路质量，抗干扰并省电。

  • 功率控制： 要求对方调大/调小发射功率。
  • AFH 信道映射： 同步干扰黑名单，避开 Wi-Fi 等频段。

深度剖析范例：SSP (Secure Simple Pairing) 安全简易配对

🔴 设计背景与解决的痛点：
在蓝牙 2.1 之前，设备配对使用“传统配对(Legacy Pairing)”，需要用户手动输入 PIN 码（如常见的 "0000" 或 "1234"）。

痛点： 1. 极易遭受被动窃听攻击。黑客只需抓取空中的配对包，利用离线字典攻击，几秒钟就能破解 "0000" 的加密链路。
2. 无屏幕/无键盘的设备输入密码极其痛苦。

SSP 的解决方案：
引入了银行级的 ECDH (椭圆曲线迪菲-赫尔曼) 公钥加密算法。黑客即使抓取了空中所有的明文公钥交换包，也绝对算不出最终的共享密钥。同时引入“数值比较(Just Works/Numeric Comparison)”防止中间人(MITM)攻击，实现既安全又无感的配对体验。

二、底层 LMP 报文格式与字段拆解

LMP 报文本身隐藏在基带载荷 (Payload) 之中。一个标准的 LMP 消息包含：
Transaction ID (1 bit) + Opcode (7 bits) + [Ext Opcode] + Parameter List

💌 1. LMP_IO_Capability_req (输入输出能力请求)
字段名称 大小 功能说明与实战含义
Opcode / Ext Opcode 7 bits + 8 bits 基带识别码 (Opcode=127, Ext Opcode=11)，告诉芯片这是一个查户口的信令。
IO_Capability 1 Byte 设备的真实硬件能力，决定配对方式。取值：
0x00: DisplayOnly (只有屏幕)
0x01: DisplayYesNo (有屏幕，有确认按键 - 触发 Numeric Comparison)
0x03: NoInputNoOutput (瞎子哑巴，如蓝牙耳机 - 触发 Just Works 无感配对)
OOB_Data_Present 1 Byte 是否有带外数据(如NFC)。如果双方支持 NFC，则直接用 NFC 交换密钥，极大加快速度。
Auth_Requirements 1 Byte 鉴权要求。表示该设备是否要求“中间人保护(MITM Protection)”，是否要保存 Link Key (Bonding)。
💌 2. LMP_encapsulated_payload (公钥封装传输)
字段名称 大小 功能说明与实战含义
Opcode 7 bits / 8 bits Opcode=127, Ext Opcode=17。因为公钥太长（P-256曲线的公钥长达384比特 / 48字节），一个基带包塞不下，所以 LMP 发明了这种封装包，将大数据切片发送。
Encapsulated_Data 16 Bytes (每包) 这里承载的正是 ECDH 椭圆曲线加密算法的公钥碎片。 由于公钥密码学的特性，哪怕这段数据在空中被嗅探器(Sniffer) 100% 截获，黑客也无法通过数学手段反推出你的私钥，完美解决了传统 PIN 码易被窃听的致命伤。
💌 3. LMP_simple_pairing_confirm / number (鉴权防伪)
字段名称 大小 功能说明与实战含义
Commitment Value 16 Bytes 在 _confirm 信令中发送。这是设备自己生成的随机数(Nonce)通过 AES-CMAC 算出的哈希值（相当于把答案放在带锁的箱子里扔给对方）。
Nonce Value 16 Bytes 在 _number 信令中发送。此时揭晓刚才的明文随机数（给对方箱子的钥匙）。对方拿到钥匙打开箱子比对哈希值，如果对上了，证明对方没被掉包，完美防止中间人(MITM)攻击。
