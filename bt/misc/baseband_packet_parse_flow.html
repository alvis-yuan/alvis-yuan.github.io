📡 经典蓝牙底层报文解析与分流引擎

结合基带头(Baseband Header)与载荷头(Payload Header)的位级(Bit)结构，揭示底层硅芯片如何依靠提取关键字段，在微秒间完成复杂的硬件路由分流。

LMP PDU 全集（Table 5.1/5.2，可搜索 Opcode） · 基带分流 + LMP 概览

  • 1. 天线捕获电磁波
    匹配 Access Code (72 bit) 同步时钟。
    随后进入基带解析流水线。
    • 2. 解析基带头 (Baseband Header)
      硬件进行 HEC 校验后，提取 TYPE 字段，作为整个链路的一级分流道岔。
      基带报头结构 (逻辑 18 bits)
      AM
      TYPE
      F
      A
      S
      HEC
      • AM_ADDR(3b): 目标从设备短地址(1-7)
      • TYPE(4b): 报文类型(分流核心标志)
      • F(1b): FLOW 流控 (0=内存满)
      • A/S(2b): ARQN/SEQN (ACK与防重传)
      • HEC(8b): 头部完整性校验码
      • TYPE = 0000/0001
        纯控制包 (NULL/POLL)
        (无载荷部分)
        直接丢弃，仅提取 ARQN/FLOW 更新硬件 ACK 状态机。
      • TYPE = 01xx (HV1/HV3)
        同步语音包 (SCO)
        (无载荷头，全为音频数据)
        物理旁路直通：
        将剩余比特流强行怼入内部 CVSD 语音解码器直接出声，绝不上报！
      • TYPE = 1xxx (DM/DH)
        异步数据包 (ACL)
        确认是数据包后，读取载荷最开头的数据，进入二级分流鉴定。
        • 3. 解析载荷头 (Payload Header)
          提取前 2 bit L_CH，决定这包数据该交给谁处理。
          载荷报头结构 (以单时隙 8 bits 为例)
          L_CH
          F
          Length
          • L_CH(2b): 逻辑通道(分拣路由标志)
          • F(1b): L2CAP 层级的流控位
          • Length(5b): 纯业务数据的字节长度
          • *若是DH5多时隙包，头为16bits(Length占9b)
          • L_CH = 11
            LMP 控制信令
            芯片内部截留：
            交给底层单片机的 LMP 固件处理配对、加密、功率控制。
            (对上层操作系统完全隐身)
          • L_CH = 10 或 01
            L2CAP 用户数据
            跨层上抛：
            剥离载荷头，将纯数据加上
            HCI PB-Flag = 10(Start)或01(Cont)
            通过 UART/USB 发给手机操作系统进行组装。
