Bluetooth Core 6.0 · Part C

Generic Access Profile (GAP)
通用访问规范详解

GAP 定义了蓝牙设备发现（空闲模式）和连接管理（连接模式）的通用流程，并规定安全级别使用方式及用户界面层可访问参数的通用格式要求。

1

前言 Foreword

蓝牙 SIG 定义的 Profile 规范确保了来自不同制造商的设备在特定服务与使用场景下的互操作性。Profile 从蓝牙 SIG 规范中选取消息与流程（统称为能力 Capabilities），描述特定服务的空中接口。

核心思想

GAP 是所有蓝牙 Profile 的基础。它定义了设备如何被发现、如何建立连接、如何处理安全，以及如何向用户呈现相关参数。任何蓝牙传输层 Profile 和应用层 Profile 都必须遵循 GAP 的规定。

🔍

BR/EDR-only

经典蓝牙核心配置，支持传统高速蓝牙通信（音频、数据传输等）

📡

LE only

低功耗蓝牙核心配置，适用于传感器、可穿戴、IoT 等低功耗场景

⚡

BR/EDR/LE

双模核心配置，同时支持经典蓝牙和低功耗蓝牙，最为通用

实现传输类型概览
实现传输类型 适用 Core 配置 典型应用
BR/EDR-only BR/EDR Core Configuration 耳机、音箱、键盘鼠标
LE only LE Core Configuration 心率带、温度传感器、信标
BR/EDR/LE BR/EDR/LE Core Configuration 智能手机、平板、电脑

2.2.2 LE 四种角色

LE 物理传输定义了四种 GAP 角色，彼此功能互补，构成完整的 LE 生态系统：

📢

Broadcaster（广播者）

持续发送广播包，不建立连接。适合信标、广告牌等场景。必须支持 Advertising 状态。

👁️

Observer（观察者）

扫描并接收广播包，不发起连接。适合扫描器、数据收集器。必须支持 Scanning 状态。

📱

Peripheral（外围设备）

发广播并接受连接请求，连接后成为 Slave。适合传感器、心率带等。

🖥️

Central（中央设备）

扫描并主动发起连接，连接后成为 Master。适合智能手机、网关设备。

用户界面方面 User Interface Aspects

GAP 对用户界面可见的参数与流程提出了统一的规范要求，以确保跨设备、跨制造商的一致体验。

📝

设备名称

«Device Name» — UTF-8 编码的设备友好名称，最长 248 字节，用于在 UI 中标识设备

🎨

设备外观

«Appearance» — 16-bit 数值描述设备类型（如手表、心率计），用于 UI 图标选择

🔑

配对流程名称

标准化配对/绑定流程的用户可见名称，确保用户理解安全操作的含义

📊

信号强度显示

RSSI 等物理层测量值的展示规范，帮助用户判断连接质量

BR/EDR 安全模式 Security Modes

安全模式 1

无安全性

不执行任何安全流程，所有通信均以明文传输，无认证无加密

安全模式 2

10.8

隐私与设备地址 Privacy & Address

GAP 定义了三种蓝牙设备地址类型，以在可追踪性与连接性之间取得平衡：

🏷️

公共地址 Public Address

由 IEEE 分配的全球唯一 48-bit MAC 地址，固定不变，最易被追踪

🎲

随机静态地址 Random Static

设备启动时随机生成，最高两位为 11，每次开机可变但会话中固定

🔄

可解析私有地址 RPA

基于 IRK（身份解析密钥）周期性生成，默认每 15 分钟轮换一次，防止位置追踪

可解析私有地址（RPA）生成与解析流程
🔑 生成 RPA
IRK (128-bit)
+
prand (24-bit 随机)
→
hash (24-bit)
RPA = prand[23:22]=10 || prand[21:0] || hash(IRK, prand)
🔓 解析 RPA
1. 取 RPA 的高 24 位（prand）
2. 用本地存储的 IRK 计算 hash'(IRK, prand)
3. 若 hash' == RPA 低 24 位 → 解析成功，设备身份确认
轮换间隔由 TGAP(private_addr_int) 控制，默认 15 分钟

LE 安全流程与绑定

LE 安全基于安全管理协议（SMP），使用 AES-128 加密，并通过 LE Secure Connections（LESC）提供椭圆曲线 Diffie-Hellman（ECDH）密钥交换，抵御中间人攻击。

12.1

LE 安全模式与级别

LE 安全模式 1（加密连接）

安全模式 1 要求使用加密链路，分为四个安全级别，从无需安全到需要 LESC 认证加密。

模式 1 级别 1

无安全要求

无需认证，无需加密。通信以明文进行。

模式 1 级别 2

未认证加密

需要加密但无需 MITM 保护。使用 Just Works 配对方式。

模式 1 级别 3

认证加密

需要认证（MITM 保护）且加密。使用 Passkey Entry 或 OOB 配对。

模式 1 级别 4

LESC 认证加密

需要 LE Secure Connections 的 MITM 保护加密，128-bit 加密强度。

LE 安全模式 2（数据签名）

安全模式 2 使用数据签名而非加密来保证数据完整性，分为未认证签名（级别 1）和认证签名（级别 2）两个级别。

等时广播 Isochronous Broadcasting

Bluetooth 5.2 引入了等时广播（Isochronous Broadcasting）机制，支持周期性广播（Periodic Advertising）和等时广播组（BIG），专为音频流传输设计。

🎵

周期广播 (PA)

Periodic Advertising — 设备以固定周期发送广播，Observer 可与其时间同步，用于广播音频流信息

📻

BIG (广播等时组)

Broadcast Isochronous Group — 允许一个设备同时向多个设备广播音频，实现真无线立体声（TWS）

🔊

Auracast™

基于 LE Audio 和 BIG，实现公共场所音频广播（如机场、影院），无需配对即可收听

⏱️

同步训练

Synchronization Train — 帮助 Observer 找到并同步到广播流的时间信息结构

GAP 定义了一系列标准定时器参数，用于规范广播、扫描和连接的时间行为。格式为 TGAP(nnn)。

1–1.2 s
后台广播时的慢速广播间隔（1M PHY）
TGAP(adv_fast_period)

30–60 ms
