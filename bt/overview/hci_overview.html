HCI 功能规范解析

Core 6.0 · Part E Host Controller Interface Functional Specification
概览
蓝牙 HCI 规范 — 总体结构
Bluetooth Core Specification 6.0 · Volume 4 · Part E

HCI（Host Controller Interface，主机控制器接口）是蓝牙协议栈中连接 Host（主机） 与 Controller（控制器） 的标准化接口层。它为 Host 提供统一的命令方法，用于访问 Controller 的能力并控制与其他蓝牙设备的连接。本规范适用于 BR/EDR 控制器、LE 控制器以及 BR/EDR/LE 双模控制器。

图 0-1: 蓝牙协议栈整体架构 — HCI 位于 Host 与 Controller 边界
📡
统一命令接口
HCI 为 Host 提供与底层 Controller 无关的统一 API，屏蔽硬件差异。
🔄
双向通信
Host 向 Controller 发送命令；Controller 通过事件异步通知 Host。
🛡️
流控制机制
内置双向流控，防止缓冲区溢出，确保数据可靠传输。
📦
标准数据格式
定义命令包、事件包、ACL/SCO/ISO 数据包的精确格式。
§ 1
引言 · Introduction
HCI 的定义、角色划分与适用范围

本节定义了 HCI 的基本概念：HCI 提供 Host 访问 Controller 能力并控制蓝牙连接的统一命令方法。对于 BR/EDR 或 LE 控制器，这些命令通常涉及 Link Manager（链路管理器） 通过 LMP 命令或 Link Layer（链路层） 通过 LL 控制包与远程设备交换信息。HCI 层跨越 Host 与 Controller 的边界，分为上层 HCI（位于 Host）和下层 HCI（位于 Controller）两个角色。

1.1 蓝牙软件栈下层结构

蓝牙协议栈下层由多个层次组成，HCI 作为分界线，上面是 Host 驱动栈，下面是 Controller 硬件。中间的传输层（Host Controller Transport Layer）负责透明传输，无需了解数据内容。

图 1-1: 端到端蓝牙软件栈 — HCI 跨越 Host 与 Controller 边界，传输层透明传递数据
1.2 跨版本兼容性（Cross-version Issues）

Host 与 Controller 不要求支持相同版本的规范。版本不匹配时，规范规定了明确的处理原则：

⚠️ 若 Controller 版本新于 Host：事件中可能出现 Host 版本中标记为"保留"的值。对于 HCI_Command_Status / HCI_Command_Complete 事件中 Status 参数的"保留"值，必须视同错误码 Unspecified Error（0x1F） 处理。
ℹ️ 若 Host 版本新于 Controller：Host 使用了旧版控制器不支持的命令特性时，Controller 将返回错误。Host 必须妥善处理这类错误，否则可能导致意外行为。
§ 2
Host Controller 传输层概述
Overview of Host Controller Transport Layer

传输层存在于 HCI 驱动和 Host 之间，其核心目标是 透明性（Transparency）：HCI 驱动无需了解底层传输技术；传输层也无需理解传递的数据内容。这使得 HCI 逻辑接口或 Controller 可以独立升级，不影响传输层。

2.1 传输层设计目标
🔍
透明性
传输层不解析、不修改 HCI 数据，只做透明转发。
🔌
多传输介质
支持 UART（H4）、USB、SDIO、SPI 等多种物理传输方式，在 Vol.4 其他 Part 中定义。
⚙️
独立演进
HCI 协议与传输层解耦，两者可独立升级，互不影响。
📨
异步事件
Host 异步接收来自 Controller 的 HCI 事件通知，无论使用哪种传输层。

§ 3
命令与事件概览
Overview of Commands and Events

HCI 命令和事件在 Host 与 Controller 之间传递。规范通过一张总表（按字母顺序）列出所有 HCI 命令和事件，包含：规范版本号、功能摘要、BR/EDR 支持要求、LE 支持要求。

3.1 命令版本机制

一条命令或事件可能有多个版本（v1、v2 等）。各版本实现相同的基础功能，但在细节上有差异（如附加参数以支持新特性）。版本信息用 [v1]、[v2] 等标注；未注明版本号时，引用适用于所有版本。

3.2 支持要求标记说明
M = Mandatory（强制支持）
O = Optional（可选支持）
C = Conditional（条件支持）
E = Excluded（不适用）

Controller 对某命令的支持要求取决于其实现类型（BR/EDR / LE / 双模）。满足至少一种传输类型的强制要求时，Controller 必须支持该命令。

§ 4
HCI 流控制
HCI Flow Control — 防止缓冲区溢出的双向机制

流控制确保 Host 发送的数据不超过 Controller 的缓冲能力，同时也允许 Controller 向 Host 反馈已完成的数据包数量。规范定义了两个方向的流控制机制，以及共享/独立缓冲区的处理方式。

4.1 Host → Controller 流控制

Host 通过 HCI_Read_Buffer_Size 命令读取 Controller 的缓冲区大小，然后跟踪已发送但尚未被 Controller 确认的数据包数量。Controller 通过 HCI_Number_Of_Completed_Packets 事件通知 Host 已完成处理的数据包数，Host 据此更新计数器，决定何时可以继续发送。

图 4-1: Host → Controller 流控制机制 — 基于缓冲区大小和已完成包数量
4.2 Controller → Host 流控制

默认情况下，Controller → Host 方向的流控制是关闭的。当 Host 需要启用时，通过 HCI_Set_Controller_To_Host_Flow_Control 命令激活，并使用 HCI_Host_Buffer_Size 命令告知 Controller 自身的缓冲区大小，此后 Host 通过 HCI_Host_Number_Of_Completed_Packets 命令向 Controller 报告消费情况。

4.3 BR/EDR 与 LE 缓冲区配置
📊
共享缓冲区模式
BR/EDR 和 LE 共用同一套 ACL 数据缓冲区。通过 HCI_Read_Buffer_Size 读取统一的缓冲区参数，流控计数器统一管理。
📦
独立缓冲区模式
BR/EDR 和 LE 各有独立缓冲区。LE 缓冲区通过 HCI_LE_Read_Buffer_Size 单独读取，各自独立计数。若 LE 缓冲区大小为 0，则复用 BR/EDR 缓冲区。
ℹ️ Controller 的具体缓冲区配置方式通过 HCI 命令在运行时动态确定，Host 不得假设默认配置。
§ 5
HCI 数据格式
HCI Data Formats — 各类数据包的精确二进制结构

本节定义 HCI 在传输层上交换的所有数据包格式。共有四种基本包类型：命令包（Command Packet）、事件包（Event Packet）、ACL 数据包、同步数据包（SCO/eSCO），以及 Bluetooth 5.2 引入的 ISO 数据包。

5.1 HCI 命令包（Command Packet）

命令包由 Host 发送给 Controller。每个命令包包含一个 OpCode（操作码，由 OGF 和 OCF 组成）、参数总长度字段，以及可变长度的命令参数。

5.2 HCI 事件包（Event Packet）

事件包由 Controller 异步发送给 Host，用于通知各种状态变化和命令完成情况。包含事件码（Event Code）、参数总长度和事件参数。

用于传输 L2CAP 数据等异步数据流。包含连接句柄（Connection Handle）、包边界标志（PB Flag）、广播标志（BC Flag）和数据长度。

§ 6
HCI 配置参数
HCI Configuration Parameters — 多命令共用的通用参数说明

本节进一步说明多个命令共用的参数定义，避免在每条命令中重复说明。主要包括：扫描参数（扫描间隔与窗口）、连接参数（连接间隔、延迟、超时）、发送功率等。

6.1 扫描参数（Scan Parameters）

扫描间隔（Scan_Interval）和扫描窗口（Scan_Window）共同决定了设备发现的效率与功耗之间的平衡。

图 6-1: 扫描间隔与扫描窗口的时序关系（扫描窗口不得超过扫描间隔）
6.2 连接参数（Connection Parameters）

LE 连接的核心参数，决定连接的能耗、延迟和可靠性：

参数名范围单位作用说明
Connection_Interval 6 – 3200 × 1.25 ms（7.5ms – 4s） 两次连接事件之间的时间间隔，影响延迟与功耗
Peripheral_Latency 0 – 499 连接事件数 外设（Peripheral）可以跳过的最大连接事件数（省电）
Supervision_Timeout 10 – 3200 × 10 ms（100ms – 32s） 超过此时间未收到数据包则认为连接已断开
CE_Length 0 – 0xFFFF × 0.625 ms 连接事件的期望长度（仅供 Controller 参考调度）
6.3 TX 功率配置

通过 HCI_LE_Read_Transmit_Power 命令可读取 Controller 支持的最小/最大 TX 功率（范围 -127 dBm 至 +20 dBm）。通过 RF 路径补偿参数（RF_TX_Path_Compensation / RF_RX_Path_Compensation）可修正天线/走线带来的增益或损耗，精度为 0.1 dB。

7.1 链路控制命令（Link Control Commands · OGF 0x01）

控制蓝牙连接的建立、接受、拒绝和断开，以及设备发现（Inquiry）、认证和加密等操作。这些命令主要针对 BR/EDR，是最基础的连接管理命令集。

🔍
HCI_Inquiry
开始查询周边 BR/EDR 设备，获取其地址和时钟偏移。
🔗
HCI_Create_Connection
向指定 BD_ADDR 设备发起 BR/EDR 连接请求。
✅
HCI_Accept_Connection_Request
接受来自远程设备的入向连接请求，可指定角色（Central/Peripheral）。
❌
HCI_Reject_Connection_Request
拒绝入向连接请求，需提供拒绝原因错误码。
🔒
HCI_Authentication_Requested
请求对指定连接句柄建立双向认证。
🔐
HCI_Set_Connection_Encryption
启用或禁用指定连接的链路层加密。
图 7-1: BR/EDR 连接建立的 HCI 命令/事件时序
7.2 链路策略命令（Link Policy Commands · OGF 0x02）

控制链路管理器（LM）的行为策略，主要用于 BR/EDR 的省电模式管理和服务质量（QoS）设置。

💤
Sniff Mode
使设备进入 Sniff 低功耗模式，仅在特定时隙醒来接收数据，大幅降低功耗。
🎯
QoS Setup
为连接设置服务质量参数（带宽、延迟、抖动），确保关键数据流的传输保障。
🔄
Role Switch
在 Central 和 Peripheral 角色之间切换，改变连接的调度主控方。
⚙️
Write Link Policy Settings
配置允许链路管理器使用哪些低功耗模式（Hold / Sniff / Park）。
7.3 控制器与基带命令（Controller & Baseband Commands · OGF 0x03）

数量最多的命令组（约 66 条），用于读写 Controller 的各种配置参数，包括事件过滤、本地名称、扫描使能、安全模式等。适用于 BR/EDR 和 LE 两种控制器。

重要命令功能说明支持类型
HCI_Reset 软复位控制器，清除所有连接、事件过滤器、配置参数 BR/EDR + LE
HCI_Set_Event_Mask 设置哪些事件需要上报给 Host（位图掩码） BR/EDR + LE
HCI_Write_Local_Name 写入本地设备名（UTF-8，最长 248 字节） BR/EDR
HCI_Write_Scan_Enable 控制 Inquiry Scan 和 Page Scan 的开关 BR/EDR
HCI_Write_Class_Of_Device 设置设备类型（CoD），决定设备图标和主要用途 BR/EDR
HCI_Write_Default_Link_Policy 为所有新建连接设置默认链路策略 BR/EDR
HCI_Read_Authenticated_Payload_Timeout 读取已认证负载超时阈值（用于防 MitM 检测） BR/EDR + LE
7.4 信息参数命令（Informational Parameters Commands · OGF 0x04）

只读命令组，用于查询 Controller 的固有属性和能力。这些参数由 Controller 固件决定，Host 无法修改。

🏷️
Read_Local_Version_Information
返回 HCI 版本、修订号、LMP/LL 版本及制造商 ID。
📍
Read_BD_ADDR
读取本地设备的蓝牙地址（6 字节，全球唯一）。
🛠️
Read_Local_Supported_Features
读取本地 LMP 特性位图，了解硬件支持的能力集合。
💾
Read_Buffer_Size
读取 Controller ACL/SCO 缓冲区大小和数量，用于流控初始化。
7.5 状态参数命令（Status Parameters Commands · OGF 0x05）

用于在运行时读取 Controller 和连接的动态状态信息，如信号强度、链路质量、时钟等。

📶
HCI_Read_RSSI
读取指定连接的接收信号强度（RSSI），单位 dBm，用于链路质量评估和测距。
🔗
HCI_Read_Link_Quality
读取链路质量指标（0–255），数值越大表示链路质量越好。
⏱️
HCI_Read_Clock
读取本地或连接的蓝牙时钟值（312.5μs 精度），用于时间同步和测距。
📊
HCI_Read_AFH_Channel_Map
读取当前自适应跳频（AFH）使用的信道图，可了解哪些 2.4GHz 信道被回避。
7.6 测试命令（Testing Commands · OGF 0x06）

用于生产测试和一致性测试（RF 认证）。这些命令在正常使用中不会调用。

⚠️ 测试命令会使 Controller 进入特殊测试模式（如 DUT 模式、直接 TX/RX 测试），此时正常的蓝牙功能将被暂停。生产环境中禁止调用这些命令。
🧪
Enable_Device_Under_Test_Mode
使 Controller 进入 DUT 测试模式，配合测试设备进行 RF 一致性测试。
📡
LE_Transmitter/Receiver_Test
LE 直接 RF 测试命令，在指定信道持续发送或接收测试数据包。
7.7 HCI 事件（Events）

Controller 通过事件异步通知 Host 各种状态变化和操作结果。每个事件有唯一的 Event Code，并携带相应的参数。重要事件一览：

事件名称Event Code触发条件关键参数
HCI_Command_Complete 0x0E 命令执行完毕（同步返回结果） Num_HCI_Commands, Command_Opcode, Return_Parameters
HCI_Command_Status 0x0F 命令已被接受，但执行异步进行 Status, Num_HCI_Commands, Command_Opcode
HCI_Connection_Complete 0x03 BR/EDR 连接建立完成（成功或失败） Status, Connection_Handle, BD_ADDR, Link_Type, Encryption
HCI_Disconnection_Complete 0x05 连接断开完成 Status, Connection_Handle, Reason
HCI_LE_Meta event 0x3E 所有 LE 子事件的统一入口（通过 Subevent_Code 区分） Subevent_Code, [各子事件参数]
HCI_Number_Of_Completed_Packets 0x13 Controller 完成数据包发送时通知 Host Num_Handles, [Handle, Num_Completed_Packets]×N
HCI_Hardware_Error 0x10 Controller 硬件故障 Hardware_Code

总结
HCI 规范要点速览
🏗️
§1–§2 架构与传输
定义 HCI 的角色（Upper/Lower HCI）、软件栈层次、传输层透明性原则和跨版本兼容规则。
📋
§3 命令/事件总表
完整列出所有 HCI 命令和事件（200+ 条），含版本信息、功能摘要、BR/EDR 和 LE 的支持要求（M/O/C/E）。
🌊
§4 流控制
双向流控机制：Host 通过 Num_Completed_Packets 事件跟踪 Controller 缓冲使用情况，支持共享或独立缓冲区模式。
📦
§5 数据格式
精确定义 5 种 HCI 数据包的二进制格式：命令包(0x01)、ACL(0x02)、SCO(0x03)、事件(0x04)、ISO(0x05)。
⚙️
§6 配置参数
扫描间隔/窗口、连接间隔/延迟/超时、TX 功率范围及 RF 路径补偿等多命令通用参数的规范化定义。
📖
§7 命令详解
8 个命令子组（7.1–7.8），LE 命令组（7.8）随蓝牙版本从 4.0 演进至 6.0，新增 ISO/Audio/测向/信道探测等能力。
📌
