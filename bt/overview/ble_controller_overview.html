Part A

物理层规范 Physical Layer Specification

▾

1 概述 (General)

物理层（PHY）是 BLE 协议栈的最底层，规定了无线信号的调制方式、频率范围、发射功率及时序参数，是所有上层通信的物理基础。BLE 工作在 2.4 GHz ISM 频段（2400–2483.5 MHz），采用 FHSS 跳频技术规避干扰。

ℹ️
BLE 物理层与经典蓝牙（BR/EDR）完全独立设计，二者互不兼容，但可共存于同一芯片。
BLE 协议栈层次结构

2 RF 信道分配 (Radio Frequency Channels)

BLE 将 2.4 GHz 频段划分为 40 个信道，编号 0–39，间隔 2 MHz。其中 3 个为广播信道（Advertising），37 个为数据信道（Data）。

40 个 BLE 信道分布图（橙色=广播信道，蓝色=数据信道）
信道类型数量编号中心频率用途
广播信道337, 38, 392402 / 2426 / 2480 MHz设备发现、广播、连接建立
数据信道370–362404–2478 MHz（偶数）已建立连接后的数据传输
💡
广播信道选在 2402/2426/2480 MHz 是为了避开 Wi-Fi 常用的 1、6、11 信道（2412/2437/2462 MHz），最大限度减少干扰。

3 调制与编码 (Modulation & Coding Scheme)

BLE 使用 GFSK（高斯频移键控）调制，对比特 0/1 分别对应不同的频偏。Core 5.0 引入了 LE 2M PHY 和 Coded PHY，大幅提升速率或距离。

4 发射功率 (Transmitter Power)

规范定义了最小发射功率 -20 dBm 以上，最大不超过 +20 dBm（100 mW）。实际实现通常支持 -20 dBm 到 +10 dBm 的功率分级，可通过 LE Power Control（BT 5.2+）动态调整。

⚠️
各国监管机构对最大 EIRP 有严格限制（欧盟 ETSI：+20 dBm，美国 FCC：+30 dBm），产品需取两者较低值。

5 时序参数 (Timing Parameters)

BLE 规范严格定义了收发切换时间、IFS 帧间间隔等时序，保证多设备共存时的有序工作。

Part B

链路层规范 Link Layer Specification

▾

1 概述 (General)

链路层（LL）是 BLE 控制器的核心，负责定义空中接口的数据包格式、状态机、流控机制、跳频算法、加密等。链路层运行在控制器端，向上通过 HCI 接口与主机通信。

BT 4.0 基础 广播 / 扫描 / 连接 BT 5.0 扩展广播 长程 Coded PHY BT 5.2 等时信道 CIS/BIS BT 6.0 Channel Sounding DBAF 决策广播过滤

2 链路层状态机 (LL State Machine)

链路层维护一个状态机，设备在各状态间根据事件（HCI命令/空中包）进行转换。

⏸️Standby

3 数据包格式 (Packet Format)

所有 BLE 空中数据包均遵循统一结构：前导码 → 接入地址 → PDU → CRC，Coded PHY 还额外增加 CI 与 TERM 字段。

Preamble 1–2 Bytes
Access Address 4 Bytes
PDU (Protocol Data Unit) 2–258 Bytes
CRC 3 Bytes
PDU 内部结构（数据信道）
LLID 2 bit
NESN 1 bit
SN 1 bit

Length 8 bit
Payload 0–251 Bytes
MIC (可选) 4 Bytes
字段长度说明
Preamble（前导码）1 Byte (1M) / 2 Bytes (2M)用于接收端同步时钟与频偏校正
Access Address（接入地址）4 Bytes广播固定值 0x8E89BED6；连接使用随机生成的唯一值
PDU Header2 Bytes包含 LLID、SN/NESN、Length 等控制字段
Payload（有效载荷）0–251 Bytes实际承载数据或控制 PDU

ℹ️
LLID 含义：00=保留，01=续包（LL Data PDU continuation），10=首包/完整包，11=LL Control PDU

4 广播 (Advertising)

广播是 BLE 发现机制的核心。Advertiser 按固定间隔在 37/38/39 信道依次发送广播包；Scanner 在其中一个或多个信道监听。BT 5.0 引入扩展广播，支持更大载荷（255 字节）和更灵活的广播配置。

5 连接建立与管理 (Connection Establishment)

当 Initiator 发现目标 Advertiser 后，向其发送 CONNECT_IND 包（包含连接参数），双方随即切换到数据信道，采用 FHSS 跳频通信，Initiator 成为 Central，Advertiser 成为 Peripheral。

连接建立三阶段
🔑
CONNECT_IND 包含的关键参数：AccessAddress（连接标识）、CRCInit（CRC初值）、Interval（连接间隔）、Latency（从端延迟）、Timeout（超时时间）、ChM（信道映射）、Hop（跳频增量）

6 跳频算法 (Channel Selection Algorithm)

BLE 定义了两种信道选择算法（CSA），CSA#1（BT 4.0）和 CSA#2（BT 5.0）。CSA#2 使用更均匀的伪随机算法，在信道条件差时抗干扰能力更强。

CSA#1 跳频示意（hop = 7, 初始 ch = 0）

7 流控与 ARQ (Flow Control & ARQ)

链路层使用 Stop-and-Wait ARQ机制，每个数据包携带序列号（SN）和期望序列号（NESN）。接收方通过设置 NESN 来表示期望的下一个序列号，实现可靠传输。

SN/NESN 停等ARQ 流程

8 链路层控制过程 (LL Control Procedures)

链路层定义了一系列控制 PDU（LL Control PDU），用于协商连接参数、加密、版本信息交换、终止连接等管理操作，不依赖 L2CAP 层直接在 LL 层执行。

控制过程相关 PDU说明
连接参数更新LL_CONNECTION_UPDATE_IND更新 Interval / Latency / Timeout
信道映射更新LL_CHANNEL_MAP_IND修改可用数据信道集合
加密启动LL_ENC_REQ / RSP / START_ENC协商会话密钥，启动 AES-CCM 加密
特性交换LL_FEATURE_REQ / RSP双方声明支持的 LL 功能集
版本信息LL_VERSION_IND交换控制器版本号
数据长度更新LL_LENGTH_REQ / RSP协商最大 PDU 长度（最大 251 字节）
PHY 更新LL_PHY_REQ / RSP / UPDATE_IND切换至 2M 或 Coded PHY
连接终止LL_TERMINATE_IND通知对端断开连接及原因码
功率控制LL_POWER_CONTROL_REQ / RSP动态调整发射功率（BT 5.2+）
Channel SoundingLL_CS_REQ / RSP / IND测距过程控制（BT 6.0+）

9 等时信道 (Isochronous Channels, BT 5.2+)

等时信道（ISO）专为音频等低延迟实时流媒体设计，分为两种：CIS（Connected Isochronous Stream，点对点连接内等时流）和 BIS（Broadcast Isochronous Stream，广播等时流）。

CIS vs BIS 架构对比
🎵
BIS 是 LE Audio（LC3 编解码器）的基础，支持超低延迟同步音频广播，一个发送端可同时对无限个接收端提供同步音频流。

10 信道测音/测距 (Channel Sounding, BT 6.0)

Channel Sounding（CS）是蓝牙 6.0 的核心新特性，提供厘米级测距能力，支持两种测量方法：PBR（相位法测距）和 RTT（往返时间法）。

Channel Sounding 测距原理
方法原理精度适用场景
RTT（往返时间）测量信号往返时间，d = t·c/2~1m门禁、汽车钥匙
PBR（相位法）多频点相位差计算距离~10cm精密资产追踪
RTT+PBR 联合两种方法融合厘米级UWB替代方案

Part E

低功耗安全规范 Low Energy Security Specification

▾

1 概述 (General)

Part E 规定了 BLE 链路层安全的密码学基础，包括密钥生成、加密算法、认证标签生成和计数器管理，为链路层加密（LL 加密）提供底层支撑。所有加密操作基于 AES-128 算法，确保符合 FIPS-197 标准。

🔑
BLE 安全分为两层：链路层加密（LL）由 Part E 定义（机密性+完整性），配对与密钥分发（SMP）由 Vol 3 Part H 定义（LE Legacy / LE Secure Connections 两种配对模式）。

2 安全架构 (Security Architecture)

BLE 安全架构由以下要素组成，每个连接建立独立的加密会话：

链路层加密密钥体系
密钥/参数长度来源说明
LTK (Long Term Key)128 bitSMP 配对生成主密钥，用于派生会话密钥
EDIV (Encrypted Diversifier)16 bitSMP Legacy用于标识 LTK（Legacy Pairing）
Rand64 bitSMP Legacy与 EDIV 一起用于查找 LTK
SKD (Session Key Diversifier)128 bitLL 加密握手SKD_C||SKD_P，各64位由两端分别生成
SK (Session Key)128 bitAES-128(LTK, SKD)实际用于 AES-CCM 的会话密钥
IV (Init Vector)64 bitLL 加密握手IV_C||IV_P，各32位，用于构造 Nonce
Nonce128 bitIV + PacketCounter + DirAES-CCM 输入，每包唯一

3 AES-CCM 加密算法 (AES-CCM Encryption)

BLE 链路层使用 AES-128-CCM（Counter with CBC-MAC）模式对数据包 Payload 进行加密和完整性保护。每个数据包携带 4 字节 MIC（Message Integrity Code）用于验证数据完整性。

AES-CCM 加密/解密流程
⚠️
MIC 校验失败时，接收方必须立即丢弃该数据包并上报 MIC_Failure 错误，控制器随后执行连接终止（发送 LL_TERMINATE_IND，ErrorCode=0x06）。

4 包计数器与 Nonce 构造 (Packet Counter & Nonce)

为防止重放攻击，每个方向的发送包维护独立的 39 位包计数器，计数器单调递增。Nonce 由 IV、包计数器和方向位构成，保证每个包的加密上下文唯一。

Nonce 构造 (128-bit = IV[63:0] || PacketCounter[38:0] || Dir[0])
IV (64 bit) IV_C[31:0] || IV_P[31:0]
PacketCounter (39 bit) 单调递增，不可回绕
Dir (1 bit) 0=M→S 1=S→M
💡
包计数器为 39 位，最大值约 5490 亿，在合理使用寿命内不会溢出，规范要求计数器溢出时必须终止连接。

6 配对模式与安全等级 (Pairing & Security Levels)

BLE 定义两大配对模式，安全等级从低到高分为四级：

LE Secure Connections (LESC) 配对流程
安全等级描述条件
Level 1无安全要求不加密，不认证
Level 2未认证加密Just Works 配对，加密但无 MITM 保护
Level 3已认证加密Passkey / OOB 配对，有 MITM 保护
Level 4已认证 LE SC 加密LE Secure Connections + FIPS 兼容

7 链路层隐私 (LL Privacy)

为防止第三方通过设备地址追踪 BLE 设备，BLE 4.2 引入链路层隐私（LL Privacy），使用 可解析私有地址（RPA）定期更换设备地址，只有持有 IRK 的对端才能解析真实身份。

RPA 生成与解析
💡
RPA 的刷新周期（RPATimeout）默认为 15 分钟，可由 Host 配置。刷新时 prand 重新随机生成，使地址不可关联，保护用户隐私。
