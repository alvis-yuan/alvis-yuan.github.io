入门

Yazi — 极速终端文件管理器

Yazi（发音 /jɑːzi/，"鸭子"之意）是一款用 Rust 编写、基于异步 I/O 的终端文件管理器，主打极速预览与 Vim 式键盘操控。

📎 官方文档原文 本页所有快捷键均来自官方文档：https://yazi-rs.github.io/docs/quick-start/

整体架构

启动方式

# 直接启动
yazi

# 使用 Shell 包装器（推荐），退出时同步工作目录
y

退出时按 q 将同步当前目录到 Shell；按 Q 退出但不同步目录。 按 F1 或 ~ 打开内置帮助菜单。

📁 文件操作
复制、剪切、粘贴、删除、重命名、创建、符号链接

🗂️ 多标签
同一窗口管理多个独立工作目录，Tab 自由切换

第一性原理：文件管理的本质

文件管理器的本质是目录树的可视化遍历 + 批量文件操作的快捷执行。 Yazi 的设计哲学是：把所有高频操作映射到单个键（如 Vim），把低频复杂操作留给命令行（;/:）， 异步 I/O 确保预览和操作互不阻塞，从而在"认知负担最低"与"操作效率最高"之间取得平衡。

第一性原理：导航的本质

目录导航的本质是在树形文件系统中移动指针。hjkl 来自 Vim，将"最高频操作"绑定到"手指最少移动距离"的键位上。 三栏布局（父/当/预览）则是把上下文（context）始终可见——你总知道自己在哪里、来自哪里、将去哪里，减少了认知负担。 快速跳转（z/Z）则是利用历史频率统计代替线性搜索，体现了"以概率换速度"的思想。

选择

文件选择

选择文件/目录是批量操作的前提。Yazi 提供多种选择模式，支持单选、范围选、全选和反选。

选择快捷键

快捷键动作使用场景
Space切换当前悬停文件/目录的选中状态逐个挑选不连续的文件
v进入可视模式（范围选择）选择连续范围内的文件
V进入可视模式（取消选择模式）对已选区域批量取消选中
Ctrl+a全选当前目录所有文件对全部文件执行批量操作前
Ctrl+r反转当前选中状态选大多数时先全选再反选少数
Esc取消选择放弃当前选择，重新来过

选择模式状态机

典型操作流程

选择多个不连续文件
在每个目标文件上按 Space 切换选中，再执行 y/x/d
选择连续范围
移动到起始位置按 v，再用 j/k 扩展范围，按 Esc 确认
选全部，再排除少数
先 Ctrl+a 全选，再逐个 Space 取消不需要的，或 Ctrl+r 反选
取消所有选中
按 Esc 清空所有选中状态，重新开始
第一性原理：选择的本质

文件选择的本质是构建操作目标集合。单选（Space）适合"枚举法"，可视模式（v）适合"范围法"，全选+反选（Ctrl+a + Ctrl+r）是"补集法"——三种方法覆盖了集合构建的全部场景。 Yazi 把 Vim 的 Visual Mode 概念引入文件管理，让用户用已有的肌肉记忆完成新任务。

文件操作

文件增删改

Yazi 提供完整的文件生命周期管理，包括打开、复制/剪切/粘贴、创建、重命名、删除及符号链接。

打开文件

快捷键动作使用场景
o / Enter用默认程序打开选中文件快速打开文件
O / Shift+Enter交互式选择打开程序需要指定特定编辑器/播放器时
Tab显示文件详细信息查看文件大小、权限、修改时间

复制 / 剪切 / 粘贴

快捷键动作说明
yYank（复制）选中文件文件标记为"复制"，不会立即移动
xYank（剪切）选中文件文件标记为"剪切"，粘贴后原始位置文件消失
p粘贴已 yank 的文件粘贴到当前目录，目标存在时跳过
P粘贴并覆盖目标文件存在时直接覆盖
Y 或 X取消 yank 状态清空剪贴板标记

创建 / 重命名 / 删除

快捷键动作使用场景与说明
a创建文件或目录以 / 结尾则创建目录，否则创建文件。支持嵌套路径，如 a/b/c.txt
r重命名选中文件支持批量重命名多个文件
d将选中文件移入回收站安全删除，可在系统回收站恢复
D永久删除选中文件⚠️ 不可恢复，谨慎使用
.切换隐藏文件显示显示/隐藏以 . 开头的文件

符号链接

快捷键动作说明
-对 yanked 文件创建绝对路径符号链接链接目标是绝对路径，移动后链接仍有效
_对 yanked 文件创建相对路径符号链接链接目标是相对路径，移动后可能失效
Ctrl+-对 yanked 文件创建硬链接硬链接，共享 inode，不随原文件删除而失效
⚠️ 注意 创建符号链接/硬链接前，需先用 y 对源文件执行 yank（复制）操作，再导航到目标目录执行链接命令。
第一性原理：文件操作的本质

文件操作的本质是对文件系统 inode 的增删改。Yazi 借鉴了编辑器的"Yank-Put"模型（来自 Vim）：先"标记意图"（y/x），再"执行意图"（p）。这种两步操作将"选择"与"执行"解耦，避免了误操作。d（回收站）与 D（永久删除）的区分则体现了"可逆性优先"原则——默认行为应是可撤销的。

路径

复制路径

快速将文件路径、目录路径或文件名复制到系统剪贴板，便于在终端或编辑器中粘贴使用。

📌 说明 c ⇒ d 表示先按 c 键，再按 d 键（序列键，不是同时按）。

复制路径快捷键

快捷键动作示例输出
c c复制完整文件路径/home/user/Projects/main.rs
c d复制所在目录路径/home/user/Projects
c f复制文件名（含扩展名）main.rs
c n复制文件名（不含扩展名）main

使用场景

在终端引用文件
用 cc 复制路径，切换到终端后粘贴，避免手动输入长路径
构建脚本路径
用 cd 复制目录路径，快速 cd 到目标目录
批量重命名参考
用 cf 或 cn 复制文件名，粘贴到重命名输入框
写文档引用
复制路径后粘贴到 Markdown/代码注释中，精确引用文件位置
第一性原理：路径复制的本质

路径复制解决的是上下文传递问题：文件管理器掌握精确的文件系统路径，而其他工具（终端、编辑器）需要这个路径作为输入。 通过键盘快捷键将路径放入系统剪贴板，实现了"零错误率"的路径传递（相比手动输入），这是工具间协作的最优解。

搜索

查找、搜索与过滤

Yazi 提供三种不同层次的定位机制：即时查找（find）、深度搜索（search）、实时过滤（filter），各有适用场景。

三种机制对比

查找文件（Find）

快捷键动作使用场景
/向后查找下一个匹配文件在当前目录列表中快速定位
?向前查找上一个匹配文件反向查找
n跳转到下一个匹配项连续浏览查找结果
N跳转到上一个匹配项反向浏览查找结果

搜索文件（Search）

快捷键动作说明
s用 fd 按文件名搜索递归搜索当前目录及子目录，需安装 fd
S用 ripgrep 按文件内容搜索搜索文件内容，需安装 ripgrep
Ctrl+s取消正在进行的搜索中止长时间运行的搜索任务

过滤文件（Filter）

快捷键动作使用场景
f输入关键词过滤当前目录文件大目录中快速缩小可见文件范围
💡 最佳实践 日常使用建议：当前目录定位用 /，跨目录找文件用 s（fd），全文搜索用 S（ripgrep），大目录整理用 f 过滤。
第一性原理：搜索的本质

搜索是在信息空间中定位目标的行为。三种机制的本质差异是搜索空间的大小与匹配维度： Find 在当前列表（一维线性）中匹配名称；Filter 在当前目录（一维线性）中过滤可见性；Search 在目录树（树形结构）中递归匹配名称或内容。 异步执行保证 UI 不阻塞，体现了"搜索不应影响浏览"的设计原则。

排序

排序

Yazi 支持多维度文件排序，使用 , 前缀键触发，序列键选择排序方式。

📌 说明 , ⇒ a 表示先按 , 键，再按 a 键（序列键）。大写字母为对应排序的反向版本。

,n按自然顺序排序（file2 < file10）,N（反向）
,m按修改时间排序（最旧在前）,M（最新在前）

排序维度可视化

使用场景

找最近修改的文件
用 ,M（修改时间倒序），最新文件排在最顶部
找最大文件清理空间
用 ,S（文件大小倒序），最大文件一目了然

处理编号文件
用 ,n（自然排序），file2 < file10，符合直觉
第一性原理：排序的本质

排序的本质是将信息重新组织以降低认知搜索成本。不同的排序维度服务于不同的"查询意图"——按时间找最新，按大小找最重，按扩展名聚类。 自然排序（Natural Sort）体现了"以人类认知习惯为标准"而非"以 ASCII 码顺序为标准"的设计哲学。

标签页

多标签页

Yazi 支持在单个实例内打开多个独立标签页，每个标签维护独立的工作目录、选中状态和预览状态。

标签页操作

快捷键动作使用场景
tt在当前工作目录创建新标签页需要同时查看多个目录时
1~9切换到第 N 个标签页快速跳转到指定标签
[切换到前一个标签页在相邻标签间循环切换
]切换到后一个标签页在相邻标签间循环切换
{将当前标签页与前一个交换位置调整标签页排列顺序
}将当前标签页与后一个交换位置调整标签页排列顺序

多标签布局示意

💡 最佳实践 在复制文件时，可开启两个标签页：一个停留在源目录，一个停留在目标目录，使用 y 复制后切换标签再 p 粘贴，无需来回导航。
第一性原理：多标签的本质

多标签的本质是并发上下文管理：人类大脑可以在多个任务上下文间切换，多标签为每个上下文保留了完整状态（目录、选中、预览）， 避免了"切换后重新导航"的成本。本质上是把"临时工作记忆"外包给程序来维护，减轻认知负担。

Shell & 任务

Shell 命令与任务管理

Yazi 可以直接在文件管理器内运行 Shell 命令，并有内置任务管理器监控异步任务的进度。

Shell 命令

快捷键动作使用场景
;运行 Shell 命令（非阻塞）执行后台命令，Yazi 继续保持响应
:运行 Shell 命令（阻塞直到完成）需要等待命令完成后继续的场景

阻塞 vs 非阻塞

退出与帮助

快捷键动作说明
q退出 Yazi（写入 CWD）使用 y 包装器时，Shell 目录同步到当前目录
Q退出 Yazi（不写入 CWD）不改变 Shell 的工作目录
F1 或 ~打开内置帮助菜单查看所有快捷键的实时帮助

Shell 包装器配置

在 Bash/Zsh 配置文件中添加以下函数，使用 y 启动 Yazi，退出时自动同步工作目录：

# Bash / Zsh (~/.bashrc 或 ~/.zshrc)
function y() {
    local tmp="$(mktemp -t "yazi-cwd.XXXXXX")" cwd
    command yazi "$@" --cwd-file="$tmp"
    IFS= read -r -d '' cwd < "$tmp"
    [ "$cwd" != "$PWD" ] && [ -d "$cwd" ] && builtin cd -- "$cwd"
    command rm -f -- "$tmp"
}
⚠️ 注意 Shell 包装器 y 是官方推荐的使用方式。直接运行 yazi 退出后不会改变终端的工作目录，而 y 包装器解决了这个限制。

完整快捷键速查卡

第一性原理：Shell 集成的本质

文件管理器与 Shell 的本质关系是图形导航与文本命令的互补：文件管理器擅长"看"（可视化目录树），Shell 擅长"做"（任意复杂操作）。 Yazi 通过 ;/: 让两者无缝衔接，用 Shell 包装器（y函数）解决"退出后目录丢失"的痛点——这是两个工具边界的最优融合点。 任务管理器则把异步操作的"黑盒"变成"透明盒"，让用户随时知道后台发生了什么。
