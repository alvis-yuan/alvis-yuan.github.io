01 · 入门基础

什么是 tmux

tmux（terminal multiplexer，终端复用器）是一个开源命令行工具，允许在单个屏幕内创建、访问和控制多个终端会话。 tmux 可以从屏幕上分离（detach）并在后台继续运行，之后再重新挂载（attach）。 —— 来源：tmux(1) man page

tmux 采用 服务端-客户端架构：启动时创建一个后台 server 进程管理所有 session，终端窗口作为 client 连接到 server。 Server 独立于任何 client 运行，因此关闭终端或 SSH 断线后，session 仍持续存在。

安装

# Ubuntu / Debian sudo apt update && sudo apt install tmux # CentOS / RHEL / Fedora sudo dnf install tmux # Arch Linux sudo pacman -S tmux # macOS (Homebrew) brew install tmux # 查看版本 tmux -V

三层层级结构

📦
Session（会话）
一组窗口的集合，持久存在于后台，可 detach / attach，SSH 断线不丢失。
🪟
Window（窗口）
占据整个屏幕的独立工作区，类似浏览器 Tab，可分割为多个 Pane。
▦
Pane（面板）
窗口中的矩形区域，每个 Pane 运行独立的 shell 进程（伪终端）。
第一性原理
终端的本质是 伪终端（pty） 上的 I/O 流。传统 SSH 会话中，连接断开意味着 pty 销毁，进程终止。 tmux 的根本洞见是：将 pty 的生命周期与网络连接解耦——Server 进程持有 pty，Client 只是显示层。 断线只是 Client 消失，Server 和 pty 继续存活，重连即恢复。这是持久化远程工作的第一性原理解法。
⌨️
02 · 入门基础

前缀键与命令提示

tmux 通过 前缀键（Prefix Key） 区分自身快捷键与普通终端输入。 默认前缀为 Ctrl+b。 按下并释放前缀键，再按命令键，tmux 截获并执行对应命令。 —— 来源：tmux(1) DEFAULT KEY BINDINGS

💡 修改前缀键
很多用户将前缀改为 Ctrl+a（GNU Screen 风格），因为更易按到。 在 ~/.tmux.conf 中添加：
unbind C-b set -g prefix C-a bind C-a send-prefix

命令提示符

按 Ctrl-b : 进入 tmux 命令提示符，可以直接输入任何 tmux 命令——无需退出当前程序（如 vim）。

三种执行方式对比

方式示例适用场景
Shell 命令行 $ tmux new-session -s work 外部启动 脚本、别名
快捷键（Prefix + key） Ctrl-b c 日常操作 最快捷
命令提示符 Ctrl-b : 然后输入命令 高级操作 不常用命令
第一性原理
前缀键的本质是一个 命名空间隔离机制：终端输入是一个共享信道，tmux 需要与运行在其中的程序共存。 前缀键就像一个"模式切换位"——平时 0，按下前缀后短暂变为 1，tmux 在这个窗口内截获下一个按键。 这与 Vim 的模式切换、Emacs 的 C-x 前缀是同一设计哲学：用有限按键空间实现无限命令集。

03 · 核心概念

会话管理 Session

Session 是 tmux 管理的一组伪终端的集合，每个 session 包含一或多个 window。 Session 持久存在，能在 SSH 断线或主动 detach 后继续运行。 —— 来源：tmux(1) CLIENTS AND SESSIONS

Shell 命令

# 创建新 session（匿名） tmux # 创建命名 session tmux new-session -s work tmux new -s work # 简写 # 创建后台 session（不立即进入） tmux new -s build -d # 列出所有 session tmux list-sessions tmux ls # 挂载到最近一个 session tmux attach tmux a # 挂载到指定 session tmux attach -t work tmux a -t work # 重命名 session tmux rename-session -t old-name new-name # 销毁指定 session tmux kill-session -t work # 销毁 tmux 服务端（所有 session） tmux kill-server

会话快捷键（Prefix + key）

快捷键功能使用场景
Ctrl-b dDetach 当前 client（session 继续运行）安全离开
Ctrl-b D选择要 detach 的 client多人共享时
Ctrl-b $重命名当前 session整理命名
Ctrl-b s交互式列表，选择切换到其他 session多项目切换
Ctrl-b (切换到上一个 session快速轮换
Ctrl-b )切换到下一个 session快速轮换
Ctrl-b L切换回上一次使用的 session在两个项目间跳转
📌 Session 的生命周期
Session 在以下情况下终止：① 所有 window 关闭；② 执行 kill-session；③ 执行 kill-server。 detach 不会终止 session，仅断开当前 client 的显示连接。
第一性原理
Session 是 tmux 对 "工作上下文" 的最高层抽象。 人类工作的本质是在多个不同任务（项目A、项目B、系统监控）之间切换，每次切换都有认知加载成本。 Session 将一个任务的所有终端状态（多个 Window、Pane、正在运行的进程）封装为一个单元， 切换 session 就是切换完整工作上下文，最小化了多任务的认知开销。

04 · 核心概念

窗口管理 Window

Window 占据整个屏幕，一个 session 可包含任意数量的 window。状态栏底部显示所有 window 列表，当前活跃的 window 用 * 标记。 —— 来源：tmux(1) WINDOWS AND PANES

Shell 命令

# 在指定 session 中新建窗口 tmux new-window -t work tmux new-window -t work -n editor # 指定窗口名 # 切换到指定 session 的第 N 个窗口 tmux select-window -t work:1 # 移动窗口到指定索引 tmux move-window -t 3 # 关闭当前窗口 tmux kill-window

窗口快捷键（Prefix + key）

快捷键功能场景
Ctrl-b c创建新 window开启新任务
Ctrl-b ,重命名当前 window语义化命名
Ctrl-b n切换到下一个 window顺序浏览
Ctrl-b p切换到上一个 window顺序浏览
Ctrl-b l切换到上一次使用的 window两个窗口间跳转
Ctrl-b 0~9直接跳转到第 N 个 window精确跳转
Ctrl-b '提示输入索引号跳转索引 > 9 时
Ctrl-b w交互式 window 选择器窗口多时概览
Ctrl-b &关闭当前 window（需确认）谨慎操作
Ctrl-b .提示移动当前 window 到指定索引重排窗口
Ctrl-b f提示在所有 window 中搜索文本查找内容
Ctrl-b i显示当前 window 信息查看状态
第一性原理
Window 是对 "任务视图" 的抽象。在没有 tmux 的时代，每个任务需要一个物理终端标签页。 Window 的本质是将多个 pty 复用到同一个物理终端时间槽上——同一时刻只有一个 window 可见， 但所有 window 的进程都在运行。这是 时分复用（TDM） 思想在终端层面的应用。 命名 window 是将无意义的数字索引升级为语义标签，降低认知负担。

05 · 核心概念

面板管理 Pane

Pane 是 window 中的矩形区域，每个 pane 是一个独立的伪终端进程。 Window 可水平或垂直分割为多个 pane，每个 pane 独立运行命令。 —— 来源：tmux(1) WINDOWS AND PANES

分割方式

Shell 命令

# 水平分割（左右） tmux split-window -h tmux splitw -h # 简写 tmux splitw -h -p 30 # 右侧占 30% # 垂直分割（上下） tmux split-window -v tmux splitw -v # 选择 pane（按方向） tmux select-pane -L # 左 tmux select-pane -R # 右 tmux select-pane -U # 上 tmux select-pane -D # 下 # 调整 pane 大小 tmux resize-pane -L 10 # 向左扩展 10 格 tmux resize-pane -R 10 tmux resize-pane -U 5 tmux resize-pane -D 5 # 关闭当前 pane tmux kill-pane exit

💡 Pane 同步输入
:setw synchronize-panes on 可让所有 pane 同时接收相同键盘输入——适合在多台服务器上执行同一命令。 用 off 关闭。
第一性原理
Pane 的本质是 空间复用：在有限的物理屏幕上，同时可视化多个 pty 的输出流。 它直接对应人类视觉的并行处理能力——左眼看代码，右眼看日志，无需在两个窗口间切换。 Zoom（Ctrl-b z）则是对"单点专注"需求的响应：需要深度工作时隐藏噪音，需要对比时恢复全局视图。 这是 注意力带宽管理 在终端界面的具体实现。

复制模式 Copy Mode

复制模式允许滚动查看终端历史、搜索文本，并将内容复制到 tmux 粘贴缓冲区。 按 Ctrl-b [ 进入，按 q 退出。 —— 来源：tmux(1) DEFAULT KEY BINDINGS

tmux 复制模式默认使用 Emacs 键位。若要使用 vi 键位，在 ~/.tmux.conf 中添加： setw -g mode-keys vi

第一性原理

07 · 进阶功能

配置文件 ~/.tmux.conf

tmux 启动时加载 ~/.tmux.conf（或 $XDG_CONFIG_HOME/tmux/tmux.conf）。 配置文件是一系列 tmux 命令，在 server 首次启动时顺序执行。 —— 来源：tmux(1) FILES

实用配置示例

### 基础设置 ###################################### # 修改前缀为 Ctrl+a（Screen 风格，更易按） set -g prefix C-a unbind C-b bind C-a send-prefix # 历史滚动行数 set -g history-limit 50000 # 窗口编号从 1 开始（键盘 1 在左边更自然） set -g base-index 1 setw -g pane-base-index 1 # 关闭窗口后自动重排编号 set -g renumber-windows on # 减少 Esc 延迟（对 Vim 用户重要） set -sg escape-time 0 # 启用鼠标支持 set -g mouse on ### 快捷键 ######################################## # 热重载配置 bind r source-file ~/.tmux.conf \; display "Config Reloaded!" # 更直觉的分割键 bind | split-window -h -c "#{pane_current_path}" bind - split-window -v -c "#{pane_current_path}" unbind '"' unbind % # Vim 风格 pane 导航 bind h select-pane -L bind j select-pane -D bind k select-pane -U bind l select-pane -R ### 状态栏 ######################################## set -g status-position top set -g status-interval 5 set -g status-left " #[bold]#S #[nobold]| " set -g status-right " %H:%M %d-%b #H " ### 复制模式（vi 风格）############################ setw -g mode-keys vi bind-key -T copy-mode-vi v send-keys -X begin-selection bind-key -T copy-mode-vi y send-keys -X copy-selection

重载配置（无需重启）

# 方法一：Shell 命令 tmux source-file ~/.tmux.conf # 方法二：tmux 命令提示符（Prefix :） source-file ~/.tmux.conf # 方法三：若已绑定 r 键（见上面配置） Prefix + r

常用选项参考

选项命令说明
set -g history-limit N设置历史记录行数（默认 2000）
set -g mouse on启用鼠标（点击选 pane、拖拽调整大小、滚轮翻页）
set -g base-index 1窗口编号从 1 开始
set -g renumber-windows on关闭窗口后自动重排编号
set -sg escape-time 0消除 Esc 键延迟
set -g default-terminal "tmux-256color"开启 256 色支持
setw -g mode-keys vi复制/帮助模式使用 vi 键位
set -g status-position top状态栏显示在顶部
set -g monitor-activity on监控其他 window 的活动
set -g visual-activity on活动时在状态栏显示提示
第一性原理
配置文件的本质是 将重复性手动操作转化为自动执行的声明。 每次 tmux 启动，配置文件中的命令都被完整重放——这是"幂等初始化"的思想。 好的配置追求最小化摩擦：让最常用操作（分割、导航、复制）以最少按键完成， 让环境在任何机器上都能快速重现。配置即工作流的代码化。

08 · 进阶功能

脚本与自动化

tmux 的每条命令都可在 shell 外部执行，支持构建可重现的开发环境脚本。 —— 来源：tmux(1) EXAMPLES、tmux.app/doc

常用脚本命令

命令说明
tmux has-session -t name检测 session 是否存在（返回 0/1）
tmux send-keys -t sess:win.pane "cmd" Enter向指定 pane 发送按键/命令
tmux new-session -d -s name创建后台 session（不附加）
tmux new-window -t sess -n winname在 session 中创建命名窗口
tmux split-window -h -t sess:win水平分割指定窗口
tmux select-window -t sess:win切换到指定窗口
tmux select-pane -t sess:win.pane切换到指定 pane
tmux attach -t sess附加到 session

实战脚本：一键搭建开发环境

#!/bin/bash # dev-session.sh — 自动搭建完整开发环境 SESSION="dev" # 若 session 已存在则直接附加 tmux has-session -t $SESSION 2>/dev/null if [ $? -eq 0 ]; then tmux attach -t $SESSION exit 0 fi # 创建后台 session，初始目录为项目根 tmux new-session -d -s $SESSION -c ~/projects/myapp # 窗口 0: 编辑器 tmux rename-window -t $SESSION:0 'editor' tmux send-keys -t $SESSION:0 'nvim .' Enter # 窗口 1: 开发服务器（左右分割） tmux new-window -t $SESSION -n 'server' tmux send-keys -t $SESSION:1 'npm run dev' Enter tmux split-window -h -t $SESSION:1 tmux send-keys -t $SESSION:1 'npm run test:watch' Enter # 窗口 2: Git 操作 tmux new-window -t $SESSION -n 'git' tmux send-keys -t $SESSION:2 'git log --oneline -10' Enter # 窗口 3: 日志监控（三等分） tmux new-window -t $SESSION -n 'logs' tmux send-keys -t $SESSION:3 'tail -f logs/app.log' Enter tmux split-window -h -t $SESSION:3 tmux send-keys -t $SESSION:3 'tail -f logs/error.log' Enter # 切换到编辑器窗口后附加 tmux select-window -t $SESSION:0 tmux attach -t $SESSION

目标地址语法

# 格式: session-name:window-index.pane-index tmux send-keys -t dev:0.1 "echo hello" Enter # 特殊 token tmux select-window -t mysession:{last} # 上一个 window tmux select-window -t mysession:{next} # 下一个 window tmux select-window -t mysession:^ # 第一个 window tmux select-window -t mysession:$ # 最后一个 window
第一性原理
脚本化 tmux 的根本价值是 将不可重现的手动步骤转化为幂等的代码。 每次启动开发环境，花费5分钟手动分割窗口、启动各服务，是一种低效的重复劳动。 脚本将这个过程编码为一次性的智识投入，此后每次执行只需秒级完成。 这是软件工程中 DRY（Don't Repeat Yourself）原则 在终端工作流中的实践。

场景一：SSH 远程开发（不怕断线）

# 在远程服务器上，SSH 登录后立即进入/创建 session ssh user@server tmux attach -t work || tmux new -s work # 或写入 ~/.bashrc，SSH 登录自动进入 tmux [ -z "$TMUX" ] && (tmux attach -t default || tmux new -s default)

场景二：全栈开发工作区

场景三：多服务器批量操作

# 创建 4 个 pane，分别 SSH 到不同服务器 tmux new-session -s ops tmux split-window -h tmux split-window -v tmux select-pane -t 0 tmux split-window -v tmux send-keys -t 0 "ssh web-01" Enter tmux send-keys -t 1 "ssh web-02" Enter tmux send-keys -t 2 "ssh db-01" Enter tmux send-keys -t 3 "ssh cache-01" Enter # 开启同步输入：所有 pane 同时执行同一命令 # Prefix : 然后输入： setw synchronize-panes on # 现在输入任何命令，4 台服务器同时执行 sudo systemctl restart nginx

# 主机：创建共享 socket tmux -S /tmp/pair new -s pair chmod 777 /tmp/pair # 同事：连接同一个 socket（完全同步视图） tmux -S /tmp/pair attach -t pair # 只读观察（-r 参数） tmux -S /tmp/pair attach -t pair -r

第一性原理
tmux 所有使用场景的共同根基是：将终端状态从瞬态（ephemeral）变为持久态（persistent），将空间从独占变为共享。 传统终端是一次性的——关掉就消失，最多一人使用，受限于物理屏幕大小。 tmux 打破了这三个约束：Session 持久化存储状态；多 Client 支持状态共享；Pane/Window 将一个屏幕无限分割。 这是对终端资源从稀缺到丰盈的根本性重构。
