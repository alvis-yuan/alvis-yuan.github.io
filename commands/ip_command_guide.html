第 1 章 · 导论

概览与背景

导读：本章介绍 ip 命令的来龙去脉——它从何而来、解决什么问题、整体架构如何，以及为什么它全面取代了老旧的 ifconfig / route 工具链。

什么是 ip 命令

ip 是 iproute2 软件包的核心工具。它的官方定义是："show / manipulate routing, network devices, interfaces and tunnels"（展示和操控路由、网络设备、接口与隧道）。ip 由 Alexey N. Kuznetsov 编写，随 Linux 2.2 内核一同加入。

iproute2 使用 Netlink 套接字直接与内核通信，相比 net-tools（ifconfig/route/arp/netstat）通过 /proc 读写的旧方式，信息更完整、更实时，支持策略路由、多路由表、网络命名空间、隧道等高级特性。

↗ 原文：ip(8) — Linux manual page，man7.org

统一语法结构

ip [ OPTIONS ] OBJECT { COMMAND | help }

与旧工具对比

旧命令 (net-tools)新命令 (iproute2)功能
ifconfig eth0ip addr show eth0查看接口地址
ifconfig eth0 upip link set eth0 up启用接口
route -nip route show显示路由表
arp -nip neigh show查看 ARP 缓存
netstat -rip route路由信息
不支持ip netns网络命名空间
不支持ip rule策略路由规则
ℹ 所有对象名称可缩写：address 可写为 addr 或 a，route 可写为 ro 或 r，以此类推。来源：ip(8) man page

第 2 章 · 核心子命令

ip link · 链路层设备管理

导读：ip link 操控的是网络设备本身（第二层链路层），包括物理网卡和各种虚拟接口。它负责接口的启停、MTU 设置、MAC 地址修改，以及创建 VLAN、VXLAN、VETH、Bridge、Bond 等虚拟设备。本章完整覆盖 ip-link(8) 手册内容。
↗ 原文：ip-link(8) — Linux manual page，man7.org

查看接口状态

ip link show                          # 显示所有网络接口状态
ip link show eth0                   # 只显示 eth0
ip link show up                      # 只显示已激活接口
ip link show type bridge             # 只显示 bridge 设备
ip link show master br0              # 显示 br0 的从属设备
ip -s link                           # 显示接口统计信息（含收发包数）
ip -d link show eth0              # 显示详细信息（-details）
ip -br link                          # 简洁表格形式输出

启停与基本配置

ip link set eth0 up                 # 启用接口
ip link set eth0 down               # 禁用接口
ip link set eth0 mtu 9000          # 设置 MTU（巨帧）
ip link set eth0 name lan0         # 重命名接口
ip link set eth0 address 00:11:22:33:44:55  # 修改 MAC 地址
ip link set eth0 promisc on         # 开启混杂模式
ip link set eth0 txqueuelen 10000  # 设置发送队列长度
ip link set ppp0 mtu 1400          # 示例来自 ip-link(8) man page

创建虚拟设备

来源：ip-link(8)，TYPE 支持列表：amt | bareudp | bond | bridge | can | dummy | erspan | geneve | gre | gretap | gtp | ifb | ip6gre | ipip | ipvlan | macsec | macvlan | nlmon | sit | vcan | veth | vlan | vrf | vti | vxlan | xfrm 等。

# --- VLAN (802.1q) ---
ip link add link eth0 name eth0.100 type vlan id 100

# --- Bridge 网桥 ---
ip link add name br0 type bridge
ip link set eth0 master br0         # 将 eth0 加入 br0
ip link set eth0 nomaster            # 脱离网桥

# --- VETH Pair（容器常用）---
ip link add veth0 type veth peer name veth1

# --- VXLAN 隧道 ---
ip link add vxlan100 type vxlan id 100 group 239.1.1.1 dev eth0 dstport 4789

# --- Dummy（测试用虚拟接口）---
ip link add dummy0 type dummy

# --- 删除虚拟接口 ---
ip link del eth0.100
ip link del vxlan100

第 3 章 · 核心子命令

ip address · 协议地址管理

导读：ip address（可缩写为 ip addr 或 ip a）管理绑定到网络设备上的 IPv4/IPv6 协议地址。一个设备可绑定多个地址。本章完整覆盖 ip-address(8) 手册内容。
↗ 原文：ip-address(8) — Linux manual page，man7.org

查看地址

ip address show                     # 显示所有接口的 IPv4/IPv6 地址（show 可省略）
ip addr show up                     # 只显示活动接口的地址
ip addr show dev eth0             # 显示 eth0 的地址（示例来自 ip-address(8)）
ip -4 addr                          # 只显示 IPv4
ip -6 addr                          # 只显示 IPv6
ip -br addr                         # 简洁输出（-brief）

典型输出

2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc pfifo_fast state UP
    link/ether 08:00:27:66:72:e1 brd ff:ff:ff:ff:ff:ff
    inet 10.1.102.59/24 brd 10.1.102.255 scope global eth0
       valid_lft forever preferred_lft forever
    inet 10.1.102.60/24 scope global secondary eth0
       valid_lft forever preferred_lft forever
    inet6 fe80::a00:27ff:fe66:72e1/64 scope link
       valid_lft forever preferred_lft forever

添加地址

# 基本语法：ip address { add | change | replace } IFADDR dev IFNAME
ip addr add 192.168.1.100/24 dev eth0
ip addr add 192.168.4.223/24 dev eth0       # 绑定第二个地址（示例来自 Red Hat 文档）
ip addr add 192.168.1.101/24 dev eth0 label eth0:1  # 带标签

# IPv6
ip addr add 2001:db8::1/64 dev eth0
ip addr add 2001:0db8:85a3::0370:7334/64 dev eth1   # 示例来自 ip-address(8)

# 带生命周期（IPv6 地址老化）
ip addr add 2001:db8::1/64 dev eth0 valid_lft 3600 preferred_lft 1800

# nodad: 跳过重复地址检测（DAD）
ip addr add 2001:db8::1/64 dev eth0 nodad

删除与清空地址

ip addr del 192.168.1.100/24 dev eth0
ip addr del 2001:0db8:85a3::0370:7334/64 dev eth1   # 示例来自 ip-address(8)
ip addr flush dev eth0                        # 清空 eth0 所有地址
ip addr flush dev eth4 scope global         # 只删全局地址，保留链路本地（示例来自 ip-address(8)）
⚠ ip address flush 是不可逆的破坏性操作，会无情地清除所有选中地址。使用 -statistics 选项可查看删除数量。来源：ip-address(8)

地址作用域（scope）说明

Scope 值含义典型例子
global地址全局有效，可路由到外部192.168.1.1，公网 IP
link仅在本链路有效fe80::/10 链路本地地址
host仅在主机内部有效127.0.0.1 环回

第 4 章 · 核心子命令

ip route · 路由表管理

导读：ip route 是 ip 命令中最复杂的子系统，用于操控内核路由表。Linux 支持多张路由表（main/local/default），还支持多路径路由、路由作用域、协议标记、MPLS 封装等高级特性。本章严格基于 ip-route(8) 手册。
↗ 原文：ip-route(8) — Linux manual page，man7.org

路由表体系

根据 ip-route(8)：Linux 2.x 可以将路由打包到多张路由表中，表由 1~2³²-1 的数字或 /etc/iproute2/rt_tables 中的名称标识。默认情况下所有普通路由插入 main 表（ID 254）。

查看路由

ip route                             # 显示 main 路由表（示例来自 ip-route(8)）
ip ro                                # 缩写
ip route show table local           # 查看 local 表
ip route show table all             # 所有路由表
ip -6 route                          # IPv6 路由表
ip route get 8.8.8.8               # 查找到达 8.8.8.8 的路由（内核视角）
ip route get 8.8.8.8 from 192.168.1.10  # 指定源地址查路由

添加、修改、删除路由

# 添加默认路由（示例来自 ip-route(8)）
ip route add default via 192.168.1.1 dev eth0

# 添加静态路由
ip route add 10.0.0.0/8 via 192.168.1.1
ip route add 20.0.0.0/24 via 192.168.0.1

# 指定出口设备
ip route add 10.1.0.0/16 dev eth1

# 带 metric（低值优先）
ip route add 10.0.0.0/8 via 192.168.1.1 metric 100

# 修改路由
ip route change 10.0.0.0/8 via 192.168.2.1
ip route replace 10.0.0.0/8 via 192.168.2.1   # 不存在则新建

# 删除路由
ip route del 10.0.0.0/8
ip route del default

# 清空路由表
ip route flush table main

路由类型

类型行为说明（来自 ip-route(8)）
unicast正常转发描述到达目标的真实路径，默认类型
blackhole静默丢弃本地发送方收到 EINVAL 错误
unreachable丢弃+ICMP发送 host unreachable，本地返回 EHOSTUNREACH
prohibit丢弃+ICMP发送 administratively prohibited，本地返回 EACCES
local本地回环目标为本机地址，数据包回环投递
broadcast广播发送目标为广播地址
throw终止表查找策略路由中终止当前表查找，返回 ENETUNREACH

多路径路由（ECMP）

# 等价多路径路由（ECMP）
ip route add 10.0.0.0/8
    nexthop via 192.168.1.1 dev eth0 weight 1
    nexthop via 192.168.2.1 dev eth1 weight 2
# weight：反映相对带宽/质量，来源：ip-route(8)

# MPLS 封装路由（示例来自 ip-route(8)）
ip route add 10.1.1.0/30 encap mpls 200/300 via 10.1.1.1 dev eth0

路由协议标记（proto）

来源：ip-route(8)。kernel — 内核自动配置；boot — 启动期间安装（路由守护进程启动后清除）；static — 管理员手动添加，路由守护进程尊重并可能向邻居通告；ra — 路由发现协议安装；redirect — ICMP 重定向安装。

第 5 章 · 核心子命令

ip neigh · 邻居表 / ARP 管理

导读：ip neigh（neighbour 的缩写）管理邻居对象——即协议地址与链路层地址之间的绑定关系。IPv4 邻居表就是我们熟知的 ARP 表，IPv6 使用 NDP（邻居发现协议）。本章内容严格来自 ip-neighbour(8)。
↗ 原文：ip-neighbour(8) — Linux manual page，man7.org

查看邻居表

ip neighbour                        # 显示当前 ARP/邻居表（示例来自 ip-neighbour(8)）
ip neigh show dev eth0            # 只显示 eth0 的邻居
ip neigh show nud reachable        # 只显示可达状态的邻居
ip neigh get 10.0.1.10 dev eth0   # 向内核查询特定邻居（示例来自 ip-neighbour(8)）

NUD 邻居状态机

NUD（Neighbour Unreachability Detection，邻居不可达检测）定义了邻居条目的状态转换。来源：ip-neighbour(8)。

状态含义（来自 ip-neighbour(8)）
permanent永久有效，只能由管理员删除
noarp有效条目，不进行可达性验证，生命周期到期后删除
reachable有效，直到可达超时
stale有效但可疑，等待下次使用时验证
incomplete尚未完成验证/解析
delay验证当前处于延迟状态
probe正在探测邻居
failed探测超出最大次数，最终失败
none伪状态，初始创建或等待释放时使用

增删改邻居条目

# 添加静态 ARP 条目
ip neigh add 192.168.1.10 lladdr 00:11:22:33:44:55 dev eth0 nud permanent

# 修改邻居状态
ip neigh change 192.168.1.10 dev eth0 nud reachable

# 删除邻居条目
ip neigh del 192.168.1.10 dev eth0

# 清空 eth0 邻居表（示例来自 ip-neighbour(8)）
ip neigh flush dev eth0

第 6 章 · 高级功能

ip netns · 网络命名空间

导读：网络命名空间（Network Namespace）是 Linux 内核提供的网络隔离机制——每个命名空间有自己独立的路由表、防火墙规则、网络设备。ip netns 是管理具名网络命名空间的标准工具，是容器技术（Docker、Kubernetes）的底层基础。本章内容来自 ip-netns(8)。
↗ 原文：ip-netns(8) — Linux manual page，man7.org

命名空间基础操作

根据 ip-netns(8)，具名网络命名空间是 /var/run/netns/NAME 处的对象，保持文件描述符打开即可保活命名空间。

ip netns list                         # 列出所有具名命名空间（示例来自 ip-netns(8)）
ip netns add vpn                      # 创建命名空间，命名为 vpn
ip netns del vpn                      # 删除命名空间
ip netns identify $$                 # 报告当前进程所在的命名空间
ip netns pids vpn                     # 列出命名空间内的所有进程
ip netns exec vpn ip link set lo up  # 在命名空间内执行命令（示例来自 ip-netns(8)）
ip netns attach myns 1234            # 将进程 1234 的命名空间附加为 myns

用 VETH 对连接两个命名空间

# 1. 创建两个命名空间
ip netns add ns1
ip netns add ns2

# 2. 创建 veth 对
ip link add veth0 type veth peer name veth1

# 3. 将各端移入命名空间
ip link set veth0 netns ns1
ip link set veth1 netns ns2

# 4. 在各命名空间内配置地址并启动
ip netns exec ns1 ip addr add 10.0.0.1/24 dev veth0
ip netns exec ns1 ip link set veth0 up
ip netns exec ns1 ip link set lo up

ip netns exec ns2 ip addr add 10.0.0.2/24 dev veth1
ip netns exec ns2 ip link set veth1 up
ip netns exec ns2 ip link set lo up

安全删除含进程的命名空间

# 来源：ip-netns(8) — 防止物理设备丢失的正确流程
ip netns pids net0 | xargs kill
ip netns del net0

第 7 章 · 高级功能

ip rule · 策略路由规则

导读：策略路由（Policy-Based Routing）让 Linux 根据数据包的来源地址、目标地址、TOS、fwmark 等属性选择不同的路由表，而不仅仅依赖目标地址。ip rule 管理路由策略数据库（RPDB）。本章内容来自 ip-route(8) 及 iproute2 文档。
↗ 原文：ip-route(8) 路由表部分 — Linux manual page，man7.org

策略路由工作原理

查看与管理规则

ip rule list                          # 列出所有 RPDB 规则（优先级从低到高匹配）
ip rule show

# 典型输出：
0:      from all lookup local
32766:  from all lookup main
32767:  from all lookup default

# 添加规则：来自 192.168.2.0/24 的包查 table 100
ip rule add from 192.168.2.0/24 table 100 priority 1000

# 添加规则：fwmark 标记路由（配合 iptables 使用）
ip rule add fwmark 0x1 table 200

# 添加规则：目标地址匹配
ip rule add to 10.0.0.0/8 table 100

# 删除规则
ip rule del from 192.168.2.0/24 table 100

策略路由完整示例（双上行链路）

# 场景：两条 ISP 链路，按来源 IP 分流
# ISP1: eth0, GW 1.2.3.1；ISP2: eth1, GW 4.5.6.1

# 1. 创建两张路由表（在 /etc/iproute2/rt_tables 中添加名称）
echo "100 isp1" >> /etc/iproute2/rt_tables
echo "200 isp2" >> /etc/iproute2/rt_tables

# 2. 在各表中添加默认路由
ip route add default via 1.2.3.1 table 100
ip route add default via 4.5.6.1 table 200

# 3. 添加策略规则
ip rule add from 1.2.3.0/24 table 100
ip rule add from 4.5.6.0/24 table 200

第 8 章 · 高级功能

全局选项与高级语法

导读：ip 命令拥有丰富的全局选项（OPTIONS），作用于所有子命令之前，控制输出格式、协议族过滤、网络命名空间切换等行为。本章还涵盖批量执行、JSON 输出、监控等高级用法，内容严格来自 ip(8) man page。
↗ 原文：ip(8) OPTIONS 部分 — Linux manual page，man7.org

全局选项速查（来自 ip(8)）

选项完整形式说明
-V-Version打印版本信息并退出
-h-human-readable以人类可读单位显示统计值（如 1K, 2M）
-s-statistics输出更多信息（可叠加 -s -s 获取更多）
-d-details输出详细信息
-4-family inet只处理 IPv4
-6-family inet6只处理 IPv6
-o-oneline每条记录输出为一行（方便 grep/wc）
-r-resolve用 DNS 名称替代 IP 地址显示
-n-netns NAME切换到指定网络命名空间执行
-N-Numeric直接打印协议/作用域编号，不转换为名称
-a-all对所有对象执行命令（命令须支持）
-c-color彩色输出（always/auto/never）
-br-brief简洁表格输出（addr/link/neigh 支持）
-j-jsonJSON 格式输出
-p-pretty美化 JSON 输出（加缩进）
-b-batch FILE从文件或标准输入批量执行命令
-force-批量模式下遇错不终止
-t-timestampmonitor 模式下显示时间戳
-iec-以 IEC 单位显示速率（1Ki=1024）

高级用法示例

# --- JSON 输出（脚本友好）---
ip -j addr                          # JSON 格式
ip -j -p route                      # 美化 JSON 路由表
ip -j neigh | python3 -m json.tool  # 进一步格式化

# --- 切换命名空间执行（来自 ip(8)）---
ip -n myns link                       # 等价于 ip netns exec myns ip link

# --- 批量执行（来自 ip(8)）---
ip -batch commands.txt               # 从文件执行命令，第一个错误即终止
ip -force -batch commands.txt       # 遇错不停止

# --- 监控网络事件 ---
ip monitor                            # 实时监控所有 Netlink 事件
ip monitor link                       # 只监控链路事件
ip monitor route                      # 只监控路由变更
ip -t monitor                        # 带时间戳

# --- 统计信息 ---
ip -s link                           # 接口统计（收发包/字节/错误）
ip -s -s link                        # 更详细统计
ip -h -s link                        # 人类可读单位（K/M/G）

# --- 单行输出（便于脚本处理）---
ip -o addr | grep eth0 | wc -l       # 统计 eth0 地址数量
ip -o -4 addr show scope global     # 脚本提取全局 IPv4 地址

第 9 章 · 实战速查

常用场景速查手册

导读：本章将前述知识汇集为实际使用场景，覆盖网络排障、接口配置、路由管理、容器网络、安全排查等日常工作场景，所有命令均来自前述官方文档内容。

场景一：网络诊断与排障

# 快速查看所有接口与地址
ip -br addr
ip -br link

# 检查路由是否正确
ip route get 8.8.8.8          # 查看访问 8.8.8.8 走哪条路由
ip route get 8.8.8.8 from 192.168.1.100  # 指定源地址

# 检查 ARP/邻居状态
ip neigh show
ip neigh show nud failed      # 找出 ARP 失败的条目

# 实时监控路由/链路变化
ip -t monitor route link

场景二：接口管理

# 临时修改 MAC 地址（断网后改）
ip link set eth0 down
ip link set eth0 address 00:11:22:33:44:55
ip link set eth0 up

# 配置 IP 地址并添加默认路由
ip addr add 192.168.1.100/24 dev eth0
ip route add default via 192.168.1.1

# 查看接口流量统计
ip -h -s link show eth0

# 清空接口地址并重新配置
ip addr flush dev eth0
ip addr add 10.0.0.1/24 dev eth0

场景三：VLAN 子接口配置

# 创建 VLAN 100 子接口（来自 ip-link(8)）
ip link add link eth0 name eth0.100 type vlan id 100
ip link set eth0.100 up
ip addr add 192.168.100.1/24 dev eth0.100

第 10 章 · 总结

第一性原理总结

导读：从第一性原理出发，抛开所有记忆技巧和表面语法，回归本质：Linux 网络通信究竟需要解决哪些根本问题？ip 命令的设计如何一一对应这些本质？理解了本质，就能从任意一条 ip 命令推导出其他所有命令。

本质一：网络是分层的状态机

Linux 网络栈是一个严格分层的状态机。每一层只关心本层的问题：链路层关心帧的收发（ip link）；网络层关心地址标识（ip addr）和路径选择（ip route）；邻居子系统负责 IP↔MAC 映射（ip neigh）。 理解了这个分层，就知道为什么 ip 命令要被拆分为这些子命令——每个子命令精确对应一个层的状态管理。

本质二：ip 命令是内核 Netlink API 的用户态代理

ip 命令并不自己实现任何网络逻辑，它只是通过 Netlink 套接字向内核发送结构化消息（RTM_NEWLINK、RTM_NEWADDR、RTM_NEWROUTE 等），内核接受消息后修改自身状态。因此：

  • 所有 ip 命令的修改都是即时生效的——因为直接操作内核数据结构
  • 所有修改都是易失的——内核状态不持久化到磁盘，重启后消失
  • JSON 输出（-j）是原始 Netlink 数据的序列化——信息完整度高于 ifconfig 等旧工具

本质三：路由是一个函数 f(packet) → nexthop

路由的本质是一个函数：给定一个数据包（含源 IP、目标 IP、TOS、fwmark 等），输出下一跳（nexthop）。Linux 的路由决策分为两步：

  1. ip rule（RPDB）：选择使用哪张路由表——这是策略层
  2. ip route（FIB）：在选定的路由表中做最长前缀匹配——这是查找层

单纯使用 ip route 只操作 main 表。当需要"不同来源的包走不同路径"时，就需要引入 ip rule 增加策略维度。这是策略路由（Policy-Based Routing）的第一性原理。

本质四：虚拟化 = 设备抽象 + 命名空间

容器网络、VPN、SDN 的底层都是两个正交的抽象：

  • 虚拟设备（ip link add type ...）：在同一内核中创建软件模拟的网络设备（veth、bridge、vxlan 等）
  • 网络命名空间（ip netns）：为每个"租户"提供独立的内核网络视图，使同一台主机上的多个网络栈互相隔离

二者组合（veth + netns）就是 Linux 容器网络的核心原语。Docker 的 bridge 网络模式，本质上就是：创建 bridge → 为每个容器创建 netns → 用 veth pair 连接 netns 与 bridge。

本质五：一切皆可观测，一切皆可变更

ip 命令的所有子命令都遵循一个统一接口：show（观测）、add（创建）、del（删除）、set/change（修改）、flush（批量清除）、monitor（实时事件流）。这个设计反映了 Unix 哲学：将系统状态暴露为可读写的数据，而非隐藏在命令的副作用中。

掌握 ip 命令的最高境界是：不记语法，只记本质——遇到任何网络问题，先问"我在操作哪一层的哪种对象？"，然后 ip <object> help 即可得到所有可用操作。

✓ 终极速查：ip link help、ip addr help、ip route help、ip neigh help、ip netns help、ip rule help — 任何时候忘记语法，都可以直接向 ip 命令本身求助。
