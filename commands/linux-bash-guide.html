Bash（Bourne-Again SHell）是 GNU 操作系统的默认 Shell，也是大多数 Linux 发行版的标准命令行解释器。 名称来源于对 Stephen Bourne 所写的经典 Unix Shell（sh）的致敬与双关。

Bash 兼容 POSIX 标准，并融合了 Korn Shell（ksh）和 C Shell（csh）的优秀特性， 同时提供命令行编辑、历史记录、作业控制、别名、函数等交互功能。

官方文档：GNU Bash Reference Manual (v5.3, 2025-05-18)
快捷键参考：ss64.com — bash keyboard shortcuts
{/* Bash 架构 SVG */}
⚡ 第一性原理

Shell 的本质是一个宏处理器——它读取用户的文本指令，展开变量、通配符、命令替换等宏， 再将最终结果作为系统调用传递给内核。理解这一点，就理解了所有命令行行为的底层逻辑： 你所看到的每一个命令，都不过是 Bash 对内核 API 的文本化封装。

{/* ── 2. NAVIGATION ── */}
{ sectionRefs.current['navigation'] = el }} id="navigation" className="section">

目录导航是 Shell 使用的基础。Bash 提供多个内建命令（builtins）直接操纵 Shell 自身状态， 如 cd、pwd，它们不会启动子进程。

{/* Navigation SVG */}
⚡ 第一性原理

文件系统是一棵有根的树（root = /）。所有导航命令的本质是改变进程的 当前工作目录（CWD）——这是内核为每个进程维护的一个指针。 cd 是内建命令，因为它必须修改 Shell 自身的 CWD，而不是启动一个子进程去修改（子进程的 CWD 变化不影响父进程）。

{/* ── 3. FILE OPS ── */}
{ sectionRefs.current['fileops'] = el }} id="fileops" className="section">

文件操作是日常使用最频繁的命令类别，涵盖创建、复制、移动、删除、压缩等操作。

{/* cp/mv 操作图示 */}
⚡ 第一性原理

Linux 文件系统的核心概念是 inode（索引节点）。文件名只是 inode 的一个引用。 cp 创建新 inode；mv 在同一文件系统内只更改目录项（不移动数据块）； 硬链接新增 inode 引用；软链接创建一个存储路径的特殊文件。rm 只是减少引用计数，当计数归零内核才真正释放数据块。

{/* ── 4. VIEW & SEARCH ── */}
{ sectionRefs.current['viewtext'] = el }} id="viewtext" className="section">
⚡ 第一性原理

Unix 哲学核心之一："一切皆文本流"。grep、awk、sed 均遵循"读取标准输入 → 处理 → 写入标准输出"的管道模型。 这使得它们可以任意组合（grep | awk | sort | uniq）， 每个工具只做一件事并做好，组合威力远超单一工具。

{/* ── 5. PROCESS ── */}
{ sectionRefs.current['process'] = el }} id="process" className="section"> {/* Signal SVG */}
⚡ 第一性原理

进程是内核的执行单元，拥有独立的地址空间和 PID。 信号（Signal）是内核向进程传递事件的机制——本质上是软件中断。 SIGKILL 和 SIGSTOP 无法被进程捕获，由内核直接处理； 而 SIGTERM 允许进程优雅退出（清理资源）。 优先用 SIGTERM（kill 默认），仅在无效时才用 SIGKILL。

{/* ── 6. PERMISSION ── */}
{ sectionRefs.current['permission'] = el }} id="permission" className="section"> {/* Permission SVG */}
⚡ 第一性原理

Linux 权限模型基于三元组：所有者（owner）、所属组（group）、其他人（others）， 各自拥有 读（r=4）、写（w=2）、执行（x=1） 三个权限位。 这是一个位掩码系统——权限检查是内核对 inode 权限字段的位运算，而非字符串匹配。 umask 从默认最大权限（文件 666，目录 777）中按位"遮罩"掉不需要的位。

{/* ── 7. NETWORK ── */}
{ sectionRefs.current['network'] = el }} id="network" className="section">
⚡ 第一性原理

网络命令的底层是 套接字（Socket）API——内核提供的端点抽象。 ping 使用 ICMP（第 3 层）；curl/wget 使用 TCP + HTTP（第 4/7 层）； ssh 在 TCP 上建立加密隧道。理解 OSI 层级，就能判断问题出在哪一层： ping 通但 HTTP 失败 → 应用层问题；ping 不通 → 网络层问题。
