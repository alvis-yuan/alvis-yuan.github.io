0. 概述与核心术语

本章是全文的入口。WiFi 关联（Association）并不是一个单一动作，而是一套有严格状态定义的协议流程。读懂术语是读懂后续每一条信令的前提：STA 是谁，AP 是谁，BSS 意味着什么？这些概念在 IEEE 802.11-2020 标准中有精确定义。

1. 被动扫描 / 主动扫描

STA 在关联之前首先必须发现可用的 AP。802.11 定义了两种扫描方式：被动扫描像收音机调台一样倾听，主动扫描则主动发出 Probe Request 叫板所有 AP。本章通过 MSC 图呈现两种扫描的信令交互差异。

被动扫描（Passive Scanning）

STA 逐信道监听 AP 周期性广播的 Beacon 帧（默认间隔 102.4 ms / 100 TU）。STA 无需发送任何帧，能耗低，但发现延迟高。Beacon 帧包含 SSID、支持速率、安全配置（RSN IE）、时间戳等关键信息。

主动扫描（Active Scanning）

STA 在每个信道发出 Probe Request 广播帧，等待 AP 回复 Probe Response。Probe Request 可携带 SSID（定向探测）或空 SSID（发现所有 AP）。探测速率选最低速（2.4GHz=1Mbps，5GHz=6Mbps）以最大覆盖。

维度被动扫描主动扫描
STA 发帧无Probe Request（每信道）
发现延迟高（取决于 Beacon 间隔）低（主动触发）
能耗低高
隐藏 SSID无法发现定向 Probe 可发现
典型场景初次启动、省电模式快速漫游、连接前

3. 802.11 开放系统认证

这是关联前的"握手礼"——形式大于内容。802.11 标准最初设计了 WEP 共享密钥认证，但由于 WEP 的严重安全缺陷，现代网络几乎全部使用"开放系统认证"（Open System Authentication），本质上是一次空认证（NULL Auth），真正的身份验证发生在关联之后的 EAP/4-Way Handshake 阶段。

信令流程（MSC）

Authentication 帧固定字段

字段长度说明
Authentication Algorithm Number2 Bytes0 = Open System；1 = Shared Key（WEP，已废弃）；3 = SAE（WPA3）
Authentication Transaction Seq Number2 BytesRequest=1，Response=2（SAE Commit=1, Confirm=2）
Status Code2 Bytes0 = 成功；其他值表示失败原因（如 1=Unspecified Failure）
Challenge Text（可选）可变仅 WEP 共享密钥认证使用，现代网络不使用
⚠️ 重要说明：开放系统认证不做任何身份验证，AP 对任何 STA 的 Auth Request 都会回复成功。真正的认证由后续的 802.1X/EAP（企业网）或 WPA2 4-Way Handshake（个人网）完成。这是 WiFi 历史遗留的设计，为了向后兼容 WEP 时代。
参考来源： Authentication, Association & Authorization in 802.11 WiFi — supernetworks.org · Why authentication before association — networkengineering.stackexchange.com

4. 关联请求 / 关联响应

关联是 STA 正式"入网"的动作——AP 为 STA 分配资源（AID）并记录其能力。关联请求帧携带 STA 完整的能力集（速率、HT/VHT/HE、安全参数），AP 审核后决定接受（Status=0）或拒绝（Status≠0）。这是 State 2 → State 3 的关键跳变。

5. 802.11 状态机与 AID

802.11 标准用四态状态机精确定义 STA 的每个阶段权限。理解状态机才能理解为什么某些帧只能在特定状态发送，以及 Disassociation/Deauthentication 帧如何触发状态回退。AID 是 AP 管理 TIM（省电通知）的关键索引。

AID（关联标识符）

AID 是 AP 在 Association Response 中分配给 STA 的 14 bit 无符号整数（1~2007），在帧头中高 2 bit 固定为 1。AP 用 AID 维护 Beacon 帧中的 TIM（Traffic Indication Map）位图：当 AP 为某 STA 缓存了下行帧时，TIM 中对应 AID 的 bit 置 1，STA 收到 Beacon 后检查自己 AID 对应的 bit，若为 1 则唤醒发送 PS-Poll 取帧。

Disassociation vs Deauthentication：
• Disassociation：从 State 3/4 → State 2（仍保留认证状态，STA 可直接发 AssocReq 重新关联，无需重新 Auth）
• Deauthentication：从任意状态 → State 1（完全重置，需重新走 Auth→Assoc 完整流程）
两者均携带 Reason Code 字段说明原因（如 Code 3 = "STA 离开 BSS"，Code 5 = "AP 不活跃"）
参考来源： 802.11 Frame Exchanges — howiwifi.com · Understanding 802.11 State Machine — Aruba Networks

6. 802.1X / EAP 认证（企业网）

WPA2-Enterprise 网络在完成 802.11 关联之后，STA 还需要通过 802.1X/EAP 框架向 RADIUS 服务器证明身份。AP 作为"认证者（Authenticator）"充当透明中继，维护受控端口（Controlled Port）和非受控端口（Uncontrolled Port）两个虚拟端口。只有 EAP 成功后，受控端口才打开。

EAP-PEAP/MSCHAPv2 信令流程（MSC）

PMK 的来源：企业网中，RADIUS Access-Accept 携带 MSK（Master Session Key，至少 64 字节）。PMK = MSK 的前 32 字节（256 bit）。个人网（WPA2-PSK）中，PMK 直接由密码短语（Passphrase）通过 PBKDF2-HMAC-SHA1 派生：PMK = PBKDF2(HMAC-SHA1, Passphrase, SSID, 4096, 256bit)

10. WiFi 安全演化：WEP → WPA → WPA2 → WPA3

WiFi 安全标准的每次演化，都是对前代协议被攻破后的防御性升级。从 WEP 的静态密钥到 WPA3 的 SAE 零知识证明，关联流程中的安全握手部分经历了根本性的变革。了解这条演化链，才能理解每个设计决策背后的安全动机。
1997 — IEEE 802.11 原始标准
WEP（Wired Equivalent Privacy）
使用 RC4 流密码，64/128 bit 静态密钥，IV 仅 24 bit（约 5000 帧后重复）。采用 CRC-32 校验（非加密完整性）。
关联流程：支持"共享密钥认证"——AP 发 Challenge Text，STA 用 WEP 密钥加密后回传。由于 RC4 密钥流可从密文推导，共享密钥认证实际上比开放认证更不安全。
致命缺陷：2001年 Fluhrer-Mantin-Shamir 攻击（FMS attack）证明 24 bit IV 使 WEP 密钥可在数分钟内被破解。
2003 — Wi-Fi Alliance / IEEE 802.11i 草案
WPA（Wi-Fi Protected Access）= TKIP
紧急替代方案，设计为可通过固件升级部署在 WEP 硬件上。引入 TKIP（Temporal Key Integrity Protocol）：每数据包动态生成 128 bit 密钥（混合基础密钥 + MAC + 序列号），64 bit MIC（"Michael"）检测篡改，IV 扩展为 48 bit。
引入 4-Way Handshake 和 PMK 机制（重大进步）。支持 PSK（个人）和 802.1X（企业）模式。
仍用 RC4：Beck-Tews 攻击（2008）可在约 12 分钟内伪造 ARP 数据包；WPA/TKIP 于 2012 年被 IEEE 禁用。
2004 — IEEE 802.11i-2004 / WPA2
WPA2 = AES-CCMP（彻底重建）
完全实现 IEEE 802.11i，放弃 RC4，采用 AES-CCMP（Counter Mode with CBC-MAC Protocol）：128 bit AES 块加密 + CBC-MAC 完整性保护。PTK/GTK 密钥生成机制成熟稳定。
关联流程变化：开放系统认证（形式）→ 关联（携带 RSN IE 协商 CCMP）→ 4-Way Handshake 建立 PTK/GTK → 数据加密。
2017年 KRACK（Key Reinstallation Attack）攻击揭示 4-Way Handshake 的重放漏洞，但通过补丁可修复。离线字典攻击始终是 PSK 模式的威胁。
2018 — Wi-Fi Alliance WPA3
WPA3 = SAE + 前向保密 + PMF 强制
WPA3-Personal：用 SAE（Simultaneous Authentication of Equals / Dragonfly）替代 PSK，基于椭圆曲线 Diffie-Hellman 的零知识证明，密码从不在空中传输，抵抗离线字典攻击，提供前向保密。
WPA3-Enterprise：192-bit 安全套件（GCMP-256 + HMAC-SHA-384）。
Enhanced Open（OWE）：开放网络自动加密，无需密码，每客户端唯一密钥。
PMF（Protected Management Frames）强制：管理帧加密，防止伪造 Deauth/Disassoc 攻击。
2019+ — Wi-Fi 6/6E (802.11ax) / Wi-Fi 7 (802.11be)
关联流程的效率革命
Wi-Fi 6 引入 TWT（Target Wake Time）：在关联时协商唤醒时间表，提升物联网省电性能。BSS Coloring 减少空间复用干扰。MU-OFDMA 允许多 STA 同时在不同子载波上接入。Wi-Fi 7（802.11be）新增 Multi-Link Operation（MLO）：STA 可同时在多个频段（2.4/5/6 GHz）建立逻辑链路，关联过程扩展为多链路协商。

11. WPA3 SAE（Dragonfly）流程详解

SAE 是 WPA3-Personal 的核心创新。它将原来简单的"开放认证"替换为真正的密码认证——基于椭圆曲线密码的 Dragonfly 握手。SAE 的关键特性是：密码本身永不在空中传输，即使攻击者监听全过程，也无法离线暴力破解密码，并且每次会话生成独立的 PMK（前向保密）。

12. 第一性原理总结

第一性原理（First Principles）要求我们剥离历史偶然性和技术细节的外壳，回归最基本的问题：WiFi 关联流程究竟在解决什么根本问题？每一个协议设计决策背后的最深层逻辑是什么？

🎯 根本问题 1：无线信道是共享且不可信的

与有线网络不同，802.11 的物理介质（射频空间）对所有设备开放，任何人都可以监听、干扰、伪造。因此，关联流程必须解决三个不可回避的根本问题：

① 发现（Discovery）：在无中心协调的射频空间里，两个设备如何"找到彼此"？→ Beacon/Probe 机制解决。

② 协商（Negotiation）：双方能力不同，如何就速率、信道宽度、安全算法等达成一致？→ Auth/Association Request/Response 帧交换解决。

③ 信任（Trust）：如何在不安全信道上建立安全通道？→ 4-Way Handshake 用公开交换的随机数（Nonce）+ 共享秘密（PMK），通过 PRF 派生会话密钥，不传输秘密本身。

🧮 根本问题 2：对称密码学的密钥分发困境

加密需要共享密钥，但在不安全信道上协商密钥本身就是悖论。802.11 的解法：

WPA2 PSK 模式：预先在带外（Out-of-Band）共享密码（物理交换），用 PBKDF2 派生 PMK，再通过 4-Way Handshake 从 PMK 派生会话密钥（PTK/GTK）。Nonce 公开交换，PMK 不暴露，攻击者得不到 PTK。

WPA3 SAE 模式：用椭圆曲线 Diffie-Hellman 进一步解决"PMK 本身是固定的"问题——每次会话的共享秘密 S 由临时随机数决定，即使密码泄露，历史会话密钥也无法推导（前向保密的本质）。

⚖️ 根本问题 3：状态机是协议复杂性的最小表达

关联流程的 4 个状态（未认证未关联 → 已认证未关联 → 已关联 → RSNA）不是任意设计，而是对"允许哪些操作"的最小化约束集合：

• 状态 1 → 2：建立"我认识你"（Identity）——历史上用于 WEP，现代用于向后兼容占位

• 状态 2 → 3：建立"我接受你"（Admission）——AP 确认 STA 能力兼容，分配资源（AID）

• 状态 3 → 4：建立"我信任你"（Trust）——密码学证明双方持有相同 PMK，派生加密密钥

每一次状态跳变都对应一个不可缺少的逻辑层次。任何简化都会破坏安全性或兼容性。

🔄 演化规律：被动适应 → 主动防御 → 零信任

WEP（1997）：假设攻击者不存在，用简单加密提供"等同有线"的安全幻觉。

WPA/WPA2（2003-2004）：承认攻击者存在，用 4-Way Handshake 分离会话密钥与长期秘密，用 MIC/CCMP 提供真实的完整性保护。但 PMK 固定导致前向保密缺失。

WPA3（2018）：接受"密码可能最终泄露"的假设，通过 SAE 的临时秘密保证前向保密；通过 PMF 保护管理帧防止脱离攻击；通过 OWE 让开放网络也有加密。

→ 第一性原理结论：WiFi 关联流程的每一次演化，本质上是对"信任假设"的逐步收紧：从假设信道可信，到假设信道不可信，再到假设长期秘密也可能泄露。这是安全工程从"边界防御"走向"零信任架构"的必然轨迹。

📡 物理层约束决定协议上界

WiFi 协议的所有设计都受制于射频物理规律：

• 半双工（不能同时收发）→ CSMA/CA 碰撞避免、ACK 确认机制的必要性

• 广播介质（所有人可听）→ 加密在协议层必须实现（不能假设物理层隔离）

• 信号衰减（覆盖有限）→ 漫游机制（Reassociation、PMK Caching、Fast BSS Transition）的必要性

• 频谱共享（干扰不可避免）→ RTS/CTS、NAV 机制；频段演化（2.4→5→6GHz）本质是逃离拥塞

→ 802.11 协议栈的每一层设计，都是对物理约束的工程化应答。理解物理约束，才能理解协议设计的必然性。
