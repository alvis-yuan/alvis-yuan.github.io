IEEE 802.11 / Wi-Fi

WiFi 协议栈深度解析

WiFi 是基于 IEEE 802.11 标准的无线局域网（WLAN）技术族，由 IEEE（电气和电子工程师协会）标准化。802.11 标准规定了物理层（PHY）和介质访问控制层（MAC）的规范，而上层（LLC、IP、TCP、应用层）与有线以太网保持一致，从而实现无缝桥接。

与蓝牙协议栈的设计对比

蓝牙（IEEE 802.15）和 WiFi（IEEE 802.11）都工作在 2.4 GHz ISM 频段，但设计目标与协议栈架构有本质区别。

🔵 蓝牙（IEEE 802.15）

  • 目标：短距离设备互联、电缆替代
  • 范围：10m（PAN 个人域网）
  • 拓扑：主从 Piconet（1主7从）
  • 信道访问：FHSS 跳频（1600次/秒）
  • MAC：主节点集中轮询调度
  • 协议栈：专有协议栈（L2CAP、RFCOMM、SDP）
  • 功耗：超低功耗设计（BLE）
  • 带宽：1~3 Mbps（经典）
  • 上层：独立应用层 Profile，不直接承载 IP

📶 WiFi（IEEE 802.11）

  • 目标：高速无线局域网，替代有线以太网
  • 范围：100m+（LAN 局域网）
  • 拓扑：基础设施 BSS（AP 接入点）或 IBSS
  • 信道访问：CSMA/CA 载波侦听碰撞避免
  • MAC：分布式 DCF 或集中 PCF
  • 协议栈：与 IEEE 802.3 兼容，标准 IP 栈
  • 功耗：相对较高（有省电模式）
  • 带宽：11 Mbps ~ 9.6 Gbps（Wi-Fi 6）
  • 上层：直接承载 TCP/IP，透明集成互联网

WiFi 整体协议栈架构

IEEE 802.11 标准定义了 OSI 模型中物理层和数据链路层的两个子层——PHY（物理层）和 MAC（介质访问控制）子层。上层与标准 IEEE 802.3 以太网兼容，通过桥接实现无缝互通。

图2：IEEE 802.11 协议栈与有线以太网的桥接关系（引用自 Tanenbaum《Computer Networks》5th Ed. P.300）

📌 关键设计思路：IEEE 802.11 通过在 LLC 层"隐藏"不同 802.11 版本（a/b/g/n/ac/ax）之间的差异，使网络层及其以上看来与有线以太网完全一致。这是与蓝牙协议栈"端到端独立"设计的根本区别。

物理层（PHY）

IEEE 802.11 物理层对应 OSI 模型的物理层，分为两个子层：PLCP（物理层会聚协议）和 PMD（物理介质相关子层）。PHY 层负责原始比特的无线传输，定义频率、调制方式和速率。

📌 来源（Tanenbaum《Computer Networks》P.301）：「所有 802.11 技术都使用短程无线电，在 2.4 GHz 或 5 GHz ISM 频段发送信号。这些频段是未授权的，与很多其他设备（如车库门遥控器、微波炉）共享。」

PMD 子层 — 调制技术演进

标准频段调制方式最高速率要解决的核心问题
802.11b 2.4 GHz DSSS（直接序列扩频）+ CCK 11 Mbps 提高抗干扰性；Barker序列自相关特性实现同步
802.11a 5 GHz OFDM（正交频分复用） 54 Mbps 利用5GHz减少拥塞；52子载波并行传输（48数据+4同步）
802.11g 2.4 GHz OFDM（兼容802.11b） 54 Mbps 在2.4GHz实现54Mbps；向下兼容802.11b
802.11n 2.4/5 GHz OFDM + MIMO（多天线） 600 Mbps 100Mbps有效吞吐；40MHz信道+MIMO多空间流
802.11ac 5 GHz VHT OFDM + MU-MIMO 6.93 Gbps 多用户同时服务；256-QAM；最大160MHz信道
802.11ax (Wi-Fi 6) 2.4/5/6 GHz HE OFDMA + MU-MIMO 9.6 Gbps 密集场景效率；OFDMA资源分配；TWT节能

速率自适应（Rate Adaptation）

所有 802.11 传输方式都定义了多种速率。如果信号弱，使用低速率（更健壮的调制）；如果信道清晰，使用最高速率。这个自动调整过程称为速率自适应（Rate Adaptation）。

来源：Tanenbaum《Computer Networks》P.301

PLCP 子层帧结构（PPDU）

PLCP（Physical Layer Convergence Protocol，物理层会聚协议）在 MAC 帧（PSDU）前添加前导码和帧头，形成 PPDU（PLCP Protocol Data Unit），完成物理层同步和参数协商。

字段大小说明解决的问题
Sync（同步） 80 bits 80位交替 0/1 序列（01010101...） 接收端锁定信号时序，完成时钟同步和频率偏移校正

Service（服务） 8 bits 保留用于扩展；部分比特标记调制方式（CCK/PBCC） 兼容性和扩展标志位
Length（长度） 16 bits PSDU 的传输所需微秒数；MAC层据此判断 PPDU 结束 接收端知道何时停止接收，防止截断或过度接收
CRC（帧校验） 16 bits 对 Signal/Service/Length 三字段的 CRC 校验 保护帧头完整性，防止头部损坏导致误解析
PSDU（数据） 可变 MAC 层传下来的帧（即 MAC MPDU） 承载实际数据负载

MAC 子层

MAC（Medium Access Control，介质访问控制）子层是 IEEE 802.11 的核心，负责管理对共享无线介质的访问，提供帧格式化、寻址、错误检测等数据链路层服务。

📌 来源（Tanenbaum P.303）：「802.11 MAC 子层与以太网 MAC 子层的不同原因有两个：一是无线电几乎总是半双工的；二是不同站点的传输范围可能不同。」

MAC 两大子层功能模式

DCF — 分布式协调功能

  • 机制：CSMA/CA 载波侦听碰撞避免
  • 控制：各站点独立、无中心控制
  • 服务：仅异步数据服务
  • 场景：通用数据传输（强制实现）
  • 优先级：使用 DIFS 帧间间隔

PCF — 点协调功能

  • 机制：集中轮询（Point Coordinator）
  • 控制：AP 集中管理，无竞争
  • 服务：异步 + 时限服务
  • 场景：时间敏感传输（如语音，可选实现）
  • 优先级：使用 PIFS（比 DIFS 短）

MAC 帧结构详解

IEEE 802.11 MAC 帧由一组固定顺序的字段构成。所有帧都包含 Frame Control、Duration/ID、Address 1 和 FCS 四个最小字段。其余字段根据帧类型和子类型有所不同。

① Frame Control 字段（2字节 = 16位）

子字段大小含义与取值解决的问题
Protocol Version 2 bits 当前固定为 00，未来版本扩展用 协议版本标识
Type 2 bits 00=管理帧 01=控制帧 10=数据帧 11=保留/扩展 区分帧的功能大类
Subtype 4 bits 细化帧类型：如 1000=Beacon，1011=RTS，1100=CTS，1101=ACK 精确标识帧用途；最多支持16种子类型
To DS 1 bit 置1表示帧目的地为分布式系统（DS） 确定帧流向：STA→AP→DS
From DS 1 bit 置1表示帧来自分布式系统（DS） 确定帧流向：DS→AP→STA；两位组合决定4个地址字段的语义
More Fragments 1 bit 置1表示后续还有分片 大帧分片传输，减少噪声影响；接收端知道等待更多片段
Retry 1 bit 置1表示这是之前已发送帧的重传 接收端过滤重复帧；配合 Sequence Control 使用
Power Management 1 bit 置1表示发送方成功传输后将进入省电模式 通知 AP 缓冲该 STA 的帧；节省终端电量
More Data 1 bit 置1表示发送方还有更多帧要发 AP 通知省电 STA 还有缓冲帧待取；避免 STA 过早休眠
Protected Frame 1 bit 置1表示帧体已加密（WEP/TKIP/CCMP） 标识数据是否受安全保护
Order 1 bit 置1表示高层要求帧必须严格按序处理 保证特定数据流的有序传输

该字段的含义取决于最高有效位（MSB）：

⑤ Frame Body（帧体）与 FCS

字段大小说明
Frame Body（数据）0~2312 Bytes承载高层数据（IP包、LLC帧等）。最大 MSDU 为 2304 字节；含加密开销最大 2312 字节。帧体的前几字节为 LLC 头，标识高层协议类型。

LLC 逻辑链路控制层（IEEE 802.2）

LLC 子层位于 MAC 子层之上，是 IEEE 802.11 协议栈中连接无线 MAC 与上层网络协议（IP）的桥梁。其核心功能是屏蔽不同 802.11 物理层版本的差异，使网络层看到统一的数据链路接口。

📌 来源（Tanenbaum P.300）：「LLC（逻辑链路层）子层对网络层屏蔽了不同 802.11 版本之间的差异。」这使得 802.11a/b/g/n/ac/ax 对上层 IP 协议完全透明。

CSMA/CA 与 DCF（分布式协调功能）

802.11 使用 CSMA/CA（载波侦听多路访问 + 碰撞避免）协议。与以太网的 CSMA/CD（碰撞检测）不同，无线信道无法在发送的同时检测碰撞，因此改为碰撞避免策略。

📌 来源（Tanenbaum P.303-304）：「无线电几乎总是半双工的……CSMA/CA 类似以太网 CSMA/CD：使用信道侦听和指数退避，但不是检测到碰撞后才退避，而是立即退避（除非发送方最近未使用信道且信道空闲）。」

RTS/CTS 机制与 NAV（虚拟载波侦听）

隐藏终端问题（Hidden Terminal Problem）：两个站可能都在 AP 范围内，但互相听不到对方的信号，从而同时发送造成碰撞。RTS/CTS + NAV 机制正是为解决此问题而设计。

💡 NAV（网络分配向量）工作原理（来源：Tanenbaum P.305；ScienceDirect NAV）：每个站维护一个 NAV 计时器。当站收到含非零 Duration 字段的帧（RTS/CTS/DATA），它将 NAV 设为该值并倒计时。NAV > 0 时不尝试物理载波侦听（虚拟载波侦听）。这是一种节能且有效的信道预约机制。

传输机会（TXOP）

来源（Tanenbaum P.309）：「此前，CSMA/CA 每次只允许发送一帧，这对高速站不公平。传输机会（Transmission Opportunity）使每个站拥有相等的信道时间，而非相等的帧数。」这确保不同速率的站都能获得公平的空中时间。

电源管理

802.11 定义了省电模式（Power Save Mode），允许 STA 在不活跃时关闭无线电，由 AP 缓冲帧，在 STA 醒来时再传输。

📖 来源：Tanenbaum P.307 — 省电模式

省电模式工作机制

1
通知 AP：STA 在发送给 AP 的帧中将 Power Management 位（Frame Control）置1，告知 AP 自己将进入省电模式。
2
AP 缓冲帧：在 STA 休眠期间，AP 缓冲所有目的为该 STA 的帧，不立即发送。
3
Beacon + TIM：AP 按间隔（默认约 102ms）广播 Beacon 帧，帧体含 TIM（Traffic Indication Map）比特图，标记哪些 STA 有缓冲帧。STA 在每个 Beacon 时刻醒来检查 TIM。
4
PS-Poll 取数：若 TIM 中有自己的比特，STA 发送 PS-Poll 控制帧（含 AID），AP 响应并发送缓冲帧，More Data=1 表示还有更多帧。
5
恢复活跃：STA 将 Power Management 位清零，恢复正常工作状态。

Wi-Fi 6 (802.11ax) 引入 TWT（Target Wake Time）进一步精细化省电调度。

第一性原理总结

从根本问题出发

WiFi 的所有设计，都围绕一个核心问题：如何在不可靠、半双工、共享的无线介质上，高效、公平、安全地传输数据——同时让上层网络协议毫无感知地运行？

1
问题：无线信道是共享的，会冲突。
→ 解法：CSMA/CA + 随机退避（DCF）。无法检测碰撞（半双工），故改为碰撞避免。退避窗口指数增长保证公平性与快速恢复。
2
问题：听不到对方发送（隐藏终端）导致碰撞。
→ 解法：RTS/CTS 握手 + NAV 虚拟载波侦听。让接收方附近的所有站知道信道繁忙，而非只依赖物理侦听。NAV 计时器无需持续侦听，还能节能。
3
问题：无线信号易出错，没有碰撞检测。
→ 解法：强制 ACK 确认机制（与以太网不同）。无 ACK 则判定失败重传。分片（Fragmentation）减小帧尺寸降低误码影响。速率自适应（Rate Adaptation）动态选择健壮调制。
4
问题：不同应用对延迟/带宽需求不同（QoS）。
→ 解法：多级帧间间隔（SIFS < PIFS < DIFS < AIFS）；EDCA 4个访问类别按 AIFS 竞争；TXOP 保证每站公平的空中时间，而非帧数。
5
问题：终端设备需要省电。
→ 解法：AP 作为"代理"缓冲帧；STA 省电休眠只在 Beacon 时刻醒来检查 TIM；Wi-Fi 6 的 TWT 更精确调度每个设备的唤醒时刻。
6
问题：无线传输不安全（广播天然可监听）。
→ 解法：WPA2/WPA3 的 AES 加密 + 4次握手密钥协商。Frame Control 的 Protected Frame 位标记加密状态；认证与关联流程保证接入控制。
7
问题：协议需要持续演进以提高速率，但不能破坏上层生态。
→ 解法：LLC 子层屏蔽物理层版本差异；PLCP 前导码设计兼容旧设备（Mixed Mode）；IP/TCP 对无线完全透明。这是 WiFi 与蓝牙最根本的架构差异——WiFi 是以太网向空中的延伸，而蓝牙是独立设计的设备互联协议。
8
问题：密集场景（多用户）效率崩溃。
→ 解法（Wi-Fi 6）：OFDMA 将信道细分为资源单元（RU），AP 集中调度多用户同时传输，彻底突破 DCF 的单次一帧限制；BSS Coloring 允许空间复用，减少不必要退避。

核心设计哲学对比（一句话）

蓝牙：「一个主人管理多个从属，在嘈杂的 ISM 频段通过跳频保证连接——为设备互联而生。」

WiFi：「在共享无线信道上，通过分布式竞争和礼貌协商，让每个设备平等地接入互联网——以太网的无线化延伸。」
