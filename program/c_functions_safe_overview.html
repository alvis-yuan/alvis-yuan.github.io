C99 标准库头文件
安全性全面分析

C99（ISO/IEC 9899:1999）定义了 24 个标准头文件，涵盖 I/O、字符串、数学、内存管理等基础设施。本文档从 可重入性、线程安全性 和 异步信号安全性 三个维度，逐一分析每个头文件的接口特性。

核心概念

§1 三大安全属性详解

导读
理解这三个概念是分析 C 标准库接口的基础。它们并非包含关系——可重入 ⊆ 异步信号安全，但线程安全与可重入是正交的。

1.1 可重入（Reentrant）

若一个函数可以在其执行过程中被中断，然后在中断上下文中被再次调用，且两次调用都能正确完成，则称该函数是可重入的。可重入函数不依赖任何全局或静态可变状态。

POSIX 定义：若一个函数被两个或更多线程并发调用，其效果与各线程顺序独立执行的效果相同，则称该函数可重入。

1.2 线程安全（Thread-Safe）

函数在多线程环境下并发调用时能产生正确结果。线程安全通常通过互斥锁（mutex）等同步机制实现。线程安全不要求可重入——使用 mutex 保护的函数在信号处理中可能死锁。

陷阱：malloc 是线程安全的（内部有锁），但不是可重入的。若信号处理函数在 malloc 持有锁期间调用 malloc，将产生死锁。

1.3 异步信号安全（Async-Signal-Safe）

函数可以安全地在信号处理函数（signal handler）中调用。POSIX.1 明确列出了必须支持异步信号安全的函数集合（约 180 个）。C99 标准库函数中绝大多数不在此列表中。

1.4 三者关系总结

属性定义关键字保护机制可在信号处理中调用？
可重入无全局/静态状态纯粹本地变量✅ 是
线程安全并发正确性mutex/原子操作⚠️ 通常不行（锁死）
异步信号安全POSIX 明确列出原子/可重入✅ 是（POSIX 保证）

§2 <assert.h> — 断言诊断

导读
assert.h 是最简单的头文件，仅提供 assert() 宏。其行为受 NDEBUG 宏控制：定义 NDEBUG 后，assert 展开为空表达式。安全性分析极为简单。
<assert.h> 运行时断言检查宏
可重入 ✓ 线程安全 △ 信号安全 ✗
接口类型可重入线程安全信号安全说明
assert(expr)宏 条件 条件 ✗ 失败时调用 fprintf+abort，均非信号安全
static_assert宏(C11) 编译期 编译期 编译期 编译期检查，无运行时开销
注意：assert() 失败时内部调用 fprintf(stderr, ...) 和 abort()。fprintf 不是异步信号安全的；abort() 在 POSIX.1-2001 TC1 后是信号安全的，但组合使用仍不安全。

用法示例

// 仅在调试构建中启用断言
#include <assert.h>

int divide(int a, int b) {
    assert(b != 0);  // 不满足则打印错误信息并终止
    return a / b;
}
// 发布版本：gcc -DNDEBUG main.c  → assert 被优化掉

§3 <complex.h> — 复数运算

导读
C99 新增的复数运算支持。所有函数操作纯数值，不访问任何全局状态（除浮点异常环境 fenv），因此整体安全性良好。但涉及浮点环境的函数在多线程下需注意。
<complex.h> 复数类型与数学运算（C99）
可重入 ✓ 线程安全 ✓ 信号安全 △

复数函数（cabs, carg, cexp, clog, csin, ccos 等）均为纯计算函数，不使用静态/全局状态。

函数组代表函数可重入线程安全信号安全
模与辐角cabs, carg 依赖 fenv
三角函数csin, ccos, ctan, casin, cacos, catan
双曲函数csinh, ccosh, ctanh …
指数/对数cexp, clog, cpow, csqrt
构造/分量creal, cimag, conj, cproj
构造与分量函数（creal, cimag, conj, cproj）纯粹操作参数，可视为异步信号安全。
#include <complex.h>
#include <stdio.h>

int main() {
    double complex z = 3.0 + 4.0 * I;
    printf("|z| = %.1f\n", cabs(z)); // 输出: |z| = 5.0
    printf("arg = %.4f rad\n", carg(z));
}

§4 <ctype.h> — 字符分类与转换

导读
字符分类函数族是最常用的接口之一。关键陷阱：这些函数的行为受当前 locale 影响。setlocale() 修改全局 locale 时，字符函数的行为也随之改变，因此在多线程环境中存在数据竞争风险。
<ctype.h> 字符分类与大小写转换
可重入 △ 线程安全 △ 信号安全 ✗
函数功能可重入线程安全信号安全说明
isalpha, isdigit, isalnum字母/数字分类受 locale 影响
isspace, isblank空白检测受 locale 影响
isupper, islower大小写检测受 locale 影响
toupper, tolower大小写转换受 locale 影响
ispunct, iscntrl, isprint, isgraph, isxdigit其他分类受 locale 影响
多线程陷阱：若一个线程正在调用 setlocale()，而另一线程正在调用 isalpha()，则行为未定义（数据竞争）。在固定 locale 的程序中，这些函数实际上是线程安全的。
#include <ctype.h>
// 安全用法：固定 C locale，函数行为确定
int is_identifier_char(int c) {
    return isalnum(c) || c == '_';
}
// 多线程中，避免并发调用 setlocale

§5 <errno.h> — 错误号

导读
errno 是 C 标准库错误处理的核心机制。在现代实现中，errno 是线程局部变量（Thread-Local Storage），因此是线程安全的。但在信号处理中使用 errno 需要特别小心。
<errno.h> 错误号宏与线程局部变量
可重入 ✓ 线程安全 ✓ 信号安全 △
接口类型可重入线程安全信号安全说明
errno宏（展开为左值）现代实现为 TLS；信号中需保存/恢复
EDOM, ERANGE, EILSEQ常量宏编译期常量
信号处理中正确使用 errno： 
void sig_handler(int sig) {
    int saved_errno = errno;   // 必须先保存
    write(1, "signal\n", 7); // 信号安全的 I/O
    errno = saved_errno;       // 必须恢复
}

POSIX.1 明确：在信号处理函数中访问/修改 errno 是合法的，前提是在进入处理函数时保存、返回前恢复其值。

§6 <fenv.h> — 浮点环境控制

导读
C99 新增，提供对浮点异常标志和舍入模式的访问。浮点控制寄存器在许多架构上是进程/线程级全局资源，在多线程环境下修改浮点舍入模式存在竞争风险。
<fenv.h> 浮点异常与舍入模式控制（C99）
可重入 ✗ 线程安全 ✗ 信号安全 ✗
函数功能可重入线程安全信号安全
fegetround / fesetround获取/设置舍入模式
feclearexcept / feraiseexcept清除/触发浮点异常
fetestexcept测试浮点异常标志
fegetenv / fesetenv保存/恢复完整浮点环境
feholdexcept / feupdateenv保存环境并清除异常
浮点状态寄存器（MXCSR 等）在 x86 架构上是每线程的，但 C 标准对此无保证。修改舍入模式会影响同线程的所有浮点计算，使用时需极度谨慎。
#include <fenv.h>
#pragma STDC FENV_ACCESS ON  // 告知编译器访问浮点环境

fesetround(FE_UPWARD);  // 设置向上舍入（非线程安全！）
double r = 1.0 / 3.0;   // 结果受舍入模式影响
fesetround(FE_TONEAREST); // 恢复默认

§7 <float.h> — 浮点数特性宏

导读
float.h 仅定义编译期常量宏，无任何函数或运行时状态。完全不存在安全性问题。
<float.h> 浮点数精度与范围的编译期常量
仅宏定义 完全安全

该头文件仅包含宏定义（FLT_MAX、DBL_EPSILON、LDBL_DIG 等），均为编译期常量。无函数，无运行时状态，在任何上下文中均安全使用。

#include <float.h>
// 所有宏都是编译期常量，完全安全
printf("double 精度: %d 位十进制\n", DBL_DIG); // 通常 15
printf("double 最大值: %e\n", DBL_MAX);       // ≈ 1.8e308
printf("double epsilon: %e\n", DBL_EPSILON);   // ≈ 2.2e-16

§11 <locale.h> — 本地化支持

导读
locale.h 是最危险的头文件之一。setlocale() 修改的是进程级全局状态，会影响所有线程中的 ctype.h、stdio.h（数字格式）等函数的行为。在多线程程序中，setlocale() 的调用必须发生在创建线程之前。
<locale.h> 本地化环境控制
可重入 ✗ 线程安全 ✗ 信号安全 ✗
函数可重入线程安全信号安全说明
setlocale(category, locale)修改进程级全局 locale 状态
localeconv()返回指向静态 lconv 结构的指针
严重警告：localeconv() 返回指向内部静态结构的指针。在多线程程序中，调用 setlocale() 后，该结构内容可能被修改，导致其他线程读取到错误数据。 
// 正确做法：在创建任何线程前完成 locale 设置
int main() {
    setlocale(LC_ALL, "");  // 在 main 开头，线程创建前
    // ... 之后创建线程 ...
    return 0;
}

§12 <math.h> — 数学函数

导读
数学函数通常是纯计算函数，可重入性和线程安全性良好。但需注意：
1. 通过 errno 或浮点异常（fenv）报告错误，信号处理中需保存 errno
2. 历史上部分实现将结果写入全局变量（已废弃），现代实现均返回值传递
3. C99 增加了大量新函数（fma, round, trunc, log2, cbrt 等）
<math.h> 数学运算函数（C99 大幅扩展）
可重入 ✓ 线程安全 ✓ 信号安全 △
函数组代表函数（含 C99 新增）可重入线程安全信号安全
三角函数sin, cos, tan, asin, acos, atan, atan2*
双曲函数sinh, cosh, tanh, asinh(C99), acosh(C99), atanh(C99)
指数/对数exp, log, log10, log2(C99), exp2(C99), expm1(C99), log1p(C99)
幂/根pow, sqrt, cbrt(C99), hypot(C99)
取整（C99新增）round, lround, llround, trunc, floor, ceil, nearbyint, rint
浮点操作（C99）fma, fmax, fmin, fdim, scalbn, logb, ilogb
分类宏（C99）isnan, isinf, isfinite, isnormal, signbit, fpclassify
错误处理math_errhandling, MATH_ERRNO, MATH_ERREXCEPT

* 异步信号安全性标记为 △ 表示：函数本身纯粹，但可能通过 errno 或 feraiseexcept 与浮点状态交互，在信号处理中需保存/恢复 errno。

C99 引入了 float 和 long double 版本（sinf, sinl 等），以及类型泛型宏（见 tgmath.h）。取整函数族（round, trunc）不修改 errno，信号安全性更好。
#include <math.h>
// C99 新增：fma 融合乘加，精度更高且通常是信号安全的
double r = fma(2.0, 3.0, 4.0); // 2*3+4 = 10.0，单次舍入

// C99 新增分类宏（宏，非函数）
if (isnan(r) || isinf(r)) {
    // 处理特殊值
}

§13 <setjmp.h> — 非局部跳转

导读
setjmp/longjmp 实现非局部跳转，常用于错误恢复。关键危险：在信号处理函数中使用 longjmp 跳出，会跳过正常的函数返回路径，导致资源泄漏、互斥锁未释放等严重问题。
<setjmp.h> 非局部跳转（setjmp/longjmp）
可重入 ✗ 线程安全 △ 信号安全 △
接口类型可重入线程安全信号安全说明
setjmp(env)宏保存调用环境到 jmp_buf；jmp_buf 本身不是线程共享的
longjmp(env, val)函数POSIX.1-2008 TC2 列为信号安全，但有重大限制
POSIX 的严重警告（signal-safety(7)）：如果信号处理函数通过 longjmp/siglongjmp 终止，且程序随后调用了任何非信号安全函数，则行为未定义。实践中极度危险，应避免在信号处理中使用。 
#include <setjmp.h>
#include <stdio.h>

jmp_buf env;

void risky_op() {
    longjmp(env, 1);  // 跳回 setjmp 处，val=1
}

int main() {
    if (setjmp(env) == 0) {
        risky_op();    // 正常执行路径
    } else {
        printf("从 longjmp 恢复\n"); // 错误恢复路径
    }
}

§14 <signal.h> — 信号处理

导读
信号处理是异步信号安全性的核心场景。signal() 本身被 POSIX 列为信号安全函数，但 C 标准规定的信号处理函数内只能调用极少数函数。信号处理函数中的可用操作非常受限。
<signal.h> 信号注册与发送
可重入 △ 线程安全 △ 信号安全 △
接口可重入线程安全信号安全说明
signal(signum, handler)POSIX 明确列为异步信号安全
raise(sig)POSIX 明确列为异步信号安全
sig_atomic_t（类型）原子访问保证，信号处理中安全读写
C 标准（C11 §7.14.1.1）规定：信号处理函数只能安全调用以下函数：signal()（用于同一信号）、_Exit()，以及对 volatile sig_atomic_t 变量的赋值。其余均为未定义行为。
#include <signal.h>
#include <stdio.h>

volatile sig_atomic_t got_sigint = 0;

void handler(int sig) {
    got_sigint = 1;  // ✅ 唯一安全的非平凡操作
    // ❌ 禁止: printf, malloc, exit, pthread_mutex_lock ...
}

int main() {
    signal(SIGINT, handler);
    while (!got_sigint) {
        /* 正常工作 */
    }
    printf("收到 SIGINT，退出\n"); // ✅ 在主程序中安全
}

§15 <stdarg.h> — 可变参数

<stdarg.h> 可变参数列表访问宏
可重入 ✓ 线程安全 ✓ 信号安全 ✓

va_list、va_start、va_arg、va_end、va_copy（C99新增）均为编译器内建宏/类型，操作的是函数调用栈上的局部数据，不访问任何全局状态。完全可重入、线程安全、信号安全。

#include <stdarg.h>
int sum(int n, ...) {  // 求 n 个整数之和
    va_list ap;
    va_start(ap, n);
    int total = 0;
    for (int i = 0; i < n; i++)
        total += va_arg(ap, int);
    va_end(ap);
    return total;
}
// va_copy(C99): 安全复制 va_list 给另一个函数使用

§18 <stdint.h> — 固定宽度整数类型

§19 <stdio.h> — 标准 I/O

导读
stdio.h 是最常用但安全性问题最多的头文件。所有缓冲 I/O 函数（printf, fread, fwrite 等）维护内部缓冲区和全局状态，一律不是异步信号安全的。线程安全性通过 flockfile/funlockfile 机制实现（POSIX）。C99 新增了 snprintf、vsnprintf 等更安全的格式化函数。
<stdio.h> 标准输入/输出库（C99 新增 snprintf/vsnprintf）
可重入 ✗ 线程安全 △ 信号安全 ✗

§23 <time.h> — 时间与日期

导读
时间函数是安全性问题最集中的区域之一。gmtime、localtime、ctime、asctime 均返回指向静态内部结构/缓冲区的指针，是教科书级别的不可重入、非线程安全函数。POSIX 提供了 _r 后缀的线程安全版本。
<time.h> 时间测量与日历时间
部分不可重入 ✗ 部分非线程安全 ✗ 信号安全 ✗
函数可重入线程安全信号安全说明
time(t)POSIX 信号安全，返回值传递
difftime(t1, t2)纯计算，完全安全
mktime(tm)依赖 locale/时区，修改传入结构
gmtime(timer)⚠️ 返回指向静态 tm 结构的指针
localtime(timer)⚠️ 返回指向静态 tm 结构的指针，依赖时区
asctime(tm)⚠️ 返回指向静态字符串的指针

gmtime、localtime、asctime、ctime 这四个函数共享同一个或两个静态缓冲区。多次调用会互相覆盖结果。POSIX 线程安全替代：gmtime_r、localtime_r、asctime_r、ctime_r。

struct tm *local = localtime(&t);  // ⚠️ 可能覆盖 gmt 指向的内容！

// ✅ 正确：使用 _r 版本（POSIX）
struct tm gmt_buf, local_buf;
gmtime_r(&t, &gmt_buf);
localtime_r(&t, &local_buf);

// strftime 是相对安全的格式化选择
char buf[64];
strftime(buf, sizeof(buf), "%Y-%m-%d %H:%M:%S", &local_buf);

关键结论与最佳实践

🏆 在信号处理函数中的黄金法则：
1. 只设置 volatile sig_atomic_t 标志，在主程序中处理信号
2. 若必须 I/O，使用 POSIX write() 而非 printf()
3. 若必须退出，使用 _Exit() 或 abort()，而非 exit()
4. 禁止：malloc/free、所有 stdio 函数、setlocale、strtok
🔒 在多线程程序中的关键原则：
1. setlocale() 必须在创建任何线程之前调用
2. 用 strtok_r、gmtime_r、localtime_r、strerror_r（POSIX）代替不安全版本
3. rand() 非线程安全，使用 rand_r()（POSIX）或 C11 <stdlib.h> 的 PRNG
4. malloc/free 线程安全，但避免频繁跨线程分配/释放（性能）
5. errno 在 POSIX 实现中为 TLS，每线程独立
