Linux 内核通知链深度解析

基于 Linux 内核官方源码（v6.x）• 涵盖原理、实现、用户态移植与实战示例
源码参考： 📄 include/linux/notifier.h 📄 kernel/notifier.c

📖1. 什么是通知链

通知链（Notifier Chain）是 Linux 内核提供的一种发布-订阅（Pub/Sub）事件通知机制， 允许内核子系统之间以松耦合的方式传递状态变化事件。其核心代码位于 kernel/notifier.c， 头文件为 include/linux/notifier.h，由 Alan Cox 最初设计引入。

从本质上讲，通知链是一个按优先级有序排列的单向链表， 链上每个节点（notifier_block）携带一个回调函数指针。 当事件发生时，内核遍历链表，依次调用每个注册者的回调函数。

ℹ️
官方注释原文（include/linux/notifier.h）：
"Routines to manage notifier chains for passing status changes to any interested routines. We need this instead of hard coded call lists so that modules can poke their nose into the innards. The network devices needed them so here they are for the rest of you." —— Alan Cox <Alan.Cox@linux.org>

🎯2. 设计背景与动机

Linux 内核各子系统之间存在强烈的相互依赖。例如，网络子系统需要知道网络设备（NIC）何时上线/下线； 电源管理子系统需要通知各驱动进入休眠；CPU 热插拔需要告知调度器、内存管理子系统等。

如果采用硬编码调用列表的方式，每次增加新的订阅者都需要修改发布者的源码， 这会导致内核模块间的耦合度极高，使得内核模块（.ko）无法独立地"插入"到已有子系统的事件流中。 通知链机制解决了这一问题。

📋
官方说明（kernel/notifier.c 注释）：
"We need this instead of hard coded call lists so that modules can poke their nose into the innards."
——即：模块需要能动态地介入内核内部事件，而不是通过静态硬编码调用。

核心设计思想

  • 1解耦：发布者与订阅者无需互相了解，只通过通知链头（notifier_head）间接交互
  • 2动态扩展：模块在运行时（insmod/rmmod）可注册或注销回调，无需重新编译内核
  • 3优先级控制：同一链上的 notifier_block 按 priority 字段从高到低排列执行
  • 4可中止：任意回调返回 NOTIFY_STOP 或 NOTIFY_BAD 可立即终止后续调用
  • 5并发安全：根据上下文提供原子/信号量/SRCU 等不同的保护机制

🔬3. notifier_block 核心结构

所有通知链的核心数据结构均定义在 include/linux/notifier.h 中。 最基础的结构是 notifier_block： 

/* include/linux/notifier.h (Linux v6.x) */

typedef int (*notifier_fn_t)(struct notifier_block *nb,
                unsigned long action, void *data);

struct notifier_block {
    notifier_fn_t             notifier_call;  /* 回调函数指针 */
    struct notifier_block __rcu *next;        /* 下一个节点（优先级降序链接）*/
    int                       priority;       /* 优先级，越大越先执行 */
};
💡
priority 优先级说明： 内核代码中实例化的 notifier_block 通常不指定 priority（默认为 0）。 此时，按先注册先执行的顺序调用——即先通过 insmod 加载的模块，其回调会先被执行。 如需强制先于其他模块执行，可以设置较大的 priority 值。

🧩4. 四种通知链类型

围绕 notifier_block，内核定义了四种通知链类型， 主要区别在于执行回调时的并发保护机制。 以下均引自 include/linux/notifier.h 官方注释：

Atomic
原子通知链
回调运行在中断/原子上下文，不允许阻塞。 注册/注销使用自旋锁，调用链使用 RCU 保护（无锁读取）。
Blocking
可阻塞通知链
回调运行在进程上下文，允许阻塞。 使用读写信号量（rwsem）保护链表访问。
Raw
原始通知链
对回调函数没有任何限制。 所有锁和保护机制由调用者自行提供。灵活但危险。
SRCU
SRCU 通知链
可阻塞通知链的变体，使用可睡眠读拷贝更新（SRCU）代替 rwsem。 调用链开销极低（无缓存抖动、无内存屏障），但注销开销较大。 
/* 四种链表头结构体定义（include/linux/notifier.h）*/

/* 1. 原子通知链 — spinlock + RCU */
struct atomic_notifier_head {
    spinlock_t  lock;
    struct notifier_block __rcu *head;
};

/* 2. 可阻塞通知链 — rwsem */
struct blocking_notifier_head {
    struct rw_semaphore  rwsem;
    struct notifier_block __rcu *head;
};

/* 3. 原始通知链 — 无保护 */
struct raw_notifier_head {
    struct notifier_block __rcu *head;
};

/* 4. SRCU 通知链 — mutex + SRCU */
struct srcu_notifier_head {
    struct mutex        mutex;
    struct srcu_struct  srcu;
    struct notifier_block __rcu *head;
};

初始化宏（官方源码）

/* 定义并初始化（一步到位）*/
#define ATOMIC_NOTIFIER_HEAD(name)  \

    struct blocking_notifier_head name = BLOCKING_NOTIFIER_INIT(name)

🔁5. 返回值语义

回调函数的返回值是通知链机制的核心控制手段，定义在 include/linux/notifier.h： 

#define NOTIFY_DONE       0x0000  /* Don't care */
#define NOTIFY_OK         0x0001  /* Suits me */
#define NOTIFY_STOP_MASK  0x8000  /* Don't call further */
#define NOTIFY_BAD        (NOTIFY_STOP_MASK | 0x0002)  /* Bad/Veto action */
#define NOTIFY_STOP       (NOTIFY_OK | NOTIFY_STOP_MASK)  /* OK + stop */
返回值数值含义后续执行
NOTIFY_DONE 0x0000 不关心此事件，忽略 ✅ 继续调用链上下一个
NOTIFY_OK 0x0001 处理成功 ✅ 继续调用链上下一个
NOTIFY_STOP 0x8001 处理成功，但要求停止后续通知 🛑 立即停止，不再调用后续
NOTIFY_BAD 0x8002 拒绝/否决此操作（Veto），相当于 EPERM 🛑 立即停止，调用者收到 BAD 后应回滚
⚠️
重要：通常情况下，回调函数应返回 NOTIFY_OK 或 NOTIFY_DONE， 确保链上后续注册的回调仍然可以被执行。除非明确需要拦截后续通知，否则不要返回 NOTIFY_STOP 或 NOTIFY_BAD。

判断是否需要停止的逻辑：ret & NOTIFY_STOP_MASK，即检查最高位（bit 15）是否置位。 NOTIFY_BAD 和 NOTIFY_STOP 都设置了此位，因此都会触发提前终止。

📝6. 注册与注销机制

内核提供了分层的 API 设计：底层核心函数（static，不对外暴露）， 上层各类型的公开包装函数。

底层核心实现（kernel/notifier.c）

/* 注册：按 priority 降序插入链表 */
static int notifier_chain_register(struct notifier_block **nl,
                                   struct notifier_block *n,
                                   bool unique_priority)
{
    while ((*nl) != NULL) {
        if (unlikely((*nl) == n)) {         /* 重复注册检查 */
            WARN(1, "notifier callback %ps already registered",
                 n->notifier_call);
            return -EEXIST;
        }
        if (n->priority > (*nl)->priority)  /* 找到插入位置 */
            break;
        if (n->priority == (*nl)->priority && unique_priority)
            return -EBUSY;              /* 唯一优先级模式：拒绝重复 */
        nl = &((*nl)->next);
    }
    n->next = *nl;
    rcu_assign_pointer(*nl, n);           /* RCU 安全写入 */
    trace_notifier_register((void *)n->notifier_call);
    return 0;
}
/* 注销：遍历链表找到节点并移除 */
static int notifier_chain_unregister(struct notifier_block **nl,
                                     struct notifier_block *n)
{
    while ((*nl) != NULL) {
        if ((*nl) == n) {
            rcu_assign_pointer(*nl, n->next);  /* RCU 安全移除 */
            trace_notifier_unregister((void *)n->notifier_call);
            return 0;
        }
        nl = &((*nl)->next);
    }
    return -ENOENT;
}

公开 API 汇总

/* ── 注册 ── */

/* ── 注销 ── */

/* ── 触发通知 ── */

⚠️
约束（来自官方注释）： atomic_notifier_chain_unregister()、blocking_notifier_chain_unregister()、 srcu_notifier_chain_unregister() 不得在回调函数执行期间被调用（即不能在 call_chain 过程中注销自身）。

⚙️7. 通知调用链路

核心调用函数 notifier_call_chain()（static，不对外暴露）实现了实际的链表遍历与回调调用： 

/**
 * notifier_call_chain - Informs the registered notifiers about an event.
 * @nl:        Pointer to head of the blocking notifier chain
 * @val:       Value passed unmodified to notifier function
 * @v:         Pointer passed unmodified to notifier function
 * @nr_to_call: Number of notifier functions to be called. -1 = all.
 * @nr_calls:  Records the number of notifications sent.
 * Return:     return value of the last notifier function called.
 */
static int notifier_call_chain(struct notifier_block **nl,
                               unsigned long val, void *v,
                               int nr_to_call, int *nr_calls)
{
    int ret = NOTIFY_DONE;
    struct notifier_block *nb, *next_nb;

    nb = rcu_dereference_raw(*nl);       /* RCU 读取链表头 */

    while (nb && nr_to_call) {
        next_nb = rcu_dereference_raw(nb->next);  /* 提前读取 next */

        trace_notifier_run((void *)nb->notifier_call);
        ret = nb->notifier_call(nb, val, v);       /* 调用回调 */

        if (nr_calls)
            (*nr_calls)++;

        if (ret & NOTIFY_STOP_MASK)    /* 检查是否需要停止 */
            break;
        nb = next_nb;
        nr_to_call--;
    }
    return ret;
}

🔄8. Robust 回滚机制

对于某些需要事务性保证的操作（如内存热插拔、CPU 热插拔）， 内核提供了 notifier_call_chain_robust()： 

/**
 * notifier_call_chain_robust - 通知并在错误时回滚
 * @val_up:   正向通知事件值（如 CPU_UP_PREPARE）
 * @val_down: 错误回滚事件值（如 CPU_UP_CANCELED）
 *
 * NOTE: 两次调用期间 @nl 链不得改变，因此不能使用 RCU
 */
static int notifier_call_chain_robust(struct notifier_block **nl,
                                     unsigned long val_up,
                                     unsigned long val_down,
                                     void *v)
{
    int ret, nr = 0;

    ret = notifier_call_chain(nl, val_up, v, -1, &nr);
    if (ret & NOTIFY_STOP_MASK)
        /* 有回调返回 BAD/STOP，对已成功执行的 nr-1 个回调发送回滚通知 */
        notifier_call_chain(nl, val_down, v, nr - 1, NULL);

    return ret;
}

🔒9. 同步与并发保护

各类型通知链在注册/注销和调用时的加锁策略不同，以下是官方实现的核心逻辑：

Atomic：spinlock + RCU

/* 注册：spinlock 保护写操作 */
int atomic_notifier_chain_register(struct atomic_notifier_head *nh, struct notifier_block *n)
{
    unsigned long flags;
    spin_lock_irqsave(&nh->lock, flags);
    ret = notifier_chain_register(&nh->head, n, false);
    spin_unlock_irqrestore(&nh->lock, flags);
    return ret;
}

/* 调用：RCU 读锁保护，无阻塞 */
int atomic_notifier_call_chain(struct atomic_notifier_head *nh, unsigned long val, void *v)
{
    int ret;
    rcu_read_lock();
    ret = notifier_call_chain(&nh->head, val, v, -1, NULL);
    rcu_read_unlock();
    return ret;
}

Blocking：rwsem

/* 注册：写锁 */
down_write(&nh->rwsem);
notifier_chain_register(&nh->head, n, false);
up_write(&nh->rwsem);

/* 调用：读锁（允许多个并发调用）*/
down_read(&nh->rwsem);
notifier_call_chain(&nh->head, val, v, -1, NULL);
up_read(&nh->rwsem);

SRCU：mutex + srcu_read_lock

/* 注册：mutex 保护 */
mutex_lock(&nh->mutex);
notifier_chain_register(&nh->head, n, false);
mutex_unlock(&nh->mutex);

/* 调用：SRCU 读锁（极低开销）*/
idx = srcu_read_lock(&nh->srcu);
notifier_call_chain(&nh->head, val, v, -1, NULL);
srcu_read_unlock(&nh->srcu, idx);

/* 注销：需调用 synchronize_srcu()，开销较大 */
mutex_lock(&nh->mutex);
notifier_chain_unregister(&nh->head, n);
mutex_unlock(&nh->mutex);
synchronize_srcu(&nh->srcu);   /* 等待所有读者退出 */
📌
SRCU 适用场景： 官方注释说明 —— "SRCU notifier chains should be used when the chain will be called very often but notifier_blocks will seldom be removed."（通知链被频繁调用，但很少注销节点时使用 SRCU）

📌11. 内核代码示例

示例1：USB 设备插拔通知（订阅者模块）

以下示例基于 OpenSourceForYou 官方示例：

#include <linux/module.h>
#include <linux/kernel.h>
#include <linux/usb.h>
#include <linux/notifier.h>

/* 1. 订阅者：实现回调函数 */
static int usb_notify(struct notifier_block *self,
                      unsigned long action, void *dev)
{
    switch (action) {
    case USB_DEVICE_ADD:
        printk(KERN_INFO "USB device added\n");
        break;
    case USB_DEVICE_REMOVE:
        printk(KERN_INFO "USB device removed\n");
        break;
    }
    return NOTIFY_OK;
}

/* 2. 声明 notifier_block */
static struct notifier_block usb_nb = {
    .notifier_call = usb_notify,
    .priority = 0,   /* 不指定优先级，按注册顺序 */
};

static int __init usbhook_init(void)
{
    usb_register_notify(&usb_nb);  /* 向 USB core 的 usb_notifier_list 注册 */
    return 0;
}

static void __exit usbhook_exit(void)
{
    usb_unregister_notify(&usb_nb);
}

module_init(usbhook_init);
module_exit(usbhook_exit);
MODULE_LICENSE("GPL");

示例2：自定义通知链（发布者 + 订阅者）

#include <linux/module.h>
#include <linux/notifier.h>

/* ── 发布者（publisher.c）── */

/* 1. 定义并初始化通知链（阻塞型，进程上下文）*/
static BLOCKING_NOTIFIER_HEAD(my_noti_chain);

/* 2. 暴露注册/注销接口 */
int my_register_notifier(struct notifier_block *nb)
{
    return blocking_notifier_chain_register(&my_noti_chain, nb);
}
EXPORT_SYMBOL(my_register_notifier);

int my_unregister_notifier(struct notifier_block *nb)
{
    return blocking_notifier_chain_unregister(&my_noti_chain, nb);
}
EXPORT_SYMBOL(my_unregister_notifier);

/* 3. 触发通知事件（val=100 表示某业务事件）*/
void my_send_event(unsigned long val, void *data)
{
    blocking_notifier_call_chain(&my_noti_chain, val, data);
}
EXPORT_SYMBOL(my_send_event);


/* ── 订阅者（subscriber.c）── */

static int my_event_handler(struct notifier_block *self,
                            unsigned long val, void *data)
{
    printk(KERN_INFO "my_event: Val=%ld\n", val);
    return NOTIFY_OK;
}

static struct notifier_block my_notifier = {
    .notifier_call = my_event_handler,
};

static int __init sub_init(void)
{
    my_register_notifier(&my_notifier);
    return 0;
}

static void __exit sub_exit(void)
{
    my_unregister_notifier(&my_notifier);
}

🛠️12. user_notifier.h 设计思路

内核通知链运行于内核空间，无法直接在用户态使用。但其设计思想完全可以移植到 用户态 C 程序中，实现一套类内核风格的通知链库 user_notifier.h。

内核 → 用户态的映射

内核概念用户态等价实现
notifier_block结构体 notifier_block（直接对应）
RCU 保护简化：使用 pthread_rwlock_t 或原子操作
spinlock_tpthread_spinlock_t 或 pthread_mutex_t
rw_semaphorepthread_rwlock_t
EXPORT_SYMBOL头文件 static inline 或普通函数
GFP_KERNEL 内存分配malloc()
NOTIFY_DONE/OK/BAD/STOP完全相同的宏定义

设计原则

  • 1完全仿照内核 API 命名约定，降低学习成本
  • 2提供 Raw（无锁）和 Blocking（rwlock）两种类型，覆盖主要用例
  • 3支持优先级排序插入，与内核行为一致
  • 4支持 Robust 回滚模式，适用于事务性操作
  • 5单头文件实现，零依赖（仅标准 C + POSIX 线程）

/*
 * user_notifier.h — 用户态通知链实现
 * 设计仿照 Linux kernel include/linux/notifier.h
 * 来源参考：https://github.com/torvalds/linux/blob/master/include/linux/notifier.h
 *
 * 使用方法：#include "user_notifier.h"
 * 依赖：标准 C99 + POSIX 线程（pthread）
 */
#pragma once
#ifndef USER_NOTIFIER_H
#define USER_NOTIFIER_H

#include <stddef.h>
#include <errno.h>
#include <pthread.h>

/* ════════════════════════════════════════════
 * 返回值定义（严格对应内核 notifier.h）
 * ════════════════════════════════════════════ */
#define NOTIFY_DONE       0x0000   /* Don't care */
#define NOTIFY_OK         0x0001   /* Suits me */
#define NOTIFY_STOP_MASK  0x8000   /* Don't call further */
#define NOTIFY_BAD        (NOTIFY_STOP_MASK | 0x0002) /* Veto */
#define NOTIFY_STOP       (NOTIFY_OK | NOTIFY_STOP_MASK)

/* ════════════════════════════════════════════
 * 核心数据结构
 * ════════════════════════════════════════════ */
struct notifier_block;

typedef int (*notifier_fn_t)(struct notifier_block *nb,
                             unsigned long action,
                             void *data);

struct notifier_block {
    notifier_fn_t         notifier_call;  /* 回调函数 */
    struct notifier_block *next;          /* 链表指针（按优先级降序）*/
    int                   priority;       /* 优先级：越大越先执行 */
};

/* Raw 通知链头（无锁，调用者自己保护）*/
struct raw_notifier_head {
    struct notifier_block *head;
};

/* Blocking 通知链头（pthread_rwlock 保护）*/
struct blocking_notifier_head {
    pthread_rwlock_t       rwlock;
    struct notifier_block *head;
};
/* ════════════════════════════════════════════
 * Raw 通知链核心实现（对应内核 static 函数）
 * ════════════════════════════════════════════ */

/* 初始化宏 */
#define RAW_NOTIFIER_HEAD_INIT   { .head = NULL }
#define RAW_NOTIFIER_HEAD(name)  \
    struct raw_notifier_head name = RAW_NOTIFIER_HEAD_INIT

static inline void
raw_notifier_head_init(struct raw_notifier_head *nh) {
    nh->head = NULL;
}

/* 注册（按 priority 降序插入）— 对应内核 notifier_chain_register */
static inline int
raw_notifier_chain_register(struct raw_notifier_head *nh,
                             struct notifier_block *n)
{
    struct notifier_block **nl = &nh->head;
    while (*nl != NULL) {
        if (*nl == n) return -EEXIST;          /* 重复注册 */
        if (n->priority > (*nl)->priority) break; /* 找到插入点 */
        nl = &(*nl)->next;
    }
    n->next = *nl;
    *nl = n;
    return 0;
}

/* 注销 — 对应内核 notifier_chain_unregister */
static inline int
raw_notifier_chain_unregister(struct raw_notifier_head *nh,
                               struct notifier_block *n)
{
    struct notifier_block **nl = &nh->head;
    while (*nl != NULL) {
        if (*nl == n) { *nl = n->next; return 0; }
        nl = &(*nl)->next;
    }
    return -ENOENT;
}

/* 调用链 — 对应内核 notifier_call_chain */
static inline int
raw_notifier_call_chain_nr(struct raw_notifier_head *nh,
                            unsigned long val, void *v,
                            int nr_to_call, int *nr_calls)
{
    int ret = NOTIFY_DONE;
    struct notifier_block *nb = nh->head, *next;
    while (nb && nr_to_call) {
        next = nb->next;
        ret = nb->notifier_call(nb, val, v);
        if (nr_calls) (*nr_calls)++;
        if (ret & NOTIFY_STOP_MASK) break;
        nb = next;
        nr_to_call--;
    }
    return ret;
}

static inline int
raw_notifier_call_chain(struct raw_notifier_head *nh,
                         unsigned long val, void *v)
{
    return raw_notifier_call_chain_nr(nh, val, v, -1, NULL);
}

/* Robust 变体 — 对应内核 notifier_call_chain_robust */
static inline int
raw_notifier_call_chain_robust(struct raw_notifier_head *nh,
                                unsigned long val_up,
                                unsigned long val_down,
                                void *v)
{
    int ret, nr = 0;
    ret = raw_notifier_call_chain_nr(nh, val_up, v, -1, &nr);
    if (ret & NOTIFY_STOP_MASK)
        raw_notifier_call_chain_nr(nh, val_down, v, nr - 1, NULL);
    return ret;
}
/* ════════════════════════════════════════════
 * Blocking 通知链实现（pthread_rwlock 保护）
 * 对应内核 blocking_notifier_chain_*
 * ════════════════════════════════════════════ */

#define BLOCKING_NOTIFIER_HEAD_INIT(name) \
    { .rwlock = PTHREAD_RWLOCK_INITIALIZER, .head = NULL }

#define BLOCKING_NOTIFIER_HEAD(name) \
    struct blocking_notifier_head name = \
        BLOCKING_NOTIFIER_HEAD_INIT(name)

static inline void
blocking_notifier_head_init(struct blocking_notifier_head *nh) {
    pthread_rwlock_init(&nh->rwlock, NULL);
    nh->head = NULL;
}

static inline void
blocking_notifier_head_destroy(struct blocking_notifier_head *nh) {
    pthread_rwlock_destroy(&nh->rwlock);
}

/* 注册（写锁保护）*/
static inline int
blocking_notifier_chain_register(struct blocking_notifier_head *nh,
                                  struct notifier_block *n)
{
    int ret;
    struct raw_notifier_head raw = { .head = nh->head };
    pthread_rwlock_wrlock(&nh->rwlock);
    raw.head = nh->head;
    ret = raw_notifier_chain_register(&raw, n);
    nh->head = raw.head;
    pthread_rwlock_unlock(&nh->rwlock);
    return ret;
}

/* 注销（写锁保护）*/
static inline int
blocking_notifier_chain_unregister(struct blocking_notifier_head *nh,
                                    struct notifier_block *n)
{
    int ret;
    struct raw_notifier_head raw = { .head = nh->head };
    pthread_rwlock_wrlock(&nh->rwlock);
    raw.head = nh->head;
    ret = raw_notifier_chain_unregister(&raw, n);
    nh->head = raw.head;
    pthread_rwlock_unlock(&nh->rwlock);
    return ret;
}

/* 调用链（读锁保护，允许并发通知）*/
static inline int
blocking_notifier_call_chain(struct blocking_notifier_head *nh,
                              unsigned long val, void *v)
{
    int ret = NOTIFY_DONE;
    if (!nh->head) return ret;
    struct raw_notifier_head raw;
    pthread_rwlock_rdlock(&nh->rwlock);
    raw.head = nh->head;
    ret = raw_notifier_call_chain(&raw, val, v);
    pthread_rwlock_unlock(&nh->rwlock);
    return ret;
}

/* Robust 变体（写锁保护，防止并发修改链表）*/
static inline int
blocking_notifier_call_chain_robust(struct blocking_notifier_head *nh,
                                     unsigned long val_up,
                                     unsigned long val_down,
                                     void *v)
{
    int ret = NOTIFY_DONE;
    if (!nh->head) return ret;
    struct raw_notifier_head raw;
    pthread_rwlock_rdlock(&nh->rwlock);
    raw.head = nh->head;
    ret = raw_notifier_call_chain_robust(&raw, val_up, val_down, v);
    pthread_rwlock_unlock(&nh->rwlock);
    return ret;
}

#endif /* USER_NOTIFIER_H */

🧪14. 用户态示例演示

以下是使用 user_notifier.h 实现一个插件式网络状态监测系统的示例：

/*
 * example_net_monitor.c
 * 模拟网络设备状态变化通知系统
 * 编译：gcc -o monitor example_net_monitor.c -lpthread
 */
#include "user_notifier.h"
#include <stdio.h>

/* 事件类型定义（仿内核 netdevice.h）*/
#define NETDEV_UP       0x01  /* 网口上线 */
#define NETDEV_DOWN     0x02  /* 网口下线 */
#define NETDEV_CHANGE   0x04  /* 状态变化 */

/* ── Publisher：发布者模块 ── */
static BLOCKING_NOTIFIER_HEAD(netdev_notifier_list);

int register_netdev_notifier(struct notifier_block *nb) {
    return blocking_notifier_chain_register(&netdev_notifier_list, nb);
}

int unregister_netdev_notifier(struct notifier_block *nb) {
    return blocking_notifier_chain_unregister(&netdev_notifier_list, nb);
}

static void call_netdev_notifiers(unsigned long event, const char *ifname) {
    blocking_notifier_call_chain(&netdev_notifier_list, event, (void*)ifname);
}

/* ── Subscribers：订阅者模块 ── */

/* 模块A：日志记录器（高优先级）*/
static int logger_handler(struct notifier_block *nb,
                           unsigned long event, void *data) {
    printf("[Logger p=100] event=%lu iface=%s\n", event, (char*)data);
    return NOTIFY_OK;
}
static struct notifier_block logger_nb = {
    .notifier_call = logger_handler, .priority = 100
};

/* 模块B：防火墙（中优先级）*/
static int firewall_handler(struct notifier_block *nb,
                             unsigned long event, void *data) {
    printf("[Firewall p=50] apply rules for %s\n", (char*)data);
    if (event == NETDEV_DOWN) {
        printf("[Firewall] blocking DOWN — return NOTIFY_BAD\n");
        return NOTIFY_BAD; /* 阻止下线操作 */
    }
    return NOTIFY_OK;
}
static struct notifier_block firewall_nb = {
    .notifier_call = firewall_handler, .priority = 50
};

/* 模块C：统计收集器（默认优先级）*/
static int stats_handler(struct notifier_block *nb,
                          unsigned long event, void *data) {
    printf("[Stats p=0] collecting metrics\n");
    return NOTIFY_OK;
}
static struct notifier_block stats_nb = {
    .notifier_call = stats_handler, .priority = 0
};

int main(void)
{
    blocking_notifier_head_init(&netdev_notifier_list);

    register_netdev_notifier(&logger_nb);
    register_netdev_notifier(&firewall_nb);
    register_netdev_notifier(&stats_nb);

    printf("=== NETDEV_UP eth0 ===\n");
    call_netdev_notifiers(NETDEV_UP, "eth0");

    printf("\n=== NETDEV_DOWN eth0 (Firewall veto) ===\n");
    int ret = blocking_notifier_call_chain(&netdev_notifier_list,
                                            NETDEV_DOWN, "eth0");
    printf("Result: %s\n", (ret & NOTIFY_STOP_MASK) ? "VETOED" : "OK");

    unregister_netdev_notifier(&logger_nb);
    unregister_netdev_notifier(&firewall_nb);
    unregister_netdev_notifier(&stats_nb);
    blocking_notifier_head_destroy(&netdev_notifier_list);
    return 0;
}

运行输出：

=== NETDEV_UP eth0 ===
[Logger p=100] event=1 iface=eth0
[Firewall p=50] apply rules for eth0
[Stats p=0] collecting metrics

=== NETDEV_DOWN eth0 (Firewall veto) ===
[Logger p=100] event=2 iface=eth0
[Firewall p=50] apply rules for eth0
[Firewall] blocking DOWN — return NOTIFY_BAD
Result: VETOED
💡
执行顺序：Logger（priority=100）→ Firewall（priority=50）→ Stats（priority=0）。 当 Firewall 返回 NOTIFY_BAD 时，Stats 模块不会被调用，符合内核通知链行为。

📊15. 优劣分析与适用场景

优势

  • ✓松耦合：发布者和订阅者完全解耦，可独立演化
  • ✓动态扩展：运行时可自由注册/注销，无需重编译
  • ✓优先级控制：精确控制回调执行顺序
  • ✓可中止与回滚：NOTIFY_BAD + Robust 模式支持事务语义
  • ✓多并发安全选项：从无保护（Raw）到 SRCU 全覆盖

劣势与注意事项

  • ✗回调执行是同步串行的，链很长时可能有延迟
  • ✗不得在回调中注销自身（可能导致链表损坏）
  • ✗Atomic 通知链中回调不可睡眠/阻塞
  • ✗SRCU 链注销开销大，频繁注销场景不适合

选型指南

场景推荐类型理由
中断处理器/定时器回调Atomic不可睡眠上下文，RCU 无锁高效
驱动/模块事件通知Blocking进程上下文，回调可能有 I/O 操作
高频通知，极少注销SRCU读路径开销极低
调用者自己管理锁Raw最大灵活性
热插拔/事务性状态变更Blocking Robust支持回滚语义
