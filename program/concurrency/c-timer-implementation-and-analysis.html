从第一性原理解析 C 语言定时器

剥去所有框架的外衣，回归计算机系统本质——时间测量与中断驱动执行

⚙️ 什么是定时器？

定时器（Timer）从第一性原理出发，本质是一个回答： 「在未来某个时刻 T，执行函数 F」 的机制。 它解决的核心问题只有两个：

  • 时间测量：系统如何知道「现在是几点」
  • 事件调度：系统如何在正确时机触发回调

📡 用户态获取时间的系统调用

time_basics.c — POSIX 时间接口 
/* 1. POSIX: clock_gettime (推荐，纳秒级精度) */
#include <time.h>

struct timespec ts;
clock_gettime(CLOCK_MONOTONIC, &ts);  // 单调时钟，不受 settimeofday 影响
// ts.tv_sec = 秒, ts.tv_nsec = 纳秒 [0, 999999999]

/* 2. gettimeofday (微秒级，但受墙钟影响) */
#include <sys/time.h>
struct timeval tv;
gettimeofday(&tv, NULL);
// tv.tv_sec, tv.tv_usec

/* 3. POSIX.1b timer_create (进程级定时器，发信号/线程) */
timer_t tid;
struct sigevent sev = { .sigev_notify = SIGEV_SIGNAL,
                          .sigev_signo  = SIGALRM };
timer_create(CLOCK_MONOTONIC, &sev, &tid);

struct itimerspec its = {
    .it_value    = { .tv_sec = 1, .tv_nsec = 0 },  // 1秒后触发
    .it_interval = { .tv_sec = 1, .tv_nsec = 0 }   // 每隔1秒重复
};
timer_settime(tid, 0, &its, NULL);

/* 4. timerfd_create (Linux特有，fd可poll/epoll) */
#include <sys/timerfd.h>
int fd = timerfd_create(CLOCK_MONOTONIC, TFD_NONBLOCK);
timerfd_settime(fd, 0, &its, NULL);
// 到期后 read(fd) 返回到期次数
💡 第一性原理总结
定时器本质是：硬件计数器（产生中断）+ 有序数据结构（管理到期事件）+ 调度机制（触发回调）。 所有上层实现（glib/libuv/nginx）都是这三个要素的不同组合与权衡。

🐧 Linux 内核低精度定时器：时间轮

📐 数据结构：timer_list

include/linux/timer.h 
/* Linux kernel: include/linux/timer.h */
struct timer_list {
    struct hlist_node entry;      /* 时间轮中的桶链表节点 */
    unsigned long     expires;   /* 到期时间（jiffies）*/
    void (*function)(struct timer_list *); /* 回调函数 */

};

🎡 时间轮（Timer Wheel）算法

Linux 内核时间轮将 32 位的超时值分割为 5 个层级（tv1~tv5）， 采用 8+6+6+6+6 位的分组方式（2.6 内核经典实现）， 共 256+64+64+64+64=512 个桶。

新内核（≥4.8，由 Thomas Gleixner 重写）切换为非级联时间轮， 采用 LVL_DEPTH=9 层 × 64槽的均匀结构，彻底消除级联开销。 🔗 LWN: timer wheel replacement

图3：经典分层时间轮结构（Linux 2.6 经典实现）

插入逻辑（经典实现核心代码）

kernel/timer.c — internal_add_timer() 
static void internal_add_timer(struct tvec_base *base, struct timer_list *timer)
{
    unsigned long expires = timer->expires;
    unsigned long idx = expires - base->timer_jiffies;
    struct list_head *vec;

    if (idx < TVR_SIZE) {
        /* tv1: 0~255 jiffies，直接用低8位索引 */
        int i = expires & TVR_MASK;
        vec = base->tv1.vec + i;
    } else if (idx < (1 << (TVR_BITS + TVN_BITS))) {
        /* tv2: 256~16383 jiffies */
        int i = (expires >> TVR_BITS) & TVN_MASK;
        vec = base->tv2.vec + i;
    } else if (idx < (1 << (TVR_BITS + 2 * TVN_BITS))) {
        /* tv3: ~16s~18min */
        int i = (expires >> (TVR_BITS + TVN_BITS)) & TVN_MASK;
        vec = base->tv3.vec + i;
    } else if (idx < (1 << (TVR_BITS + 3 * TVN_BITS))) {
        /* tv4 */
        int i = (expires >> (TVR_BITS + 2 * TVN_BITS)) & TVN_MASK;
        vec = base->tv4.vec + i;
    } else {
        /* tv5: 最远的未来 */
        int i = (expires >> (TVR_BITS + 3 * TVN_BITS)) & TVN_MASK;
        vec = base->tv5.vec + i;
    }
    /* FIFO: 追加到链表尾 */
    list_add_tail(&timer->entry, vec);
}

/* 级联：当低层bucket用尽时，从高层迁移下来 */
static int cascade(struct tvec_base *base, struct tvec *tv, int index)
{
    struct timer_list *timer, *tmp;
    struct list_head tv_list;
    list_replace_init(tv->vec + index, &tv_list);
    /* 将这个bucket中的所有定时器重新插入（插到更低层） */
    list_for_each_entry_safe(timer, tmp, &tv_list, entry)
        internal_add_timer(base, timer);
    return index;
}

⚡ 新内核时间轮（Linux ≥ 4.8）

Thomas Gleixner 在 2016 年重写了时间轮（ patch 13/20 ），新设计特点：

  • 9 层 × 64 槽，每层精度翻倍：1, 8, 64, 512, 4096, 32768, ... ticks
  • 彻底无级联：用位图（bitmap）跳过空槽，O(1)定位
  • 与 hrtimer 更好集成，支持 tickless（NO_HZ）
✅ timer_list 适用场景：连接超时、重传定时器、keepalive、硬件watchdog—— 这类定时器数量大（数千~数万）、精度要求低（毫秒级）、且大多在触发前被取消。 时间轮在这种场景下的 O(1) 插入/删除优势极为突出。

⚡ Linux hrtimer：纳秒级高精度定时器

红黑树 + ktime_t + 硬件中断直驱，由 Thomas Gleixner 于 Linux 2.6.16 引入

数据来源： 🔗 Documentation/timers/hrtimers.txt 🔗 kernel/time/hrtimer.c 🔗 kernel-internals.org

🏗️ 设计动机

官方文档原文（hrtimers.txt）核心思路：

"data structure not bound to jiffies or any other granularity. All the kernel logic works at 64-bit nanoseconds resolution - no compromises."

"after looking at several possible solutions such as radix trees and hashes, we chose the red black tree as the basic data structure. Rbtrees are available as a library in the kernel and are used in various performance-critical areas of e.g. memory management and file systems."

📐 核心数据结构

include/linux/hrtimer.h 
/* include/linux/hrtimer.h */
struct hrtimer {
    struct timerqueue_node  node;         /* 红黑树节点，存储到期时间 */
    ktime_t                  _softexpires; /* 最早可触发时间（slack的下界）*/
    enum hrtimer_restart   (*function)(struct hrtimer *); /* 回调 */
    struct hrtimer_clock_base *base;      /* per-CPU时钟基 */
    u8  state;     /* HRTIMER_STATE_INACTIVE / ENQUEUED */
    u8  is_rel;   /* 相对时间? */
    u8  is_soft;  /* softirq 投递? */
    u8  is_hard;  /* hardirq 投递? */
};

/* ktime_t: 64位纳秒值，内核统一时间单位 */
typedef s64 ktime_t; /* nanoseconds since boot (CLOCK_MONOTONIC) */

/* 每个时钟基（per-CPU per-clockid）维护一棵红黑树 */
struct hrtimer_clock_base {
    struct hrtimer_cpu_base *cpu_base;
    unsigned int            index;   /* HRTIMER_BASE_MONOTONIC 等 */
    clockid_t               clockid;
    struct timerqueue_head   active;  /* 红黑树根 */
    ktime_t                  (*get_time)(void);
    ktime_t                  offset;
};

🌳 红黑树工作原理

图4：hrtimer 红黑树结构

关键机制

  • 最左节点永远是最近到期的定时器，O(1) 获取
  • 插入/删除：O(log N)
  • 硬件中断直接设置为最近到期时刻，无需周期性tick扫描
  • _softexpires + node.expires 形成一个「宽松窗口」，内核可在窗口内合并中断

🔄 工作流程

内核驱动代码示例 
#include <linux/hrtimer.h>

static enum hrtimer_restart my_handler(struct hrtimer *timer)
{
    do_work();

    /* 重新触发：前进10ms（避免漂移累积）*/
    hrtimer_forward_now(timer, ms_to_ktime(10));
    return HRTIMER_RESTART;

    /* 或单次：return HRTIMER_NORESTART; */
}

/* 初始化 + 启动 */
struct hrtimer ht;
hrtimer_setup(&ht, my_handler, CLOCK_MONOTONIC, HRTIMER_MODE_REL);
hrtimer_start(&ht, ms_to_ktime(10), HRTIMER_MODE_REL);

/* 取消（等待正在执行的回调完成）*/
hrtimer_cancel(&ht);

🏗️ 最小堆（Binary Min-Heap）

libuv 使用二叉最小堆管理所有定时器， 堆顶永远是最近到期的定时器。 这与 hrtimer 的红黑树类似，但最小堆的缓存局部性更好（数组实现）。

图6：libuv 最小堆结构

📝 核心源码（libuv/src/timer.c）

libuv/src/timer.c — uv__run_timers() 
/* 来源：https://github.com/libuv/libuv/blob/v1.x/src/timer.c */

void uv__run_timers(uv_loop_t* loop) {
    struct heap_node*  heap_node;
    uv_timer_t*        handle;
    struct uv__queue* queue_node;
    struct uv__queue  ready_queue;

    uv__queue_init(&ready_queue);

    /* 阶段1：将所有已到期的定时器从堆中摘出 */
    for (;;) {
        heap_node = heap_min(timer_heap(loop));
        if (heap_node == NULL)
            break;
        handle = container_of(heap_node, uv_timer_t, node.heap);
        if (handle->timeout > loop->time)  /* 堆顶未到期，退出 */
            break;
        uv_timer_stop(handle);  /* 从堆中移除 */
        uv__queue_insert_tail(&ready_queue, &handle->node.queue);
    }

    /* 阶段2：执行所有就绪的回调 */
    while (!uv__queue_empty(&ready_queue)) {
        queue_node = uv__queue_head(&ready_queue);
        uv__queue_remove(queue_node);
        handle = container_of(queue_node, uv_timer_t, node.queue);
        uv_timer_again(handle);  /* 若 repeat != 0，重新入堆 */
        handle->timer_cb(handle);
    }
}

/* 时间精度：毫秒级，底层使用 uv__hrtime(UV_CLOCK_FAST)/1000000 */
static void uv__update_time(uv_loop_t* loop) {
    loop->time = uv__hrtime(UV_CLOCK_FAST) / 1000000;
}

/* 计算 poll 等待时间 = 堆顶定时器剩余时间 */
int uv__next_timeout(const uv_loop_t* loop) {
    heap_node = heap_min(timer_heap(loop));
    if (heap_node == NULL) return -1; /* 无定时器，永久阻塞 */
    handle = container_of(heap_node, uv_timer_t, node.heap);
    if (handle->timeout <= loop->time) return 0; /* 已到期，立即返回 */
    diff = handle->timeout - loop->time;
    return (int) diff; /* 毫秒 */
}

🦅 APR（Apache Portable Runtime）定时器

数据来源： 🔗 APR: apr_time.h 🔗 APR Design Document

🏗️ APR 时间系统设计

APR 的核心设计原则（官方 APR Design Document 原文）：

"The current design of APR requires that most APR types be incomplete. It is not possible to write flexible portable code if programs can access the internals of APR types. This is because different platforms are likely to define different native types. The first exception to the incomplete type rule is if the type can only reasonably be implemented one way. For example, time is a complete type because there is only one reasonable time implementation."

APR 将时间定义为 apr_time_t（int64，微秒级），统一跨平台行为：

⚠️ APR 定时器局限性：
APR 的定时器主要是时间测量和睡眠工具， 不提供事件驱动的定时回调机制（没有 g_timeout_add 等价物）。 如需回调驱动定时器，需配合 APR 的线程池（apr_thread_pool）或自行实现事件循环。 APR 更多用于 Apache httpd 中的超时控制（如请求超时）而非周期性定时器。

    struct aeTimeEvent *next; /* 链表：指向下一个（无序！） */
} aeTimeEvent;

/* 事件循环主结构 */
typedef struct aeEventLoop {
    int           maxfd;
    int           setsize;
    long long     timeEventNextId;

🔄 事件处理核心逻辑

ae.c — processTimeEvents() + aeProcessEvents() 
/* 处理定时器：O(N) 线性扫描所有定时器 */
static int processTimeEvents(aeEventLoop *eventLoop) {
    aeTimeEvent *te = eventLoop->timeEventHead;
    while (te) {
        long now_sec, now_ms;
        aeGetTime(&now_sec, &now_ms);
        /* 检查是否到期 */
        if (now_sec > te->when_sec ||
            (now_sec == te->when_sec && now_ms >= te->when_ms)) {
            int retval = te->timeproc(eventLoop, te->id, te->clientData);
            if (retval == AE_NOMORE) {
                aeDeleteTimeEvent(eventLoop, te->id); /* 单次：删除 */
            } else {
                aeAddMillisecondsToNow(retval, &te->when_sec, &te->when_ms);
            }
        }
        te = te->next;
    }
}

/* 计算 epoll 超时 = 最近定时器剩余时间（线性搜索）*/
static aeTimeEvent* aeSearchNearestTimer(aeEventLoop *el) {
    aeTimeEvent *te = el->timeEventHead;
    aeTimeEvent *nearest = NULL;
    while (te) {
        if (!nearest || te->when_sec < nearest->when_sec ||
            (te->when_sec == nearest->when_sec && te->when_ms < nearest->when_ms))
            nearest = te;
        te = te->next;
    }
    return nearest; /* O(N) 扫描！ */
}

🤔 为什么 Redis 用链表而不用堆/树？

官方解释（Redis Event Loop Internals, 2026）： "Time events are stored in an unsorted linked list. Each iteration, the loop scans the entire list to find and run overdue timers. This is O(N) but acceptable because there are typically very few time events (often just serverCron)."

Redis 通常只有 1~2 个定时器（主要是 serverCron，默认每100ms执行一次）， O(N) 扫描 = O(1)，没有任何问题。 简单链表带来的代码可读性和维护性远超引入复杂数据结构的收益。

📐 数据结构

Nginx 使用全局红黑树管理所有定时器事件，key 为绝对到期毫秒时间戳：

ngx_event_timer.c — 核心实现 
/* Nginx 全局定时器红黑树 */
ngx_thread_volatile ngx_rbtree_t  ngx_event_timer_rbtree;

/* 插入定时器（关键：惰性更新优化）*/
static ngx_inline void
ngx_event_add_timer(ngx_event_t *ev, ngx_msec_t timer)
{
    ngx_msec_t     key;
    ngx_msec_int_t diff;

    key = ngx_current_msec + timer;  /* 绝对到期时间 */

    if (ev->timer_set) {
        diff = (ngx_msec_int_t) (key - ev->timer.key);
        /* ★ 惰性优化：差值小于300ms则不更新，减少红黑树操作 */
        if (ngx_abs(diff) < NGX_TIMER_LAZY_DELAY) {  /* 300ms */
            return;
        }
        ngx_del_timer(ev);  /* 先删旧的 */
    }

    ev->timer.key = key;
    ngx_rbtree_insert(&ngx_event_timer_rbtree, &ev->timer);
    ev->timer_set = 1;
}

/* 获取最近到期时间（红黑树最左节点=最小值）*/
ngx_msec_t ngx_event_find_timer(void) {
    if (ngx_event_timer_rbtree.root == &ngx_event_timer_sentinel)
        return NGX_TIMER_INFINITE;
    node = ngx_rbtree_min(root, sentinel);  /* O(log N) → 实际O(1)从缓存 */
    timer = (ngx_msec_int_t) (node->key - ngx_current_msec);
    return (timer > 0 ? timer : 0);
}

/* 处理所有已到期定时器 */
void ngx_event_expire_timers(void) {
    for (;;) {
        node = ngx_rbtree_min(root, sentinel);
        /* node->key > ngx_current_msec: 最小的都没到期，退出 */
        if ((ngx_msec_int_t)(node->key - ngx_current_msec) > 0)
            return;

        ev = (ngx_event_t *)((char*)node - offsetof(ngx_event_t, timer));
        ngx_rbtree_delete(&ngx_event_timer_rbtree, &ev->timer);
        ev->timer_set = 0;
        ev->timedout  = 1;   /* 标记超时 */
        ev->handler(ev);      /* 调用事件handler */
    }
}

🌟 Nginx 定时器特色：惰性更新

Nginx 源码注释原文： "Use a previous timer value if difference between it and a new value is less than NGX_TIMER_LAZY_DELAY milliseconds: this allows to minimize the rbtree operations for fast connections."

对于高并发下大量连接快速重置定时器的场景（如 keep-alive）， 如果新旧超时时间差 < 300ms，直接复用旧的红黑树节点， 避免了 delete + reinsert 两次 O(log N) 操作。

⚠️ 实践注意事项

精度幻觉： 用户态定时器永远不能保证精确触发。 nanosleep(1μs) 在繁忙系统上可能睡 100μs+。 真正的实时需要 PREEMPT_RT 内核 + 实时优先级进程。
时钟选择： 永远用 CLOCK_MONOTONIC 计算时间间隔， 不要用 CLOCK_REALTIME（NTP/手动校时会导致定时器跳变）。 只有需要显示「几点几分」时才用 REALTIME。
事件循环集成： 如果程序已有 select/epoll 事件循环， 将最近定时器的剩余时间作为 epoll_wait 的 timeout 参数， 这是零开销的定时器集成方式（libuv/nginx/redis 都这么做）。

🛠 用户态 C 语言定时器实现

不依赖任何框架，仅用 POSIX 标准库，实现最小堆定时器

实现说明： 以下实现使用最小堆 + clock_gettime(CLOCK_MONOTONIC) + epoll， 是工程中最实用的轻量级定时器方案。纯 C99，无外部依赖，可直接嵌入项目。

📁 完整实现

timer.h — 接口定义

timer.h 
/**
 * timer.h — 轻量级用户态定时器
 * 基于最小堆 + clock_gettime(CLOCK_MONOTONIC)
 * 纯 C99，无框架依赖
 */
#pragma once
#include <stdint.h>
#include <stdbool.h>
#include <time.h>

#define TIMER_MAX_COUNT 4096   /* 最大定时器数量 */

typedef void (*timer_cb_t)(void *userdata);

typedef struct {
    uint64_t   expire_ms;    /* 绝对到期时间（毫秒，MONOTONIC）*/
    uint64_t   interval_ms;  /* 周期间隔（0=单次）*/
    timer_cb_t cb;           /* 回调函数 */
    void      *userdata;     /* 用户数据 */
    int        id;           /* 定时器ID（堆索引）*/
    bool       active;       /* 是否有效 */
} timer_entry_t;

typedef struct {
    timer_entry_t  heap[TIMER_MAX_COUNT]; /* 最小堆数组 */
    int            size;                  /* 当前元素数量 */
    int            next_id;              /* ID计数器 */
} timer_heap_t;

/* 初始化 */
void     timer_heap_init(timer_heap_t *h);

/* 添加定时器：timeout_ms后触发，interval_ms=0单次，>0周期 */
/* 返回 timer id（失败返回-1）*/
int      timer_add(timer_heap_t *h, uint64_t timeout_ms,
                    uint64_t interval_ms, timer_cb_t cb, void *ud);

/* 删除定时器 */
void     timer_del(timer_heap_t *h, int id);

/* 处理所有已到期定时器，返回处理数量 */
int      timer_run(timer_heap_t *h);

/* 获取下次到期剩余毫秒（用于 epoll_wait timeout）*/
/* 返回 -1 表示无定时器（永久阻塞），0 表示立即 */
int      timer_next_ms(const timer_heap_t *h);

/* 获取当前单调时间（毫秒）*/
uint64_t timer_now_ms(void);

timer.c — 实现

timer.c 
#include "timer.h"
#include <string.h>
#include <assert.h>

/* ===== 时间工具 ===== */
uint64_t timer_now_ms(void) {
    struct timespec ts;
    clock_gettime(CLOCK_MONOTONIC, &ts);
    return (uint64_t)ts.tv_sec * 1000ULL
         + (uint64_t)ts.tv_nsec / 1000000ULL;
}

/* ===== 最小堆操作 ===== */

/* 堆性质：parent.expire_ms <= child.expire_ms */
static inline bool heap_less(const timer_entry_t *a, const timer_entry_t *b) {
    return a->expire_ms < b->expire_ms;
}

static void heap_swap(timer_heap_t *h, int i, int j) {
    timer_entry_t tmp = h->heap[i];
    h->heap[i] = h->heap[j];
    h->heap[j] = tmp;
    /* 更新 id 以支持 O(1) 定位（此处简化：id即初始序号）*/
}

/* 上浮（sift up）：新插入元素从末尾向上调整 */
static void sift_up(timer_heap_t *h, int i) {
    while (i > 0) {
        int parent = (i - 1) / 2;
        if (heap_less(&h->heap[i], &h->heap[parent])) {
            heap_swap(h, i, parent);
            i = parent;
        } else break;
    }
}

/* 下沉（sift down）：移除堆顶后重新调整 */
static void sift_down(timer_heap_t *h, int i) {
    while (1) {
        int smallest = i;
        int l = 2*i + 1, r = 2*i + 2;
        if (l < h->size && heap_less(&h->heap[l], &h->heap[smallest]))
            smallest = l;
        if (r < h->size && heap_less(&h->heap[r], &h->heap[smallest]))
            smallest = r;
        if (smallest == i) break;
        heap_swap(h, i, smallest);
        i = smallest;
    }
}

/* ===== 公开接口 ===== */

void timer_heap_init(timer_heap_t *h) {
    memset(h, 0, sizeof(*h));
}

int timer_add(timer_heap_t *h, uint64_t timeout_ms,
               uint64_t interval_ms, timer_cb_t cb, void *ud) {
    if (h->size >= TIMER_MAX_COUNT) return -1;

    int idx = h->size++;
    timer_entry_t *e = &h->heap[idx];
    e->expire_ms   = timer_now_ms() + timeout_ms;
    e->interval_ms = interval_ms;
    e->cb          = cb;
    e->userdata    = ud;
    e->id          = h->next_id++;
    e->active      = true;

    sift_up(h, idx);
    return e->id;
}

void timer_del(timer_heap_t *h, int id) {
    /* 线性查找（小规模可接受；大规模可用 id→index 哈希表优化）*/
    for (int i = 0; i < h->size; i++) {
        if (h->heap[i].id == id && h->heap[i].active) {
            h->heap[i].active = false;
            /* 用末尾元素填补，然后重新调整 */
            h->heap[i] = h->heap[--h->size];
            sift_up(h, i);
            sift_down(h, i);
            return;
        }
    }
}

int timer_run(timer_heap_t *h) {
    int      fired = 0;
    uint64_t now   = timer_now_ms();

    while (h->size > 0 && h->heap[0].expire_ms <= now) {
        /* 取堆顶 */
        timer_entry_t e = h->heap[0];

        /* 移除堆顶：末尾填补 + 下沉 */
        h->heap[0] = h->heap[--h->size];
        sift_down(h, 0);

        /* 执行回调 */
        if (e.active) {
            e.cb(e.userdata);
            fired++;

            /* 周期定时器：重新入堆 */
            if (e.interval_ms > 0 && e.active) {
                e.expire_ms = now + e.interval_ms;
                int idx = h->size++;
                h->heap[idx] = e;
                sift_up(h, idx);
            }
        }
    }
    return fired;
}

int timer_next_ms(const timer_heap_t *h) {
    if (h->size == 0) return -1;           /* 无定时器 */
    uint64_t now  = timer_now_ms();
    uint64_t next = h->heap[0].expire_ms;
    if (next <= now) return 0;             /* 已到期 */
    uint64_t diff = next - now;
    return (diff > INT32_MAX) ? INT32_MAX : (int)diff;
}

main.c — 集成示例（epoll 事件循环）

main.c — epoll + 定时器集成 
#include <stdio.h>
#include <sys/epoll.h>
#include "timer.h"

static void on_ping(void *ud) {
    printf("[%lums] ping! count=%d\n", timer_now_ms(), *(int*)ud);
    (*(int*)ud)++;
}

static void on_oneshot(void *ud) {
    printf("[%lums] one-shot fired!\n", timer_now_ms());
}

int main() {
    timer_heap_t timers;
    timer_heap_init(&timers);

    int count = 0;
    /* 每500ms触发一次（周期）*/
    timer_add(&timers, 500, 500, on_ping, &count);
    /* 2000ms后触发一次（单次）*/
    timer_add(&timers, 2000, 0, on_oneshot, NULL);

    int epfd = epoll_create1(0);

    /* 事件循环 */
    while (1) {
        int timeout_ms = timer_next_ms(&timers); /* ← 核心！*/

        struct epoll_event events[64];
        int nev = epoll_wait(epfd, events, 64, timeout_ms);

        /* 处理 IO 事件 */
        for (int i = 0; i < nev; i++) {
            /* ... handle events[i] ... */
        }

        /* 处理定时器 */
        timer_run(&timers);

        /* 停止条件 */
        if (count >= 5) break;
    }
    return 0;
}

🔧 进阶优化方向

  • O(1) 删除：维护 id→heap_index 哈希表，支持任意位置删除
  • 时间轮版本：定时器数量 > 10万时，改用时间轮以获得 O(1) 插入
  • timerfd 集成（Linux）：用 timerfd_create 替代 epoll timeout， 可将定时器精确唤醒集成到标准 epoll 模型，无需在每次 epoll_wait 返回后主动调用 timer_run
  • 多线程：加 pthread_mutex 或改用 per-thread 堆 + 线程安全唤醒机制
  • 惰性时钟更新：参考 libuv，每次事件循环开始时统一更新 now，减少 clock_gettime 调用次数
