CHAPTER 1

概述与背景

Linux 内核 kfifo 的设计哲学：用最少的同步原语，实现最高效的队列

什么是 kfifo？

kfifo（Kernel FIFO）是 Linux 内核提供的一种通用环形缓冲区（Ring Buffer / Circular Buffer）实现， 位于内核源码的 include/linux/kfifo.h 和 lib/kfifo.c。 它由 Stelian Pop 在 2004 年首次引入，后经 Stefani Seibold 在 2013 年重写为泛型版本。

📌 官方注释原文（摘自 torvalds/linux master）
/*
 * Note about locking: There is no locking required until only one reader
 * and one writer is using the fifo and no kfifo_reset() will be called.
 * kfifo_reset_out() can be safely used, until it will be only called
 * in the reader thread.
 * For multiple writer and one reader there is only a need to lock the writer.
 * And vice versa for only one writer and multiple reader there is only a need
 * to lock the reader.
 */
🔗 github.com/torvalds/linux — include/linux/kfifo.h

核心设计目标

🔒

SPSC 无锁

单生产者 + 单消费者场景下，无需任何互斥锁，仅依靠内存屏障保证正确性

⚡

高性能

利用 2 的幂次大小，将取模运算替换为位与（&）运算，极大提升速度

♾️

自然溢出

in/out 指针不做范围限制，依靠 unsigned int 自然溢出（wrap around）消除判断

🧩

泛型支持

现代版本通过宏和联合体支持任意类型元素，并支持定长/变长记录模式

演进历史

版本/时间作者主要变化
v2.6.10 (2004)Stelian Pop首次引入，简单字节流 FIFO，含 spinlock
v2.6.33 (2010)Stefani Seibold重构 API，引入 __kfifo 内部结构，支持泛型
v3.x+Stefani Seibold完整宏体系，支持变长记录（rec_ptr_1/rec_ptr_2）
v6.x (现代)社区维护引入 smp_load_acquire/smp_store_release 等现代屏障原语

应用场景

  • 驱动层：键盘/串口/网卡的中断上下文 → 进程上下文数据传递
  • 内核日志：tracing/ftrace 环形日志缓冲区
  • 用户态移植：高性能服务器、实时系统的线程间通信
  • 音视频：媒体管道中的帧缓冲队列

kfifo 数据结构

理解 __kfifo 结构体的每个字段，以及泛型宏体系的设计

核心结构体 struct __kfifo

摘自 Linux kernel master 分支 include/linux/kfifo.h（作者：Stefani Seibold）：

🔗 github.com/torvalds/linux — include/linux/kfifo.h (Copyright © 2013 Stefani Seibold) 
kernel/include/linux/kfifo.hstruct __kfifo {
    unsigned int  in;    /* 写入位置（生产者更新）*/
    unsigned int  out;   /* 读取位置（消费者更新）*/
    unsigned int  mask;  /* 缓冲区大小减一（size - 1，用于位与取模）*/
    unsigned int  esize; /* 每个元素的字节大小 */
    void         *data;  /* 指向实际存储缓冲区的指针 */
};

字段详解

字段类型含义所有者
inunsigned int下一个写入位置的绝对偏移，单调递增，不清零生产者
outunsigned int下一个读取位置的绝对偏移，单调递增，不清零消费者
maskunsigned int等于 size - 1，用于 idx & mask 替代取模只读
esizeunsigned int每个元素字节数（字节流时为 1）只读
datavoid *指向实际环形缓冲区内存只读

旧版结构（v2.6.x，更直观）

来自 spotify/linux（早期内核镜像），便于理解基本概念：

kernel v2.6.x — include/linux/kfifo.h/* Copyright (C) 2004 Stelian Pop <stelian@popies.net> */
struct kfifo {
    unsigned char  *buffer; /* 数据缓冲区指针 */
    unsigned int    size;   /* 缓冲区大小（必须为 2 的幂）*/
    unsigned int    in;     /* 写位置：data is added at offset (in % size) */
    unsigned int    out;    /* 读位置：data is extracted from off. (out % size) */
    spinlock_t     *lock;   /* 多写者时使用锁 */
};
✅ 关键洞见

旧版用 size 字段，新版用 mask = size - 1 字段，这样在访问元素时只需 in & mask 一次位操作，比 in % size 更快。

CHAPTER 3

环形缓冲区原理

索引自然溢出技巧、2 的幂次魔法、满/空判断的精妙设计

为什么必须是 2 的幂次大小？

来自 kernel/kfifo.c 注释：

kernel/kfifo.c — kfifo_alloc/*
 * round up to the next power of 2, since our 'let the indices
 * wrap' technique works only in this case.
 */
if (size & (size - 1)) {
    BUG_ON(size > 0x80000000);
    size = roundup_pow_of_two(size);
}
💡 位运算替代取模

当 size = 2^N 时：index % size ≡ index & (size-1) ≡ index & mask
位与运算是 CPU 单周期指令，取模运算需要除法指令（慢 5~20 倍）。

索引自然溢出（Wrap-Around）技巧

kfifo 使用 unsigned int 类型的 in 和 out，它们永不清零、单调递增，利用无符号整数溢出（C 标准定义的行为）自动回绕。

满/空判断

kfifo 满/空判断逻辑/* 队列中已使用的元素数 */
used = fifo->in - fifo->out;          // 无符号减法，自动处理溢出

/* 队列是否为空 */
is_empty = (fifo->in == fifo->out);   // in == out 时为空

/* 队列是否已满 */
is_full = (used == size);             // used == size 时满
                                      // size = mask + 1

/* 剩余可写空间 */
unused = size - (fifo->in - fifo->out);  // size - used
⚠️ 为何 size 必须是 2 的幂？

若 size 不是 2 的幂，则 in & (size-1) 不等于 in % size，索引计算会出错。内核代码会强制 roundup_pow_of_two。

写入操作原理（__kfifo_put）

来自 kernel/kfifo.c：

kernel/kfifo.c — __kfifo_put（简化版）unsigned int __kfifo_put(struct kfifo *fifo,
                        const unsigned char *buffer, unsigned int len)
{
    unsigned int l;

    /* 1. 限制写入长度，不超过剩余空间 */
    len = min(len, fifo->size - fifo->in + fifo->out);

    /* 2. ★ 写屏障：确保读取 fifo->out 在数据写入之前 */
    smp_wmb();

    /* 3. 计算从 in 到缓冲区末尾的连续空间 */
    l = min(len, fifo->size - (fifo->in & (fifo->size - 1)));

    /* 4. 第一段：写到缓冲区末尾 */
    memcpy(fifo->buffer + (fifo->in & (fifo->size - 1)), buffer, l);

    /* 5. 第二段：若有剩余，从缓冲区头部继续写（环绕）*/
    memcpy(fifo->buffer, buffer + l, len - l);

    /* 6. ★ 写屏障：确保数据写入完成后再更新 in 索引 */
    smp_wmb();

    /* 7. 更新写指针（消费者可见）*/
    fifo->in += len;
    return len;
}

无锁核心：内存屏障

SPSC 模型下，为什么两个 smp_wmb/smp_rmb 就足够了？

SPSC（Single Producer Single Consumer）模型

来自 Stack Overflow 对内核代码的权威解析（原文链接）：

📌 官方锁注释（摘自 kfifo.h 头部）
/*
 * Note about locking: There is no locking required until only one reader
 * and one writer is using the fifo and no kfifo_reset() will be called.
 */

内存屏障的作用

🛡️

smp_wmb()（写屏障）

确保屏障前的所有写操作对其他 CPU 可见，再执行屏障后的写操作。
防止编译器/CPU 乱序：先写数据，再更新 in 索引

👁️

smp_rmb()（读屏障）

确保屏障前的读操作完成后，再执行屏障后的读操作。
防止乱序：先读 in 索引，再读数据

两次屏障的完整时序

用户态等价原语

内核原语用户态等价（C11）用户态等价（GCC）
smp_wmb()atomic_thread_fence(memory_order_release)__sync_synchronize()
smp_rmb()atomic_thread_fence(memory_order_acquire)__sync_synchronize()
smp_mb()atomic_thread_fence(memory_order_seq_cst)__sync_synchronize()
READ_ONCE(x)atomic_load_explicit(&x, memory_order_relaxed)*(volatile typeof(x)*)&x
WRITE_ONCE(x,v)atomic_store_explicit(&x, v, memory_order_relaxed)*(volatile typeof(x)*)&x = v
✅ 用户态选择建议

在用户态 C11 程序中，推荐使用 stdatomic.h 中的原子操作和 atomic_thread_fence，语义更精准，编译器可生成最优代码（acquire/release 通常比 seq_cst 少一条屏障指令）。

CHAPTER 5

user_kfifo.h 完整实现

严格遵照 Linux kfifo 设计，移植到用户态 C11 的无锁队列头文件

设计原则（与内核版本对应）

  • in/out 单调递增：不清零，自然溢出，与 struct __kfifo 完全一致
  • mask = size - 1：size 必须是 2 的幂，位与取模
  • C11 原子操作：用 _Atomic unsigned int + atomic_thread_fence 替代内核屏障
  • SPSC 无锁：单生产者单消费者场景下，无需任何锁
  • 泛型支持：通过宏支持任意 POD 类型
user_kfifo.h — 完整实现（仿 Linux kernel kfifo，SPSC 无锁）/*
 * user_kfifo.h — User-space SPSC lock-free ring buffer
 *
 * Design based on Linux kernel kfifo (include/linux/kfifo.h, lib/kfifo.c)
 * Copyright (C) 2013 Stefani Seibold <stefani@seibold.net>
 * Original source: https://github.com/torvalds/linux/blob/master/include/linux/kfifo.h
 *
 * User-space adaptation using C11 stdatomic.h
 * - Single Producer, Single Consumer (SPSC) lock-free
 * - in/out indices never reset, rely on unsigned wrap-around
 * - size MUST be a power of 2
 * - Uses acquire/release semantics (matches kernel smp_rmb/smp_wmb)
 */
#ifndef USER_KFIFO_H
#define USER_KFIFO_H

#include <stdint.h>
#include <stddef.h>
#include <string.h>
#include <stdlib.h>
#include <stdatomic.h>  /* C11 — 替代内核 smp_wmb/smp_rmb */
#include <assert.h>

/* ────────────────────────────────────────────────────────────
 * 判断是否为 2 的幂（对应内核 is_power_of_2）
 * ──────────────────────────────────────────────────────────── */
#define KFIFO_IS_POWER_OF_2(n)  ((n) != 0 && (((n) & ((n) - 1)) == 0))

/* ────────────────────────────────────────────────────────────
 * 核心结构体（对应内核 struct __kfifo）
 *
 * 关键设计：in/out 使用 _Atomic，保证跨线程可见性；
 * 使用 C11 acquire/release 语义，精确匹配内核 smp_wmb/smp_rmb
 * ──────────────────────────────────────────────────────────── */
struct user_kfifo {
    _Atomic unsigned int  in;    /* 写入偏移（生产者更新）*/
    _Atomic unsigned int  out;   /* 读取偏移（消费者更新）*/
    unsigned int           mask;  /* size - 1（只读）*/
    unsigned int           esize; /* 元素字节大小（只读）*/
    void                  *data;  /* 数据缓冲区（只读）*/
};

/* ────────────────────────────────────────────────────────────
 * 初始化（对应内核 __kfifo_init）
 *
 * @fifo  : 指向 user_kfifo 结构的指针
 * @buf   : 用户提供的缓冲区（大小必须为 2 的幂次 * esize）
 * @size  : 缓冲区能容纳的元素个数（必须是 2 的幂）
 * @esize : 每个元素的字节大小
 * 返回 0 成功，-1 失败
 * ──────────────────────────────────────────────────────────── */
static inline int
user_kfifo_init(struct user_kfifo *fifo, void *buf,
                unsigned int size, unsigned int esize)
{
    if (!KFIFO_IS_POWER_OF_2(size) || size < 2 || !buf)
        return -1;
    atomic_init(&fifo->in,  0);
    atomic_init(&fifo->out, 0);
    fifo->mask  = size - 1;
    fifo->esize = esize;
    fifo->data  = buf;
    return 0;
}

/* ────────────────────────────────────────────────────────────
 * 动态分配（对应内核 kfifo_alloc）
 * ──────────────────────────────────────────────────────────── */
static inline int
user_kfifo_alloc(struct user_kfifo *fifo,
                  unsigned int size, unsigned int esize)
{
    /* roundup to next power of 2（对应内核 roundup_pow_of_two）*/
    if (!KFIFO_IS_POWER_OF_2(size)) {
        size--;
        size |= size >> 1;  size |= size >> 2;
        size |= size >> 4;  size |= size >> 8;
        size |= size >> 16; size++;
    }
    void *buf = malloc((size_t)size * esize);
    if (!buf) return -1;
    return user_kfifo_init(fifo, buf, size, esize);
}

/* 释放动态分配的缓冲区 */
static inline void
user_kfifo_free(struct user_kfifo *fifo)
{
    free(fifo->data);
    fifo->data = NULL;
    fifo->mask = fifo->esize = 0;
}

/* ────────────────────────────────────────────────────────────
 * 内部查询辅助
 * ──────────────────────────────────────────────────────────── */

/* 当前已使用的元素数（对应 kfifo_len）*/
static inline unsigned int
user_kfifo_len(const struct user_kfifo *fifo)
{
    /* 无符号减法：自然处理 wrap-around，与内核完全一致 */
    return atomic_load_explicit(&fifo->in, memory_order_acquire)
         - atomic_load_explicit(&fifo->out, memory_order_acquire);
}

/* 缓冲区总容量（元素数）*/
static inline unsigned int
user_kfifo_size(const struct user_kfifo *fifo) { return fifo->mask + 1; }

/* 剩余可写空间（元素数，对应 kfifo_avail）*/
static inline unsigned int
user_kfifo_avail(const struct user_kfifo *fifo)
{
    return user_kfifo_size(fifo) - user_kfifo_len(fifo);
}

/* 是否为空（对应 kfifo_is_empty）*/
static inline int
user_kfifo_is_empty(const struct user_kfifo *fifo)
{
    return atomic_load_explicit(&fifo->in, memory_order_acquire)
        == atomic_load_explicit(&fifo->out, memory_order_acquire);
}

/* 是否已满（对应 kfifo_is_full）*/
static inline int
user_kfifo_is_full(const struct user_kfifo *fifo)
{
    return user_kfifo_len(fifo) == user_kfifo_size(fifo);
}

/* ────────────────────────────────────────────────────────────
 * 内部：将 src 复制到环形缓冲区的 off 位置（对应 kfifo_copy_in）
 *
 * 对应内核实现：
 *   static void kfifo_copy_in(struct __kfifo *fifo, const void *src,
 *                             unsigned int len, unsigned int off)
 *   {
 *       unsigned int size = fifo->mask + 1;
 *       unsigned int l;
 *       off &= fifo->mask;
 *       l = min(len, size - off);
 *       memcpy(fifo->data + off, src, l);
 *       memcpy(fifo->data, src + l, len - l);
 *       smp_wmb();   ← 确保数据写入完成后再更新 in
 *   }
 * ──────────────────────────────────────────────────────────── */
static inline void
_kfifo_copy_in(struct user_kfifo *fifo, const void *src,
               unsigned int nelem, unsigned int off)
{
    unsigned int size = fifo->mask + 1;
    unsigned int esize = fifo->esize;
    unsigned int l;
    char *base = (char *)fifo->data;

    off &= fifo->mask;              /* 对应内核 off &= fifo->mask */
    l = (nelem < size - off) ? nelem : (size - off);  /* min(nelem, size-off) */

    /* 第一段 memcpy：from off to end */
    memcpy(base + off * esize, src, l * esize);
    /* 第二段 memcpy：wrap around（若有剩余）*/
    memcpy(base, (const char *)src + l * esize, (nelem - l) * esize);

    /* ★ 写屏障（对应内核 smp_wmb）
     * 保证数据写入完成后，in 的更新才对消费者可见 */
    atomic_thread_fence(memory_order_release);
}

/* 内部：从环形缓冲区的 off 位置复制到 dst（对应 kfifo_copy_out）*/
static inline void
_kfifo_copy_out(struct user_kfifo *fifo, void *dst,
                unsigned int nelem, unsigned int off)
{
    unsigned int size = fifo->mask + 1;
    unsigned int esize = fifo->esize;
    unsigned int l;
    const char *base = (const char *)fifo->data;

    off &= fifo->mask;
    l = (nelem < size - off) ? nelem : (size - off);

    memcpy(dst, base + off * esize, l * esize);
    memcpy((char *)dst + l * esize, base, (nelem - l) * esize);

    /* ★ 全内存屏障（对应内核 smp_mb）
     * 保证数据读取完成后，out 的更新才对生产者可见 */
    atomic_thread_fence(memory_order_seq_cst);
}

/* ────────────────────────────────────────────────────────────
 * user_kfifo_in — 写入 nelem 个元素（对应内核 __kfifo_in）
 *
 * @fifo  : 队列指针
 * @buf   : 数据源
 * @nelem : 要写入的元素个数
 * 返回实际写入的元素个数
 *
 * 对应内核：
 *   unsigned int __kfifo_in(struct __kfifo *fifo,
 *                           const void *buf, unsigned int len)
 *   仅生产者调用，无需锁（SPSC 模式）
 * ──────────────────────────────────────────────────────────── */
static inline unsigned int
user_kfifo_in(struct user_kfifo *fifo, const void *buf, unsigned int nelem)
{
    unsigned int avail, in;

    /* 1. 读取消费者的 out（acquire，对应内核 smp_wmb 前读 out）*/
    unsigned int out = atomic_load_explicit(&fifo->out, memory_order_acquire);
    in  = atomic_load_explicit(&fifo->in,  memory_order_relaxed);

    /* 2. 计算可写空间 */
    avail = (fifo->mask + 1) - (in - out);
    if (nelem > avail) nelem = avail;  /* 截断到可写上限 */
    if (nelem == 0) return 0;

    /* 3. 写入数据 + release fence（内部调用 _kfifo_copy_in）*/
    _kfifo_copy_in(fifo, buf, nelem, in);

    /* 4. 更新 in（release，消费者可见）
     *    对应内核：fifo->in += len; 配合前面的 smp_wmb() */
    atomic_store_explicit(&fifo->in, in + nelem, memory_order_release);

    return nelem;
}

/* ────────────────────────────────────────────────────────────
 * user_kfifo_out — 读取 nelem 个元素（对应内核 __kfifo_out）
 *
 * @fifo  : 队列指针
 * @buf   : 目标缓冲区
 * @nelem : 要读取的元素个数
 * 返回实际读取的元素个数
 * ──────────────────────────────────────────────────────────── */
static inline unsigned int
user_kfifo_out(struct user_kfifo *fifo, void *buf, unsigned int nelem)
{
    unsigned int used, out;

    /* 1. 读取生产者的 in（acquire，对应内核 smp_rmb 前读 in）*/
    unsigned int in = atomic_load_explicit(&fifo->in, memory_order_acquire);
    out = atomic_load_explicit(&fifo->out, memory_order_relaxed);

    /* 2. 计算可读数量 */
    used = in - out;  /* 无符号减法，自动处理溢出 */
    if (nelem > used) nelem = used;
    if (nelem == 0) return 0;

    /* 3. 读取数据 + seq_cst fence */
    _kfifo_copy_out(fifo, buf, nelem, out);

    /* 4. 更新 out（release，生产者可见）*/
    atomic_store_explicit(&fifo->out, out + nelem, memory_order_release);

    return nelem;
}

/* ────────────────────────────────────────────────────────────
 * user_kfifo_peek — 仅查看不消费（对应内核 kfifo_out_peek）
 * ──────────────────────────────────────────────────────────── */
static inline unsigned int
user_kfifo_peek(struct user_kfifo *fifo, void *buf, unsigned int nelem)
{
    unsigned int in = atomic_load_explicit(&fifo->in,  memory_order_acquire);
    unsigned int out= atomic_load_explicit(&fifo->out, memory_order_relaxed);
    unsigned int used = in - out;
    if (nelem > used) nelem = used;
    if (nelem == 0) return 0;
    atomic_thread_fence(memory_order_acquire);
    _kfifo_copy_out(fifo, buf, nelem, out);
    return nelem;  /* out 不更新 */
}

/* ────────────────────────────────────────────────────────────
 * 泛型便捷宏（对应内核 kfifo_put / kfifo_get 宏）
 *
 * KFIFO_PUT(fifo_ptr, val)  — 写入单个元素
 * KFIFO_GET(fifo_ptr, ptr)  — 读取单个元素到 *ptr
 * ──────────────────────────────────────────────────────────── */
#define KFIFO_PUT(fifo, val) \
    user_kfifo_in((fifo), &(val), 1)

#define KFIFO_GET(fifo, ptr) \
    user_kfifo_out((fifo), (ptr), 1)

#endif /* USER_KFIFO_H */
CHAPTER 6

关键 API 解析

memory_order 语义选择

正确性证明（非形式化）

  1. 生产者写数据：_kfifo_copy_in 内的 release fence 确保所有 memcpy 在 store in 之前对其他核可见
  2. 消费者读数据：load in 使用 acquire，保证"看到新 in 值" 的同时，也能看到对应的 data 写入
  3. 反向对称：消费者 store out 使用 release，生产者 load out 使用 acquire，同步释放的空间信息
  4. 无数据竞争：生产者只写 in 和 data[in..in+n-1]，消费者只写 out 和读 data[out..out+n-1]，两者无交集（队列满时生产者停止写入）
🚫 严格限制
  • 本实现仅保证 SPSC（单生产者-单消费者） 无锁安全
  • 多生产者或多消费者场景必须在外部加锁（如 pthread_mutex）
  • 不要在信号处理器中调用（异步信号安全性需额外保证）
  • 不要调用 user_kfifo_reset() 期间有其他线程访问队列
CHAPTER 7

生产者消费者示例

完整的多线程示例：使用 user_kfifo.h 实现无锁生产者消费者模型

示例场景描述

模拟一个数据采集系统：一个采集线程（生产者）持续产生传感器数据包，一个处理线程（消费者）读取并处理。使用 user_kfifo 作为中间缓冲队列，SPSC 无锁通信。

完整示例代码（producer_consumer_demo.c）

producer_consumer_demo.c — 完整可编译示例（gcc -std=c11 -pthread -o demo demo.c）#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>
#include <unistd.h>
#include <stdatomic.h>
#include "user_kfifo.h"  /* 引入我们的无锁队列 */

/* ── 传感器数据包定义 ── */
typedef struct {
    uint32_t   seq;         /* 序列号（用于验证顺序）*/
    float      temperature; /* 模拟温度值 */
    uint64_t   timestamp;   /* 时间戳（微秒）*/
} SensorPacket;

/* ── 全局队列（缓冲 64 个 SensorPacket）── */
#define QUEUE_SIZE  64    /* 必须是 2 的幂 */
#define MSG_COUNT   200   /* 总共生产/消费的消息数 */

static struct user_kfifo g_fifo;
static _Atomic int       g_producer_done = 0; /* 生产者完成标志 */

/* ── 生产者线程函数 ── */
static void *producer_thread(void *arg)
{
    (void)arg;
    for (uint32_t i = 0; i < MSG_COUNT; i++) {
        SensorPacket pkt = {
            .seq         = i,
            .temperature = 25.0f + (i % 10) * 0.5f,
            .timestamp   = (uint64_t)i * 1000
        };

        /* 队列满时自旋等待（实际项目可用 semaphore 阻塞）*/
        while (user_kfifo_in(&g_fifo, &pkt, 1) == 0) {
            /* 让出 CPU，避免忙等耗尽资源 */
            sched_yield();
        }

        if (i % 50 == 0)
            printf("[Producer] 已发送 seq=%u, temp=%.1f°C\n",
                   pkt.seq, pkt.temperature);

        /* 模拟采集间隔（生产比消费稍慢，测试缓冲效果）*/
        usleep(500);
    }

    /* 通知消费者生产完成 */
    atomic_store_explicit(&g_producer_done, 1, memory_order_release);
    printf("[Producer] 全部 %d 条消息发送完毕\n", MSG_COUNT);
    return NULL;
}

/* ── 消费者线程函数 ── */
static void *consumer_thread(void *arg)
{
    (void)arg;
    SensorPacket pkt;
    uint32_t consumed = 0;
    uint32_t expected_seq = 0;
    int      errors = 0;

    for (;;) {
        if (user_kfifo_out(&g_fifo, &pkt, 1) == 1) {
            /* 验证序列号连续（无锁安全性验证）*/
            if (pkt.seq != expected_seq) {
                fprintf(stderr, "[Consumer] !! 顺序错误: 期望 %u, 得到 %u\n",
                        expected_seq, pkt.seq);
                errors++;
            }
            expected_seq++;
            consumed++;

            if (consumed % 50 == 0)
                printf("[Consumer] 已处理 seq=%u, temp=%.1f°C, 队列剩余=%u\n",
                       pkt.seq, pkt.temperature,
                       user_kfifo_len(&g_fifo));

            /* 模拟处理耗时 */
            usleep(300);

        } else {
            /* 队列空，检查是否所有消息已处理完毕 */
            if (atomic_load_explicit(&g_producer_done, memory_order_acquire)
                && user_kfifo_is_empty(&g_fifo))
            {
                break;
            }
            sched_yield();
        }
    }

    printf("[Consumer] 全部处理完毕，共 %u 条，错误 %d 处\n", consumed, errors);
    return NULL;
}

int main(void)
{
    /* 初始化 user_kfifo（64 个 SensorPacket 槽位）*/
    if (user_kfifo_alloc(&g_fifo, QUEUE_SIZE, sizeof(SensorPacket)) != 0) {
        fprintf(stderr, "队列初始化失败\n");
        return 1;
    }

    printf("=== user_kfifo SPSC 无锁队列演示 ===\n");
    printf("队列容量: %u 个 SensorPacket\n", user_kfifo_size(&g_fifo));
    printf("元素大小: %u 字节\n\n", g_fifo.esize);

    pthread_t prod_tid, cons_tid;
    pthread_create(&prod_tid, NULL, producer_thread, NULL);
    pthread_create(&cons_tid, NULL, consumer_thread, NULL);
    pthread_join(prod_tid, NULL);
    pthread_join(cons_tid, NULL);

    user_kfifo_free(&g_fifo);
    printf("\n演示完成。\n");
    return 0;
}

内存屏障等价性分析

📊 性能说明

在 x86_64 架构上，memory_order_release store 和 memory_order_acquire load 通常不产生额外屏障指令（x86 的 TSO 模型已保证），性能接近无屏障版本。在 ARM/RISC-V 等弱序架构上，会生成 dmb/fence 等显式指令，但这正是正确性所必需的。

CHAPTER 9

注意事项与局限

使用 user_kfifo 时必须了解的约束与最佳实践

使用约束

🚫 禁止事项
  • 不允许多生产者并发调用 user_kfifo_in()，否则 in 索引存在竞争
  • 不允许多消费者并发调用 user_kfifo_out()，否则 out 索引存在竞争
  • 不要从信号处理器（signal handler）中调用，需额外 async-signal safety 保证
  • 不要在有其他线程访问时调用 reset（atomic_store in=out=0），可能导致数据损坏
  • 元素类型必须是 POD（Plain Old Data），不要使用含指针的复杂结构（指针在跨线程传递时可能悬空）

最佳实践

🔢

选择合适队列容量

队列大小应大于生产/消费速率差值的峰值，避免频繁忙等。推荐：capacity ≥ 2 × burst_size

😴

避免忙等

生产者/消费者循环中加入 sched_yield() 或 nanosleep()，或配合 semaphore 实现阻塞式等待

📦

批量操作

尽量批量写入/读取（nelem > 1），减少屏障指令和缓存行竞争次数

🏷️

缓存行对齐

将 in/out 分别放在不同缓存行（64字节对齐），避免 False Sharing

缓存行伪共享（False Sharing）优化

内核中通过 cache_aligned 属性将 in/out 放在不同缓存行，用户态可参考此优化：

避免 False Sharing 的优化版结构体#define CACHE_LINE_SIZE  64
#define CACHE_ALIGNED    __attribute__((__aligned__(CACHE_LINE_SIZE)))

struct user_kfifo_aligned {
    /* 生产者独占缓存行 */
    _Atomic unsigned int  in    CACHE_ALIGNED;

    /* 消费者独占缓存行 */
    _Atomic unsigned int  out   CACHE_ALIGNED;

    /* 只读字段（初始化后不变）*/
    unsigned int           mask  CACHE_ALIGNED;
    unsigned int           esize;
    void                  *data;
};
⚠️ 关于 kfifo_len() 等查询函数的注意事项

user_kfifo_len()、user_kfifo_is_empty() 等函数返回的是近似值。 由于 in/out 由不同线程更新，读取到的值可能已过时。 例如：消费者调用 is_empty() 返回 true 后，生产者可能立即写入了新数据。 这些函数仅适用于统计/监控，不能用于并发控制判断。

扩展场景：配合 semaphore 实现阻塞队列

阻塞式生产者（避免忙等）#include <semaphore.h>

sem_t g_slots_sem;   /* 空槽位计数（初始=QUEUE_SIZE）*/
sem_t g_items_sem;   /* 已有元素计数（初始=0）*/

/* 初始化 */
sem_init(&g_slots_sem, 0, QUEUE_SIZE);
sem_init(&g_items_sem, 0, 0);

/* 生产者：阻塞等待有空槽 */
void blocking_produce(SensorPacket *pkt) {
    sem_wait(&g_slots_sem);          /* 等待空槽位 */
    user_kfifo_in(&g_fifo, pkt, 1); /* 无锁写入 */
    sem_post(&g_items_sem);          /* 通知有新元素 */
}

/* 消费者：阻塞等待有数据 */
void blocking_consume(SensorPacket *pkt) {
    sem_wait(&g_items_sem);           /* 等待有元素 */
    user_kfifo_out(&g_fifo, pkt, 1); /* 无锁读取 */
    sem_post(&g_slots_sem);           /* 通知有空槽 */
}
