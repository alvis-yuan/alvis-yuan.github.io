server/print 入门导览：文件职责、ESC/POS 与打印术语

这份文档面向第一次接触本打印模块的人：先解释 Epson ESC/POS 官方概念，再把 server/print 目录里的文件按职责分组，最后给出术语表和阅读路径。它不是 API 手册，而是一张“怎么进入这块代码”的地图。

一句话理解：server/print 是 PrintServer 进程。它接收 USB/TCP/蓝牙/业务侧数据，把数据排成任务队列，解析 ESC/POS 或 TSPL 指令，生成点阵，最后通过 write()/ioctl() 交给内核打印驱动。
第一性原理：打印机软件要把“外部发来的字节”变成“热敏头某一行哪些点发热”。目录里的所有文件，都可以按这个转换链条定位。

ESC/POS 官方概念

Epson 官方 ESC/POS Command Reference 说明，ESC/POS 是面向 POS 打印机的命令系统，提供标准命令语法和协议；命令可按代码顺序、功能、机型等方式查看。官方数据处理模型把接收数据分为预处理、主处理和实时命令处理：实时命令接收时处理，普通命令进入接收缓冲后顺序解释。

官方概念含义本仓库对应
Command打印机控制指令，例如 LF 打印并走纸，ESC @ 初始化，GS V 切纸。Instruct_Proc.c 入口，ESC_Instruct.c/GS_Instruct.c 等分模块实现。
Command Notation官方用 ASCII/Hex/Decimal 描述命令格式；n 是 1 字节参数，nL/nH 是 2 字节数值，pL/pH 表示后续参数长度。Cmd_List*、Cmd_Len、状态函数负责收这些参数。
Function Classification命令按打印、字符、位置、纸张、状态、条码、位图、机械控制等功能分类。代码按前缀拆文件，同时在函数名里体现功能，例如 barcode、bit image、paper layout。
Real-time Command实时命令接收时处理，例如 DLE EOT 传输实时状态。Realtime.c 与 print_mqueue_thread() 的 PRINT_CMD_HANDLE_REALTIME_COMMAND。
Main Processing普通数据顺序解释；字符数据会查字体并写入打印缓冲。printer_send_data() 逐字节调用 Cmd_Proc()，字符进入 LineBuffer。
ESC/POS 是 Epson 的官方命令体系，本仓库还支持 TSPL 标签语言和商米私有扩展。读代码时要区分：ESC/POS 是收据/热敏打印主线，TSPL 是标签模式主线，私有命令是产品扩展。
第一性原理：官方协议解释“字节应该代表什么”，本仓库代码解释“这些字节在这台设备上如何落到硬件”。两者结合，才不会只看见 switch，却看不见设计意图。

心智模型

新同学读代码时可以先抓这条主线：print_server.c 启动进程，usb.c/tcpserver.c/serial.c/蓝牙模块 接收数据，print_queue.c 排队调度，printer.c 逐字节送 Cmd_Proc()，Instruct_Proc.c 分发命令，LineBuffer.c/PageBuffer.c 生成点阵，最后 PrnDotLine() 写驱动。

第一性原理：不要先背所有命令。先知道数据从哪里来、在哪里排队、在哪里解释、在哪里变成点阵、在哪里写硬件。

根目录

路径功能入门提示

PrintServer已构建出的 PrintServer 可执行文件。这是产物，不是源码入口；源码入口在 src/print_server.c。
PrintServer.map链接 map 文件，记录符号布局。排查二进制体积或符号来源时使用。

TSPL/标签打印语言 TSPL 的独立实现与库。标签模式从这里理解。
inc/Cairo、FreeType、Fontconfig、HarfBuzz 等第三方头文件。主要服务字体渲染和复杂文字排版。
test/状态/资源管理相关测试和 stub。了解如何隔离部分模块测试。
*.o、libtspl.so构建中间产物或动态库。读源码时通常可以跳过。
第一性原理：根目录先区分源码、产物、第三方依赖、测试。读错对象会浪费大量时间。

src 分层

进程与入口

print_server.c、arg.c、glbvar.c、simple_timer.c。负责启动、参数、全局变量、定时器。

外部输入

usb.c、usb_hid.c、tcp.c、tcpserver.c、serial.c。把外部连接收到的数据送入打印通道。

队列与 IPC

print_queue.c、print_queue.h。管理共享内存任务池、buffer 池、通道、全局队列、消息队列、状态通知。

ESC/POS 解析

Instruct_Proc.c、DLE_Instruct.c、ESC_Instruct.c、FS_Instruct.c、GS_Instruct.c、Realtime.c。把字节解释成命令。

文本与图像渲染

CharBuffer.c、WordSet.c、HarfBuzz.c、LineBuffer.c、PageBuffer.c、Util.c。把字符/图像命令变成点阵。

硬件边界

printer.c、printer.h。封装 PrnDotLine()、PrnIoctl()、PrnGetStatus()，最终调用 write()/ioctl()。

第一性原理：分层不是目录强制给出的，而是数据职责自然形成的：接收、排队、解释、渲染、输出、反馈。

src 文件速查

文件主要职责ESC/POS/打印概念
print_server.c/hPrintServer 进程入口，初始化全局变量、打印机、队列、USB/TCP/串口和 TSPL 线程。把所有输入通道和解析/驱动层拉起来。
arg.c/h解析 PrintServer 启动参数。运行形态配置，不直接处理命令。
glbvar.c/h加载/保存全局项目信息和设备配置。影响机型差异、项目差异。
simple_timer.c/h轻量定时器。用于合盖走纸切纸、标签定位等延迟动作。
usb.c/hUSB 打印数据接收和回复。主机通过 USB 发送 ESC/POS 字节。
usb_hid.cUSB HID 相关处理。偏输入设备/控制通道支持。
tcp.c/hTCP 连接基础支持。网络打印入口配套。
tcpserver.c/hTCP socket 服务端，收到网络数据后写打印通道。局域网 ESC/POS 打印。
serial.c/h串口接收打印数据。串口外设输入。
print_queue.c/h共享内存初始化、任务/缓冲池、通道队列、全局队列、状态采样、消息队列控制、缺纸重打。官方“接收缓冲/主处理”的本项目实现。
printer.c/h底层打印设备封装，写点阵、发送 embedded command、ioctl 控制、状态查询。把解析结果落成驱动动作。
Instruct_Proc.c/hESC/POS 主状态机入口，Cmd_Proc() 逐字节处理，根状态识别 HT/LF/ESC/FS/GS/DLE。官方命令表的字节流解析入口。
DLE_Instruct.c/hDLE 前缀命令。实时状态类命令，如 DLE EOT。
ESC_Instruct.c/hESC 前缀命令。初始化、字体、行距、位置、切换模式、钱箱脉冲等。
FS_Instruct.c/hFS 前缀命令。官方常用于 Kanji/纸张布局/标签黑标控制。
GS_Instruct.c/hGS 前缀命令和多功能命令。条码、二维码、图像、切刀、状态、用户设置等大量扩展命令。
Realtime.c/h小型实时命令解析器，处理需要快速回复或控制的命令。对应官方 real-time command 思路。
Runtime_Data.c/h运行时打印设置：模式、行距、字宽、页模式、标签模式、纸张布局等。很多 ESC/POS 命令本质是在改这些设置。
CharBuffer.c/h单个字符的字体点阵缓冲。字符数据转点阵的第一站。
WordSet.c/h字库/编码查找，ASCII、CodePage、Unicode、CJK 等。对应 ESC/POS 字符集、代码页概念。
HarfBuzz.c/h复杂文字 shaping，例如连写、RTL 等。让文本排版不只限于简单点阵字库。
LineBuffer.c/h行缓冲，负责对齐、下划线、旋转、反白、行输出。LF、打印并走纸等命令最终触发行输出。
PageBuffer.c/h页模式缓冲，把内容先画到页面区域再输出。对应 ESC/POS Page mode。
Util.c/h点阵、图像、数据转换等工具函数。支撑位图、条码、图形处理。
Config.h打印宽度、点数、常量等配置。决定点阵一行有多少点、缓冲尺寸等。
host_cmd.c/h、host_cmd_def.h主机命令定义和处理。偏设备私有控制面，不是标准 ESC/POS 主线。
src/tspl/PrintServer src 下的 TSPL 适配头文件。标签语言入口声明。
第一性原理：文件名里的前缀就是最快线索：ESC/FS/GS/DLE 是协议前缀，Buffer 是渲染缓冲，queue 是调度，printer 是硬件边界。

TSPL 目录

server/print/TSPL 是标签打印模式的另一套语言处理栈。TSPL 不是 Epson ESC/POS 官方命令体系，它更偏标签打印：尺寸、间隙、文本坐标、条码、二维码、打印份数等通常以文本命令表达。

文件功能入门解释
tspl_parser.c/hTSPL 命令解析入口。相当于 TSPL 版 Instruct_Proc.c。
tspl_api.c/hTSPL 对外接口。连接 PrintServer 主流程和 TSPL 库。
tspl_basic.c/h基础 TSPL 命令实现。如 SIZE、GAP、CLS、TEXT、PRINT 等。
tspl_calculator.c/h坐标、尺寸、单位等计算。标签打印高度依赖坐标系统。
tspl_label_func.c/h标签相关功能。定位、出纸、标签处理。
tspl_LineBuffer.c/h、tspl_PageBuffer.c/hTSPL 的行/页缓冲。和 ESC/POS 的 LineBuffer/PageBuffer 类似，但服务标签布局。
tspl_HarfBuzz.c/hTSPL 文本 shaping。复杂文字支持。
tspl_util.c/hTSPL 工具函数。解析、转换、辅助绘制。
tspl_config.hTSPL 常量配置。标签模式参数。
Makefile、libtspl.soTSPL 库构建文件和产物。读源码优先看 .c/.h。
第一性原理：ESC/POS 更像“顺序小票流”，TSPL 更像“先描述一张标签版面，再打印”。所以 TSPL 代码更强调坐标、页面和标签定位。

专业术语

术语解释代码位置
ESC/POSPOS 打印机控制命令体系，字节流协议。Instruct_Proc.c 与各 *_Instruct.c。
ESC/FS/GS/DLE命令前缀字节。后续字节决定具体命令。Cmd_Start() 分派到对应模块。
实时命令接收时应快速处理的状态/控制命令。Realtime.c。
状态机根据当前状态解释下一字节；本项目用 pNextFun 表示下一处理函数。Instruct_Proc.c。
Code Page单字节字符编码表，决定 0x80 以上字符怎么显示。WordSet.c、Runtime_Data.c。
CJK中文、日文、韩文字符处理。FS_Instruct.c、WordSet.c。
HarfBuzz / shaping把 Unicode 文本变成正确字形序列，例如连字、RTL。HarfBuzz.c。
LineBuffer行缓冲。先把一行字符和图形拼成点阵，再输出。LineBuffer.c。
Page mode页模式。先在页面区域定位绘制，再统一输出。PageBuffer.c。
Raster / Bit Image栅格图/位图，按点阵数据打印。GS_Instruct.c、PrnDotLine()。
HRIHuman Readable Interpretation，条码下方可读文本。GS_Instruct.c 条码相关命令。
ASBAutomatic Status Back，自动状态回传。状态类命令和共享内存状态。
NV memoryNon-Volatile memory，非易失存储，用于图像、用户配置等。GS ( L、GS ( E 等扩展命令。
Task / Buffer打印任务和数据块。任务由多个 250 字节 buffer 链起来。print_queue.c、print_api.h。
ioctlLinux 设备控制系统调用。PrnIoctl()。
第一性原理：术语多是因为打印跨了协议、排版、图像、硬件四个领域。把每个词归到领域里，就不乱了。

推荐阅读顺序

  1. 先读 [docs/print-end-to-end-dataflow.html](print-end-to-end-dataflow.html)，建立数据从外部到驱动的主线。
  2. 再读 server/print/src/print_server.c 的 main()，知道进程启动了哪些模块。
  3. 读 libcommon/print_api.c 的 print_channel_send_data() 和 server/print/src/print_queue.c 的 print_queue_run()、send_next_buffer()。
  4. 读 server/print/src/Instruct_Proc.c 的 Cmd_Start() 和 Cmd_Proc()，理解函数指针状态机。
  5. 按命令前缀读 ESC_Instruct.c、GS_Instruct.c、FS_Instruct.c、DLE_Instruct.c。
  6. 读 LineBuffer.c 和 printer.c，理解“字符/图像如何变成点阵行并写驱动”。
  7. 如果目标是标签模式，再读 server/print/TSPL/tspl_parser.c、tspl_basic.c 和 TSPL 的 buffer 文件。
新手最容易卡在 GS_Instruct.c，因为它命令最多。建议先不要通读，而是拿一个具体命令，例如 GS V 切纸、GS k 条码、GS v 0 栅格位图，顺着状态函数读到底。
第一性原理：学习复杂代码不要从最大文件开始，而要从“最短闭环”开始：入口、队列、解析一个命令、输出一行、查一次状态。

总结

server/print 可以被看成一个嵌入式打印引擎：外部输入模块负责收数据，print_queue.c 负责排队和异常恢复，ESC/POS/TSPL 解析器负责理解打印语言，Buffer 模块负责排版和点阵化，printer.c 负责与内核驱动交互。
