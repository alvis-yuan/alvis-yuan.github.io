TLS 1.2 协议图解导读

RFC 5246 定义 TLS 第二稳定版：HMAC-SHA256 PRF、AEAD 密码套件、扩展框架合并进主规范，是理解 TLS 1.3 之前生态的基础文档。

PRF-SHA256master_secretChangeCipherSpec2-RTT

1 · 简介

TLS 1.2 在 TLS 1.1 基础上强化密码学：PRF 不再固定 MD5+SHA-1，改为按密码套件指定哈希；支持 GCM/CCM 等 AEAD；扩展机制正式纳入规范。

协议四层（§6）

  • Handshake — 协商参数、认证、密钥交换
  • ChangeCipherSpec — 独立内容类型，触发密钥切换
  • Alert — 错误/关闭通知
  • Application Data — 上层数据

2 · 相对 TLS 1.1 的主要变化（§1.2）

方面变化
PRFMD5/SHA-1 组合 → 密码套件指定 PRF（默认 SHA-256）
Finishedverify_data 长度随 PRF 哈希变化（12 字节起）
密码套件新增 AES-GCM、AES-CCM 等 AEAD 套件
扩展与 AES 套件定义合并进本文档
版本号{3, 3}（0x0303）

3 · 记录层（§6）

记录层对数据分片 → 可选压缩 → MAC → 加密 → 传输。TLSPlaintext 结构：

struct { ContentType type; ProtocolVersion version; uint16 length; opaque fragment[TLSPlaintext.length]; } TLSPlaintext;
ContentType值说明
change_cipher_spec20单字节 1，触发激活新密钥
alert21告警
handshake22握手消息
application_data23应用数据

记录的类型与长度在加密前可见，可能泄露流量特征；TLS 1.3 通过统一外层类型缓解此问题。

  • ClientHello：版本、随机数、Session ID、密码套件列表、压缩方法、扩展
  • ServerKeyExchange：DHE/ECDHE 等需要服务器额外 DH/EC 参数时发送
  • ClientKeyExchange：传递 premaster secret（RSA 加密或 DH/ECDH 公钥）

5 · 会话恢复 — Figure 2

Abbreviated handshake（Session ID 匹配）

Session ID 不匹配则回退完整握手。TLS 1.3 改用 PSK / NewSessionTicket 机制（见 RFC 8446 导读）。

6 · ChangeCipherSpec

独立 TLS 内容类型（非 Handshake 子消息），载荷固定为单字节 1。收到对端 CCS 后，后续记录使用刚协商的密钥保护。

Finished 消息在 CCS 之后发送，且用新密钥加密/ MAC，用于确认密钥切换成功及握手完整性。

7 · PRF 与 HMAC（§5）

P_hash(secret, seed) = HMAC_hash(secret, A(1)+seed) + HMAC_hash(secret, A(2)+seed) + …
A(0)=seed; A(i)=HMAC_hash(secret, A(i-1))

PRF(secret, label, seed) = P_<hash>(secret, label + seed)

TLS 1.2 默认 PRF 使用 SHA-256（tls_prf_sha256）。Finished、key expansion、master secret 派生均依赖此 PRF。

8 · master_secret 与 key_block（§6.3 · §8.1）

master_secret = PRF(premaster_secret, "master secret", client_random + server_random)[0..47] key_block = PRF(master_secret, "key expansion", server_random + client_random) Finished.verify_data = PRF(master_secret, "client finished" / "server finished", Hash(handshake_messages))[0..verify_length-1]
密钥派生链

key_block 顺序切分：client_write_MAC → server_write_MAC → client_write_key → server_write_key → client_write_IV → server_write_IV。

9 · 密码套件结构（Appendix A.5）

每个 CipherSuite 是 2 字节值，定义四元组：

组件说明示例
Key ExchangeRSA、DHE、ECDHE 等ECDHE_RSA
AuthenticationRSA、ECDSA 等ECDSA
Bulk CipherAES_CBC、AES_GCM 等AES_128_GCM
MAC / PRFHMAC-SHA256 或 AEAD 内置SHA256

ECC 套件见 RFC 4492 导读（如 TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256）。

10 · 扩展框架（§7.4.1.4）

ClientHello / ServerHello 末尾可附 extensions 块。未知扩展应被忽略（非致命），除非扩展规范另有要求。

常用扩展定义见 RFC 6066（SNI、OCSP Stapling 等）与 RFC 4492（elliptic_curves、ec_point_formats）。
