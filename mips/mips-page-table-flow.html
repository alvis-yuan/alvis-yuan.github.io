MIPS 页表结构，以及一次 VA 到 PA 和缺页建表的完整路径

本文严格按当前仓库源码解释。外部背景先取 Linux kernel 官方文档；本地源码不作为引用跳转，而是直接复制到页面内逐段说明。当前配置下，Linux 通用接口看起来是 PGD/P4D/PUD/PMD/PTE 五层，但 32-bit MIPS 实际物理结构是 PGD 指向 PTE 页，P4D/PUD/PMD 都是编译期折叠层。

主题官方原文链接本文使用的事实
Linux 页表通用模型Linux kernel Page Tables页表把 CPU 看到的虚拟地址映射到外部内存总线看到的物理地址；Linux 当前定义五级层级，具体体系结构可以折叠不用的层级。
MMU、TLB、缺页MMU, TLB, and Page FaultsCPU 访问 VA 时先查 TLB/页表缓存；没有可用翻译时触发 page fault，体系结构入口最终进入 `handle_mm_fault()`，再由 `__handle_mm_fault()` 分配页表层级。
页表折叠Page Table Folding如果体系结构没有使用所有页表层级，未使用层级可以被 folded，也就是访问下一层时编译期跳过该层。
MIPS 官方内核文档入口MIPS-specific Documentationkernel.org 提供 MIPS 架构文档入口；本文的页表细节来自当前仓库 MIPS 源码，因为该入口没有展开本仓库所需的具体页表实现。
第一性原理概述：虚拟内存的核心问题是“进程发出一个地址，硬件最终必须访问某个物理字节”。页表是稀疏映射表，TLB 是这个映射表的高速缓存，缺页异常是“当前映射不存在或权限不满足”时让内核补齐或拒绝映射的控制面。

量源码公式当前值含义
`PAGE_SHIFT``CONFIG_PAGE_SIZE_4KB` 下为 1212页内偏移占 12 位，页大小 4096 字节。
`PTE_T_LOG2``ffs(sizeof(pte_t))-1`2普通 32-bit `pte_t` 为 4 字节。
`PGDIR_SHIFT``2 * PAGE_SHIFT + PTE_ORDER - PTE_T_LOG2`22一个 PGD 项覆盖 4MB VA 范围。
`PTRS_PER_PTE``PAGE_SIZE / sizeof(pte_t)`1024一个 PTE 页有 1024 个 PTE，每个映射 4KB。
`USER_PTRS_PER_PGD``0x80000000 / PGDIR_SIZE`512低 2GB 用户空间需要 512 个 PGD 项。
`PTRS_PER_PGD``USER_PTRS_PER_PGD * 2`1024整个 4GB VA 空间有 1024 个 PGD 项。
第一性原理概述：页表规模来自地址位数拆分。4KB 页固定消耗 12 位偏移；剩下 20 位如果分成两段 10 位，就自然得到 1024 个 PGD 项和 1024 个 PTE 项。

页表类型与二级物理结构

/* arch/mips/include/asm/page.h */
#ifdef CONFIG_PHYS_ADDR_T_64BIT
  #ifdef CONFIG_CPU_MIPS32
    typedef struct { unsigned long pte_low, pte_high; } pte_t;
    #define pte_val(x)    ((x).pte_low | ((unsigned long long)(x).pte_high << 32))
    #define __pte(x)      ({ pte_t __pte = {(x), ((unsigned long long)(x)) >> 32}; __pte; })
  #else
     typedef struct { unsigned long long pte; } pte_t;
     #define pte_val(x) ((x).pte)
     #define __pte(x)   ((pte_t) { (x) } )
  #endif
#else
typedef struct { unsigned long pte; } pte_t;
#define pte_val(x)      ((x).pte)
#define __pte(x)        ((pte_t) { (x) } )
#endif
typedef struct page *pgtable_t;

/*
 * Right now we don't support 4-level pagetables, so all pud-related
 * definitions come from <asm-generic/pgtable-nopud.h>.
 */

/*
 * Finall the top of the hierarchy, the pgd
 */
typedef struct { unsigned long pgd; } pgd_t;
#define pgd_val(x)      ((x).pgd)
#define __pgd(x)        ((pgd_t) { (x) } )

逐段解释：第一段为 PTE 类型做配置分支。当前没有 `CONFIG_PHYS_ADDR_T_64BIT`，所以实际落在 `typedef struct { unsigned long pte; } pte_t;`，一个 PTE 就是一个机器字，`pte_val()` 取出这个机器字。

第二段把 `pgtable_t` 定义成 `struct page *`。这说明 PTE 表页本身按页分配，通用 mm 代码拿到的是“承载 PTE 数组的页”。

第三段定义 `pgd_t`。当前 32-bit MIPS 的 PGD 项存放的是下一层 PTE 页的内核虚拟地址，空项则指向 `invalid_pte_table`。这样 TLB refill 或软件遍历能通过 PGD 找到 PTE 数组。

图 1：Linux 名义层级很多，但当前 MIPS 物理页表只有 PGD 与 PTE 页真正占内存。
第一性原理概述：页表项不是“页”的同义词，而是“下一步怎么走”的记录。PGD 项回答 PTE 数组在哪里；PTE 项回答目标物理页框在哪里、能不能读写执行。

P4D/PUD/PMD 为什么存在又不存在

/* arch/mips/include/asm/pgtable-32.h */
#include <asm-generic/pgtable-nopmd.h>

/*
 * Basically we have the same two-level (which is the logical three level
 * Linux page table layout folded) page tables as the i386.  Some day
 * when we have proper page coloring support we can have a 1% quicker
 * tlb refill handling mechanism, but for now it is a bit slower but
 * works even with the cache aliasing problem the R4k and above have.
 */

/* include/asm-generic/pgtable-nop4d.h */
#define __PAGETABLE_P4D_FOLDED 1
typedef struct { pgd_t pgd; } p4d_t;
static inline p4d_t *p4d_offset(pgd_t *pgd, unsigned long address)
{
        return (p4d_t *)pgd;
}

/* include/asm-generic/pgtable-nopud.h */
#define __PAGETABLE_PUD_FOLDED 1
typedef struct { p4d_t p4d; } pud_t;
static inline pud_t *pud_offset(p4d_t *p4d, unsigned long address)
{
        return (pud_t *)p4d;
}

/* include/asm-generic/pgtable-nopmd.h */
#define __PAGETABLE_PMD_FOLDED 1
typedef struct { pud_t pud; } pmd_t;
static inline pmd_t * pmd_offset(pud_t * pud, unsigned long address)
{
        return (pmd_t *)pud;
}

逐段解释：MIPS 32 位头文件直接包含 `pgtable-nopmd.h`，而 `nopmd` 又包含 `nopud`，`nopud` 再包含 `nop4d`。这条 include 链把 P4D、PUD、PMD 都声明为 folded。

`p4d_offset()`、`pud_offset()`、`pmd_offset()` 都只是把上一级指针强制视作下一级指针返回，没有按地址索引，也没有分配新的页表页。它们的设计目标是让通用 mm 代码总能写成五级遍历，同时让二级页表体系结构零成本跳过无用层。

图 2：folded 层是类型和接口，不是额外内存对象。
第一性原理概述：抽象层的价值在于隔离差异。Linux 让所有体系结构共享同一套 mm 遍历代码；MIPS 通过折叠层级告诉编译器“这些层在本硬件上没有实际工作”。

VA 位切分：10 位 PGD + 10 位 PTE + 12 位偏移

/* arch/mips/include/asm/page.h */
#ifdef CONFIG_PAGE_SIZE_4KB
#define PAGE_SHIFT      12
#endif
#define PAGE_SIZE       (_AC(1,UL) << PAGE_SHIFT)
#define PAGE_MASK       (~((1 << PAGE_SHIFT) - 1))

/* arch/mips/include/asm/pgtable-32.h */
#define PGDIR_SHIFT     (2 * PAGE_SHIFT + PTE_ORDER - PTE_T_LOG2)
#define PGDIR_SIZE      (1UL << PGDIR_SHIFT)
#define PGDIR_MASK      (~(PGDIR_SIZE-1))

#define PGD_ORDER       (__PGD_ORDER >= 0 ? __PGD_ORDER : 0)
#define PTE_ORDER       0

#define PTRS_PER_PGD    (USER_PTRS_PER_PGD * 2)
#define PTRS_PER_PTE    ((PAGE_SIZE << PTE_ORDER) / sizeof(pte_t))
#define USER_PTRS_PER_PGD       (0x80000000UL/PGDIR_SIZE)

/* include/linux/pgtable.h */
static inline unsigned long pte_index(unsigned long address)
{
        return (address >> PAGE_SHIFT) & (PTRS_PER_PTE - 1);
}

#define pgd_index(a)  (((a) >> PGDIR_SHIFT) & (PTRS_PER_PGD - 1))

逐段解释：`PAGE_SHIFT=12` 固定页内偏移为低 12 位；`PTRS_PER_PTE=1024` 让 PTE 索引占 10 位；`PGDIR_SHIFT=22` 说明 PGD 索引从 VA 第 22 位开始，也占 10 位。

`USER_PTRS_PER_PGD=0x80000000/4MB=512`，因此用户空间低 2GB 用 PGD 的前 512 项；`PTRS_PER_PGD=1024`，整个 4GB 空间由 1024 个 4MB 区间覆盖。

图 3：当前配置下，一个 PGD 项覆盖 4MB，一个 PTE 项覆盖 4KB。
第一性原理概述：地址翻译就是把一个整数拆成几段。高位选择更粗的表，中间位选择更细的表，低位不参与映射选择而保留为页内偏移。

PTE 位布局：软件权限位与硬件 EntryLo 位

/* arch/mips/include/asm/pgtable-bits.h */
/* Page table bits used for r4k systems */
enum pgtable_bits {
        /* Used only by software (masked out before writing EntryLo*) */
        _PAGE_PRESENT_SHIFT,
#if !defined(CONFIG_CPU_HAS_RIXI)
        _PAGE_NO_READ_SHIFT,
#endif
        _PAGE_WRITE_SHIFT,
        _PAGE_ACCESSED_SHIFT,
        _PAGE_MODIFIED_SHIFT,
#if defined(CONFIG_MIPS_HUGE_TLB_SUPPORT)
        _PAGE_HUGE_SHIFT,
#endif
#if defined(CONFIG_ARCH_HAS_PTE_SPECIAL)
        _PAGE_SPECIAL_SHIFT,
#endif
#if defined(CONFIG_HAVE_ARCH_SOFT_DIRTY)
        _PAGE_SOFT_DIRTY_SHIFT,
#endif
        /* Used by TLB hardware (placed in EntryLo*) */
#if defined(CONFIG_CPU_HAS_RIXI)
        _PAGE_NO_EXEC_SHIFT,
        _PAGE_NO_READ_SHIFT,
#endif
        _PAGE_GLOBAL_SHIFT,
        _PAGE_VALID_SHIFT,
        _PAGE_DIRTY_SHIFT,
        _CACHE_SHIFT,
};

/* Used only by software */
#define _PAGE_PRESENT          (1 << _PAGE_PRESENT_SHIFT)
#define _PAGE_WRITE            (1 << _PAGE_WRITE_SHIFT)
#define _PAGE_ACCESSED         (1 << _PAGE_ACCESSED_SHIFT)
#define _PAGE_MODIFIED         (1 << _PAGE_MODIFIED_SHIFT)

/* Used by TLB hardware (placed in EntryLo*) */
#define _PAGE_NO_READ          (1 << _PAGE_NO_READ_SHIFT)
#define _PAGE_GLOBAL           (1 << _PAGE_GLOBAL_SHIFT)
#define _PAGE_VALID            (1 << _PAGE_VALID_SHIFT)
#define _PAGE_DIRTY            (1 << _PAGE_DIRTY_SHIFT)
#define _CACHE_MASK            (7 << _CACHE_SHIFT)
#define _PFN_SHIFT             (PAGE_SHIFT - 12 + _CACHE_SHIFT + 3)

static inline uint64_t pte_to_entrylo(unsigned long pte_val)
{
#ifdef CONFIG_CPU_HAS_RIXI
        if (cpu_has_rixi) {
                int sa;
#ifdef CONFIG_32BIT
                sa = 31 - _PAGE_NO_READ_SHIFT;
#else
                sa = 63 - _PAGE_NO_READ_SHIFT;
#endif
                return (pte_val >> _PAGE_GLOBAL_SHIFT) |
                        ((pte_val & (_PAGE_NO_EXEC | _PAGE_NO_READ)) << sa);
        }
#endif

        return pte_val >> _PAGE_GLOBAL_SHIFT;
}

逐段解释：当前是 R4K 风格且 `CONFIG_CPU_HAS_RIXI` 打开，低位先放软件状态：present、write、accessed、modified；随后放硬件会写入 EntryLo 的执行/读取禁止、global、valid、dirty、cache 属性。

`_PFN_SHIFT` 在当前配置下等于 12，所以 `pfn_pte(pfn, prot)` 会把 PFN 左移 12 位再与权限位合并。`pte_to_entrylo()` 在写 TLB EntryLo 前会把 PTE 调整成 MIPS TLB 需要的格式；这解释了为什么 PTE 里既有 Linux 软件状态，也有硬件能直接消费的位。

图 4：PTE 同时服务 Linux 软件状态机与 MIPS TLB EntryLo 格式。
第一性原理概述：硬件只关心“能否翻译、能否访问、物理页在哪”；内核还关心“这个映射是否存在于软件语义、是否被访问/修改、是否允许写”。PTE 必须同时编码这两类事实。

软件页表遍历：从 `mm->pgd` 找到 PTE

/* include/linux/pgtable.h */
static inline pte_t *pte_offset_kernel(pmd_t *pmd, unsigned long address)
{
        return (pte_t *)pmd_page_vaddr(*pmd) + pte_index(address);
}

#define pte_offset_map(dir, address)    pte_offset_kernel((dir), (address))
#define pte_unmap(pte) ((void)(pte))    /* NOP */

static inline pgd_t *pgd_offset_pgd(pgd_t *pgd, unsigned long address)
{
        return (pgd + pgd_index(address));
};

#define pgd_offset(mm, address)         pgd_offset_pgd((mm)->pgd, (address))

static inline pmd_t *pmd_off(struct mm_struct *mm, unsigned long va)
{
        return pmd_offset(pud_offset(p4d_offset(pgd_offset(mm, va), va), va), va);
}

/* arch/mips/include/asm/pgtable.h */
#define pmd_page_vaddr(pmd)     pmd_val(pmd)

逐段解释：`pgd_offset(mm, address)` 从进程的 `mm->pgd` 基址加上 `pgd_index(address)`，定位 PGD 项。折叠层级让 `p4d_offset/pud_offset/pmd_offset` 原样穿透同一个地址。

`pte_offset_kernel()` 读取 PMD 值作为 PTE 页虚拟地址，再加 `pte_index(address)` 得到目标 PTE 指针。当前没有 `CONFIG_HIGHPTE`，所以 `pte_offset_map()` 只是这个内核直接映射版本，不需要临时 kmap。

图 5：软件 walk 最终拿到的是 PTE 指针，后续再看 present/权限/PFN。
第一性原理概述：页表遍历不是复制数据，而是沿着索引逐级缩小范围。每一级都把“整个地址空间”切成更小窗口，直到定位唯一的 4KB 页。

TLB 与页表：快路径缓存，慢路径查表

官方 Page Tables 文档说明：CPU 访问内存时提供虚拟地址，MMU 先查 TLB 或页表缓存；没有翻译时触发 page fault 或体系结构的 TLB refill 路径。当前仓库 MIPS 还会运行时合成 TLB refill handler，注释说明如下。

/* arch/mips/mm/tlbex.c */
/*
 * Synthesize TLB refill handlers at runtime.
 */

/*
 * do_page_fault remains normal asm.
 */
extern void tlb_do_page_fault_0(void);
extern void tlb_do_page_fault_1(void);

逐段解释：MIPS TLB miss 的高频路径尽量由汇编 refill handler 从当前页表取 PTE 并写入 TLB；如果页表中没有有效映射或访问非法，才落到正常 `do_page_fault` 异常处理。这样把“已有映射但 TLB 没缓存”的情况与“真的需要内核分配/拒绝”的情况分开。

第一性原理概述：TLB 是缓存，不是真相来源。真相来源是内存中的页表；缓存命中直接翻译，缓存未命中才查表，查表也失败才让内核改变系统状态。

一个 VA 到 PA 的流程示例

以 `VA = 0x12345678` 为例，按当前位切分得到：PGD index 为 `0x48`，PTE index 为 `0x345`，页内偏移为 `0x678`。假设目标 PTE 的 PFN 为 `0x001ab`，则物理页基址是 `0x001ab000`，最终 PA 是 `0x001ab678`。

第一性原理概述：VA 到 PA 不是“换个基址”这么简单，而是一次受权限约束的查表。只有页表项声明映射存在且访问合法，页内偏移才被拼接到物理页基址上。

进程 PGD 创建：用户半区初始化为空，内核半区复制模板

/* arch/mips/mm/pgtable.c */
pgd_t *pgd_alloc(struct mm_struct *mm)
{
        pgd_t *ret, *init;

        ret = (pgd_t *) __get_free_pages(GFP_KERNEL, PGD_ORDER);
        if (ret) {
                init = pgd_offset(&init_mm, 0UL);
                pgd_init((unsigned long)ret);
                memcpy(ret + USER_PTRS_PER_PGD, init + USER_PTRS_PER_PGD,
                       (PTRS_PER_PGD - USER_PTRS_PER_PGD) * sizeof(pgd_t));
        }

        return ret;
}
EXPORT_SYMBOL_GPL(pgd_alloc);

/* arch/mips/mm/pgtable-32.c */
void pgd_init(unsigned long page)
{
        unsigned long *p = (unsigned long *) page;
        int i;

        for (i = 0; i < USER_PTRS_PER_PGD; i+=8) {
                p[i + 0] = (unsigned long) invalid_pte_table;
                p[i + 1] = (unsigned long) invalid_pte_table;
                p[i + 2] = (unsigned long) invalid_pte_table;
                p[i + 3] = (unsigned long) invalid_pte_table;
                p[i + 4] = (unsigned long) invalid_pte_table;
                p[i + 5] = (unsigned long) invalid_pte_table;
                p[i + 6] = (unsigned long) invalid_pte_table;
                p[i + 7] = (unsigned long) invalid_pte_table;
        }
}

逐段解释：`pgd_alloc()` 为新地址空间分配 PGD 页。当前 `PGD_ORDER=0`，所以分配一个 4KB 页，正好容纳 1024 个 4 字节 PGD 项。

`pgd_init(ret)` 只初始化 `USER_PTRS_PER_PGD`，也就是用户半区 512 项，把它们都指向 `invalid_pte_table`。设计含义是：新进程开始时用户地址空间没有任何实际 PTE 页，空 PGD 项也有统一哨兵。

`memcpy()` 从 `init_mm` 复制内核半区 PGD 项。这样每个进程都有独立用户映射，但共享同一套内核虚拟地址映射模板；系统调用、异常、中断进入内核后不需要切换到另一个内核页表。

第一性原理概述：进程隔离要求用户映射私有；内核常驻要求内核映射稳定可达。把 PGD 分成用户半区和内核半区，就是同时满足隔离与内核可进入性的最小结构。

PTE 页创建：分配、清零、发布、并发竞争处理

/* include/asm-generic/pgalloc.h */
static inline pgtable_t __pte_alloc_one(struct mm_struct *mm, gfp_t gfp)
{
        struct page *pte;

  pte = alloc_page(gfp);
        if (!pte)
                return NULL;
        if (!pgtable_pte_page_ctor(pte)) {
                __free_page(pte);
                return NULL;
        }
        return pte;
}

static inline pgtable_t pte_alloc_one(struct mm_struct *mm)
{
        return __pte_alloc_one(mm, GFP_PGTABLE_USER);
}

/* mm/memory.c */
int __pte_alloc(struct mm_struct *mm, pmd_t *pmd)
{
        spinlock_t *ptl;
        pgtable_t new = pte_alloc_one(mm);
        if (!new)
                return -ENOMEM;

        smp_wmb(); /* Could be smp_wmb__xxx(before|after)_spin_lock */

        ptl = pmd_lock(mm, pmd);
        if (likely(pmd_none(*pmd))) {   /* Has another populated it ? */
                mm_inc_nr_ptes(mm);
                pmd_populate(mm, pmd, new);
                new = NULL;
        }
        spin_unlock(ptl);
        if (new)
                pte_free(mm, new);
        return 0;
}

/* arch/mips/include/asm/pgalloc.h */
static inline void pmd_populate(struct mm_struct *mm, pmd_t *pmd,
        pgtable_t pte)
{
        set_pmd(pmd, __pmd((unsigned long)page_address(pte)));
}

逐段解释：`pte_alloc_one()` 分配一个承载 PTE 数组的物理页，并通过 `pgtable_pte_page_ctor()` 做页表页构造。这个页之后会被当成 1024 个 `pte_t` 的数组使用。

`__pte_alloc()` 先在锁外分配，避免持锁期间做可能睡眠或耗时的内存分配；发布前用 `smp_wmb()` 保证页表页初始化先于指针进入页表。随后拿 `pmd_lock()`，再次检查 `pmd_none()`，因为别的 CPU/线程可能已经抢先安装了 PTE 页。

`pmd_populate()` 在 MIPS 上把新 PTE 页的内核虚拟地址写入 folded PMD/PGD 项。如果发现别人已经填充，就释放刚才多分配的 `new`，避免泄漏。

第一性原理概述：页表是多 CPU 会同时读取的数据结构。创建页表页的关键不是“分配一页”本身，而是按正确顺序把已初始化对象发布给无锁读路径，并处理并发重复创建。

缺页建表主流程：MIPS 异常入口到通用 mm

/* arch/mips/mm/fault.c */
static void __kprobes __do_page_fault(struct pt_regs *regs, unsigned long write,
        unsigned long address)
{
        struct vm_area_struct * vma = NULL;
        struct task_struct *tsk = current;
        struct mm_struct *mm = tsk->mm;
        int si_code;
        vm_fault_t fault;
        unsigned int flags = FAULT_FLAG_DEFAULT;

        si_code = SEGV_MAPERR;

        if (unlikely(address >= VMALLOC_START && address <= VMALLOC_END))
                goto VMALLOC_FAULT_TARGET;

        if (faulthandler_disabled() || !mm)
                goto bad_area_nosemaphore;

        if (user_mode(regs))
                flags |= FAULT_FLAG_USER;

        perf_sw_event(PERF_COUNT_SW_PAGE_FAULTS, 1, regs, address);
retry:
        mmap_read_lock(mm);
        vma = find_vma(mm, address);
        if (!vma)
                goto bad_area;
        if (vma->vm_start <= address)
                goto good_area;
        if (!(vma->vm_flags & VM_GROWSDOWN))
                goto bad_area;
        if (expand_stack(vma, address))
                goto bad_area;
good_area:
        si_code = SEGV_ACCERR;

        if (write) {
                if (!(vma->vm_flags & VM_WRITE))
                        goto bad_area;
                flags |= FAULT_FLAG_WRITE;
        } else {
                if (cpu_has_rixi) {
                        if (address == regs->cp0_epc && !(vma->vm_flags & VM_EXEC))
                                goto bad_area;
                        if (!(vma->vm_flags & VM_READ) &&
                            exception_epc(regs) != address)
                                goto bad_area;
                } else {
                        if (unlikely(!vma_is_accessible(vma)))
                                goto bad_area;
                }
        }

        fault = handle_mm_fault(vma, address, flags, regs);
        ...
}

asmlinkage void __kprobes do_page_fault(struct pt_regs *regs,
        unsigned long write, unsigned long address)
{
        enum ctx_state prev_state;

        prev_state = exception_enter();
        __do_page_fault(regs, write, address);
        exception_exit(prev_state);
}

逐段解释：MIPS `do_page_fault()` 先进入异常上下文，再调用 `__do_page_fault()`。`address` 是 BadVAddr，`write` 表示写异常，`regs` 保存异常现场。

函数先特殊处理 vmalloc 区，因为内核 vmalloc 映射要从 `init_mm` 同步到当前 CPU/任务页表；用户普通缺页则必须有 `mm` 且不能在禁用 fault handler 的上下文里处理。

`find_vma()` 验证 VA 是否落在某个 VMA 内；如果刚好在向下增长的栈附近，会尝试 `expand_stack()`。这一步解决的是“地址是否属于进程合法虚拟地址空间”的问题，还没有创建 PTE。

权限检查把 VMA 语义转换成 fault flags。当前 CPU 有 RIXI，所以读、执行权限可分别检查；写异常必须要求 `VM_WRITE`。只有 VMA 存在且权限允许，才调用 `handle_mm_fault()` 进入通用建表/装页流程。

/* mm/memory.c */
vm_fault_t handle_mm_fault(struct vm_area_struct *vma, unsigned long address,
                           unsigned int flags, struct pt_regs *regs)
{
        vm_fault_t ret;

        __set_current_state(TASK_RUNNING);

        count_vm_event(PGFAULT);
        count_memcg_event_mm(vma->vm_mm, PGFAULT);

        if (!arch_vma_access_permitted(vma, flags & FAULT_FLAG_WRITE,
                                            flags & FAULT_FLAG_INSTRUCTION,
                                            flags & FAULT_FLAG_REMOTE))
                return VM_FAULT_SIGSEGV;

        if (flags & FAULT_FLAG_USER)
                mem_cgroup_enter_user_fault();

        if (unlikely(is_vm_hugetlb_page(vma)))
                ret = hugetlb_fault(vma->vm_mm, vma, address, flags);
        else
                ret = __handle_mm_fault(vma, address, flags);

        if (flags & FAULT_FLAG_USER) {
                mem_cgroup_exit_user_fault();
                if (task_in_memcg_oom(current) && !(ret & VM_FAULT_OOM))
                        mem_cgroup_oom_synchronize(false);
        }

        mm_account_fault(regs, address, flags, ret);

        return ret;
}

逐段解释：`handle_mm_fault()` 是体系结构无关入口。它统计 fault、检查架构允许性、进入 memcg 用户 fault 作用域，然后在普通页路径调用 `__handle_mm_fault()`。当前没有 huge page 配置命中，因此重点是普通页路径。

/* mm/memory.c */
static vm_fault_t __handle_mm_fault(struct vm_area_struct *vma,
                unsigned long address, unsigned int flags)
{
        struct vm_fault vmf = {
                .vma = vma,
                .address = address & PAGE_MASK,
                .flags = flags,
                .pgoff = linear_page_index(vma, address),
                .gfp_mask = __get_fault_gfp_mask(vma),
        };
        unsigned int dirty = flags & FAULT_FLAG_WRITE;
        struct mm_struct *mm = vma->vm_mm;
        pgd_t *pgd;
        p4d_t *p4d;
        vm_fault_t ret;

        pgd = pgd_offset(mm, address);
        p4d = p4d_alloc(mm, pgd, address);
        if (!p4d)
                return VM_FAULT_OOM;

        vmf.pud = pud_alloc(mm, p4d, address);
        if (!vmf.pud)
                return VM_FAULT_OOM;
retry_pud:
        ...
        vmf.pmd = pmd_alloc(mm, vmf.pud, address);
        if (!vmf.pmd)
                return VM_FAULT_OOM;

        ...
        return handle_pte_fault(&vmf);
}

逐段解释：通用代码仍写成 PGD、P4D、PUD、PMD、PTE 的完整流程。对当前 MIPS 来说，`p4d_alloc()`、`pud_alloc()`、`pmd_alloc()` 经过折叠层后不会分配真实中间页；它们只是保证接口一致并最终进入 `handle_pte_fault()`。

`handle_pte_fault()` 内部会在需要 PTE 页时通过 `pte_alloc_one_map()`/`pte_alloc()` 落到前一节的 `__pte_alloc()`；需要物理页时再走匿名页、文件页、COW 或共享映射处理。本文聚焦页表创建，所以止于页表页创建与 PTE 安装。

第一性原理概述：缺页不是错误的同义词，而是延迟工作的触发器。内核先证明访问属于合法 VMA，再按需创建页表与页面；如果证明失败，才把异常转成信号或内核 oops。

vmalloc fault：同步内核参考页表

/* arch/mips/mm/fault.c */
#ifndef CONFIG_64BIT
vmalloc_fault:
        {
                /*
                 * Synchronize this task's top level page-table
                 * with the 'reference' page table.
                 *
                 * Do _not_ use "tsk" here. We might be inside
                 * an interrupt in the middle of a task switch..
                 */
                int offset = pgd_index(address);
                pgd_t *pgd, *pgd_k;
                p4d_t *p4d, *p4d_k;
                pud_t *pud, *pud_k;
                pmd_t *pmd, *pmd_k;
                pte_t *pte_k;

                pgd = (pgd_t *) pgd_current[raw_smp_processor_id()] + offset;
                pgd_k = init_mm.pgd + offset;

                if (!pgd_present(*pgd_k))
                        goto no_context;
                set_pgd(pgd, *pgd_k);

                p4d = p4d_offset(pgd, address);
                p4d_k = p4d_offset(pgd_k, address);
                if (!p4d_present(*p4d_k))
                        goto no_context;

                pud = pud_offset(p4d, address);
                pud_k = pud_offset(p4d_k, address);
                if (!pud_present(*pud_k))
                        goto no_context;

                pmd = pmd_offset(pud, address);
                pmd_k = pmd_offset(pud_k, address);
                if (!pmd_present(*pmd_k))
                        goto no_context;
                set_pmd(pmd, *pmd_k);

                pte_k = pte_offset_kernel(pmd_k, address);
                if (!pte_present(*pte_k))
                        goto no_context;
                return;
        }
#endif

逐段解释：32-bit MIPS 对 vmalloc 地址采用“按需同步当前页表”的策略。`init_mm.pgd` 是内核参考页表；如果当前 CPU 正在使用的页表缺少对应内核映射，就从参考页表复制 PGD/PMD 信息。

注释强调不能用 `tsk`，因为 fault 可能发生在中断或任务切换中间。代码改用 `pgd_current[raw_smp_processor_id()]` 找当前 CPU 正在使用的 PGD，避免把映射同步到错误任务。

最后检查参考页表里的 PTE 是否 present。只有参考页表确认映射有效，才认为 vmalloc fault 已被修复；否则落到 `no_context` 处理内核非法访问。

第一性原理概述：内核映射是全局语义，但每个地址空间有自己的顶层页表副本。vmalloc fault 的本质是把全局参考事实同步到当前 CPU 的可见页表，而不是为用户进程创建私有映射。

/* arch/mips/mm/pgtable-32.c */
void __init pagetable_init(void)
{
        unsigned long vaddr;
        pgd_t *pgd_base;

        /* Initialize the entire pgd.  */
        pgd_init((unsigned long)swapper_pg_dir);
        pgd_init((unsigned long)swapper_pg_dir
                 + sizeof(pgd_t) * USER_PTRS_PER_PGD);

        pgd_base = swapper_pg_dir;

        /*
         * Fixed mappings:
         */
        vaddr = __fix_to_virt(__end_of_fixed_addresses - 1);
        fixrange_init(vaddr & PMD_MASK, vaddr + FIXADDR_SIZE, pgd_base);
}

逐段解释：`swapper_pg_dir` 是内核初始 PGD，`init_mm.pgd` 指向它。后续新进程复制内核半区时，复制的源头就是 `init_mm`。

`pagetable_init()` 对 `swapper_pg_dir` 的前半和后半都调用 `pgd_init()`，把整个 PGD 初始化为空哨兵；然后为 fixmap 地址区间调用 `fixrange_init()` 建立早期固定映射所需页表。

这一步发生在普通进程页表创建之前，解决的是内核自己必须先拥有一套可用参考页表的问题。没有这个参考页表，就无法在 `pgd_alloc()` 中复制内核半区，也无法在 vmalloc fault 中同步内核映射。

第一性原理概述：任何按需创建都需要一个最早的根。`swapper_pg_dir` 是内核页表世界的根模板；进程页表从它继承内核部分，运行时同步也以它为准。

整体总结

结构

当前 32-bit MIPS 使用 PGD + PTE 页的二级物理页表。P4D/PUD/PMD 是为了兼容 Linux 通用五级接口而折叠出来的类型层。

翻译

VA 按 `10/10/12` 位切分。PGD 项找到 PTE 页，PTE 项给出 PFN 和权限，PFN 左移 12 位后拼接页内偏移得到 PA。

创建

新进程先分配 PGD，用户半区置空，内核半区复制 `init_mm`。真正的 PTE 页通常在合法缺页时按需分配并发布。

异常

TLB miss 或访问异常进入 MIPS fault 路径；合法 VMA 交给 `handle_mm_fault()`，非法访问转成 `SIGSEGV`、`SIGBUS` 或内核 oops。

从第一性原理看，页表系统只做三件事：描述“有没有映射”，约束“能不能访问”，并在缺失时决定“是补齐状态还是拒绝访问”。MIPS 的实现把这三件事拆成 TLB 快路径、页表内存结构和通用 fault 状态机，从而兼顾性能、隔离与跨体系结构复用。
