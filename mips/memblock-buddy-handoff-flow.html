本文只按当前仓库源码解释 MIPS 启动路径。外部依据先取 kernel.org 官方文档；本地源码直接复制进页面逐段说明。核心结论：memblock 管“物理区间事实”，`free_area_init()` 建“伙伴系统元数据”，`memblock_free_all()` 把 `memory - reserved` 的低端页逐批释放进 buddy。

官方依据

主题官方原文链接本文使用的事实
boot-time memory managementBoot time memory management普通内存分配器未建立时使用 `memblock`；架构代码在 `setup_arch()` 设置它，并在 `mem_init()` 中把内存释放给 buddy。

buddy free_areaPhysical Memory: Zones`zone->free_area[]` 是 buddy allocator 管理不同 order 空闲块的核心结构；每个 order 是 2 的幂大小。
第一性原理概述：启动早期最缺的是“可分配内存”，但恰好普通分配器本身也要内存才能建立。memblock 用静态小数组先记录物理区间，避免在分配器尚未存在时递归依赖分配器。

启动总时序

图 1：`free_area_init()` 建好框架，`memblock_free_all()` 才真正给 buddy 填充空闲页。
/* init/main.c */
setup_arch(&command_line);
...
build_all_zonelists(NULL);
page_alloc_init();
...
mm_init();
...
setup_per_cpu_pageset();

逐段解释：`setup_arch()` 是 MIPS 建立 memblock 内存图和 zone 元数据的地方。之后 `build_all_zonelists()` 建立分配 fallback 顺序，`page_alloc_init()` 注册 page allocator 的 CPU hotplug 状态。真正释放页给 buddy 发生在后面的 `mm_init()` 里，而 per-cpu pageset 的正式初始化更晚，在 `setup_per_cpu_pageset()`。

第一性原理概述：一个分配器的“可用”分两步：先有描述结构和路由规则，再有可分配对象。伙伴系统的结构在 `paging_init()` 建好，但空闲页要等 memblock 移交后才进入 free list。

memblock 的结构：memory 与 reserved 两个有序区间集合

/* include/linux/memblock.h */
struct memblock_region {
        phys_addr_t base;
        phys_addr_t size;
        enum memblock_flags flags;
#ifdef CONFIG_NEED_MULTIPLE_NODES
        int nid;
#endif
};

struct memblock_type {
        unsigned long cnt;
        unsigned long max;
        phys_addr_t total_size;
        struct memblock_region *regions;
        char *name;
};

struct memblock {
        bool bottom_up;  /* is bottom up direction? */
        phys_addr_t current_limit;
        struct memblock_type memory;
        struct memblock_type reserved;
};

/* mm/memblock.c */
static struct memblock_region memblock_memory_init_regions[INIT_MEMBLOCK_REGIONS] __initdata_memblock;
static struct memblock_region memblock_reserved_init_regions[INIT_MEMBLOCK_RESERVED_REGIONS] __initdata_memblock;

struct memblock memblock __initdata_memblock = {
        .memory.regions         = memblock_memory_init_regions,
        .memory.cnt             = 1,    /* empty dummy entry */
        .memory.max             = INIT_MEMBLOCK_REGIONS,
        .memory.name            = "memory",

        .reserved.regions       = memblock_reserved_init_regions,
        .reserved.cnt           = 1,    /* empty dummy entry */
        .reserved.max           = INIT_MEMBLOCK_RESERVED_REGIONS,
        .reserved.name          = "reserved",

        .bottom_up              = false,
        .current_limit          = MEMBLOCK_ALLOC_ANYWHERE,
};

逐段解释：`memblock_region` 只记录一个物理区间的起点、大小、属性，以及多节点配置下的 node id。它没有页粒度链表，也不维护 order。

`memblock_type` 是某类区间的数组包装，维护当前区间数、数组容量和总大小。`memblock` 自身只有两个核心集合：`memory` 与 `reserved`。

全局 `memblock` 在编译期静态初始化，两个数组也是静态早期数据。这个设计解决了“分配器初始化前不能依赖动态分配”的问题。后续如果允许 resize，扩容也必须避开 initrd 等将要保留的区域。

图 2：memblock 的“空闲”是集合差，不是已经挂到 buddy free list 的页。
第一性原理概述：早期内存管理只需要回答“哪些物理范围存在、哪些不能用、从哪里切一段出来”。区间表比页级结构小得多，也更适合在内核自举阶段使用。

MIPS 如何建立 memblock 内存图

/* arch/mips/kernel/setup.c */
void __init detect_memory_region(phys_addr_t start, phys_addr_t sz_min, phys_addr_t sz_max)
{
        void *dm = &detect_magic;
        phys_addr_t size;

        for (size = sz_min; size < sz_max; size <<= 1) {
                if (!memcmp(dm, dm + size, sizeof(detect_magic)))
                        break;
        }

        pr_debug("Memory: %lluMB of RAM detected at 0x%llx (min: %lluMB, max: %lluMB)\n",
                ((unsigned long long) size) / SZ_1M,
                (unsigned long long) start,
                ((unsigned long long) sz_min) / SZ_1M,
                ((unsigned long long) sz_max) / SZ_1M);

        memblock_add(start, size);
}

static int __init early_parse_mem(char *p)
{
        phys_addr_t start, size;

        if (usermem == 0) {
                usermem = 1;
                memblock_remove(memblock_start_of_DRAM(),
                        memblock_end_of_DRAM() - memblock_start_of_DRAM());
        }
        start = 0;
        size = memparse(p, &p);
        if (*p == '@')
                start = memparse(p + 1, &p);

        memblock_add(start, size);

        return 0;
}
early_param("mem", early_parse_mem);

逐段解释：`detect_memory_region()` 是平台可调用的探测 helper，最终用 `memblock_add(start, size)` 把探测到的 RAM 加进 `memblock.memory`。

`mem=` 命令行会覆盖自动发现的内存图。第一次解析用户内存时，它先把当前 DRAM 范围从 memblock 中移除，再按用户指定的 `size[@start]` 添加回来。这保证用户命令行是权威输入。

/* arch/mips/kernel/setup.c */
static void __init arch_mem_init(char **cmdline_p)
{
        /* call board setup routine */
        plat_mem_setup();
        memblock_set_bottom_up(true);

        bootcmdline_init();
        strlcpy(command_line, boot_command_line, COMMAND_LINE_SIZE);
        *cmdline_p = command_line;

        parse_early_param();

        if (usermem)
                pr_info("User-defined physical RAM map overwrite\n");

        check_kernel_sections_mem();

        early_init_fdt_reserve_self();

#ifndef CONFIG_NUMA
        memblock_set_node(0, PHYS_ADDR_MAX, &memblock.memory, 0);
#endif
        bootmem_init();

        memblock_set_current_limit(PFN_PHYS(max_low_pfn));
        ...
        device_tree_init();
        early_init_fdt_scan_reserved_mem();
        ...
        sparse_init();
        ...
        dma_contiguous_reserve(PFN_PHYS(max_low_pfn));

        memblock_reserve(__pa_symbol(&__nosave_begin),
                __pa_symbol(&__nosave_end) - __pa_symbol(&__nosave_begin));

        memblock_dump_all();
}

逐段解释：`arch_mem_init()` 由 `setup_arch()` 调用。`plat_mem_setup()` 负责平台早期内存发现，随后 `parse_early_param()` 允许 `mem=`、`memmap=` 等命令行修正 memblock 图。

非 NUMA 下，代码把全部 memblock memory 标记到 node 0。`bootmem_init()` 计算 `min_low_pfn/max_low_pfn/max_pfn`，并保留内核镜像、initrd 等关键范围。之后 `memblock_set_current_limit(PFN_PHYS(max_low_pfn))` 禁止 memblock 分配高端内存，保证早期分配能被内核直接访问。

`device_tree_init()`、`early_init_fdt_scan_reserved_mem()`、`dma_contiguous_reserve()` 和 nosave 保留区继续往 `reserved` 集合写入不可释放范围。最后 `memblock_dump_all()` 打印当前区间图，供调试启动内存布局。

第一性原理概述：memblock 的第一阶段是“事实收集”：固件、平台代码、命令行和 DT 都可能提供物理内存事实。内核必须先把这些事实合并成一个可计算的区间图，后面的 zone 和 buddy 才有边界依据。

memblock_add 与 memblock_reserve：同一个插入算法，写入不同集合

/* mm/memblock.c */
static int __init_memblock memblock_add_range(struct memblock_type *type,
                                phys_addr_t base, phys_addr_t size,
                                int nid, enum memblock_flags flags)
{
        bool insert = false;
        phys_addr_t obase = base;
        phys_addr_t end = base + memblock_cap_size(base, &size);
        int idx, nr_new;
        struct memblock_region *rgn;

        if (!size)
                return 0;

        /* special case for empty array */
        if (type->regions[0].size == 0) {
                WARN_ON(type->cnt != 1 || type->total_size);
                type->regions[0].base = base;
                type->regions[0].size = size;
                type->regions[0].flags = flags;
                memblock_set_region_node(&type->regions[0], nid);
                type->total_size = size;
                return 0;
        }
repeat:
        base = obase;
        nr_new = 0;

        for_each_memblock_type(idx, type, rgn) {
                phys_addr_t rbase = rgn->base;
                phys_addr_t rend = rbase + rgn->size;

                if (rbase >= end)
                        break;
                if (rend <= base)
                        continue;
                if (rbase > base) {
                        nr_new++;
                        if (insert)
                                memblock_insert_region(type, idx++, base,
                                                       rbase - base, nid,
                                                       flags);
                }
                base = min(rend, end);
        }

        if (base < end) {
                nr_new++;
                if (insert)
                        memblock_insert_region(type, idx, base, end - base,
                                               nid, flags);
        }

        if (!nr_new)
                return 0;

        if (!insert) {
                while (type->cnt + nr_new > type->max)
                        if (memblock_double_array(type, obase, size) < 0)
                                return -ENOMEM;
                insert = true;
                goto repeat;
        } else {
                memblock_merge_regions(type);
                return 0;
        }
}

逐段解释：`memblock_add_range()` 先处理空数组，再进入两遍算法。第一遍只计算需要插入多少个新片段，必要时扩容；第二遍真正插入。

它允许新区间与已有区间重叠。重叠部分不重复插入，只插入已有区间覆盖不到的洞，最后调用 `memblock_merge_regions()` 合并相邻兼容区间。这样 memblock 始终保持紧凑、有序、可遍历。

/* mm/memblock.c */
int __init_memblock memblock_add(phys_addr_t base, phys_addr_t size)
{
        phys_addr_t end = base + size - 1;

        memblock_dbg("%s: [%pa-%pa] %pS\n", __func__,
                     &base, &end, (void *)_RET_IP_);

        return memblock_add_range(&memblock.memory, base, size, MAX_NUMNODES, 0);
}

int __init_memblock memblock_reserve(phys_addr_t base, phys_addr_t size)
{
        phys_addr_t end = base + size - 1;

        memblock_dbg("%s: [%pa-%pa] %pS\n", __func__,
                     &base, &end, (void *)_RET_IP_);

        return memblock_add_range(&memblock.reserved, base, size, MAX_NUMNODES, 0);
}

逐段解释：`memblock_add()` 与 `memblock_reserve()` 都复用同一个区间插入算法，区别只是目标集合不同。前者声明“这段 RAM 存在且可纳入内核管理”，后者声明“这段 RAM 暂不能释放给普通分配器”。

第一性原理概述：“存在”和“可用”是两件事。memblock 用 `memory` 记录存在的 RAM，用 `reserved` 扣除暂不可用部分，避免把内核自身、设备树、initrd 等关键数据释放掉。

早期分配约束：低端、方向、直接可访问

/* arch/mips/kernel/setup.c */
bootmem_init();

/*
 * Prevent memblock from allocating high memory.
 * This cannot be done before max_low_pfn is detected, so up
 * to this point is possible to only reserve physical memory
 * with memblock_reserve; memblock_alloc* can be used
 * only after this point
 */
memblock_set_current_limit(PFN_PHYS(max_low_pfn));
...
memblock_set_bottom_up(false);
sparse_init();
memblock_set_bottom_up(true);

逐段解释：`bootmem_init()` 之后才知道 `max_low_pfn`，因此在此之前只能安全地做 `memblock_reserve()` 这种记录操作，不能随意做 `memblock_alloc*()`。

`memblock_set_current_limit(PFN_PHYS(max_low_pfn))` 把早期分配限制在低端内存中。对 32 位 MIPS 来说，低端内存可被内核直接映射访问；如果早期分配跑到 highmem，普通线性地址可能无法直接访问。

`sparse_init()` 前临时切到 top-down 分配，是为了让稀疏内存模型的元数据分配策略减少对低地址的挤占；结束后切回 bottom-up。这里体现 memblock 不只是区间表，也提供受方向和上限约束的早期分配策略。

第一性原理概述：早期分配必须满足“分配出来马上能被当前内核访问”。因此 memblock 不只找空洞，还要受 `current_limit`、分配方向、对齐和 node 等约束控制。

伙伴系统创建入口：MIPS `paging_init()`

/* arch/mips/mm/init.c */
void __init paging_init(void)
{
        unsigned long max_zone_pfns[MAX_NR_ZONES];

        pagetable_init();

#ifdef CONFIG_HIGHMEM
        kmap_init();
#endif
#ifdef CONFIG_ZONE_DMA
        max_zone_pfns[ZONE_DMA] = MAX_DMA_PFN;
#endif
#ifdef CONFIG_ZONE_DMA32
        max_zone_pfns[ZONE_DMA32] = MAX_DMA32_PFN;
#endif
        max_zone_pfns[ZONE_NORMAL] = max_low_pfn;
#ifdef CONFIG_HIGHMEM
        max_zone_pfns[ZONE_HIGHMEM] = highend_pfn;

        if (cpu_has_dc_aliases && max_low_pfn != highend_pfn) {
                printk(KERN_WARNING "This processor doesn't support highmem."
                       " %ldk highmem ignored\n",
                       (highend_pfn - max_low_pfn) << (PAGE_SHIFT - 10));
                max_zone_pfns[ZONE_HIGHMEM] = max_low_pfn;
        }
#endif

        free_area_init(max_zone_pfns);
}

逐段解释：`paging_init()` 先初始化内核页表，再准备 `max_zone_pfns[]`，告诉通用 page allocator 每个 zone 的结束 PFN。

当前普通配置重点是 `ZONE_NORMAL`，其结束 PFN 是 `max_low_pfn`，这个值来自前面的 `bootmem_init()` 对 memblock DRAM 范围的扫描。最后调用 `free_area_init(max_zone_pfns)`，进入通用 buddy 元数据初始化。

第一性原理概述：伙伴系统不能凭空知道自己管理哪段物理页。架构代码必须先把硬件/平台约束转换成 zone 边界，通用代码才能按 zone 建立 free_area。

`free_area_init()`：用 memblock PFN 范围计算 node/zone

/* mm/page_alloc.c */
void __init free_area_init(unsigned long *max_zone_pfn)
{
        unsigned long start_pfn, end_pfn;
        int i, nid, zone;
        bool descending;

        memset(arch_zone_lowest_possible_pfn, 0,
                                sizeof(arch_zone_lowest_possible_pfn));
        memset(arch_zone_highest_possible_pfn, 0,
                                sizeof(arch_zone_highest_possible_pfn));

        start_pfn = find_min_pfn_with_active_regions();
        descending = arch_has_descending_max_zone_pfns();

        for (i = 0; i < MAX_NR_ZONES; i++) {
                if (descending)
                        zone = MAX_NR_ZONES - i - 1;
                else
                        zone = i;

                if (zone == ZONE_MOVABLE)
                        continue;

                end_pfn = max(max_zone_pfn[zone], start_pfn);
                arch_zone_lowest_possible_pfn[zone] = start_pfn;
                arch_zone_highest_possible_pfn[zone] = end_pfn;

                start_pfn = end_pfn;
        }

        find_zone_movable_pfns_for_nodes();

        pr_info("Early memory node ranges\n");
        for_each_mem_pfn_range(i, MAX_NUMNODES, &start_pfn, &end_pfn, &nid) {
                pr_info("  node %3d: [mem %#018Lx-%#018Lx]\n", nid,
                        (u64)start_pfn << PAGE_SHIFT,
                        ((u64)end_pfn << PAGE_SHIFT) - 1);
                subsection_map_init(start_pfn, end_pfn - start_pfn);
        }

        mminit_verify_pageflags_layout();
        setup_nr_node_ids();
        for_each_online_node(nid) {
                pg_data_t *pgdat = NODE_DATA(nid);
                free_area_init_node(nid);

                if (pgdat->node_present_pages)
                        node_set_state(nid, N_MEMORY);
                check_for_memory(pgdat, nid);
        }
}

逐段解释：`free_area_init()` 先根据架构给出的 `max_zone_pfn[]` 计算每个 zone 的最低/最高可能 PFN。zone 边界是 page allocator 的地址空间分区。

`for_each_mem_pfn_range()` 遍历早期 memblock memory 范围，打印 node 内存范围并初始化 subsection map。这个循环说明：buddy 的 node/zone 初始化仍然以 memblock 提供的物理 PFN 图为输入。

最后对每个 online node 调 `free_area_init_node()`，它会计算 node 总页数、分配 `struct page` 数组并初始化 zone。

第一性原理概述：zone 是“物理页的管理边界”，不是内存发现机制。memblock 发现和修正物理图，`free_area_init()` 把这张图投影成 allocator 可以使用的 node/zone 数据结构。

zone 核心结构：managed_pages 与 free_area[]

/* include/linux/mmzone.h */
struct free_area {
        struct list_head        free_list[MIGRATE_TYPES];
        unsigned long           nr_free;
};

struct zone {
        ...
        atomic_long_t           managed_pages;
        unsigned long           spanned_pages;
        unsigned long           present_pages;
        ...
        const char              *name;
        ...
        struct free_area        free_area[MAX_ORDER];
        unsigned long           flags;
        spinlock_t              lock;
        ...
};

逐段解释：`free_area` 是伙伴系统最核心的数据结构。每个 order 对应一个 `free_area`，每个 `free_area` 又按迁移类型分多个链表，`nr_free` 记录该 order 的空闲块数量。

`zone` 记录一个物理内存区域的边界与状态。`spanned_pages` 是 zone 覆盖范围，包括洞；`present_pages` 是实际存在页；`managed_pages` 是 buddy 真正管理的页数，它会在 memblock 释放页面时重新计数。

图 3：伙伴系统创建的目标，是让 zone 能按 order 快速拆分/合并空闲页块。
第一性原理概述：伙伴系统的基本单位不是任意字节区间，而是 2 的幂个连续页。用 order 链表组织空闲块，可以在分配时拆大块、释放时按地址关系合并 buddy。

}

逐段解释：源码注释直接说明：初始所有页都是 reserved，空闲页要等 `memblock_free_all()` 后才释放。这个设计避免在元数据尚未完整初始化时，页面被普通分配路径拿走。

`zone_init_free_lists()` 对所有 order、所有 migratetype 初始化链表头，并把 `nr_free` 置 0。此时 buddy 的结构已经有了，但里面没有页块。

`init_currently_empty_zone()` 设置 zone 起始 PFN，初始化 free list，并标记 zone 已初始化。名字里的 “empty” 很准确：这是一个还没接收空闲页的 zone。

第一性原理概述：数据结构初始化必须先于对象发布。buddy 先建立空链表和 `struct page` 状态，再由 memblock 按真实空闲范围发布页面，避免把未初始化页暴露给分配器。

逐段解释：`mm_init()` 中先调用体系结构的 `mem_init()`，再初始化 slab。也就是说 slab 普通缓存建立前，buddy 必须已经拿到可用页。

MIPS `mem_init()` 设置 `max_mapnr` 和 `high_memory` 后，调用 `memblock_free_all()`。这是本文的交接点：memblock 不再只是早期分配器，它会把自己认为空闲的低端页释放进 page allocator。

第一性原理概述：slab、vmalloc、后续内核子系统都依赖页分配器。因此 `mem_init()` 必须在这些系统之前完成 memblock 到 buddy 的交接。

`memblock_free_all()`：遍历 free ranges，把页释放给 buddy

/* mm/memblock.c */
static void __init __free_pages_memory(unsigned long start, unsigned long end)
{
        int order;

        while (start < end) {
                order = min(MAX_ORDER - 1UL, __ffs(start));

                while (start + (1UL << order) > end)
                        order--;

                memblock_free_pages(pfn_to_page(start), start, order);

                start += (1UL << order);
        }
}

static unsigned long __init __free_memory_core(phys_addr_t start,
                                 phys_addr_t end)
{
        unsigned long start_pfn = PFN_UP(start);
        unsigned long end_pfn = min_t(unsigned long,
                                      PFN_DOWN(end), max_low_pfn);

        if (start_pfn >= end_pfn)
                return 0;

        __free_pages_memory(start_pfn, end_pfn);

        return end_pfn - start_pfn;
}

static unsigned long __init free_low_memory_core_early(void)
{
        unsigned long count = 0;
        phys_addr_t start, end;
        u64 i;

        memblock_clear_hotplug(0, -1);

        for_each_reserved_mem_range(i, &start, &end)
                reserve_bootmem_region(start, end);

        for_each_free_mem_range(i, NUMA_NO_NODE, MEMBLOCK_NONE, &start, &end,
                                NULL)
                count += __free_memory_core(start, end);

        return count;
}

unsigned long __init memblock_free_all(void)
{
        unsigned long pages;

        reset_all_zones_managed_pages();

        pages = free_low_memory_core_early();
        totalram_pages_add(pages);

        return pages;
}

逐段解释：`memblock_free_all()` 先把所有 zone 的 `managed_pages` 归零，随后调用 `free_low_memory_core_early()`。这说明 `free_area_init_core()` 中的 `managed_pages` 只是近似值，真正计数要在释放时重新计算。

`free_low_memory_core_early()` 先遍历 reserved 区间并调用 `reserve_bootmem_region()`，把这些页保持为 reserved。然后 `for_each_free_mem_range()` 遍历 `memory - reserved` 的空闲区间，只释放这些范围。

`__free_memory_core()` 将物理地址按页对齐，并裁剪到 `max_low_pfn` 以下。`__free_pages_memory()` 再把连续 PFN 范围切成尽可能大的、对齐的 order 块，调用 `memblock_free_pages()` 释放。

图 4：交接不是复制内存，而是把 memblock 认定可释放的 PFN 变成 buddy free list 节点。
第一性原理概述：memblock 到 buddy 的移交本质是所有权转移：同一批物理页不移动，只是管理权从“区间差集”转到“zone/order 链表”。

`__free_pages_core()`：清 reserved、清 refcount、进入 buddy

/* mm/page_alloc.c */
void __free_pages_core(struct page *page, unsigned int order)
{
        unsigned int nr_pages = 1 << order;
        struct page *p = page;
        unsigned int loop;

        /*
         * When initializing the memmap, __init_single_page() sets the refcount
         * of all pages to 1 ("allocated"/"not free"). We have to set the
         * refcount of all involved pages to 0.
         */
        prefetchw(p);
        for (loop = 0; loop < (nr_pages - 1); loop++, p++) {
                prefetchw(p + 1);
                __ClearPageReserved(p);
                set_page_count(p, 0);
        }
        __ClearPageReserved(p);
        set_page_count(p, 0);

        atomic_long_add(nr_pages, &page_zone(page)->managed_pages);

        /*
         * Bypass PCP and place fresh pages right to the tail, primarily
         * relevant for memory onlining.
         */
        __free_pages_ok(page, order, FPI_TO_TAIL);
}

void __init memblock_free_pages(struct page *page, unsigned long pfn,
                                                        unsigned int order)
{
        if (early_page_uninitialised(pfn))
                return;
        __free_pages_core(page, order);
}

逐段解释：`struct page` 初始化时引用计数为 1，含义是“已分配/不可自由使用”。释放给 buddy 前，`__free_pages_core()` 对整个 order 块逐页清 `PageReserved`，并把 refcount 置 0。

随后 `managed_pages` 增加 `1 << order`，表示这些页现在由 buddy 管理。最后调用 `__free_pages_ok()`，它会进入普通释放路径，把页块按 order 和 migratetype 放入 zone 的 `free_area[]`，并在可能时合并 buddy。

`memblock_free_pages()` 还会跳过 `early_page_uninitialised()` 的 PFN，避免把尚未初始化 `struct page` 的页释放给普通分配器。

第一性原理概述：要让一个页能被分配，必须同时满足三个条件：它不再是 reserved，引用计数表明它空闲，并且它被挂入 allocator 能找到的数据结构。

逐段解释：`build_all_zonelists()` 在 `mm_init()` 前运行，建立从哪些 zone 以及按什么 fallback 顺序分配的路由表。`page_alloc_init()` 注册 CPU dead 回调，用于 CPU hotplug 时处理 page allocator 状态。

`setup_per_cpu_pageset()` 在中后段启动时运行，为每个 zone 建立正式 per-cpu pageset。此后小批量页分配可以先走 PCP，减少频繁触碰 zone 全局锁；PCP 与 buddy 之间批量转移页面。

`kmem_cache_init()` 位于 `mem_init()` 之后，说明 slab 建立在 buddy 已可用的基础上。memblock 负责早期粗粒度区间分配，buddy 负责页级分配，slab 再在页之上提供对象级缓存。

第一性原理概述：内存管理是分层的。memblock 解决自举，buddy 解决页框，PCP 解决多 CPU 热路径，slab 解决小对象。每一层都依赖下一层先建立足够稳定的供给。

整体总结

memblock 管什么

它管理物理区间集合：`memory` 表示可纳入管理的 RAM，`reserved` 表示不可释放的保留范围，空闲范围是二者差集。

buddy 怎么创建

MIPS `paging_init()` 给出 zone 边界，`free_area_init()` 创建 node、zone、mem_map、usemap，并初始化 `free_area[MAX_ORDER]` 空链表。

如何移交

`mem_init()` 调 `memblock_free_all()`，遍历 `for_each_free_mem_range()`，把低端空闲 PFN 切成 order 块，清 reserved/refcount 后释放进 buddy。

为什么这样设计

早期只能用小而静态的区间表自举；buddy 需要完整 `struct page` 和 zone 元数据；二者分阶段交接避免循环依赖和未初始化页暴露。

从第一性原理看，memblock 到 buddy 的变化不是内存内容搬运，而是“管理模型”的切换：从少量物理区间记录，切换到按页、按 order、按 zone 的可并发分配结构。
