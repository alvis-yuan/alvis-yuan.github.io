第一性原理：为何需要 SFC

SPI Flash 是 X2600 常见的启动介质与固件存储，但 bit 级 SPI 协议要求精确的命令—地址—哑周期—数据时序，且需支持 Standard/Dual/Quad 等多种线宽模式。若由 CPU 软件 bit-bang GPIO，既无法满足 tSETUP/tHOLD/tSH 等 AC 时序，也无法在高速时钟下通过可调采样点（SMP_DELAY）可靠接收数据。

手册 7.1 定义 SFC 的本质：SPI 主设备，在 AHB 从端口（寄存器/CPU 控制模式）与 AHB 主端口（DMA）之间插入 64×32 FIFO、6 phase 流控与 CDT/DMA 描述符链，使软件以「配置寄存器 → 触发 → 等待 END」的方式完成 Flash 读写，而非逐 bit 操作引脚。

必要 每次传输前必须配置

  • SFC_DEV_CONF — CPOL/CPHA、CE 时序、SMP_DELAY 采样点
  • SFC_CGE — 清所有 clock gate 位（手册 Note）
  • SFC_GLB0 — phase 数、方向、slave/DMA 模式、FIFO 阈值
  • 各 phase 的 TRAN_CFG0/1、DEV_ADDR、TRAN_LEN 等
  • SFC_TRIG.START 启动；轮询/中断等待 SR.END

增强 运行期可选能力

  • 7 种传输格式（Standard 至 Full Quad I/O）
  • CDT 命令描述符表（Base+0x0800~0x0C00）
  • DMA 描述符链（DES_ADDR + 内存中描述符）
  • Polling 等待 Flash status（POLL_EN + DEV_STA_EXP/MSK）
  • 双片选 GLB1.CHIP_SEL / CHIP1_SEL

接口用途约束
AHB Slave寄存器配置；slave 模式传数（写/读 DR @ Base+0x1000）数据与地址须 4 字节对齐；仅支持 SINGLE 传输
AHB MasterDMA 模式在 DDR 与 FIFO/Flash 间搬移数据支持 SINGLE、INCR、INCR4/8/16/32
两类 CPU 访问要区分：
① 寄存器访问 — MMIO 读写 0x13440000 区域，配置 DEV_CONF/GLB/TRIG 等。
② 数据传输 — Slave 模式 CPU 按 THRESHOLD 读写 DR；DMA 模式 SFC 作为 AHB Master 访问 MEM_ADDR 或描述符指定的 DDR 物理地址。

配置完成后如何工作

一次传输（flow）由最多 6 个 phase 组成；每个 phase 含 CMD、ADDR、DUMMY、DATA 四段中的若干段，且全 flow 仅一个 phase 可有 DATA（7.8.3）。Flow Control 按 PHASE_NUM 顺序执行各 phase，SPI Control 产生符合所选 TRAN_MD 的线宽时序。

图 2 — 单次 flow：多 phase 经 FIFO 与 SPI 交互，结束置 END（7.7.3.14–15）

传输格式（7.2 / 7.6.1）

7 种格式：Standard SPI、Dual-Output/Input、Dual I/O、Full Dual I/O、Quad-Output/Input、Quad I/O、Full Quad I/O。由 TRAN_CFG1.TRAN_MD 选择。字节序：命令/地址 MSB first；数据字节间 LSB first（7.2）；可用 TRAN_CFG1.DATA_ENDIAN / DEV_CONF.STA_ENDIAN 调整。

Phase 格式（7.8.3）

  • Format0：CMD → ADDR → DUMMY → DATA
  • Format1：CMD → ADDR/DUMMY 交错 → DATA
  • 一次 flow 仅支持一种传输方向（读或写）

状态与中断（SR / SCR / INTC）

SR 位含义
END传输结束，产生 INT_END
TRAN_REQ (TREQ)FIFO 空位 ≥ THRESHOLD，写方向请求 CPU/DMA 填 FIFO
RECE_REQ (RREQ)FIFO 数据 ≥ THRESHOLD，读方向请求取 FIFO
OVER / UNDERFIFO 溢出 / 欠载
WORKING / BUSYSFC / SPI 子状态机

清中断：写 SCR 对应 CLR_* 位。INTC 各位默认屏蔽（MASK=1）；清标志后若条件仍满足会立即再次产生中断（7.7.3.17 NOTE）。

采样与时序（7.4.4）

高速时钟下传播延迟可能导致采样错位，需配置 DEV_CONF.SMP_DELAY（参考 Figure 7-3）。AC 时序 tSETUP(tSLCH)、tHOLD(tCHSH)、tSH(tSHSL) 由 DEV_CONF 的 TSETUP/THOLD/TSH_H/TSH_L 配置（Figure 7-4）。

7.8 软件配置与启动流程

手册未单独列出「上电初始化」章节，但 7.8.4 Figure 7-8 定义了多 phase flow的标准步骤，Boot ROM 与驱动均遵循「硬件参数 → 传输参数 → 触发 → 等待 END」模式。

图 3 — 手册 7.8.4 Multi phases flow（Figure 7-8）
  1. 配置与设备硬件相关的参数（IO 默认电平、时序等）— SFC_DEV_CONF；并清 SFC_CGE 所有位 disable clock gate
  2. 配置本次 flow：phase 数、命令格式、模式、源/目标地址、方向与长度 — GLB、DEV_STA_EXP/MSK、TRAN_CFG0/1、TRAN_LEN、DEV_ADDR、MEM_ADDR 等
  3. 写 SFC_TRIG.START 启动
  4. 等待 END 中断或轮询 SR.END
  5. 写 SCR 清除 END；其他 flow 重复步骤 2

各阶段与三种工作模式

支持模糊匹配；按 Esc 清除

阶段 ① — 设备硬件参数（7.8.4 Step 1）

SFC_DEV_CONF：CPOL/CPHA、CE/HOLD/WP 默认无效电平、TSETUP/THOLD/TSH、SMP_DELAY 采样延迟、CMD_TYPE/STA_TYPE。

SFC_CGE：手册 Note — please disable all the clock gate function（clear all bits of SFC_CGE）。

阶段 ②a — 全局 flow 参数（GLB0）

  • PHASE_NUM — 001~110 对应 1~6 phase
  • OP_MODE — 0 slave（CPU 控制）；1 DMA
  • TRAN_DIR — 0 从 Flash 读；1 向 Flash 写
  • THRESHOLD — FIFO 阈值（word），不可为 0
  • CDT_EN / DES_EN — 使能 CDT 或 DMA 描述符链
  • BURST_MD — DMA burst：INCR8/16/32

阶段 ②b — 各 Phase 传输参数

每 phase 一对 TRAN_CFG0_n + TRAN_CFG1_n，配合 DEV_ADDRn、必要时 DEV_PLUSn。

  • TRAN_CFG0：CMD_EN、ADDR_WIDTH、DMY_BITS、DATA_EN、POLL_EN、PHASE_FORMAT、CMD
  • TRAN_CFG1：TRAN_MD（7 种 SPI 格式）、DATA_ENDIAN、WORD_UNIT
  • TRAN_LEN — 字节数；无 data phase 时必须为 0
  • Polling：DEV_STA_EXP + DEV_STA_MSK；POLL_EN 优先级高于 tran_dir 与 data_en

模式 A — Slave（CPU 控制，7.8.7 / 7.8.11）

GLB0.OP_MODE=0。每个 data phase 启动前必须 TRIG.FLUSH。

  • 写方向：等 SR.TREQ → 按 THRESHOLD 向 DR 写数据 → 循环至传完 → 等 END
  • 读方向：等 SR.RREQ → 按 THRESHOLD 从 DR 读 → 循环 → 等 END
  • 每次 AHB 传输数据量须与 THRESHOLD 匹配

模式 B — DMA（7.8.6 / 7.8.9）

GLB0.OP_MODE=1。额外配置 GLB（BURST_MD 等）与 MEM_ADDR（DDR 物理地址）。

  • 简单 DMA：DES_EN=0，SFC 作 AHB Master 在 MEM_ADDR 与 FIFO 间搬移
  • 描述符链：内存中建描述符 → DES_ADDR 指向首描述符 → DES_EN=1（不再使用 MEM_ADDR 寄存器）
  • 描述符结构（7.8.2）：Word0 LINK/TRAN_MODE/…；Word1 命令字段；Word2 DEV_STA_EXP；Word3 DEV_STA_MSK；随后 NEXT_DES_ADDR、MEM_ADDR、TRAN_LEN

模式 C — CDT（7.8.1 / 7.8.10）

GLB0.CDT_EN=1。每条命令 4 word，可链接；硬件每次执行一条 command chain，每条 chain 至多一次 data 传输。

  • 写 CDT 寄存器区（Base+0x0800~0x0C00）
  • 按 ADDR_KIND 配置 COL_ADDR / ROW_ADDR / STA_ADDR0 / STA_ADDR1
  • CMD_IDX：CMD_IDX 索引 + CDT_DATA_EN + CDT_DIR
  • 流程：DEV_CONF + GLB → 配 CDT → 配地址寄存器 → CMD_IDX → START → 等 END 并清除

阶段 ③④⑤ — 触发与完成

TRIG.START 写 1 启动；TRIG.STOP 写 1 停止；TRIG.FLUSH 写 1 复位 FIFO。

轮询 SR.END 或使能 INTC 后等中断 → SCR.CLR_END 清除。

CPU 与 SFC 的交互

MMIO 配置路径

CPU 通过 AHB Slave 访问 0x13440000 配置 SFC。手册 7.7.1：Register Address = Base + Offset；寄存器由 AHB2 总线读写。这是 Bootloader/驱动在每次 Flash 操作前的标准路径。

数据传输的两条路径

模式CPU 角色SFC 角色数据接口
Slave主动轮询 TREQ/RREQ，读写 DRSPI 主 + FIFODR @ Base+0x1000，4 字节对齐 SINGLE
DMA仅配置寄存器/描述符AHB Master 访问 DDRMEM_ADDR 或描述符内 MEM_ADDR

Boot ROM 中的 SFC（第 42 章）

复位后 CPU 从内部 Boot ROM 启动。Table 42-1：boot_sel 为 001/010/110 等时从 SFC（SPI NOR/NAND）启动。Boot 程序初始化引脚 SFC_CLK、SFC_CE、SFC_DR、SFC_DT、SFC_WP、SFC_HOL，将 SPL 加载到 Cache 并跳转（42.2）。Boot ROM 占用 Cache 0x80000000–0x80020000（128KB）。
