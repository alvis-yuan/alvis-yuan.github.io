第一性原理：为何需要 I2C 控制器

传感器、EEPROM、PMIC、触摸屏等外设普遍使用 I2C 两线总线（SDA + SCL）。协议要求主设备产生时钟、发 START/STOP/RESTART、逐字节 ACK/NACK——若由 CPU 软件 bit-bang，既难满足 tSU:DAT、tHD:DAT 等时序，也无法高效处理多字节突发。

手册 30.1：I2C 控制器由软件配置为主或从（不可同时），内置可编程 SCL 发生器、Tx/Rx FIFO、中断与 DMA，把「位级时序」下沉到硬件，CPU 只需配置寄存器并向 I2C_DC 写入命令/数据。

必要 使用前必须配置

  • I2C_ENABLE=0 下写 CON/TAR/SAR、SCL 计数寄存器
  • 按 SPEED 选 SHCNT/SLCNT 或 FHCNT/FLCNT 或 HHCNT/HLCNT
  • 主模式：CON.MD=1 且 SLVDIS=1；从模式：MD=0 且 SLVDIS=0
  • 使能后写 I2C_DC（STOP/RESTART/CMD/DAT）
  • 禁用时轮询 ENBST.I2CEN=0（30.3.4）

增强 运行期特性

  • 7/10 位寻址、General Call、START byte
  • High-Speed 模式与 HS_MADDR
  • FIFO 阈值中断 + 独立 clear 寄存器
  • Tx/Rx DMA（DMACR、DMATDLR、DMARDLR）
  • 4 路独立通道 I2C0–I2C3

使能后如何工作

手册 30.3.1：主设备产生 SCL 并控制传输；从设备响应地址并收发数据。控制器通过 I2C_DC 向 Tx FIFO 填入「命令字」（含 CMD 读/写、STOP、RESTART）与数据字节，硬件按 FIFO 顺序在总线上执行。

图 2 — 主模式：DC→Tx FIFO→总线；读操作数据进 Rx FIFO（30.2.2.5、30.3.2.3）

SCL 频率公式（30.3.2.1）

Tscl = TI2C_clk × ((I2C*HCNT + 8) + (I2C*LCNT + 1))
(I2C*HCNT + 8) × TI2C_clk ≥ Tmin_scl_h
(I2C*LCNT + 1) × TI2C_clk ≥ Tmin_scl_l
I2C*HCNT ≥ 6 · I2C*LCNT ≥ 8

CON.SPEED=01 配 SHCNT/SLCNT；=10 配 FHCNT/FLCNT；=11 配 HHCNT/HLCNT。

STOP / RESTART（30.3.6）

  • STOP 仅当 I2C_DC[9]=1 时产生；Tx FIFO 空且 STOP=0 时 SCL 拉低等待新命令
  • RESTART 需 CON[5]=1（能力）且 DC[10]=1（本字节前发 RESTART）；否则发 STOP+START
  • CON[5] 决定能否 repeat start；DC[10] 决定本次是否发 RESTART

30.3 初始化与操作流程

图 3 — 30.3.2.1 主模式 Configuration 七步

主模式（30.3.2.1）

  1. 写 I2C_ENABLE=0，等待 I2C_ENBST.I2CEN=0
  2. 写 I2C_CON（SPEED、RESTART、MD=1、SLVDIS=1）；MATP 只读，寻址由 TAR 控制
  3. 按 SPEED 配置 SCL 高/低计数寄存器（见上节公式）
  4. 写 I2C_TAR（目标地址、7/10 位 MATP）
  5. 写 I2C_ENABLE=1，等待 I2C_ENBST.I2CEN=1
  6. 写 I2C_DC 发起传输（未使能前写入的数据会丢失）

从模式（30.3.3.1）

  1. I2C_ENABLE=0
  2. 写 I2C_SAR 从地址
  3. I2C_CON：SATP 选 7/10 位；SLVDIS=0，MD=0
  4. I2C_ENABLE=1

从发（RDREQ）：主读时 SCL 拉低，软件写 DC（CMD=0）后清 RDREQ/TXABT。从收（RXFL）：读 DC 取字节，注意 RXOF 溢出。

安全禁用（30.3.4.1）

tI2C_poll = 10 × 最快 SCL 周期（400 kHz → 25 µs）；写 ENABLE=0 后轮询 ENBST.I2CEN 直至为 0。

关键寄存器与模式

支持模糊匹配；按 Esc 清除

全局 — I2C_CON / I2C_ENABLE

CON 仅在 ENABLE=0 可写。主：MD=1、SLVDIS=1。从：MD=0、SLVDIS=0。ENABLE 清 FIFO；ABORT 位仅主模式且 ENABLE=1 时有效。

SCL 时序 — *HCNT / *LCNT

Standard/Fast/High 三套计数；禁用时写；硬件强制最小 HCNT≥6、LCNT≥8。另：SDAHD（hold）、SDASU（从发 setup）、FSPKLEN（毛刺抑制）。

寻址 — I2C_TAR / I2C_SAR

TAR：主模式目标地址 + MATP + General Call/START byte（SPECIAL/GC_OR_START）。SAR：从模式本机地址；禁用时写。TAR/SAR 可动态更新（30.3.2.2 条件）。

数据传输 — I2C_DC

写 Tx：DAT + CMD(0=写/1=读) + STOP + RESTART。读 Rx：读 DC 低 8 位。每命令 8 位数据 + 1 位 CMD。

中断 — INTST / INTM / CR*

INTM 低有效屏蔽（0=屏蔽）。RDREQ：从被读须写 DC。TXABT：读 CTXABT 释放 Tx FIFO。各 CR* 读清对应位。

DMA — I2C_DMACR

TDEN/RDEN 独立于 ENABLE 可配。TDLR：Tx 有效条目 ≤ 阈值时 dma_tx_req。RDLR：Rx 有效条目 ≥ RDLR+1 时 dma_rx_req。主模式步骤 6 可由 DMA 写 DC/FIFO。

禁用确认 — I2C_ENBST

ENABLE 1→0 后读 I2CEN；SLVDISB/SLVRDLST 报告禁用时的从机异常状态。

CPU 与 I2C 的交互

MMIO 配置与 PIO 传输

CPU 通过 APB 对各路基址做 load/store，无内存映射的 I2C 从设备空间——访问外设 EEPROM/传感器仍由 I2C 控制器在 SDA/SCL 上发协议。软件路径：配置寄存器 → 写 DC 填 Tx FIFO → 中断或轮询 ST/INTST → 读 DC 取 Rx。

DMA 路径

使能 DMACR 后，系统 DMA 根据 FIFO 水位自动读写 I2C_DC 或等效数据端口，减轻 CPU 逐字节负担（30.3.2.1 步骤 6 注明可由 DMA 写 DC）。

图 4 — CPU 经 APB 配置 I2C；可选 DMA 搬运 DC/FIFO；物理外设仍接 SDA/SCL（30.1、30.2.2.35）
与 DDR/SFC 的区别：I2C 外设数据不在 CPU 物理地址空间内直接 load/store；CPU 只访问控制器寄存器，由硬件在 I2C 总线上完成对外设的读写。
