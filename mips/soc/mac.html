第一性原理：为何需要 MAC 与 DMA

以太网帧可达 1500+ 字节，若由 CPU 逐字节搬运并与 PHY bit 级交互，将占用大量周期且难以满足线速。MAC（媒体访问控制器） 在主机与外部 PHY（经 RMII）之间完成 IEEE 802.3 成帧、地址过滤、流控与 MDIO 管理；DMA 则通过描述符链表在系统内存与 MAC 内部 FIFO 间搬运整帧，使 CPU 仅在描述符完成时介入。

图 1 — 主机经 DMA 与 MAC 核心交换帧数据，MAC 经 RMII/MDIO 连接外部 PHY（Figure 36-1）

必要 最小以太网收发集合

  • MAC_CR / MAC_FFR — 速率、双工、收发使能与地址过滤。
  • GMII_AR / GMII_DR — MDIO 配置外部 PHY。
  • MAC_A0HR / MAC_A0LR — 本机 MAC 地址。
  • DMA_TDLAR / DMA_RDLAR + DMA_OMR — 描述符链与 Tx/Rx 启停。
  • DMA_SR / DMA_IER — 完成与异常中断。

增强特性 手册列出的扩展能力

  • 全/半双工、802.3x 流控、Jumbo（2KB）、VLAN 检测与插入/替换。
  • 哈希/额外 MAC 地址过滤、远程唤醒与 Magic Packet 检测。
  • EEE LPI（LPI_CSR/LPI_TCR）、MMC/PMT 子模块中断。

36.1 概述（Overview）

处理器内含一个快速以太网 MAC，支持 10/100 Mbps。MAC 在主机应用与 PHY 层之间通过 RMII 提供接口；PHY 器件在处理器外部。

36.2 特性（Features）

  • 10/100 Mbps，RMII 连接外部 Fast Ethernet PHY
  • 全双工：IEEE 802.3x 流控、零配额 Pause 帧自动发送、转发接收到的 Pause 帧
  • 半双工：CSMA/CD、100 Mbps 下半双工帧突发与帧扩展
  • 发送路径插入 Preamble/SFD；接收路径删除 Preamble/SFD
  • 可按帧控制自动 CRC/Pad 生成；接收可选自动剥离 Pad/CRC
  • 灵活地址过滤：最多 31 个额外 48 位完美 DA 过滤、64 位哈希、混杂模式等
  • 标准或 Jumbo 以太网帧（最大 2 KB）；IEEE 802.1Q VLAN 标签检测
  • MDIO 主接口配置 PHY；发送可 CRC 替换、SA 插入/替换、VLAN 插入/替换/删除
  • 接收路径可编程看门狗超时；可检测远程唤醒帧与 AMD Magic Packet

36.4 功能描述（Functional Description）

MAC 使主机能够按 IEEE 802.3-2008 标准在以太网上收发数据。MAC 为可配置 IP，可在门数与延迟间权衡，适用于交换机、网卡等应用。

36.8 操作流程（Operation Flow）

手册 Figure 36-2 建议的 MAC 配置与启动网络流程如下（文本顺序来自 PDF）：

图 3 — 描述符驱动的 Tx/Rx 数据路径与中断（36.8 操作流程）
  1. 配置 MAC 地址与设备地址
  2. 编程 MDC 时钟（GMII_AR.CR）
  3. 初始化 PHY 寄存器（MDIO 读写）
  4. 注册中断处理程序
  5. 建立发送/接收描述符（Setup descriptor）
  6. 配置 DMA（DMA_BMR、DMA_OMR、链表地址等）
  7. 配置 MAC（MAC_CR、MAC_FFR 等）
  8. 使能 Tx 与 Rx 路径（MAC_CR.TE/RE、DMA_OMR.ST/SR）
  9. 为接收描述符分配 buffer
  10. 使能中断与 DMA（DMA_IER）
  11. 启动网络（Start network）
