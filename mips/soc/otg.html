第一性原理：为何需要 OTG

传统 USB 假设单一主机连接多个从设备。OTG（On-The-Go）允许便携设备互为宿主与设备，通过 SRP/HNP 协议协商角色。X2600 集成双端口 USB 控制器：Port 0 支持完整 OTG 与 BC1.2 充电检测，Port 1 仅基本 USB 功能。

图 1 — 双 USB 端口：OTG0 支持 DRD/BC1.2，USB1 仅基本功能

必要 全局控制寄存器

  • GOTGCTL / GOTGINT — OTG 会话与 HNP/SRP 控制
  • GAHBCFG / GUSBCFG — AHB 与 USB 全局配置
  • GINTSTS / GINTMSK — 全局中断状态与屏蔽
  • GRSTCTL — 软复位与 FIFO 刷新

增强特性 Host/Device 路径

  • Host: HCFG, HPRT, HCCHARn, HCTSIZn（通道调度）
  • Device: DCFG, DCTL, DIEPCTLn, DOEPCTLn（端点管理）
  • PHY: Reg00~Reg23 模拟参数调校
  • Port 0 BC1.2：BCCR 寄存器 + 35.7 检测流程

35.1–35.4 概述与特性

35.1 Overview

本章描述处理器中实现的 USB On-The-Go (OTG)。USB 支持主机与多种便携外设之间的串行数据交换；OTG 使设备可互连。需熟悉 USB 1.1 规范与 OTG supplement。

35.2 Features

  • 集成两个端口：Port 0 支持 OTG，Port 1 仅基本功能
  • 符合 USB 2.0 高速（480 Mbps）与 OTG supplement
  • 可作为高速/全速外设功能控制器，或点对点/多点 Host
  • 支持 SRP 与 HNP
  • UTMI+ Level 3 收发器接口
  • 软连接/断开、16 端点、控制/中断/ISO/Bulk 传输
  • Port 0 支持 BC1.2 端口检测（SDP/CDP/DCP）

35.3 Functional Description

OTG 为双角色设备（DRD）控制器，完全兼容 OTG Supplement Rev 1.3a 与 Rev 2.0，亦可配置为纯 Host 或纯 Device。

35.4 Data Format

USB-Phy 接口支持 UTMI 8/16 位。

35.5 寄存器约定

Register Address = Base + Address offset。属性：R、W、RW、RCW、RSW、RWC、RWS、RWSC、RWSSC、RWCSC、RWSSSC、RWSSCSS、RC、RS。复位值 1/0/?。

35.7 Battery Charging 1.2 端口检测

仅 USB0 端口支持 BC1.2 检测。流程：

  1. 设置 OTG 为 device 模式，DCTL.SftDiscon = 1 软断开
  2. Data pin contact detect：BCCR.IdpSrcEn=1, BCCR.RdmPdwnEn=1，检查 BCCR.DpAttached
  3. SDP primary detection：BCCR.IdmSinkEn=1, BCCR.VdpSrcEn=1，检查 BCCR.CpDetected
  4. DCP/CDP secondary detection：BCCR.IdpSinkEn=1, BCCR.VdmSrcEn=1，检查 BCCR.DcpDetected
  5. 清除 BCCR[31:26]，退出 BC1.2 检测
