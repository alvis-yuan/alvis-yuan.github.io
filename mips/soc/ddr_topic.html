第一性原理：为何需要 DDR 控制器

CPU 与视频、DMA、GPU 等主控需要共享大容量、高带宽的外部 DRAM。DDR 颗粒遵循 JEDEC 协议（时序、刷新、Mode Register），无法被 CPU 直接以简单 load/store 驱动。DDRC 的存在，就是把 SoC 侧统一的总线事务翻译成符合 DDR2/DDR3(L) 规范的命令流，并在多主控并发时维持数据完整性。

手册 5.1 定义 DDRC 的核心能力：可配置 QoS、自动/自刷新、掉电模式、可编程时序与行列地址、逻辑到物理地址重映射——这些正是「让多主控安全、高效共享 DRAM」所必需的硬件支撑。

必要 上电必须完成

  • PHY 复位与 PLL 锁定（5.7）
  • DFI 初始化握手（5.4.2）
  • 颗粒配置：DCFG + DTIMING + DLMR 发 JEDEC 命令
  • DDR3 需 Write-Leveling 与读校准（5.5/5.6）
  • 使能 DREFCNT 自动刷新后方可正常读写

增强 运行期特性

  • 8 端口 QoS 调度（CCHC0–7、CSCHC）
  • 自刷新/掉电/深度掉电（DCTRL、DDLP）
  • 地址重映射 DREMAP、带宽监视 DBWM
  • DWU 内存保护区与非法访问中断（5.8）

SoC 架构中的位置

DDRC 在芯片内分为三层：控制器核（DDRC_AHB @ 0x134F0000 + DDRC_APB @ 0x13012000）、PHY（PHY_APB @ 0x13011000）、以及片外 DDR2/DDR3 颗粒。控制器通过 DFI 接口连接 PHY，PHY 驱动 DQ/DQS/CK 等物理信号。

图 1 — 多主控经 DDRC 8 端口汇聚，经 PHY/DFI 访问外部 DRAM（依据 5.1、5.8.2）

CPU 如何访问 DDR？

手册第 6 章（EMC）说明：虚拟地址与物理地址均为 32 位，MMU 将 CPU 虚拟地址翻译为物理地址，物理地址再映射到 SDRAM、静态存储或内部 I/O 区域。CPU 执行 load/store 时并不直接写 DDRC 寄存器，而是向 SDRAM 物理地址区间发起总线读写的内存事务；DDRC 在 AXI/AHB 侧作为从设备/内存控制器响应这些事务。

两类访问要区分：
① 内存访问 — 访问 0x00000000 等 SDRAM 地址，走 DDRC 数据通路（Port 6 = CPU）。
② 寄存器配置 — 访问 0x134F0000 / 0x13012000 / 0x13011000，由 Bootloader/驱动在初始化阶段完成，正常运行期 DCFG 不应 on-the-fly 修改（5.2.2）。

各 memory bank 基址与容量可软件重配（EMC 6.4）。DDRC 侧通过 DMMAP0/DMMAP1 配置 CS0/CS1 的地址匹配：当 (BUS_ADDR[31:24] & MASK) == BASE 时选中对应片选（5.2.14）。DCFG 配置颗粒类型（TYPE：100=DDR2，110=DDR3）、行列位宽（ROW/COL）、CS0EN/CS1EN 等（5.2.2）。

可选 DREMAP0–5 对 address[31:8] 做逻辑→物理位重映射（低 2KB 不变），用于优化 PCB 布线或 Bank 交错（5.2.47）。

初始化完成后如何工作

DDRC 进入正常运行后，对软件透明的核心机制如下（均来自 5.1 Features 与 5.2 寄存器语义）：

图 2 — 读/写事务经端口仲裁与调度器转为 DRAM 命令；DREFCNT 维持刷新

1. 事务调度与 QoS

  • 最多 8 个端口（CCHC0–CCHC7），每端口可设固定优先级、QoS 压缩、读写超时与带宽限制（5.2.49）。
  • CSCHC 调度器在读写 pending 队列间仲裁，支持 bypass 与 entry 超时（5.2.50–5.2.53）。
  • CPAC 配置端口读写动态/固定仲裁权重（5.2.48）。

2. 自动刷新

DREFCNT 内 8 位计数器与 CON 比较，匹配时向所有 CS 发 Refresh；REF_EN=1 使能。CLK_DIV 选择 dclk 分频驱动计数（5.2.8）。初始化完成后必须使能，否则 DRAM 数据会丢失。

3. 低功耗

  • DCTRL.SR — 自刷新；DASR_EN/DASR_CNT — 总线空闲超时自动进自刷新。
  • DCTRL PDTIMER/ACTPD — Power-Down；DPD — Deep Power Down（退出需完整重新初始化，5.2.3）。
  • DDLP — DFI 低功耗握手时序（5.2.5）。
  • 状态可读 DSTS：AREF/SREF/PD/DPD 等（5.2.1）。

4. 一致性提示

DSTS.AHB_COH_WR：AHB 写后紧跟 AXI 读发生 coherence 时置位（读自清）。CCGUC1.AHB_COH_PROT 可启用 AHB 写后 AXI 读保护（5.2.1、5.2.60）。

5.4.2 RESET 与 DFI 握手

涉及信号：presetn、system_rstn、dfi_init_start、dfi_init_complete。时序约束（手册 Note）：

  • a — 至少 2 个 pclk 周期
  • b — 等待 dfi_clk1x 稳定，至少 2 个 dfi_clk1x 周期
  • c — dfi_init_start 高电平至少 2 个 dfi_clk1x 周期
  • d — 不超过 1024 个 dfi_clk1x 周期

软件侧：DWCFG.DFI_INIT_START 控制 dfi_init_start；轮询 DWSTATUS.DFI_INIT_COMPLETE 与 PHY_PLLLOCK（5.2.45–5.2.46）。PHY_RST 释放 PHY 数字/模拟复位（5.3.1）。DCTRL.PHY_RST / DFI_RST 控制控制器与 DFI 复位（5.2.3）。

各阶段寄存器要点

支持模糊匹配；按 Esc 清除

阶段 ① — RESET flow

释放 PHY/DFI/控制器复位，启动 DFI 初始化握手。Poll DFI_INIT_COMPLETE、PHY_PLLLOCK。

关键寄存器：PHY_RST、DCTRL(PHY_RST/DFI_RST)、DWCFG(DFI_INIT_START)、DWSTATUS(DFI_INIT_COMPLETE/PHY_PLLLOCK)

阶段 ② — PHY PLL 初始化（5.7）

PLL 接收 dfi_clk1x，输出 pll_1xclk（如 1866 Mbps 时 933 MHz）供 CMD、2xclk 供数据。

配置流程（5.7.1）：

  1. Power on PLL
  2. Assert PD=1
  3. Retain divider control bits
  4. Release PD（PD=0，等待 ≥1μs）

频率关系（5.7.2）：VCO_FREQ = REFCLK × POSTDIV × 2；PFD_FREQ = REFCLK / REFDIV；PLL_2XCLK = VCO_FREQ / POSTDIV；PLL_1XCLK = PLL_2XCLK / 2。约束：2000M ≤ VCO_FREQ ≤ 3800M，100M ≤ PFD_FREQ ≤ 933M。

寄存器：PLL_FBDIV、PLL_FBDIV_H、PLL_PDIV、PLL_CTRL(PD/POST_DIV)、PLL_LOCK

阶段 ③ — PHY 与控制器参数配置

  • MEM_CFG — MEMSEL：000=DDR2，010=DDR3 PHY 模式（5.3.2）
  • CL / AL / CWL — 与 SDRAM Mode Register 一致（5.3.7–5.3.9）
  • DCFG — TYPE/ROW/COL/CS0EN/CS1EN/ODTEN（5.2.2，运行时勿改）
  • DTIMING1–5 — tRP、tRCD、tRAS、tRFC 等 JEDEC 时序（5.2.9–5.2.13）
  • DMMAP0/1 — CS 地址窗口（5.2.14）
  • DLMR — 通过 CMD 字段发 Precharge / Auto-Refresh / Load Mode / ZQ 命令，START 上升沿触发（5.2.4）

阶段 ④ — DFI initialization

在 RESET 时序约束下完成 DFI 初始化；确认 DWSTATUS.DFI_INIT_COMPLETE=1 后进入训练阶段。

阶段 ⑤ — Write-Leveling（仅 DDR3，5.5）

  1. 配置 WL_MODE1、WL_MODE2 为将写入 SDRAM Mode Register 的值
  2. 配置 TRAIN_CTRL[7:6] 选择目标 SDRAM（CS）
  3. 配置 TRAIN_CTRL[2] 启动 write leveling
  4. 读 WL_DONE；读值为 0x3 表示 PHY 完成校验

使能后硬件自动发 Load Mode：进入时 MR1[7]=1，退出时 MR1[7]=0。Bypass：TRAIN_CTRL[3]=1（5.5.2）。

阶段 ⑥ — PHY read calibration / Soft Training（5.6）

Per-bit de-skew 补偿 PCB 走线 skew，每位 64 级可调（PBDS_* 寄存器，5.3.37）。

  • RX — 直接写寄存器调整 RX Per-bit De-skew（5.6.1.1）
  • TX — 先完成 write-leveling；置 TRAIN_CTRL[3]=1 选 bypass，以 WL 结果作 DQS TX skew，再调 DQ TX skew（5.6.1.2）

轮询 CALIB_DONE、CALIB_ERR（5.3.11、5.3.21）。

收尾 — 使能正常运行

  • 配置并使能 DREFCNT.REF_EN 开启硬件自动刷新
  • 可选配置 CCHC/CSCHC QoS、DASR 自动自刷新
  • 关闭 DDBGEN 调试逻辑（5.2.22：正常工作前建议关闭）

CPU 内存保护与 DWU（5.8）

DWU 监视并保护内存区间：非法访问向 CPU 发中断并记录地址/设备号。配置流程（5.8.1）：

  1. 写 DDWU_EN 选择监视 AHB 或 AXI 设备
  2. 写 DDWU_SADDR、DDWU_EADDR、DDWU_BADDR 配置保护地址
  3. 写 DDWU_CTRL0~3 配置被授权设备
  4. 使能 DDR 中断
  5. 若 DIS[2] 与 DDWU_IRQ_STS[0] 置位，写 1 清 DDWU_IRQ_CLR[0]
  6. 读 DDWU_ERR_INFO、DDWU_ERR_ADDR 获取非法访问信息
