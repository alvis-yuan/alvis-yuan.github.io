第一性原理

I2C 是两线串行总线（SDA/SCL），多设备共享，靠唯一地址与主从角色完成读写。X2600 集成 4 路独立 I2C 控制器（I2C0–I2C3），软件可配置为主或从（不可同时），通过 FIFO + 中断/DMA 避免 CPU bit-bang。

30.1 概述与特性

设备时钟与 pclk（APB 时钟）相同；可编程 SCL 发生器；7/10 位寻址；General Call / START byte；电平 Tx/Rx FIFO；中断；APB 接口。

  • Standard 100 kb/s、Fast 400 kb/s、High 3.4 Mb/s
  • 同一 I2C 总线设备数受 400pF 总电容限制（需板级上拉）
  • 主从不可同时：I2C_CON 的 SLVDIS 与 MD 不可分别为 0 和 1（30.3 NOTE）

30.2.2 各寄存器详细说明

支持模糊匹配；按 Esc 清除

30.3 操作流程摘要

30.3.2.1 主模式配置

  1. 写 I2C_ENABLE=0，等待 I2C_ENBST.I2CEN=0
  2. 写 I2C_CON（SPEED 等）；MATP 只读，寻址由 I2C_TAR 控制
  3. 按 SPEED 配置 SHCNT/SLCNT 或 FHCNT/FLCNT 或 HHCNT/HLCNT（见 SCL 公式）
  4. 写 I2C_TAR 目标地址
  5. 写 I2C_ENABLE=1，等待 I2C_ENBST.I2CEN=1
  6. 写 I2C_DC 发起传输（可先 DMA 填 FIFO）

30.3.3.1 从模式配置

  1. I2C_ENABLE=0
  2. 写 I2C_SAR 从地址
  3. I2C_CON：SATP 选 7/10 位；SLVDIS=0，MD=0
  4. I2C_ENABLE=1

30.3.4 禁用 I2C

写 ENABLE=0 后轮询 ENBST.I2CEN；tI2C_poll ≈ 10×最快 SCL 周期（400kHz 时 25µs）。

30.3.5 / 30.3.6

TX FIFO 刷新条件：ENABLE=0、TX_ABRT、从机读命令 flush 旧 TX 数据。STOP 仅当 I2C_DC[9]=1；RESTART 需 CON[5]=1 且 DC[10]=1。

完整流程见 i2c_topic.html。
