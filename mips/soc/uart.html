第一性原理：为何需要 UART

CPU 与外设、调试终端、Modem、IrDA 设备之间需要异步串行通信：发送方与接收方无共用时钟，靠起始位、数据位、可选奇偶校验与停止位界定一帧。UART（Universal Asynchronous Receiver/Transmitter）在 APB 总线与物理引脚之间完成并/串转换、波特率产生、FIFO 缓冲及中断/DMA 请求，使 CPU 不必 bit-bang GPIO。

图 1 — UART 从 APB 取数/交数，经 FIFO 与移位器驱动串行引脚（Figure 33-1）

必要 最小收发三元组

  • URBR / UTHR — 接收读缓冲 / 发送写保持（DLAB=0 时有效）。
  • ULCR + UDLLR/UDLHR — 帧格式与波特率除数（DLAB=1 访问除数）。
  • ULSR — 线路状态：DRY、错误、TDRQ、TEMP 等，驱动轮询或中断。

增强特性 手册列出的扩展能力

  • 64×8 TX FIFO、64×11 RX FIFO；UFCR 控制 FIFO/DMA/模块使能。
  • UIER/UIIR 五类独立中断；Modem 流控（UMCR/UMSR）。
  • IrDA SIR（ISR）；非标准时钟 M/UACR 微调位宽。
  • URCR/UTCR/URFCR/URTDCR 扩展计数与分离 RX/TX DMA。

33.1 概述（Overview）

本章描述通用异步收发器（UART）串口。芯片内 UART 使用相同编程模型；各串口可在中断模式或 DMA 模式下工作。UART 兼容 16550 工业标准，亦可用作符合 IrDA 串行红外规范 1.1 的慢速红外异步接口。

33.2 特性（Features）

  • 全双工操作
  • 5/6/7/8 位字符，可选无/奇/偶校验，1、1½ 或 2 停止位
  • 64×8 位发送 FIFO 与 64×11 位接收 FIFO
  • 发送、接收与线路状态中断独立
  • 内部诊断：环回、Break、奇偶、溢出与帧错误
  • FIFO 模式下发送/接收独立 DMA 请求
  • 软件或硬件 Modem 流控
  • 符合 IrDA 规范的慢速红外异步接口

33.3 框图（Block Diagram）

如图 Figure 33-1，UART 数据控制器包含接收与发送两部分：从 PIN 接收数据经 rFIFO 处理后送上 APB；发送方向从 APB 取数经 tFIFO、encoder 送至 PIN。

33.4 功能描述（Functional Description）

33.4.1 全双工

UART 双向通信，可实现全双工收发。

33.4.2 各位含义

  1. 起始位（start）：先发逻辑 0，表示字符传输/接收开始。
  2. 数据位（data bit）：起始位之后，5/6/7/8 位构成字符，从最低位开始传输。
  3. 奇偶位（parity bit）：可选，用于校验传输准确性。
  4. 停止位（stop bit）：标记字符结束；可为 1、1.5 或 2 个高电平，并提供时钟同步机会。
  5. 空闲位（idle bit）：逻辑 1，表示线路上无数据传输。

33.4.3 RFIFO 与 TFIFO

TFIFO 为 64×8bit 发送 FIFO；RFIFO 为 64×11bit 接收 FIFO，用于缓存数据。数据先进入 UART 寄存器（临时存储块），再经 FIFO 传至串行设备；无 FIFO 时信息可能来不及传给 Modem。

33.4.4 发送、接收与线路状态独立

发送、接收（数据就绪或超时）、线路状态中断独立控制。

33.4.5 慢速红外异步接口

UART 含 IrDA SIR 编解码模块，在异步 UART 数据流与半双工 SIR 接口间转换；UART 信号脚接红外收发器实现 IrDA SIR 物理层。

33.8 操作流程（Operation Flow）

33.8.1 UART 配置

FIFO 模式下先设 UFCR.FME=1、复位 RX/TX FIFO，再：

  1. 清 UFCR.UME=0。
  2. 写 UDLLR/UDLHR 产生波特率时钟。
  3. 写 ULCR 定义数据格式。
  4. FIFO 模式：设 FME 及其他 UFCR 控制并复位 FIFO；非 FIFO 跳过。
  5. 中断模式设 UIER；DMA 模式设 UFCR.DME（仅 FIFO），再设 UFCR.UME。

33.8.2 数据发送

  1. 读 ULSR.TDRQ 或等待发送数据请求中断。
  2. TDRQ=1 时写 UTHR 启动发送。
  3. 重复直至待发数据写完。
  4. 等待 ULSR.TEMP=1 表示全部发完。
  5. 需 Break 时设 ULCR.SBK，等待至少 1 位间隔后清 SBK。
  6. 清 UFCR.UME 结束发送。

33.8.3 数据接收

  1. 读 ULSR.DRY 或等待接收数据就绪中断。
  2. 若 ULSR.FIFOE=1 进入错误处理，否则读 URBR。
  3. FIFO 模式按阈值读 URBR；使能中断时可能产生超时中断，读 URBR 至 DRY=0。
  4. 全部收完且 DRY=0 后清 UFCR.UME。

33.8.4 接收错误处理

FIFOE=1 时依次检查 OVER、BI、FMER、PARER。

33.8.5 Modem 传输

UMCR.MDCE=1 时，CTS_ 低电平暂停发送；UMCR.RTS 控制 RTS_ 输出表示可否接收。

33.8.6 DMA 传输

UFCR.DME=1（仅 FIFO）：DMA 请求替代数据收发中断；除超时与错误外应关闭收发中断。出错时先读 ULSR/UIIR，关 DMA 通道并读空错误数据后重启 DMA。

33.8.7 慢速 IrDA

设 ISR.XMITIR/RCVEIR（不可同时为 1，SIR 非全双工）；速率最高 115.2Kbps；SIR 下固定 8 数据位、无校验、1 停止位，忽略 Modem；光学节点停止发送后至少 10ms 才能恢复接收。

33.8.8 任意频率时钟

未设 UMR/UACR 时按 Baud Rate = (UART device clock) / (16 × Divisor)。设 M/UACR 可在每 bit 插入额外 CLKBR 周期以逼近非整数 Ma；Ma=CLKBR/baud rate，M=floor(Ma)，UACR 各位选 M 或 M+1 周期。
