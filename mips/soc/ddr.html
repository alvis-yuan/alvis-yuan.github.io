第一性原理：为何需要 DDR 控制器

CPU 与各类主控（DMA、GPU、视频编解码等）需要共享大容量、高带宽的外部 DRAM。DDR 控制器（DDRC）是 SoC 与 DDR2/DDR3(L) 颗粒之间的唯一标准接口，承担手册 5.1 Overview 所述职责：可配置 QoS、时序与地址映射、刷新/自刷新/掉电、以及 PHY 训练与校准。

图 1 — 主控经 DDRC（AHB/APB）访问 PHY 与外部 DDR 颗粒

必要 初始化最小寄存器集

  • DCFG / DMMAP — 颗粒类型、行列/片选映射
  • DTIMING1–5 — JEDEC 时序参数
  • DLMR + DCTRL — 模式寄存器加载与控制
  • DREFCNT — 自动刷新
  • PHY_RST / PLL / TRAIN_CTRL — PHY 复位、PLL、训练

增强特性 QoS / 监控 / 保护

  • CCHC / CSCHC / CPAC — 端口 QoS 与调度
  • DBWM / DPORT_BW — 带宽监控
  • DDWU — 内存区域访问监视与保护
  • DREMAP — 逻辑到物理地址重映射

5.1 概述（Overview）

DDRC（DDR Controller）为通用 IP，提供 DDR2 与 DDR3 颗粒接口，面向 SoC 可配置、可扩展。

特性（Features）

  • Highly configurable Quality of Service system（高度可配置的 QoS 系统）
  • Support clock-stop mode（支持 clock-stop 模式）
  • Support auto-refresh and self-refresh（支持自动刷新与自刷新）
  • Support power-down mode and deep-power-down mode（支持掉电与深度掉电）
  • Programmable DDR timing parameters（可编程 DDR 时序参数）
  • Programmable DDR row and column address width（可编程行列地址宽度）
  • Programmable remapping from Logic address to Physical address（逻辑到物理地址可编程重映射）

Supported DDR SDRAM Types

  • DDR2
  • DDR3(L)

下列为 5.3.1–5.3.37 各 PHY 寄存器；PBDS_* 系列（5.3.37）共 78 个寄存器结构相同：Bits[7:6] Reserved，Bits[5:0] DE_SKEW。

5.4 上电与初始化流程（Power-up and Initialize Sequence）

5.4.1 主初始化流程

  1. RESET flow（复位流程）
  2. PHY PLL initialization（PHY PLL 初始化）
  3. PHY register configuration（PHY 寄存器配置）
  4. DFI initialization（DFI 初始化）
  5. PHY Write-leveling training（仅 DDR3）
  6. PHY read calibration（PHY 读校准）

5.4.2 RESET 流程要点

presetn / system_rstn / dfi_init_start / dfi_init_complete 时序需满足：a) 至少 2 个 pclk 周期；b) 等待 dfi_clk1x 稳定至少 2 周期；c) dfi_init_start 高电平至少 2 个 dfi_clk1x 周期；d) 不超过 1024 个 dfi_clk1x 周期。

5.5 WRITE-LEVELING

5.5.1 Write-leveling 步骤

  1. 配置 WL_MODE1/WL_MODE2 为将写入 SDRAM Mode Register 的值。
  2. 配置 TRAIN_CTRL[7:6] 选择目标 SDRAM（CS）。
  3. 配置 TRAIN_CTRL[2] 启动 write leveling。
  4. 读 WL_DONE；读值为 0x3 表示 PHY 完成校验。

启动后硬件自动发送 Load Mode 命令，进入时 MR1[7]=1，退出时 MR1[7]=0。

5.5.2 Bypass 功能

置 TRAIN_CTRL[3]=1 启用 bypass，可通过寄存器直接控制 write-leveling 结果（见相关寄存器说明）。

5.6 SOFT TRAINING（Per-bit De-skew）

Per-bit de-skew 用于补偿 PCB 走线不匹配。DDR PHY 对所有 PHY 信号 individually 可调，每位 64 级步进；通过 inverter chain 延迟实现，TX/RX 共用延迟线。

5.6.1.1 RX 软件扫描

INNO DDR PHY 通过寄存器直接控制 RX Per-bit De-skew，用户可直接写寄存器调整。

5.6.1.2 TX 软件扫描

TX Per-bit De-skew 前须完成 write-leveling 对齐 DQS 与 Command；置 TRAIN_CTRL[3]=1 进入 bypass 模式，PHY 以 write-leveling 结果作为 DQS TX skew，再调整 DQ TX skew 寄存器。

5.7 DDR PHY PLL

PLL 作为高速时钟发生器，仅 DDR PHY 使用。接收 dfi_clk1x，输出 pll_1xclk（如 1866 Mbps 时 933 MHz）供 CMD，2xclk（如 1866 MHz）供数据。

5.7.1 PLL 配置流程

  1. Power on PLL
  2. Assert PD=1
  3. Retain divider control bits
  4. Release PD（PD=0，等待不少于 1μs）

5.7.2 PLL 频率关系

VCO_FREQ = REFCLK × POSTDIV × 2；PFD_FREQ = REFCLK / REFDIV；PLL_2XCLK = VCO_FREQ / POSTDIV；PLL_1XCLK = PLL_2XCLK / 2。约束：2000M ≤ VCO_FREQ ≤ 3800M，100M ≤ PFD_FREQ ≤ 933M。

5.8 DWU（Data Watch Unit）

DWU 监视并保护内存空间：可设定保护区间、授权最多四个设备访问；非法访问产生中断并记录 DDWU_ERR_INFO / DDWU_ERR_ADDR。

5.8.1 DWU 配置流程

  1. 写 DDWU_EN 选择监视 AHB 或 AXI 设备
  2. 写 DDWU_SADDR、DDWU_EADDR、DDWU_BADDR 配置保护地址
  3. 写 DDWU_CTRL0~3 配置被监视设备
  4. 使能 DDR 中断
  5. 若 DIS[2] 与 DDWU_IRQ_STS[0] 置位，写 1 清 DDWU_IRQ_CLR[0]
  6. 读 DDWU_ERR_INFO、DDWU_ERR_ADDR 获取非法访问信息
