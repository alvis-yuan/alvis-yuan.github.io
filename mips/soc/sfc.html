第一性原理：为何需要 SFC

SPI Flash 是系统启动与固件存储的常见介质，但 bit 级 SPI 时序、多线模式（Standard/Dual/Quad）、命令-地址-哑周期-数据各阶段组合若由 CPU bit-bang，既慢又难满足 tSETUP/tHOLD/tSH 等 AC 时序。SFC（SPI Flash Controller） 作为 SPI 主设备，在 AHB 从端口（寄存器/slave 模式）与 AHB 主端口（DMA）之间插入 FIFO、流控与可编程 6 phase，使软件通过寄存器或 CDT/DMA 描述符链完成 Flash 读写。

图 1 — SFC 框图：AHB 从/主接口、FIFO、流控与 SPI 控制（Figure 7-1）

必要 一次传输最小配置

  • SFC_DEV_CONF — CPOL/CPHA、CE/WP/HOLD 默认电平、tSETUP/tHOLD/tSH、采样延迟 SMP_DELAY。
  • SFC_TRAN_CFG0_n / TRAN_CFG1_n — 各 phase 命令、地址宽、哑周期、传输模式（Standard/Dual/Quad…）。
  • SFC_GLB0 + SFC_TRIG.START — phase 数、方向、阈值、slave/DMA 模式与启动。

增强特性 手册扩展

  • 64×32 FIFO；CDT 命令描述符表（Base+0x0800~0x0C00）。
  • DMA 描述符链（SFC_DES_ADDR）；双 Flash 片选（GLB1.CHIP_SEL/CHIP1_SEL）。
  • 设备状态轮询（DEV_STA_EXP/MSK 与 POLL_EN）；FSM/CGE 调试与时钟门控。

7.1 概述 · 7.2 特性

SFC 为控制 SPI Flash 的 SPI 主模块；数据可在 CPU 控制（slave）模式或 DMA 模式下传输。

  • SPI 协议：Standard、Dual、Quad SPI
  • 仅发送或仅接收操作
  • 字节内 MSB 先；数据字节间 LSB 先；命令/地址字节间 MSB 先
  • 64 entries × 32bit FIFO；单 device select（可扩展 chip1）
  • 可配置接收采样点 SMP_DELAY
  • 可配置 tSLCH(tSETUP)、tCHSH(tHOLD)、tSHSL(tSH)
  • 可配置 Flash 地址宽度
  • 7 种传输格式：Standard、Dual-Out/In、Quad-Out/In、Dual-I/O、Quad-I/O、Full Dual-I/O、Full Quad-I/O
  • Slave 与 DMA 两种数据传输模式；软件可配置 6 个 phase；CDT；DMA 描述符链；SDR

7.3 框图 · 7.4 总线功能

AHB 接口含从接口（寄存器配置、slave 模式传数）与主接口（DMA）。FIFO 双向缓冲 AHB 与设备间数据。Flow Control 控制 6 phase；Transfer mode Control 处理 slave/DMA。

7.4.2 AHB Slave

  • 参数配置；slave 模式传数；数据与地址须 4 字节对齐；仅支持 SINGLE 传输。

7.4.3 AHB Master

  • DMA 模式传数；支持 SINGLE、INCR、INCR4/8/16/32。

7.4.4 可配置时序

7.4.4.1 传播延迟与采样点

传播延迟不影响发送；低速接收无影响。高速时 incoming 数据可能无法在正确时钟沿采样，须 adjustable 采样点（Figure 7-2 无延迟，Figure 7-3 延迟一 SFC_CLK，参考 SMP_DELAY）。smp_clk 为内部采样时钟。

7.4.4.2 AC 时序（Figure 7-4）

tSETUP(tSLCH)、tHOLD(tCHSH)、tSH(tSHSL) 可配置：CE 断言到首 CLK 沿为 tSETUP；末 CLK 沿到 CE 撤销为 tHOLD；两次 CE 断言间隔为 tSH。

7.6 数据格式

7.6.1 传输格式（Figure 7-5 ~ 7-7）

典型阶段：command → address → dummy → data。手册定义七种 SPI 模式波形：

  • Standard SPI — 写/读均在单线 dt 上传命令、地址、哑周期与数据（Figure 7-5）。
  • Dual Input/Dual Output SPI — 命令/地址单线，数据双线 dt/dr（Figure 7-6）。
  • Dual I/O / Full Dual I/O — 地址或命令亦在双线传输。
  • Quad Input/Output / Quad I/O / Full Quad I/O — 利用 dt/dr/wp/hd 四线（Figure 7-7）。

7.6.2 字节序（Endian）

地址 MSB 先（见 Figure 7-5）。Flash 状态寄存器字节序由 SFC_DEV_CONF.STA_ENDIAN 配置 LSB/MSB；数据字节序与单位由 TRAN_CFG1.DATA_ENDIAN 与 TRAN_CFG1.WORD_UNIT 配置。

7.8 软件指南

7.8.1 CDT 描述

每条命令 4 word，可链接下一条。硬件每次执行一条 command chain；每条 chain 至多一次 data 传输。字段含 LINK、ADDR_KIND（0 Column/1 Row/2 STA_ADDR0/3 STA_ADDR1）等，与其余寄存器定义一致。

7.8.2 DMA 描述符

每个描述符 4 word，可成链。Word0：LINK、ENDIAN、WORD_UNIT、TRAN_MODE、ADDR_KIND 等；Word1：CLK_MODE、ADDR_WIDTH、POLL_EN、CMD_EN、PHASE_FORMAT、DMY_BITS、DATA_EN、TRAN_CMD；Word2 DEV_STA_EXP；Word3 DEV_STA_MSK；随后 NEXT_DES_ADDR、MEM_ADDR、TRAN_LEN。

7.8.3 Phase 描述

  • Phase 由 CMD、ADDR、DUMMY、DATA 四段组成；每 phase 最多含其中若干段，每段每 phase 至多一次。
  • 所有 phase 中仅一个 phase 可有 DATA（data phase，其余为 command phase）。
  • Format0：CMD→ADDR→DUMMY→DATA；Format1：CMD→ADDR/DUMMY→DATA 交错（见手册 Figure）。
  • 一次 flow 仅支持一种传输方向。

7.8.4 多 phase 流程（Figure 7-8）

  1. 配置设备硬件相关参数（SFC_DEV_CONF）。
  2. 配置各 phase 传输参数与 DMA 参数（GLB、DEV_STA_EXP/MSK、TRAN_CONF0/1、TRAN_LEN、DEV_ADDR、MEM_ADDR 等）。
  3. 写 SFC_TRIG 启动。
  4. 等待 END 中断并清除 END 位；其他 flow 重复步骤 2。

7.8.5 单 phase 流程（Figure 7-9）

可仅用一个 phase 完成 PL/WE/PE/CHK 等 NAND 操作序列（手册以 NAND 为例）。

7.8.6 DMA 模式注意

  • 配置 SFC_GLB（BURST_LEN、OP_MODE 等）与 SFC_MEM_ADDR。

7.8.7 Slave 模式注意

  • 每个 data transfer phase 启动前须 FLUSH。
  • SFC 按 THRESHOLD 产生请求；CPU 轮询 SFC_SR 或中断时，每次 AHB 传输数据量须匹配 THRESHOLD。
