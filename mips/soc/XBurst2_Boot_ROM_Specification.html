第一性原理：为何需要 Boot ROM

X2600 内含 24KB 内部 Boot ROM。CPU 复位后从 Boot ROM 启动。Boot ROM 在无外部引导程序或 Flash 尚未初始化 SDRAM 之前，承担最小启动环境的建立与外设引导职责。

图 1 — 复位后 Boot ROM 根据 boot_sel 从外设加载 SPL 至 Cache 并跳转执行

必要 启动最小集合

  • boot_sel[2:0] — 硬件引脚决定启动方式（MSC、SFC、USB）。
  • efuse 读取 — 决定系统时钟提升、MSC 4bit 传输、MSC 等待状态、GPIO HI-Z、USB 信息等。
  • SPL 加载与跳转 — 将 spl size 字节载入 Cache（0x80000000–0x80020000，Boot ROM 占 128KB Cache），跳转 0x80001000 执行。
  • SPL 结构 — SC Boot KEY（1.5KB）+ SPL signature（512B）+ SPL.text（≤110KB，512 字节对齐）。

增强特性 可选与扩展

  • SPL Parameters（INGE） — 可变 SPL 长度、CPU 频率、NAND Timing、CPM 描述符（仅 SFC/SD/MMC，可选）。
  • 安全启动 — SC_BOOT_EN、EFUSE_SECBOOT_EN；校验 security key 与 SPL code。
  • USB 两阶段下载 — VR_PROGRAM_START1/2、VR_FLUSH_CACHES 支持 SDRAM 初始化后再加载第二阶段程序。
  • 启动失败回退 — SFC/USB 失败时按 Figure 42-1 尝试其他启动模式；连续失败 3 次后看门狗与 While(1)。

42.2 启动流程（Boot Procedure）

复位后，内部 Boot ROM 上的启动程序按以下步骤执行：

  1. 关闭所有中断，准备程序运行环境。
  2. 读取并保存 efuse 值，用于决定是否提升系统时钟频率、设置 MSC 4bit 传输数据、MSC 等待状态、GPIO HI-Z、隐藏 USB 设备信息、USB eye diagram 等。
  3. 初始化时钟并读取 boot_sel[2:0] 确定启动方式。

按启动方式分支

  • MSC0 MMC/SD 启动：从 MMC/SD 卡加载 spl size 字节数据到 Cache 并跳转。初始时钟 EXTCLK/120；读数据时 EXTCLK/4。若 efuse msc_bus_width_4 为 1，初始化 MSC_D0–D3 为 4bit 传输；否则仅使用 MSC_D0。
  • USB 启动：通过 USB 线缆从主机 PC 接收代码块并存入 Cache，然后跳转至 Cache 中该区域。
  • SFC SPI NOR/NAND 启动：初始化 SFC_CLK、SFC_CE、SFC_DR、SFC_DT、SFC_WP、SFC_HOL 功能引脚；从 NOR/NAND 加载 spl size 字节代码到 Cache 并跳转。

Cache 占用：X2600 的 Boot ROM 占用 128KB Cache，地址范围 0x80000000 至 0x80020000。

Figure 42-1 展示了完整启动序列图，包括：各 boot_sel 分支、启动失败回退、SC_BOOT_EN 安全启动路径、连续失败 ≥3 次后看门狗开启与 While(1) 等。

42.3 SPL 结构（SPL Structure）

为满足特殊需求，SPL（Second Program Loader）数据需按不同启动模式包含不同信息。基本结构如下：

  • SC_Boot KEY（1.5KB） — 用于安全启动（security boot）。
  • SPL signature（512B） — 存储各类启动相关信息；各启动模式有唯一 SPL signature。
  • SPL.text（不超过 110KB，512 字节对齐） — 实际代码段。

Cache 地址空间前 4KB 用于栈与数据空间。所有启动模式必须跳转至 0x80001000 执行 SPL 代码，且均包含 Figure 42-2 所示 SPL 结构。

42.4 SPL 参数（SPL Parameters）

为支持可变 SPL 长度并通过改变时钟频率提升加载速度，引入 SPL parameter 机制。仅用于 SFC/SD/MMC 启动，且为可选。

参数以特定结构存放在存储空间中，大小为 256 字节。SD 卡启动时，参数放在 17K + 256 偏移处。

42.5 MSC 启动规范（MSC Boot Specification）

MSC 启动程序从 MMC/SD 卡地址 0x4400 起加载 spl size 字节数据到 Cache。

  1. 根据 efuse 值确定数据传输宽度：1bit 初始化 MSC_D0、MSC_CLK、MSC_CMD；4bit 初始化 MSC_D0–D3、MSC_CLK、MSC_CMD。
  2. 发送 CMD55 判断 SD 或 MMC 卡并初始化卡。
  3. 从卡加载 spl size 字节到 CPU Cache 并分支执行。

初始化卡时时钟 EXTCLK/120；读数据时 EXTCLK/4。

Figure 42-4 典型流程：efuse 4bit？→ MSC reset/init → 确认 SD/MMC → 从 0x4400 加载 512 字节配置到 Cache → 按配置改 CPU 频率 → 从 0x4400 加载 SPL → 检查首字是否为 MSPL → 执行 SPL 或失败跳转其他启动。

42.6 SFC 启动规范（SFC Boot Specification）

boot_sel[2:0] 选择 SFC 启动时，CPU 进入 SFC 启动。首先从 SPI 设备读取 16 字节 flag 信息表（SPL signature 前 16 字节），检查 flag 以决定 SPI 地址长度、SPI 设备类型（NOR 或 NAND）、接收代码长度及 crc7 校验数据。详细流程见 Figure 42-5「SFC Boot Procedure」。

注意：启动步骤任何异常，SPI_boot 将禁用 SFC 控制器并按 Figure 42-1 跳转其他启动模式。

安全启动（Security Boot）

手册在多处描述安全启动相关机制（无独立章节编号，内容摘自 42.2、42.3、42.6 及 Figure 42-1、42-5）：

  • SC_BOOT_EN — Figure 42-1 中，启用时进入 Security boot 路径；校验通过后跳转 Cache 中 SPL。
  • SC_Boot KEY（1536B / 1.5KB） — SPL 结构中用于 security boot 的密钥区域（42.3）。
  • EFUSE_SECBOOT_EN — SFC 启动流程中，若启用则检查 security key 与 SPL code 有效性；未启用则仅检查 SPL code（Figure 42-5）。
  • SFC NOR/NAND 启动均从 Flash 读取 security key 与 SPL code，并进行数据有效性校验。

42.7 USB 启动规范（USB Boot Specification）

boot_sel[2:0] 选择 USB 启动时，内部 Boot ROM 从 USB 端口下载用户程序到内部 Cache 并跳转执行。

  • 支持外部主晶振频率 24MHz。
  • 支持高速（480MHz）传输模式。
