第一性原理：为何需要中断控制器

SoC 内数十个外设（UART、DMA、GPIO、定时器等）可能在任意时刻完成工作并请求 CPU attention。若 CPU 轮询每个外设，将浪费大量周期。中断机制让外设主动通知 CPU；而中断控制器（Interrupt Controller） 则是硬件与软件之间的汇聚层，承担手册 25.1 Overview 所述核心职责：

  • 控制可供处理器使用的中断源；
  • 保存中断源位置信息，使软件能确定所有中断的来源；
  • 决定是否产生 IRQ，并对中断进行屏蔽（mask）。
图 1 — 中断源经 INTC 汇聚后分别送达 CPU 核、PDMA MCU 与 RISC-V（LEP）

为实现上述目的，哪些寄存器是「必要」的？

必要 最小中断管理三元组

对每一组 32 位中断源，手册定义了三种语义，构成软件识别与响应中断的最小集合：

  • Source（ICSR / DSR） — 只读，反映外设原始 pending 状态（1 = pending）。
  • Mask（ICMR / DMR / LMR） — 决定哪些源可向处理器发出 IRQ（1 = masked，默认全屏蔽）。
  • Pending（ICPR / DPR / LPR） — 只读，Source 经 Mask 后的有效 pending（1 = active 且未 masked）。

软件在 25.3 中通过读取 ICPRx / DPRx / LPRx 确定中断源，这是实现「定位中断来源」目的所不可或缺的寄存器。

增强特性 手册列出的扩展能力

  • ICMSR / ICMCR — 写 1 置位/清零 Mask 位，避免读-改-写竞态（Write-only 辅助寄存器）。
  • 多实例 INTC — INTC0/INTC1（CPU）、EXT_INTC（PDMA MCU）、LEP_INTC（RISC-V），同一 64 源映射到不同总线基址。
  • LSIP — LEP 软件中断 pending，BIT0 写 1 产生软件中断。
  • 独立使能、优先级指示、睡眠唤醒 — Overview Features 所述（优先级由软件在 ISR 中决定，见 25.3 步骤 4）。

25.1 概述（Overview）

本章描述 XBurst2 处理器内置的中断控制器，说明其工作模式并定义相关寄存器。中断控制器控制处理器可用的中断源，并包含中断源位置信息，以便软件确定所有中断的来源；同时决定是否产生 IRQ 并对中断进行屏蔽。

特性（Features）

  • 共 64 个中断源
  • INTC0、INTC1 供 CPU 核使用
  • EXT_INTC 供 PDMA MCU 使用
  • LEP_INTC 供 RISC-V 使用
  • 每个中断源可独立使能
  • 优先级机制用于指示最高优先级中断
  • INTC 4 的未屏蔽中断可在 sleep 模式下唤醒芯片

25.3 软件考虑（Software Considerations）

中断控制器反映外设中中断源的状态。软件应执行的任务是：从 ICPRx / DPRx / LPRx 确定中断源。

在本芯片中，pending 中断在结构上有两个层级。系统中包含多个中断源的模块，需要软件通过读取其内部中断状态寄存器来决定如何服务。

在中断处理程序中，已服务的中断源需要在外设中清除。为确保清除后的源请求状态已反映到对应的 ICPRx / DPRx / LPRx 位，软件应在退出中断状态前等待足够时间。

图 4 — 手册 25.3 中断服务流程（步骤 1–6，续见下文）
  1. 中断产生。
  2. CPU 查询中断源，保存当前环境，然后进入中断公共服务例程。
  3. 读取 ICPRx / DPRx / LPRx。
  4. 找到最高优先级中断并向量跳转。（由软件决定哪个优先级最高）
  5. 通过写 ICMSRx / DMRx / LMRx 屏蔽所选中断。
  6. 使能系统中断以允许中断嵌套。（由软件决定）
  7. 执行中断处理程序，退出时通过写 ICMCRx / DMRx / LMRx 取消屏蔽。
  8. CPU 恢复保存的环境并退出中断状态。
