MQTT 全流程抓包序列图

基于 mqtt_full_flow.pcap（15 帧 · 约 2.3 ms）还原 TCP 建连 → MQTT 会话 → QoS 2 发布 → 断开 的完整交互。

Client 192.168.56.103:41586 Broker 192.168.56.107:1883 mosq_pub_10060_xiaoyua topic=test · QoS 2

1 · 抓包摘要

客户端使用 mosquitto_pub 类工具连接 Broker，建立 TCP 后发送 CONNECT，以 QoS 2 向 test 发布 Hello MQTT，完成四步握手后 DISCONNECT，双方关闭 TCP。

阶段帧号时间 (Δt)内容
TCP三次握手#1–#30 – 0.3 msSYN → SYN,ACK → ACK
MQTT建连#4–#60.3 – 0.5 msCONNECT → CONNACK (rc=0)
QoS2发布#8–#110.9 – 1.4 msPUBLISH → PUBREC → PUBREL → PUBCOMP
MQTT断开#121.9 msDISCONNECT
TCP四次挥手#13–#152.0 – 2.3 msFIN → FIN → ACK
本抓包无 SUBSCRIBE：发布端仅向 Broker 投递消息；若有订阅者在线，Broker 会在 PUBCOMP 之后另行转发（不在本连接中体现）。

从 SYN 到 FIN 的端到端时序，箭头旁标注 Wireshark 帧号与相对时间。

3 · MQTT 应用层序列图

仅 MQTT 控制报文（帧 #4–#12），省略 TCP 纯 ACK 空载荷帧。

4 · QoS 2 四步握手（恰好一次）

对应规范 §4.1：PUBLISH → PUBREC → PUBREL → PUBCOMP，Message ID 均为 1。

7 · PUBLISH 报文解析（帧 #8 · QoS 2）

/* 20 bytes MQTT PUBLISH */ 34 Fixed: Type=PUBLISH(3), DUP=0, QoS=2, RETAIN=0 12 Remaining Length = 18 00 04 74 65 73 74 Topic Name = "test" 00 01 Message ID = 1 48 65 6c 6c 6f 20 4d 51 54 54 Payload = "Hello MQTT"
