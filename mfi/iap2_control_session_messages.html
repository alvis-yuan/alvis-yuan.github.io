iAP2 Control Session Messages 概述

iAP2_Control_Session_Messages.pdf 是 Accessory Interface Specification 的 Section 101： 按功能域列出所有 Control Session Message 的名称、Message ID、方向（Device / Accessory）及参数定义。 这些消息作为 Link 层 Control Session Payload 传输；Link 帧格式见 iap2_protocol.pdf。

图 1 · Control Session Messages 承载于 Link 层 Control Session 之上

理解 · 本文档 vs 专题章节

Section 101 是消息字典（ID + 参数）；Authentication/Identification 的流程、时序、失败场景见各专题 PDF 及本站 Authentication、 Identification 等页面。 EA Session 生命周期中的 0xEA03 StatusExternalAccessoryProtocolSession 在 External Accessory Protocol 中定义，未出现在本 PDF 消息列表。

Control Session Message 结构

每条 CSM 在 Link Payload 内自包含；与 iap2_protocol.pdf Control Session 章节一致：

字段长度说明
Start of Message20x4040（大端）
Message Length2整消息字节数（含本头）
Message ID216-bit，如 0xAA00、0x1D01
Parameters变长重复：Param Length (2) + Param ID (2) + Param Data

图 2 · Control Session Message 字节布局

PDF 中方向列：Device = iOS（Client）发起；Accessory = 配件发起。部分消息双向可用，以各消息 Usage 为准。

参数类型（TLV）

各消息参数表标注类型，与 iAP2 Control Session 通用解析规则一致：

类型说明示例参数
none无数据体StartIdentification、Stop 类消息
utf8零结尾 UTF-8 字符串Name、AppBundleID、DeviceName
blob原始字节AuthenticationCertificate、Challenge、Response
bool / enum / uint8 / uint16标量IsEnabled、AppLaunchMethod、ExternalAccessoryProtocolIdentifier
group嵌套子参数组ExternalAccessoryAppCategories、TransportComponent

Identification 须在 MessagesSent / MessagesReceived 中声明

配件在 0x1D01 IdentificationInformation 的 MessagesSentByAccessory / MessagesReceivedFromDevice 中列出实际会收发的 Message ID 列表；未声明却发送的消息会导致 ATS 失败（如未声明 0xEA02 却 RequestAppLaunch）。

理解要点（总结）

  1. 字典文档：Section 101 定义「发什么 ID、带什么参数」；Link 层与会话 multiplex 见 iap2_protocol。
  2. 先声明后使用：0x1D01 中的 MessagesSent/Received 是后续 CSM 的「白名单」。
  3. 两类交互：订阅型（Start/Update/Stop）与握手型（Auth、Identification、EA Session 管理）。
  4. 方向列：Device = iOS 发起；Accessory = 配件发起；Update 类多为 Device → Accessory。
  5. 版本敏感：App Discovery (0xAD vs 0x4E)、Identification Cancel/Update ID 以当前 PDF 为准。
  6. 不完整覆盖：本 PDF 未含 0xEA03、HID、Now Playing 等；见 AIS 其它 Section 与专题 HTML。
