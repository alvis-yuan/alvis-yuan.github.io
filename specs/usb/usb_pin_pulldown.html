上下拉电阻与主从角色

深入解析 USB 设备如何通过简单的电阻网络实现 Host（主机） 与 Device（设备） 角色协商、速度识别、插入检测及电源管理的完整电气原理。

核心思路

USB 没有独立的"角色协商"引脚（USB 2.0），而是巧妙地利用 D+/D− 上的上拉（Pull-up）和下拉（Pull-down）电阻构成分压网络，通过检测线上电压判断对端角色和速度。

USB 2.0

D+/D− 上下拉电阻机制

USB 2.0 使用极简的电阻设计完成设备检测、速度识别，不需要额外的协商协议。以下按"从未连接到建立通信"完整拆解每一步。

Host 端内部电路（无设备连接）

⚠️ SE0（Single-Ended Zero）状态：当 Host 无设备连接时，D+ 和 D− 两条线都通过各自 15kΩ 下拉电阻接地，线上电压均为约 0V。Host 控制器检测到 D+ = D− ≈ 0V，判定为"无设备接入"。

Host 端为何要下拉？

1
定义"空闲"基准

USB 是热插拔总线，Host 需要在任何时候都能感知总线状态。将 D+/D− 通过 15kΩ 拉到 GND，使"无连接"时两线均稳定在 0V（SE0 状态），避免浮空引起误判。

2
形成分压基础

当 Device 侧接入上拉电阻时，Host 的 15kΩ 和 Device 的 1.5kΩ 构成分压器，产生一个特定电压，Host 检测此电压即可知道设备已接入。

3
ESD 保护配合

15kΩ 还有限流作用，当 D+/D− 受到静电冲击时，电流经过此电阻泄放到 GND，保护内部 IC。

Full-Speed Device 插入后的电路

Device 端内部电路职责

1
3.3V 稳压器

Device 从 VBUS（5V）经内部 LDO 稳压器降至 3.3V，作为上拉电阻的参考电压源。USB 规范明确要求上拉电压必须来自 VBUS 衍生，当 VBUS 移除时上拉必须同步断开，防止向总线反向注入电流。

2
1.5kΩ 上拉到 D+（Full Speed）

Full Speed 设备将 1.5kΩ 接在 3.3V 和 D+ 之间，此上拉"战胜"了 Host 侧 15kΩ 下拉，使 D+ 浮至约 3V。Host 检测到 D+ 升高，立即启动枚举流程。

3
D− 保持低电平

Full Speed 设备的 D− 无上拉，经 Host 侧 15kΩ 下拉继续保持 0V。两线不对称的高低状态（D+ 高 / D− 低）即是 USB Full Speed 的 J 状态（空闲态）。

🔑 Host 如何区分 Full Speed vs Low Speed？
Full Speed 设备：上拉在 D+（D+ 高 / D− 低 = J 态）
Low Speed 设备：上拉在 D−（D− 高 / D+ 低 = K 态，即反 J）
Host 在检测到某条线升高后，判断是 D+ 还是 D−，即可立即确定设备速度，无需任何通信。

📐 分压网络计算

设备插入后，Device 侧 Rpu 和 Host 侧 Rpd 构成串联分压器。

分压公式

VD+ = Vpu × Rpd / (Rpu + Rpd)

代入数值（Full Speed）

VD+ = 3.3V × 15kΩ / (1.5kΩ + 15kΩ)
     = 3.3V × 15,000 / 16,500
     = 3.3V × 0.909
     ≈ 3.0V ✓

流过节点电流

I = (Vpu - VD+) / Rpu
  = (3.3 - 3.0) / 1500
  = 0.2mA （极小，不损耗设备供电）

为何选 1.5kΩ 和 15kΩ？

10:1 比例的设计智慧
Rpd/Rpu = 15k/1.5k = 10。这使得分压电压接近上拉电压的 91%，足够高让 Host 可靠识别（阈值约 0.8V），同时比例不过大以节省 PCB 上的精度要求。
低功耗考虑
1.5kΩ 上拉电阻在 3.3V 下静态消耗约 0.2mA，远小于 USB 设备可用的 500mA，即使是电池供电设备也完全可以接受。
阻抗匹配
USB 差分对的特性阻抗为 90Ω（差分），上下拉电阻阻值远大于传输线阻抗，不干扰高频信号传输，但在 DC/低频下提供稳定偏置。

🐢

Low Speed（1.5 Mbps）

上拉接在 D−，Host 检测到 D− 上升至 ~3V，D+ 保持 0V，识别为低速设备。常见于键盘、鼠标。

🚗

Full Speed（12 Mbps）

上拉接在 D+，Host 检测到 D+ 上升至 ~3V，D− 保持 0V，识别为全速设备。最常见类型。

🚀

High Speed（480 Mbps）

先以 Full Speed 方式接入，然后通过 Chirp 序列（特殊 K/SE0 脉冲）与 Host 协商升级到高速模式，上拉电阻随后断开。

⚡ High Speed 上拉断开原因：USB 2.0 High Speed（480Mbps）使用 400mV 差分幅度（而非 3.3V），对应 45Ω 匹配终端。若保留 1.5kΩ 上拉，会导致阻抗失配和信号反射。因此 HS 模式一旦协商成功，Device 必须关断 D+ 上的 1.5kΩ 上拉（通过 MOSFET 开关实现），转而依靠内部 45Ω 片上终端。

总线状态定义（Full Speed）

状态名D+D−含义
J（空闲） HIGH ≈3V LOW ≈0V Full Speed 默认空闲态；设备已连接
K（数据翻转） LOW ≈0V HIGH ≈3V J 的反相；NRZI 编码中的"翻转"
SE0（单端零） LOW <0.3V LOW <0.3V 复位信号（≥2.5ms）/ 无设备 / EOP
SE1（禁止） HIGH HIGH USB 协议中不允许出现的非法状态
NRZI 编码：USB 数据使用 Non-Return-to-Zero Inverted 编码——逻辑 "0" 时 D+/D− 翻转（J↔K），逻辑 "1" 时保持不变。同时使用"位填充"（连续 6 个 1 后强制插入 0）保证足够的信号跳变，供接收端恢复时钟。

数据包结构与上下拉的配合

● 空闲态（J）：由上下拉电阻维持，Host 和 Device 收发器均为高阻，不主动驱动。

● 传输期间：收发器接管驱动，上下拉电阻的 DC 影响被有源驱动器覆盖（驱动电流远大于电阻偏置）。

● EOP（包尾）：驱动器产生 SE0（≥2 bit），随后释放总线，上拉电阻将 D+ 恢复 J 态。

USB OTG

ID 引脚：Mini/Micro USB 角色切换

USB On-The-Go 在 Mini/Micro USB 的第 4 针增加了 ID 引脚，通过其接地或浮空来切换主从角色，使手机等移动设备既能作为 Device 也能作为 Host。

A-Device（Host） ID 引脚接 GND
✓ID 引脚直接短接到连接器 GND（Mini-A 插头物理接线）
✓设备检测到 ID = 0V，加载 Host 软件栈，开启 VBUS 输出
✓在 D+/D− 挂 Host 侧 15kΩ 下拉，等待 Device 连入
B-Device（Peripheral） ID 引脚浮空
✓ID 引脚在 Mini-B 插头内部保持浮空（无连接）
✓设备内部对 ID 有上拉，浮空时读取为高电平
✓ID = High → 加载 Device 软件栈，D+ 接 1.5kΩ 上拉等待 Host 枚举

USB Type-C

CC 引脚：Rp / Rd 分压协商

USB Type-C 废弃了老式的 ID 引脚，引入两条 CC（Configuration Channel）引脚，通过精确阻值的上拉/下拉电阻在任意插入方向下完成角色识别、电流能力广播和电缆检测。

DFP（Host/Source） CC 引脚挂 Rp 上拉
Rp 阻值编码电流能力：
56 kΩ→ Default USB（500mA/900mA）
22 kΩ→ Type-C 1.5A
10 kΩ→ Type-C 3.0A
UFP（Device/Sink） CC 引脚挂 Rd 下拉
为什么 Rd 固定 5.1kΩ？
与 DFP 三种 Rp 形成分压，产生不同的 CC 电压：
56k+5.1k → V_CC ≈ 0.41V（标准 USB 电流）
22k+5.1k → V_CC ≈ 0.77V（1.5A 模式）
10k+5.1k → V_CC ≈ 1.13V（3.0A 模式）
DFP 侧比较器读取 CC 电压即可判断 UFP 存在并知晓自身 Rp 档位被正确接受。

DRP 双角色端口（Dual Role Port）

现代设备（如笔记本、手机）的 Type-C 接口通常是 DRP，可动态切换 Host/Device：

1
交替切换

DRP 在 CC 上以 ~50ms 周期交替呈现 Rp（尝试做 Host）和 Rd（尝试做 Device），直到对端有响应。

2
角色固定

DFP+UFP 配对成功后停止切换。若两端均为 DRP，优先级由时序决定（先切换到 Rp 的一方胜出）。

3
PD 协议再协商

建立后还可通过 USB Power Delivery（在 CC 上运行 BMC 调制）发送 PR_Swap 消息，动态互换 Power Role（谁供电）和 Data Role（谁做 Host）。

🧠 核心原理总结 USB 的角色识别本质是一个分压器检测系统：上拉电阻产生高电平信号，下拉电阻拉低为检测基准，两者形成分压，检测端通过比较电压高低完成角色/速度/方向判定，无需任何数字通信即可完成物理层握手。
⚡ 电源安全设计 "Cold Socket"原则：Type-C Host 的 VBUS 在未检测到 Rd 之前不输出 5V，避免对空气、异物或非 USB 设备放电造成安全问题。这与老式 USB-A 接口（VBUS 始终带电）是一个重要的安全改进。
📡 进化方向 从 USB 2.0 的固定 D+ 上拉 → OTG ID 引脚 → Type-C CC 引脚，每一代在保持简洁电阻网络思路的同时，增加了更多维度（方向、电流、角色协商），最终 PD 协议在 CC 上叠加数字通信实现了完全动态的电源与角色管理。
