USB 2.0 标准设备请求

依据 USB 2.0 Specification Revision 2.0 第 9 章（Chapter 9 · USB Device Framework）整理。 所有标准请求经默认控制管道（EP0）以 SETUP → [DATA] → STATUS 控制传输完成。

主机在控制传输的 SETUP 阶段 发送 8 字节 Setup Data，定义本次请求的参数。后续 DATA / STATUS 阶段的方向由 bmRequestType 的 Direction 位与 wLength 决定。

9.3.4 · wIndex 字段编码

指定端点时（Figure 9-2）

低字节：D0–D3 为端点号；D7 为方向（0=OUT，1=IN）。控制端点请求应设 D7=0，设备可接受 0 或 1。

指定接口时（Figure 9-3）

低字节 D0–D7 为接口号；高字节保留为 0。

字符串描述符的 GET/SET_DESCRIPTOR 中，wIndex 为 Language ID（如 0x0409 美式英语）。

9.2.6 · 请求处理时序（摘要）

场景时限
任意请求上限5 s（整体）
Reset/Resume 后主机提供 10 ms 恢复间隔后再传数据
SET_ADDRESSSTATUS 须在 50 ms 内完成；之后 2 ms 恢复间隔
无 DATA 的标准请求50 ms 内完成（含 STATUS）
IN DATA（设备→主机）首包 500 ms 内；后续包间隔 ≤500 ms；最后 STATUS ≤50 ms
OUT DATA（主机→设备）受 5 s 总限约束
不支持或非法请求：在 DATA 或 STATUS 阶段返回 STALL；优先在 SETUP 之后尽早 STALL。不会因单次非法请求而将控制管道置于 Halt 特性。
