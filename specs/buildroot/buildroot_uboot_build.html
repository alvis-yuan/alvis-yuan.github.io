1. 简介与整体架构

来源：Buildroot 用户手册 2026.02 · boot/uboot/uboot.mk

Buildroot 是一套简化嵌入式 Linux 系统构建的工具，可以自动生成交叉编译工具链、根文件系统、Linux 内核镜像以及 Bootloader。U-Boot（Das U-Boot）是其中最常用的 Bootloader，官方主页为 https://docs.u-boot.org/en/latest/。

Buildroot 对 U-Boot 的支持集中在以下两个文件：

文件作用
boot/uboot/Config.in定义所有 Kconfig 菜单选项，控制 menuconfig 中的配置项
boot/uboot/uboot.mk定义下载、配置、编译、安装规则，是实际构建逻辑的核心

整体架构图

📌 本指南侧重 外部 U-Boot（自定义 Git 仓库 / 自定义 Tarball / 自定义版本号），这是企业级产品开发中最常见的使用场景。所有内容严格遵循 Buildroot 官方手册（2026.02）及源代码。

2. 在 Buildroot 中启用 U-Boot

来源：boot/uboot/Config.in

进入 Buildroot 根目录，运行菜单配置：

make menuconfig

然后导航到 Bootloaders 菜单，勾选 U-Boot（对应配置变量 BR2_TARGET_UBOOT=y）。

💡 Config.in 原文定义
# boot/uboot/Config.in（原文）
menuconfig BR2_TARGET_UBOOT
    bool "U-Boot"
    help
      Build "Das U-Boot" Boot Monitor
      https://docs.u-boot.org/en/latest/
ℹ️ 官方手册明确要求：所有构建操作必须以普通用户身份执行，无需 root 权限，这样可以防止编译过程中行为异常的软件包破坏系统。

3. 构建系统选择（Build System）

来源：boot/uboot/Config.in

Buildroot 支持两套 U-Boot 构建系统，通过 Build system 选项控制：

📋 Config.in 原文定义
choice
    prompt "Build system"
    default BR2_TARGET_UBOOT_BUILD_SYSTEM_KCONFIG if BR2_TARGET_UBOOT_LATEST_VERSION
    default BR2_TARGET_UBOOT_BUILD_SYSTEM_LEGACY

config BR2_TARGET_UBOOT_BUILD_SYSTEM_KCONFIG
    bool "Kconfig"
    help
      Select this option if you use a recent U-Boot version
      (2015.04 or newer), so that we use the Kconfig build system.

config BR2_TARGET_UBOOT_BUILD_SYSTEM_LEGACY
    bool "Legacy"
    help
      Select this option if you use an old U-Boot (older than
      2015.04), so that we use the old build system.
endchoice
构建系统适用版本配置方式配置变量
Kconfig U-Boot ≥ 2015.04 in-tree defconfig 或 自定义 config 文件 BR2_TARGET_UBOOT_BUILD_SYSTEM_KCONFIG=y
Legacy U-Boot < 2015.04 board name（_config 后缀） BR2_TARGET_UBOOT_BUILD_SYSTEM_LEGACY=y
⚠️ Legacy 模式下，板卡名称通过 BR2_TARGET_UBOOT_BOARDNAME 指定，系统会自动添加 _config 后缀。新项目推荐使用 Kconfig 模式，与 Linux 内核配置方式一致。

Legacy 模式专属配置

# boot/uboot/Config.in 原文
if BR2_TARGET_UBOOT_BUILD_SYSTEM_LEGACY
config BR2_TARGET_UBOOT_BOARDNAME
    string "U-Boot board name"
    help
      One of U-Boot supported boards to be built.
      This will be suffixed with _config to meet U-Boot standard
      naming. See boards.cfg in U-Boot source code for the list
      of available configurations.
endif

uboot.mk 中 Legacy 配置命令

# boot/uboot/uboot.mk 原文
ifeq ($(BR2_TARGET_UBOOT_BUILD_SYSTEM_LEGACY),y)
define UBOOT_CONFIGURE_CMDS
    $(TARGET_CONFIGURE_OPTS) \
    $(UBOOT_MAKE) -C $(@D) $(UBOOT_MAKE_OPTS) \
        $(UBOOT_BOARD_NAME)_config
endef
endif

4. U-Boot 版本来源配置

来源：boot/uboot/Config.in · boot/uboot/uboot.mk

Buildroot 提供五种 U-Boot 来源选择，在 menuconfig 的 U-Boot Version 菜单中设置：

📋 Config.in 原文版本选择定义
choice
    prompt "U-Boot Version"
    help
      Select the specific U-Boot version you want to use

config BR2_TARGET_UBOOT_LATEST_VERSION
    bool "2026.04"

config BR2_TARGET_UBOOT_CUSTOM_VERSION
    bool "Custom version"
    help
      This option allows to use a specific official versions

config BR2_TARGET_UBOOT_CUSTOM_TARBALL
    bool "Custom tarball"

config BR2_TARGET_UBOOT_CUSTOM_GIT
    bool "Custom Git repository"

config BR2_TARGET_UBOOT_CUSTOM_HG
    bool "Custom Mercurial repository"

config BR2_TARGET_UBOOT_CUSTOM_SVN
    bool "Custom Subversion repository"
endchoice

uboot.mk 中的下载源配置

不同版本选择对应不同的下载地址配置（原文）：

# boot/uboot/uboot.mk 原文
ifeq ($(BR2_TARGET_UBOOT_CUSTOM_TARBALL),y)
UBOOT_TARBALL = $(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_TARBALL_LOCATION))
UBOOT_SITE    = $(patsubst %/,%,$(dir $(UBOOT_TARBALL)))
UBOOT_SOURCE  = $(notdir $(UBOOT_TARBALL))
else ifeq ($(BR2_TARGET_UBOOT_CUSTOM_GIT),y)
UBOOT_SITE        = $(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_REPO_URL))
UBOOT_SITE_METHOD = git
else ifeq ($(BR2_TARGET_UBOOT_CUSTOM_HG),y)
UBOOT_SITE        = $(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_REPO_URL))
UBOOT_SITE_METHOD = hg
else ifeq ($(BR2_TARGET_UBOOT_CUSTOM_SVN),y)
UBOOT_SITE        = $(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_REPO_URL))
UBOOT_SITE_METHOD = svn
else
# Handle stable official U-Boot versions
UBOOT_SITE   = https://ftp.denx.de/pub/u-boot
UBOOT_SOURCE = u-boot-$(UBOOT_VERSION).tar.bz2
endif

5. 外部 Git 仓库配置（重点）

来源：boot/uboot/Config.in · boot/uboot/uboot.mk

使用外部 Git 仓库是企业项目中最常见的方式，可以使用内部私有仓库或 fork 后的公开仓库。

5.1 Config.in 原文：Git 仓库选项

if BR2_TARGET_UBOOT_CUSTOM_GIT || BR2_TARGET_UBOOT_CUSTOM_HG \
                               || BR2_TARGET_UBOOT_CUSTOM_SVN
config BR2_TARGET_UBOOT_CUSTOM_REPO_URL
    string "URL of custom repository"

config BR2_TARGET_UBOOT_CUSTOM_REPO_VERSION
    string "Custom repository version"
    help
      Revision to use in the typical format used by
      Git/Mercurial/Subversion E.G. a sha id, a tag, branch, ..
endif

5.2 .config 示例（外部 Git）

# 启用 U-Boot
BR2_TARGET_UBOOT=y

# 使用 Kconfig 构建系统
BR2_TARGET_UBOOT_BUILD_SYSTEM_KCONFIG=y

# 选择外部 Git 仓库
BR2_TARGET_UBOOT_CUSTOM_GIT=y

# Git 仓库地址（支持 https、git、ssh 协议）
BR2_TARGET_UBOOT_CUSTOM_REPO_URL="https://github.com/mycompany/u-boot.git"

# 指定 Git 分支、tag 或 commit SHA
BR2_TARGET_UBOOT_CUSTOM_REPO_VERSION="my-board-v2024.04"

# 使用 in-tree defconfig 文件
BR2_TARGET_UBOOT_USE_DEFCONFIG=y
BR2_TARGET_UBOOT_BOARD_DEFCONFIG="my_board"

# 许可证文件（外部仓库必须指定）
BR2_TARGET_UBOOT_LICENSE_FILES="Licenses/gpl-2.0.txt"
✅ BR2_TARGET_UBOOT_CUSTOM_REPO_VERSION 可以是 Git branch 名、tag 名或完整的 commit SHA，Buildroot 会调用 git clone --depth=1 或 git fetch 来获取指定版本。

5.3 uboot.mk 中 Git 下载处理（原文）

else ifeq ($(BR2_TARGET_UBOOT_CUSTOM_GIT),y)
UBOOT_SITE        = $(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_REPO_URL))
UBOOT_SITE_METHOD = git

5.4 menuconfig 操作路径

  • 1
    进入 Bootloaders 菜单
    运行 make menuconfig → Bootloaders → U-Boot
  • 2
    选择构建系统
    Build system → Kconfig
  • 3
    选择版本来源
    U-Boot Version → Custom Git repository
  • 4
    填写 Git URL
    URL of custom repository → 输入完整 Git 地址
  • 5
    填写版本
    Custom repository version → 输入 branch/tag/SHA
  • 6
    配置 defconfig
    U-Boot configuration → Using an in-tree board defconfig file → Board defconfig

5.5 版本校验逻辑

⚡ uboot.mk 构建时校验（原文）
ifeq ($(BR2_TARGET_UBOOT_CUSTOM_GIT)$(BR2_TARGET_UBOOT_CUSTOM_HG),y)
ifeq ($(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_REPO_URL)),)
$(error No custom U-Boot repository URL specified. \
        Check your BR2_TARGET_UBOOT_CUSTOM_REPO_URL setting)
endif
ifeq ($(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_REPO_VERSION)),)

6. 自定义 Tarball 配置

来源：boot/uboot/Config.in · boot/uboot/uboot.mk

适用于离线环境或需要使用特定二进制发布包的场景。

Config.in 原文

config BR2_TARGET_UBOOT_CUSTOM_TARBALL_LOCATION
    string "URL of custom U-Boot tarball"
    depends on BR2_TARGET_UBOOT_CUSTOM_TARBALL

uboot.mk 处理逻辑（原文）

ifeq ($(BR2_TARGET_UBOOT_CUSTOM_TARBALL),y)
UBOOT_TARBALL = $(call qstrip,\
                $(BR2_TARGET_UBOOT_CUSTOM_TARBALL_LOCATION))
UBOOT_SITE    = $(patsubst %/,%,$(dir $(UBOOT_TARBALL)))
UBOOT_SOURCE  = $(notdir $(UBOOT_TARBALL))

.config 示例（自定义 Tarball）

BR2_TARGET_UBOOT=y
BR2_TARGET_UBOOT_BUILD_SYSTEM_KCONFIG=y
BR2_TARGET_UBOOT_CUSTOM_TARBALL=y
# 可以是本地路径或 http/ftp URL
BR2_TARGET_UBOOT_CUSTOM_TARBALL_LOCATION="https://myserver.com/u-boot-myboard-2024.04.tar.gz"
BR2_TARGET_UBOOT_USE_DEFCONFIG=y
BR2_TARGET_UBOOT_BOARD_DEFCONFIG="myboard"
⚠️ 使用自定义 Tarball 或 Git 仓库时，必须提供 BR2_TARGET_UBOOT_LICENSE_FILES，否则 Buildroot 的 legal-info 功能无法正常工作。

7. 自定义 U-Boot 配置文件

来源：Buildroot 手册 · boot/uboot/Config.in

官方手册（Configuration of other components 章节）原文说明：

📖 官方手册原文（英文）

"Configuration of U-Boot (version 2015.04 or newer) is done in the same way as for the Linux kernel. The corresponding configuration variables are BR2_TARGET_UBOOT_USE_CUSTOM_CONFIG and BR2_TARGET_UBOOT_USE_DEFCONFIG. To open the configuration editor, use make uboot-menuconfig."

来源：buildroot.org/downloads/manual/manual.html - Configuration of other components 章节

两种 Kconfig 配置方式

📋 Config.in 原文定义
if BR2_TARGET_UBOOT_BUILD_SYSTEM_KCONFIG
choice
    prompt "U-Boot configuration"
    default BR2_TARGET_UBOOT_USE_DEFCONFIG

config BR2_TARGET_UBOOT_USE_DEFCONFIG
    bool "Using an in-tree board defconfig file"

config BR2_TARGET_UBOOT_USE_CUSTOM_CONFIG
    bool "Using a custom board (def)config file"
endchoice

config BR2_TARGET_UBOOT_BOARD_DEFCONFIG
    string "Board defconfig"
    depends on BR2_TARGET_UBOOT_USE_DEFCONFIG
    help
      Name of the board for which U-Boot should be built,
      without the _defconfig suffix.

config BR2_TARGET_UBOOT_CUSTOM_CONFIG_FILE
    string "Configuration file path"
    depends on BR2_TARGET_UBOOT_USE_CUSTOM_CONFIG
    help
      Path to the U-Boot configuration file.
endif
配置方式变量说明示例值
in-tree defconfig BR2_TARGET_UBOOT_USE_DEFCONFIG=y
BR2_TARGET_UBOOT_BOARD_DEFCONFIG		 使用 U-Boot 源码自带的 configs/ 目录下的 defconfig，无需 _defconfig 后缀 "rpi_4"、"am62x_evm_a53"
自定义配置文件 BR2_TARGET_UBOOT_USE_CUSTOM_CONFIG=y
BR2_TARGET_UBOOT_CUSTOM_CONFIG_FILE		 指向 Buildroot 树内的自定义配置文件路径 "board/mycompany/myboard/uboot.config"

uboot.mk 中 Kconfig 处理（原文）

ifeq ($(BR2_TARGET_UBOOT_USE_DEFCONFIG),y)
UBOOT_KCONFIG_DEFCONFIG = \
    $(call qstrip,$(BR2_TARGET_UBOOT_BOARD_DEFCONFIG))_defconfig
else ifeq ($(BR2_TARGET_UBOOT_USE_CUSTOM_CONFIG),y)
UBOOT_KCONFIG_FILE = \
    $(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_CONFIG_FILE))
endif

UBOOT_KCONFIG_EDITORS = menuconfig xconfig gconfig nconfig
UBOOT_KCONFIG_OPTS = $(UBOOT_MAKE_OPTS) \
    HOSTCC="$(HOSTCC_NOCCACHE)" \
    HOSTLDFLAGS=""

用法：修改 U-Boot 配置

# 打开 U-Boot 的 menuconfig 界面进行配置修改
make uboot-menuconfig

# 保存并更新 defconfig
make uboot-update-defconfig
ℹ️ 官方手册在 "Storing the configuration of other components" 章节中明确：使用 make uboot-update-defconfig 将配置保存回 BR2_TARGET_UBOOT_CUSTOM_CONFIG_FILE 指定的文件。如果使用 in-tree defconfig，该命令会更新对应的 defconfig 文件。

8.2 自定义补丁

📋 Config.in 原文
config BR2_TARGET_UBOOT_PATCH
    string "Custom U-Boot patches"
    help
      A space-separated list of patches to apply to U-Boot.
      Each patch can be described as an URL, a local file path,
      or a directory. In the case of a directory, all files
      matching *.patch in the directory will be applied.
      Most users may leave this empty
⚙️ uboot.mk 补丁应用逻辑（原文）
UBOOT_PATCHES = $(call qstrip,$(BR2_TARGET_UBOOT_PATCH))
# 远程补丁（URL）
UBOOT_PATCH = $(filter ftp://% http://% https://%,$(UBOOT_PATCHES))

define UBOOT_APPLY_LOCAL_PATCHES
for p in $(filter-out ftp://% http://% https://%,$(UBOOT_PATCHES)); do \
    if test -d $$p ; then \
        $(APPLY_PATCHES) $(@D) $$p \*.patch || exit 1 ; \
    else \
        $(APPLY_PATCHES) $(@D) `dirname $$p` `basename $$p` \
            || exit 1; \
    fi \
done
endef
UBOOT_POST_PATCH_HOOKS += UBOOT_APPLY_LOCAL_PATCHES

10. SPL / TPL 配置

来源：boot/uboot/Config.in

SPL（Secondary Program Loader）是 U-Boot 的第一阶段 Bootloader，负责初始化 DDR 等外部存储后加载完整的 U-Boot 镜像。

📋 Config.in 原文定义
config BR2_TARGET_UBOOT_SPL
    bool "Install U-Boot SPL binary image"
    help
      Install the U-Boot SPL binary image to the images directory.
      SPL is a first stage bootloader loaded into internal memory
      in charge of enabling and configuring the external memory
      (DDR), and load the u-boot program into DDR.

config BR2_TARGET_UBOOT_SPL_NAME
    string "U-Boot SPL/TPL binary image name(s)"
    default "spl/u-boot-spl.bin"
    depends on BR2_TARGET_UBOOT_SPL
    help
      A space-separated list of SPL/TPL binaries, generated during
      u-boot build. For most platform SPL name is spl/u-boot-spl.bin
      and TPL name is tpl/u-boot-tpl.bin but not always.
      SPL name is MLO on OMAP and SPL on i.MX6 for example.

常见平台 SPL 文件名

平台SPL 文件名BR2_TARGET_UBOOT_SPL_NAME
通用u-boot-spl.bin"spl/u-boot-spl.bin"（默认）
TI OMAPMLO"MLO"
NXP i.MX6SPL"SPL"
带 TPLSPL + TPL"tpl/u-boot-tpl.bin spl/u-boot-spl.bin"
Xilinx ZynqMPboot.bin"spl/boot.bin"

11. 主机依赖项（Host Dependencies）

来源：boot/uboot/Config.in · boot/uboot/uboot.mk

根据目标板卡配置，U-Boot 可能需要额外的主机工具。这些依赖项在 Config.in 中以 BR2_TARGET_UBOOT_NEEDS_* 形式定义：

选项配置变量触发条件（原文 help）依赖包
需要 DTC BR2_TARGET_UBOOT_NEEDS_DTC=y U-Boot board 配置需要设备树编译器 host-dtc
需要 Python3 BR2_TARGET_UBOOT_NEEDS_PYTHON3=y U-Boot 2020.01 后部分配置需要宿主机 Python 3.x host-python3, host-python-setuptools
需要 pylibfdt BR2_TARGET_UBOOT_NEEDS_PYLIBFDT=y 某些板卡配置需要 Python libfdt 库 host-python-pylibfdt
需要 pyelftools BR2_TARGET_UBOOT_NEEDS_PYELFTOOLS=y Rockchip SoC（如 rk3399）ELF 解析 host-python-pyelftools
需要 OpenSSL BR2_TARGET_UBOOT_NEEDS_OPENSSL=y CONFIG_FIT_SIGNATURE 启用时需要 host-openssl
需要 lzop BR2_TARGET_UBOOT_NEEDS_LZOP=y CONFIG_SPL_LZO 启用时需要 host-lzop
需要 GnuTLS BR2_TARGET_UBOOT_NEEDS_GNUTLS=y CONFIG_TOOLS_MKEFICAPSULE 启用时需要 host-gnutls
需要 util-linux BR2_TARGET_UBOOT_NEEDS_UTIL_LINUX=y CONFIG_TOOLS_MKEFICAPSULE（-luuid）需要 host-util-linux
需要 xxd BR2_TARGET_UBOOT_NEEDS_XXD=y CONFIG_USE_DEFAULT_ENV_FILE 启用时需要 host-vim（含 xxd）
使用 binman BR2_TARGET_UBOOT_USE_BINMAN=y 使用 binman 工具生成/签名引导镜像 host-python-jsonschema, host-python-pyyaml, host-python-yamllint

CROSS_COMPILE 和架构设置（uboot.mk 原文）

# The kernel calls AArch64 'arm64', but U-Boot calls it just 'arm',
# so we have to special case it. Similar for i386/x86_64 -> x86
ifeq ($(NORMALIZED_ARCH),arm64)
UBOOT_ARCH = arm
else ifneq ($(filter $(NORMALIZED_ARCH),i386 x86_64),)
UBOOT_ARCH = x86
else
UBOOT_ARCH = $(NORMALIZED_ARCH)
endif

UBOOT_MAKE_OPTS += \
    CROSS_COMPILE="$(TARGET_CROSS)" \
    ARCH=$(UBOOT_ARCH) \
    HOSTCC="$(HOSTCC) $(subst -I/,-isystem /,\
        $(subst -I /,-isystem /,$(HOST_CFLAGS)))" \
    HOSTLDFLAGS="$(HOST_LDFLAGS)" \
    $(call qstrip,$(BR2_TARGET_UBOOT_CUSTOM_MAKEOPTS))

13.1 常用 Make 目标

命令作用
make uboot只编译 U-Boot（若已编译过则跳过）
make uboot-rebuild强制重新编译 U-Boot（不重新下载）

14. ATF / OP-TEE / OpenSBI 集成

来源：boot/uboot/Config.in · boot/uboot/uboot.mk

某些平台（如 Allwinner A64、Rockchip RK3399、SiFive HiFive）需要在 U-Boot 中集成固件二进制文件。

14.1 ARM Trusted Firmware (ATF) BL31

📋 Config.in 原文
config BR2_TARGET_UBOOT_NEEDS_ATF_BL31
    bool "U-Boot needs ATF BL31"
    depends on BR2_TARGET_ARM_TRUSTED_FIRMWARE
    depends on !BR2_TARGET_ARM_TRUSTED_FIRMWARE_UBOOT_AS_BL33
    select BR2_TARGET_ARM_TRUSTED_FIRMWARE_BL31
    help
      Some specific platforms (such as Allwinner A64/H5) encapsulate
      the BL31 part of ATF inside U-Boot. This option makes sure ATF
      gets built prior to U-Boot, and that the BL31 variable pointing
      to ATF's BL31 binary, is passed during the Buildroot build.
⚙️ uboot.mk 中 ATF 处理（原文）
ifeq ($(BR2_TARGET_UBOOT_NEEDS_ATF_BL31),y)
UBOOT_DEPENDENCIES += arm-trusted-firmware
ifeq ($(BR2_TARGET_UBOOT_NEEDS_ATF_BL31_ELF),y)
UBOOT_MAKE_OPTS += BL31=$(BINARIES_DIR)/bl31.elf
define UBOOT_COPY_ATF_FIRMWARE
    cp $(BINARIES_DIR)/bl31.elf $(@D)/
endef
else
UBOOT_MAKE_OPTS += BL31=$(BINARIES_DIR)/bl31.bin
define UBOOT_COPY_ATF_FIRMWARE
    cp $(BINARIES_DIR)/bl31.bin $(@D)/
endef
endif
UBOOT_PRE_BUILD_HOOKS += UBOOT_COPY_ATF_FIRMWARE
endif

14.2 OP-TEE TEE（信任执行环境）

📋 Config.in 原文（OP-TEE 格式说明）
config BR2_TARGET_UBOOT_NEEDS_OPTEE_TEE
    bool "U-Boot needs OPTEE TEE"
    depends on BR2_TARGET_OPTEE_OS_CORE
    help
      Some platforms (such as Rockchip) encapsulate the TEE inside

# tee.elf - ELF 格式，仅适用于 OP-TEE < 3.8.0

15. 默认环境变量文件

来源：boot/uboot/Config.in · U-Boot 环境变量文档
📋 Config.in 原文（完整）
config BR2_TARGET_UBOOT_DEFAULT_ENV_FILE
    string "Text file with default environment"
    depends on BR2_TARGET_UBOOT_BUILD_SYSTEM_KCONFIG
    help
      Text file containing the variables to be used as the default
      environment in U-Boot.

      If empty, let U-Boot generate the default environment from the
      source code and other U-Boot configuration values, which is
      the default behaviour.

      If set to the path to a file, instructs U-Boot to define the
      entire U-Boot default environment from the provided file,
      disabling all the default behaviour.

      From the U-Boot documentation:

      The format is the same as accepted by the mkenvimage tool,
      with lines containing key=value pairs. Blank lines and
      lines beginning with '#' are ignored.

      Based on the USE_DEFAULT_ENV_FILE and DEFAULT_ENV_FILE U-Boot
      configuration variables.

      For more info see:
      https://docs.u-boot.org/en/latest/usage/environment.html#external-environment-file

      Requires U-Boot >= v2018.05.

uboot.mk 中的处理（原文）

ifeq ($(BR2_TARGET_UBOOT_DEFAULT_ENV_FILE_ENABLED),y)
UBOOT_DEFAULT_ENV_FILE = \
    $(call qstrip,$(BR2_TARGET_UBOOT_DEFAULT_ENV_FILE))
define UBOOT_KCONFIG_DEFAULT_ENV_FILE
    # Pre-2025.10
    $(call KCONFIG_SET_OPT,CONFIG_USE_DEFAULT_ENV_FILE,y)
    $(call KCONFIG_SET_OPT,CONFIG_DEFAULT_ENV_FILE,\
        "$(shell readlink -f $(UBOOT_DEFAULT_ENV_FILE))")
    # 2025.10 and later
    $(call KCONFIG_SET_OPT,CONFIG_ENV_USE_DEFAULT_ENV_TEXT_FILE,y)
    $(call KCONFIG_SET_OPT,CONFIG_ENV_DEFAULT_ENV_TEXT_FILE,\
        "$(shell readlink -f $(UBOOT_DEFAULT_ENV_FILE))")
endef
endif

使用示例

# .config 中启用
BR2_TARGET_UBOOT_DEFAULT_ENV_FILE="board/mycompany/myboard/uboot-env.txt"

# uboot-env.txt 内容格式（与 mkenvimage 工具相同）
# 空行和 # 开头的行会被忽略
bootcmd=run distro_bootcmd
bootdelay=2
ethaddr=00:11:22:33:44:55
ipaddr=192.168.1.100
serverip=192.168.1.1
✅ 此特性需要 U-Boot ≥ v2018.05，且仅在 Kconfig 构建系统下可用。设置后会自动选中 BR2_TARGET_UBOOT_NEEDS_XXD=y（依赖 xxd 工具）。

16. Defconfig 管理与项目定制

📖 官方快速指南原文

"Set the following options to board/<company>/<board>/<board>.config (as far as they are relevant): ...
BR2_TARGET_UBOOT_CUSTOM_CONFIG_FILE

Write the configuration files:
make uboot-update-defconfig"

来源：buildroot.org/downloads/manual/customize-quick-guide.txt

推荐项目目录结构

# Buildroot 官方推荐的项目定制目录结构
board/
└── mycompany/
    └── myboard/
        ├── uboot.config          # U-Boot defconfig（BR2_TARGET_UBOOT_CUSTOM_CONFIG_FILE）
        ├── uboot-fragment.config # 额外配置片段（BR2_TARGET_UBOOT_CONFIG_FRAGMENT_FILES）
        ├── uboot-env.txt         # 默认环境变量（BR2_TARGET_UBOOT_DEFAULT_ENV_FILE）
        ├── patches/
        │   └── uboot/
        │       └── 0001-myboard-support.patch  # 本地补丁
        ├── myboard.dts           # 自定义 DTS（BR2_TARGET_UBOOT_CUSTOM_DTS_PATH）
        └── post-image.sh         # 镜像后处理脚本

configs/
└── mycompany_myboard_defconfig   # Buildroot 顶层 defconfig

配置持久化工作流

  1. 1
    设置 BR2_TARGET_UBOOT_CUSTOM_CONFIG_FILE 指向文件 
    # make menuconfig 中配置，或直接写入 .config
BR2_TARGET_UBOOT_USE_CUSTOM_CONFIG=y
BR2_TARGET_UBOOT_CUSTOM_CONFIG_FILE="board/mycompany/myboard/uboot.config"
  2. 2
    修改 U-Boot 配置 
    make uboot-menuconfig
  3. 3
    保存回自定义文件 
    make uboot-update-defconfig
# 精简 defconfig（仅保存与默认值不同的选项）
make uboot-savedefconfig
  4. 4
    保存 Buildroot 顶层配置 
    make savedefconfig BR2_DEFCONFIG=configs/mycompany_myboard_defconfig
