TLS 1.3 解密抓包协议分析

基于 tls13_decode.pcapng（7 个 PDU），结合 RFC 8446 解读 1-RTT 握手、加密握手飞行与应用层 MQTT。

1 · 抓包摘要

PDUΔt方向内容
#10 msC→SClientHello（含 supported_versions、key_share）
#21.08 msS→CServerHello + CCS + 5×加密握手记录（外层 application_data）
#33.28 msC→SCCS + 3×加密握手记录

TLS 1.3 在记录层上不再发送明文 Certificate/SKE；握手消息封装在 application_data 记录内，用 handshake_traffic_secret 保护。本导出中部分记录仍为密文，#5 已可见解密后 MQTT。

标注长度为外层 TLS 记录密文大小；握手语义按 RFC 8446 消息顺序与长度推断。

8 · 应用数据与 MQTT（PDU #4–#7）

PDU #5（1066 B）已解密可见 MQTT CONNACK：首字节 0x20 为 MQTT 连接确认；说明 server_application_traffic_secret_0 解密成功，TLS 握手完成且 Broker 接受连接。
PDU长度说明
#41066 B同为应用数据记录，可能含 NewSessionTicket 与早期应用数据（导出仍为密文）
#51066 BMQTT CONNACK（明文可见）
#6–#719 B短控制帧，可能为加密 Alert（close_notify）或 MQTT 心跳

MQTT CONNACK 结构提示

CONNACK 固定头 0x20，剩余长度 2，随后为 Connect Acknowledge Flags 与 Return Code（0 = 接受）。完整 MQTT 交互需结合更长的应用数据抓包。
