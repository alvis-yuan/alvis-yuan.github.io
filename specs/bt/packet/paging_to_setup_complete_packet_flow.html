从 Paging 到 LMP Setup Complete

第二次完整建链（honor50_sunmi_bt_ota.csv 约 28.373s → 28.404s）： HONOR 50 Pro（Central）对 CloudPrint_0159（Peripheral）完成基带连接与 LMP 建链初始化。

Paging（基带） LMP 序列（CSV） 必要性汇总 Setup Complete 之后

范围说明

Paging 属于 BR/EDR 基带层，CSV 从 LMP Version Exchange 开始，说明 page 已成功。 Setup Complete 标志 LMP「连接建立阶段」结束；此后仍有 Max Slot、AFH、时钟偏移等链路参数协商（见文末），不属于 Setup Complete 之前必须完成的信令。

角色设备CSV 依据
Central / Page 发起方HONOR 50 ProLMP Version: Central 5.2；主动 Host Connection、Packet Type Table
Peripheral / Page 响应方CloudPrint_0159Peripheral 6.0；接受连接

基带 阶段 A：Paging（CSV 未导出）

Inquiry 得到 BD_ADDR 与 clock 估计后，Central 进入 Page 状态，用 Page ID train 命中 Peripheral 的 Page Scan。

Page / Page Response / FHS 协议推断

时间：≈ 28.37s 之前 · CSV 无对应行
必要性：在空口上建立物理 ACL 链路——分配主从角色、同步时钟、确定跳频序列与访问码。没有 paging，就不会有后续任何 LMP PDU。

LMP 阶段 B：Paging 之后 → Setup Complete（CSV 第 6–12 行）

下列 7 步均来自第二次建链，时间跨度约 31 ms（28.372662 → 28.403912）。

① LMP Version Exchange 28.372662

CSV 第 6 行 · Central: 5.2 · Peripheral: 6.0

交换 LMP 协议版本与厂商扩展版本号，双方确认能理解的 LMP PDU 集合。

必要性：后续所有 LMP 语义都依赖版本兼容。若版本不匹配，控制器应拒绝或降级，避免误解析 Feature、SSP、加密等 PDU。这是 paging 成功后第一条 LMP 交换，标志逻辑链路开始工作。

② LMP Features Exchange 28.376411

CSV 第 7 行 · 46 Features · 51 Features

交换 Feature Page 0 等基础特性位图（3-slot、5-slot、加密、时钟偏移、RSSI、通道质量等经典能力）。

必要性：决定双方是否支持 3/5-slot 包、链路加密、AFH、功率控制等基础链路能力。没有 Features，无法安全地发起 Max Slot、Set AFH、加密模式等后续 LMP（部分在 Setup Complete 之后）。数字 46/51 为解码器统计的支持特性数，体现两端能力不对称但可互通。

③ LMP Extended Features Exchange（Host）28.391412

CSV 第 8 行 · SSP Host, LE Host, SC Host

扩展特性页：主机栈声明支持 Secure Simple Pairing、LE、BR/EDR Secure Connections。

必要性：为稍后 28.524s 的 IO Capability / P-256 公钥 / SSP 配对 提供能力前提。若未声明 SSP Host / SC Host，只能走 Legacy PIN 配对，与本抓包后续流程不符。LE Host 表示双模设备栈能力（本段 BR/EDR 建链仍可进行）。

④ LMP Extended Features Exchange（Controller）28.395162

CSV 第 9 行 · SC Controller, Ping

控制器侧扩展能力：Secure Connections 控制器算法、Ping（链路存活检测）。

必要性：SC Controller 表示空口侧能处理 P-256 ECDH、DHKey Check 等 SC 配对 PDU；Ping 用于加密后链路维护（抓包后期可见 LMP Ping）。与上一条 Host 特性配合，构成完整 SC 配对能力链。

⑤ LMP Host Connection 28.398911 · Accepted

CSV 第 10 行

Central 侧请求对端 Host 接受 本 ACL 连接；Peripheral Host 返回 Accepted。

必要性：Paging 只代表控制器层连通，Host 可能因策略、资源、配对列表拒绝连接。Host Connection 是主机策略门控：CloudPrint 的 Host 同意保留链路，后续 LMP/L2CAP 才可继续。若 Not Accepted，链路会在建立阶段被拆除。

⑥ LMP Packet Type Table 28.403287 · 2/3Mbps · Accepted

CSV 第 11 行

协商 ACL 可使用的包类型：启用 EDR 2 Mbps / 3 Mbps 等（相对纯 Basic Rate）。

必要性：在发送 Setup Complete 前明确物理层数据包类型能力，避免一端发 EDR 包对端无法解析。对本 OTA/SDP 大流量场景，后续 Preferred Rate、Auto Rate 会进一步选速率；Packet Type Table 是 EDR 的开关级协商。放在 Setup Complete 前完成，保证“建链完成”时双方对包类型有一致预期。

⑦ LMP Setup Complete 28.403912

CSV 第 12 行 · 本阶段终点

宣告 LMP 连接建立（Connection establishment） 子状态结束。

必要性：给双方栈一个明确同步点——“建链初始化已做完，可进入已连接态的常规 LMP 与上层流程”。未收到 Setup Complete，主机通常不会认为 ACL 可用于配对/业务。它不表示加密已完成，只表示建立阶段结束。
