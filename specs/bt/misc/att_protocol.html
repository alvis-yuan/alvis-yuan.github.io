蓝牙 ATT 协议指令分类大全 (Core v5.2)

基于 Bluetooth Core Specification v5.2 [Vol 3] Part F, Section 3.4 规范
完整涵盖请求(REQ)、响应(RSP)、命令(CMD)、通知(NTF)、指示(IND)、确认(CFM) 六类 PDU。

📡 ATT 交互规则图解 (PDU 通信流)

请求与响应 (Request & Response)
客户端
── [REQ] ──▶ ◀── [RSP/ERR] ─
服务端

必然成对出现，保证可靠传输

指示与确认 (Indication & Confirm)
客户端
◀── [IND] ── ─ [CFM] ──▶
服务端

服务端主动推送，需客户端确认

单向传输 (Command & Notify)
客户端
── [CMD] ──▶ ◀── [NTF] ──
服务端

无需响应，效率最高

1. 错误处理 (Error Handling)

当客户端发送的请求(Request)因权限、格式或状态错误无法被服务端处理时，服务端返回此响应包。
Opcode指令名称PDU 类型中文释义
0x01 ATT_ERROR_RSP Error Rsp 错误响应

2. MTU 交换 (MTU Exchange)

用于客户端与服务端在连接初期协商 ATT 最大传输单元 (Maximum Transmission Unit) 的大小。
Opcode指令名称PDU 类型中文释义
0x02ATT_EXCHANGE_MTU_REQRequest交换 MTU 请求
0x03ATT_EXCHANGE_MTU_RSPResponse交换 MTU 响应

3. 查找信息 (Find Information)

用于发现 GATT 服务结构，查找特定范围内的属性句柄 (Handle) 及其对应的 UUID (类型)。
Opcode指令名称PDU 类型中文释义
0x04ATT_FIND_INFORMATION_REQRequest发现信息请求
0x05ATT_FIND_INFORMATION_RSPResponse发现信息响应
0x06ATT_FIND_BY_TYPE_VALUE_REQRequest按类型与值发现属性请求
0x07ATT_FIND_BY_TYPE_VALUE_RSPResponse按类型与值发现属性响应

4. 读取属性 (Reading Attributes)

客户端通过此类指令向服务端请求获取指定句柄或特定类型的属性值。包括了 v5.2 新增的多变长属性读取功能。
Opcode指令名称PDU 类型中文释义
0x08ATT_READ_BY_TYPE_REQRequest按类型读取请求
0x09ATT_READ_BY_TYPE_RSPResponse按类型读取响应
0x0AATT_READ_REQRequest读取请求 (单句柄)
0x0BATT_READ_RSPResponse读取响应
0x0CATT_READ_BLOB_REQRequest读取长属性值请求 (分块)
0x0DATT_READ_BLOB_RSPResponse读取长属性值响应
0x0EATT_READ_MULTIPLE_REQRequest读取多个属性请求
0x0FATT_READ_MULTIPLE_RSPResponse读取多个属性响应
0x10ATT_READ_BY_GROUP_TYPE_REQRequest按组类型读取请求 (如 Primary Service)
0x11ATT_READ_BY_GROUP_TYPE_RSPResponse按组类型读取响应
0x20ATT_READ_MULTIPLE_VARIABLE_REQRequest读取多个变长属性请求 (v5.2 新增)
0x21ATT_READ_MULTIPLE_VARIABLE_RSPResponse读取多个变长属性响应 (v5.2 新增)

5. 写入属性 (Writing Attributes)

客户端通过此类指令修改服务端的属性值。包含需要响应的 Write Request 和无需响应的 Write Command。
Opcode指令名称PDU 类型中文释义
0x12ATT_WRITE_REQRequest写入请求 (需响应)
0x13ATT_WRITE_RSPResponse写入响应
0x52ATT_WRITE_CMDCommand写入命令 (无响应，俗称 Write Without Response)
0xD2ATT_SIGNED_WRITE_CMDCommand带签名的写入命令 (用于未加密链路的安全写入)

6. 队列写入 (Queued Writes)

当写入的数据长度超过 MTU 限制，或需要保证多个写入操作的原子性时，客户端会使用队列写入（分包准备，一次执行）。
Opcode指令名称PDU 类型中文释义
0x16ATT_PREPARE_WRITE_REQRequest准备写入请求
0x17ATT_PREPARE_WRITE_RSPResponse准备写入响应
0x18ATT_EXECUTE_WRITE_REQRequest执行写入请求 (提交或取消)
0x19ATT_EXECUTE_WRITE_RSPResponse执行写入响应

7. 服务端发起 (Server Initiated)

服务端属性发生变化时，主动向订阅了该属性的客户端推送数据。
Opcode指令名称PDU 类型中文释义
0x1BATT_HANDLE_VALUE_NTFNotify句柄值通知 (无需回复)
0x1DATT_HANDLE_VALUE_INDIndicate句柄值指示 (必须回复确认)
0x1EATT_HANDLE_VALUE_CFMConfirm句柄值确认 (对 Indication 的回复)
0x23ATT_MULTIPLE_HANDLE_VALUE_NTFNotify多个句柄值通知 (v5.2 新增)
📌 核心规范总结：
  • 所有的 REQ (请求) 必须得到一个 RSP (响应) 或 ERROR_RSP (错误响应)。
  • 所有的 IND (指示) 发出后，接收方必须回复一个 CFM (确认)。
  • CMD (命令) 和 NTF (通知) 为单向传输，不需要对方在 ATT 协议层进行回包确认，适合对速率要求较高的场景（如音频流、高频传感器数据）。
