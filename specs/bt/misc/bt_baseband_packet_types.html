经典蓝牙 (BR/EDR) 基带数据包类型全解析

命名小提示： 经典蓝牙的报文命名是有规律的。
• 字母 D 代表 Data（数据包），H 代表 High rate（高速率，无纠错码），M 代表 Medium rate（中速率，带纠错码），V 代表 Voice（语音包）。
• 数字 1, 3, 5 代表该数据包在空中占用的时隙数量（Slots）。占用越多，传输越快，但也越容易受干扰。
• 前缀 2- / 3- 代表这是 EDR 时代的包，使用 2Mbps 或 3Mbps 调制。

1. 通用控制与链路管理包 (Common Packets)

这类数据包不区分具体的业务通道，主要负责底层的呼叫、寻呼、保持同步、维持连接不掉线以及底层的应答机制。

包类型 包含内容 / 结构 核心作用与应用场景
ID (ID包) 仅 68 bits
只有接入码 (Access Code)，无报文头(Header)和载荷(Payload)		 寻呼与查询的“敲门砖”：用于设备发现(Inquiry)和建立连接前的寻呼(Paging)。因为非常短促，能快速唤醒或确认对方的存在。
NULL 仅有报文头 (Packet Header)，无数据载荷，
不占用重传机制		 链路维持与应答反馈：用于回传 ACK(确认收妥) 或 NAK(要求重传) 标志，以及流量控制。如果收到 NULL 包，不需要给对方回复。
POLL 仅有报文头，无载荷
(类似 NULL，但带有强制属性)		 轮询查岗：主设备(Master)发给从设备(Slave)，强制要求从设备必须在下一个时隙回复一个报文（即使从设备没数据也要回个NULL），用于防止链路假死和判断是否掉线。
FHS
(跳频同步包)		 包含 蓝牙MAC地址、设备类(Class of Device)、时钟相位信息、寻呼扫描模式等 交换名片与对齐时钟：在寻呼响应或查询响应时发送。收到FHS包后，两台设备才能对齐跳频频率(AFH)和时钟，这是物理层建链最关键的一步。
DM1 带 2/3 比例的 FEC (前向纠错)，最大装载 17 字节数据，占用 1 个时隙 控制指令承载者：这是极其核心的包，我们上文讲到的 LMP 控制指令，绝大多数都是塞在 DM1 数据包的载荷里进行传输的。

2. ACL 异步连接数据包 (Asynchronous Data Packets)

负责绝大多数普通数据的传输，如 A2DP 听高音质音乐、文件传输、控制信令 (L2CAP 协议数据)。特点是支持重传 (ARQ)，保证数据绝对不会错，但对延时没有极端的强保障。

包类型 调制方式 特性与槽位 (Slots) 应用场景与优缺点
DH1 / DH3 / DH5 BR (1 Mbps)
GFSK 调制		 无 FEC 纠错 占 1, 3, 5 时隙 大吞吐量明文传输：去掉了前向纠错以换取更大的载荷空间。DH5 封包最大可传 339 字节，适合在没有严重干扰的情况下高速传文件。
DM1 / DM3 / DM5 BR (1 Mbps) 带 2/3 FEC 占 1, 3, 5 时隙 抗干扰传输：数据中混入了冗余纠错码。如果环境很吵（Wi-Fi干扰大），底层会自动降级使用 DM 系列包，牺牲速度来换取稳定性。
2-DH1 / 2-DH3 / 2-DH5 EDR (2 Mbps)
π/4-DQPSK		 无 FEC 占 1, 3, 5 时隙
最高传输 ~1.4 Mbps		 主流音频传输：蓝牙 2.0 引入，速度翻倍。当前绝大多数蓝牙耳机听 SBC/AAC 音乐时，底层跑的都是 2-DH3 或 2-DH5 数据包。
3-DH1 / 3-DH3 / 3-DH5 EDR (3 Mbps)
8DPSK 调制		 无 FEC 占 1, 3, 5 时隙
最高传输 ~2.1 Mbps		 极速/无损音频传输：利用 8 种相位承载更高数据密度。传输 LDAC、aptX HD 等高码率无损音频文件时，底层主要依靠 3-DH5 封包保障吞吐率。

3. SCO / eSCO 同步数据包 (Synchronous Audio/Voice)

专用于 实时打电话、发微信语音 (HFP/HSP协议)。它会在信道里强行预留出固定的时间片，保证声音不卡顿。传统 SCO 不支持重传（听不清直接丢弃变杂音），较新的 eSCO 支持重传，音质大为改观。

包类型 链路类型 纠错与重传能力 功能说明
HV1 传统 SCO 1/3 FEC 无重传 极度嘈杂环境的语音保障。只有极少真实数据，其余全是用作修复错误的校验码。（现已极少使用）
HV2 / HV3 传统 SCO 2/3 / 无 FEC 无重传 传统蓝牙耳机最常用的 64kbps 语音通话通道。因绝对不允许重传，如果信号差，你听到的就是破音和“滋滋滋”的杂音。
DV 数据/语音混合包 载荷包含一段语音 + 一段数据 一边打电话一边回传少量异步数据时使用，属于过渡性产物。
EV3 / EV4 / EV5 eSCO (扩展同步) 带 FEC 或 无FEC
支持重传 (Retransmission)		 蓝牙 1.2 的革命：打破了传统 SCO 丢包即破音的窘境。EV 语音包预留了重传窗口期，如果错过了可以重发，大大降低了蓝牙耳机的杂音率。
2-EV3 / 2-EV5
3-EV3 / 3-EV5		 eSCO + EDR 高带宽 + 支持重传 高清通话 (HD Voice / mSBC)：得益于 EDR 技术的 2~3 Mbps 带宽，能够传输 16kHz 采样率的宽带语音包。现在我们用真无线耳机清晰打电话，底层的功臣就是它们。
