Section 11. 加密与认证概述 (Encryption and authentication overview)

本节作用：介绍链路层安全机制的总体架构 [1.1]。链路层（Link Layer）通过引入对称加密与消息完整性验证，保护数据物理信道和等时流物理信道的载荷。本节明确了安全保障的核心基础：

  • 核心算法：采用基于 128 位高级加密标准（AES-128）的计数器与密码块链接消息认证码模式（CCM Mode），实现兼顾机密性与完整性的认证加密（AEAD）[1.1]。
  • 参数定义：
    • M = 4：消息完整性检测（MIC）字段长度为 4 字节，用于提供数据真实性认证 [1.1]。
    • L = 2：PDU 的 Length 长度指示字段长度固定为 2 字节 [1.1]。
    • CCM Nonce：为了确保安全防范重放攻击，每次会话启动加密都需要唯一的会话密钥，且传输的每一帧 PDU 均必须使用 13 字节的唯一随机数（Nonce） [1.1]。

1.1 密码学工具箱 (Cryptographic Toolbox)

链路层安全包含若干个核心工具箱函数，用于在密钥管理、等时广播流（BIS）等高级安全场景中导出必要的临时密钥 [1.1]。

1.1.1 组会话密钥导出函数 h8

h8 算法：专门用于在加密等时流（BIS）通信中，根据广播方宣告的广播码（Broadcast_Code）导出 128 位的组会话密钥（Group Session Key, GSK）。h8 内部使用对称的 AES-CMAC 算法作为基本构建块：

GSK = h8(K, S, keyID, IK)
  • 输入：
    • K：128 位的输入密钥（通常衍生自 Broadcast_Code）。
    • S：128 位的盐值（Salt）。
    • keyID：32 位的密钥标识符。
    • IK：128 位的中间密钥（Intermediate Key）。

1.1.2 组会话密钥的导出机制 (Derivation of Group Session Key)

本节描述了如何将用户或上层提供的 Broadcast_Code，逐步通过 AES-CMAC 与盐值（使用预定义字符串如 "BIG1"、"BIG2" 等）混淆转换，最终生成 128 位的 GSK 组会话密钥。接收端只有输入相同的 Broadcast_Code 才能得出 GSK，完成对组广播音频载荷的实时解密。

Section 22. CCM 保护模式

本节作用：详细规定在物理层传输时，CCM 机制所对应的三种组成构件：Nonce、Counter 模式块及 Encryption 块。确保发送设备产生的每一帧密文包均具有强随机性，防御选择明文攻击与恶意篡改 [2.1]。

2.1 CCM 随机数 (CCM nonce)

为了保证加密的唯一性，每帧传输都需准备一个 13 字节的 Nonce [2.1]。根据规范定义，Nonce 结构如下：

  • 包计数器 (packetCounter) - 39 bits：由设备链路层维护的单调递增计数器。在加密启动之初清零，此后每加密一帧 PDU 递增 1 [2.1]。此计数器是抵御“重放攻击（Replay Attack）”的最主要屏障。
  • 方向位 (directionBit) - 1 bit：指明数据包的传输方向。
    • 若由主设备（Central）向从设备（Peripheral）传输，则设为 1；
    • 若由从设备向主设备传输，则设为 0。
  • 初始化向量 (IV) - 8 字节 (64 bits)：在连接加密协商阶段（发送 LL_ENC_REQ 和 LL_ENC_RSP 控制包）由双方协商出的伪随机初始熵，由两端设备各自提供 4 字节合并而成。

2.2 - 2.4 计数器块、加密块与会话密钥

2.2 计数器模式块 (Counter mode blocks)

在 AES-CTR（计数器模式）阶段，链路层将 13 字节的 Nonce 拼装成符合 NIST SP 800-38C 要求的 16 字节计数器块 \(A_i\)：

  • 格式：[Flags (1 字节) | Nonce (13 字节) | Counter (2 字节)]。
  • 作用：\(A_0\) 用于产生加密或验证 MIC 的核心基准块，随后的 \(A_1, A_2...\) 与递增的 Counter 共同通过 AES 内核，源源不断生成伪随机密钥流，并与 PDU 载荷进行 XOR 异或。

2.3 加密块 (Encryption blocks)

在认证阶段（CBC-MAC），格式化输入块为 \(B_0, B_1...\)：

  • 第一块 \(B_0\) 整合了 Flags 标志位、Nonce 以及即将被保护的 Payload 的精确长度字段。
  • 后续的分块 \(B_i\) 将明文 PDU 与上一步的密码块进行 CBC 链接式加密，最终得出的最后一个块作为 4 字节 MIC 完整性字段，并拼装在 Payload 的尾部发送。

2.4 会话密钥 (Session Keys)

会话密钥（Session Key，即 K）是一个 128 位的密钥，专门用于该特定连接期间的加密运算。该密钥由 Host 层下发的 LTK（长效密钥）与两端控制器（Controller）在握手期间随机产生的 SKD_C、SKD_P（会话密钥分散码）共同衍生获得，可有效杜绝重放会话的可能性。

Section 33. 确定性随机比特生成器 (DRBG) Core 6.0 核心安全更新

本节作用：在 Core 6.0 规格中，由于高精度、高防伪物理测距功能——信道探测（Channel Sounding, CS）的引入，链路层对防预测、高安全性的随机数产生提出了极高的要求。本节详细规范了符合 NIST SP 800-90Ar1 推荐、安全强度达到 128 位的确定性随机比特生成器（DRBG）及其算法模型。

该 DRBG 用于：

  • 产生信道探测专用接入地址（CS Access Address）；
  • 生成物理载荷并决定射频相位测距中微秒级的音调调制状态；
  • 作为伪随机种子参与信道探测过程（跳频算法 CSA #3）中的物理射频信道列表生成。

3.1 密码学工具箱 (Cryptographic toolbox)

定义了 DRBG 状态演进所需要的基本密码工具，涵盖从输入变换、链化处理到防追踪回溯的全部生命周期数学函数：

3.1.1 字节与比特传输顺序 (Octet and bit ordering)

规定了在调用底层 AES 分组密码算法时，128 位寄存器的多字节数据高低位映射。规定了除状态向量 \(V_{DRBG}\) 采用反向（字节 15 存储最高位、字节 0 存储最低位）以外，其余参数均按照最左字节为第 0 字节的常规传输序。

3.1.2 DRBG 链化函数 f7 (DRBG chain function f7)

使用密码分组链接（CBC）级联机制，输入 128 位 Key 以及任意长度的输入字符串，最终产生级联后的 128 位输出，该函数提供了良好的密码杂凑扩散效果。

3.1.3 DRBG 导出函数 f8 (DRBG derivation function f8)

作用：负责在握手流程开始阶段，将输入的三个安全种子 CS_IV (128 bit)、CS_IN (64 bit) 以及 CS_PV (128 bit) 组合，并通过 f7 链化，计算并导出一个 256 位的种子材料（Seed Material, SM）。这是整个测距过程随机化流程的最原始基石。

3.1.4 DRBG 更新函数 f9 (DRBG update function f9)

作用：用来刷新 DRBG 当前内存中的 128 位状态密钥 \(K_{DRBG}\) 和 128 位状态向量 \(V_{DRBG}\)。它调用安全加密函数 \(e\)（AES-128 内核），混入 256 位的种子材料（SM）以引入最新的环境熵。在无新熵输入时，该函数会以零值作为种子，更新状态参数以消除历史足迹。

3.1.5 DRBG 实例化函数 h9 (DRBG instantiation function h9)

作用：初始化 DRBG。在信道探测安全启动（CS Security Start）阶段执行。通过输入握手导出的 CS_IV、CS_IN、CS_PV，调用 $f8$ 转换成 $SM$，随后调用 $f9$ 充当内部算法参数，将全零的 \(K_{DRBG}\) 与 \(V_{DRBG}\) 成功拉起为运行时的加密密钥和状态向量：

SM = f8( CS_IV || CS_IN || CS_PV )
K_DRBG, V_DRBG = f9( SM, 0x00...00, 0x00...00 )

3.1.6 随机比特生成函数 CS_DRBG

作用：该函数是信道探测测距过程中实时被调用的核心。每次在执行 CS 测距反射步骤（Reflect Step）前，控制器都会调用该方法获取具有抗攻击、防猜测能力的随机位，用以控制信道跳变与物理射频相位，确保即使面对高算力欺骗攻击（Spoofing Attack），攻击者也无法通过预测随机序列而干扰距离估计。

3.1.7 DRBG 后向抗性 (DRBG backtracking resistance)

后向安全：为了避免设备当前的物理状态一旦被物理手段读取而泄露过去已经产生和传输的机密测距序列，DRBG 在生成完一次随机序列后，会利用 \(f9\) 并传入零输入进行强制性的自我状态更新。这确保了历史随机状态即便在设备被攻破后依然能维持极佳的不可恢复性。

3.2 基于分组密码的 DRBG 机制 (DRBG based on a block cipher)

描述了如何使用对称的分组密码技术来实现高强度的随机位。本节严格遵循了 NIST 推荐的基础方案，并通过链路层的特定电路优化保证执行的高效性。

3.2.1 确定性随机数内部工作向量 V_DRBG (DRBG nonce V)

内部的 128 位工作向量 \(V_{DRBG}\)，在算法每一次迭代生成一组随机比特流后都需要进行更新，以确保外部无法建立状态推导关系。

3.2.2 DRBG 自增方法 (DRBG nonce increment function)

规定了在两组 AES 数据处理周期之间，需要对工作向量 \(V_{DRBG}\) 进行加一自增（Modulo \(2^{128}\) 大端整数递增）的位级操作方法。该自增规则严格保证了在不引入新物理种子时，生成器的多轮迭代不会陷入局部碰撞循环中，具备完美的逻辑周期寿命。
