Section 11. 通用描述 (General Description)

本节作用：定义链路层的基础架构。它规定了链路层状态机的核心工作机制、射频物理信道的划分、设备物理与随机地址格式以及比特传输顺序的基础要求 [1.1, 1.2, 1.3, 1.4]。

1.1 链路层状态 (Link Layer States)

链路层操作可以使用一个包含了以下七种基本状态的状态机来描述 [1.1]：

  • 就绪状态 (Standby State)：不发送或接收任何包。
  • 广播状态 (Advertising State)：发送广播物理信道包，并可能监听或回应扫描请求、连接请求。
  • 扫描状态 (Scanning State)：监听并处理来自广播物理信道的广播数据。
  • 发起状态 (Initiating State)：监听对端特定广播，并对可连接的广播发起连接请求。
  • 连接状态 (Connection State)：两个已连接设备建立点对点通信（分为 Central 主机和 Peripheral 从机角色）。
  • 同步状态 (Synchronization State)：监听并同步到特定的周期性广播源。
  • 等时广播状态 (Isochronous Broadcasting State)：无需连接即可广播同步等时数据流。

1.2 比特传输顺序 (Bit Ordering)

除非另有声明，在链路层传输中采用小端字节序（Little-Endian），即最低有效位 (LSB) 首先在空中传输 [1.2]。

1.3 设备地址 (Device Address)

链路层依靠一个 48 位的设备地址对节点进行标识，地址可以分为两大类 [1.3]：

  1. 公共设备地址 (Public Device Address)：使用 IEEE 组织分配的 OUI 全球唯一组织标识符，出厂后静态不可更改。
  2. 随机设备地址 (Random Device Address)：可在运行时变更，又细分为：
    • 静态随机地址 (Static Address)：保持不变直至设备断电重启。
    • 私有解析地址 (Resolvable Private Address, RPA)：基于密钥 (IRK) 定期更新生成，用于防止第三方持续追踪。
    • 私有不可解析地址 (Non-resolvable Private Address)：完全随机，用于临时数据传输。

1.4 物理信道 (Physical Channels)

BLE 在 2.4 GHz ISM 频段共划分了 40 个 RF 信道（编号 0-39，信道带宽 2 MHz） [1.4]：

  • 主要广播物理信道 (Primary Advertising Channels)：信道 37、38 和 39，分布于 ISM 频段的不同频点上，用于建立连接、发起发现等。
  • 第二/辅助物理信道 (Secondary Channels/Data Channels)：信道 0 - 36，用于传输已连接的业务数据包、扩展广播包和等时流等。

Section 22. 空中接口数据包 (Air Interface Packets)

本节作用：详细规定了空中射频帧的数据包（Packet）和协议数据单元（PDU）的二进制格式。它定义了未编码物理层（Uncoded PHY，如 1M 和 2M）与编码物理层（Coded PHY）的数据帧解构差异 [2.1, 2.2]。

2.2 编码 PHY 数据包结构 (LE Coded PHY)

为支持远距离通信，Coded PHY 包被拆分为 FEC 编码块 [2.2]。其中：

  • 前导码 (Preamble)：10 字节。
  • FEC 编码级别：在编码指示器（CI）中指定选择 S=2 (500 kbps) 还是 S=8 (125 kbps)。
  • TERM1 与 TERM2：用于闭合卷积编码的状态转换。

2.3 链路层 PDU 类型划分

链路层 PDU 视所处物理信道不同而异 [2.3]：

  1. 广播物理信道 PDU：如传统广播（ADV_IND）、扩展广播（ADV_EXT_IND、AUX_ADV_IND）。Core 6.0 在广播物理信道中，引入了决策性广播过滤机制 (Decision-Based Advertising Filtering)，使扫描节点在解析完主信道特征后，自主判定是否有必要到辅助信道继续接收大包。
  2. 数据物理信道 PDU：当设备进入连接后在数据物理信道上交互，包含空包/数据包（LL Data PDU）和交互底层逻辑的控制包（LL Control PDU，如加密、更新物理通道信息等）。
  3. 等时物理信道 PDU：支持连接（CIS）或无连接广播（BIS）的无损/多声带音频通道。

Section 33. 比特流处理 (Bit Stream Processing)

本节作用：描述发射端和接收端链路层硬件对二进制数据流进行转换的过程。该机制可确保空中传输的可靠性、射频频谱的连贯性以及对原始报文的加密保护 [3.1, 3.2, 3.3]。

3.1 差错检验 (CRC Generation)

为了保障包传输可靠度，所有 PDU 计算产生的 24 位 CRC 校验和将被拼接在数据包的最后部，接收端采用相同的多项式线性反馈移位寄存器（LFSR）验证数据完整性 [3.1]。

3.2 数据白化 (Data Whitening)

如果连续传输大片全是“0”或全是“1”的数据流，发射信号的直流分量发生偏移，容易使接收端锁相环和AGC失真。链路层会使用生成多项式为 \(G(x) = x^7 + x^4 + 1\) 的伪随机序列发生器对 PDU 的有效载荷进行 XOR 异或白化运算，在空中形成能量分布均匀的数据谱 [3.2]。

图 3.1: 发送端物理层流处理（Bitstream Processing Chain）

3.3 加密与前向纠错 (Encryption & FEC)

  • 硬件加密：在白化之前，若使能安全加密传输，链路层会通过硬件 AES-CCM 协处理器对 PDU 整体进行对称性加密计算，并追加 4 字节消息完整性检测码 (MIC)。
  • 前向纠错（FEC）：在 Coded PHY 下，使用卷积码计算冗余位，使接收端在负信噪比恶劣工况下无需重发，直接重构恢复正确的位流 [2.2]。

Section 44. 空中接口协议 (Air Interface Protocol)

本节作用：本节是链路层的时序中心。它详细控制数据包之间的静默间隔、节点时钟误差对齐机制、状态跳转的触发时机，以及跳频算法（CSA #1 & CSA #2）的计算规则 [4.1, 4.2, 4.3, 4.4, 4.5]。

4.1 帧间隔 (Frame Spaces)

链路层协议精确定义了两个射频封包之间的空隙时序，用以完成天线收发切换及频率锁定的校正 [4.1]：

  • T_IFS (Inter-Frame Space)：帧间隔。

    从上一个包的最后一比特（位）结束，到下一个包的第一比特开始的绝对时间。Core 6.0 创新 不再固定为传统的 150 µs。通过“帧空间更新程序 (Frame Space Update)”，两个设备现在可以协商更短的间隔时间，大大提升了等时音频传输流与高速传输业务的数据吞吐，同时缩短芯片唤醒时长。

  • T_MSS (Minimum Subevent Spacing)：等时通道中子事件（Subevent）之间的最小静默间隔。
  • T_IP1 / T_IP2：Core 6.0 在物理测距 Channel Sounding（信道探测）时子事件交互所需的专用隔离期。
图 4.1: T_IFS 帧间隔与蓝牙 6.0 的动态协商概念

4.2 时钟和窗口展宽 (Clock & Window Widening)

为确保低功耗睡眠期间唤醒时仍能对齐连接，协议定义了 [4.2]：

  • 活动时钟精度 (Active Clock Accuracy)：必须在 ±50 ppm 以内。
  • 睡眠时钟精度 (Sleep Clock Accuracy)：在 ±500 ppm 以内。
  • 窗口展宽 (Window Widening)：在每次连接事件开启前，接收端由于晶振时钟偏差、多径 range delay 等多方影响，必须要提前且展宽打开接收天线窗口的算法，用以安全兜住对端的发包前导。

4.3 主动连接管理与抗干扰跳频算法

  • Channel Selection Algorithm #1 & #2：链路层内置随机跳频算法，防止 2.4 GHz 局部同频干扰（如与 Wi-Fi 信号碰撞）。CSA #2 在信道地图频繁改变和多连接场景下提供了更为出色的伪随机离散性。

Section 55. 链路层控制 (Link Layer Control)

本节作用：本节详细描述了主机控制器接口（HCI）下，链路层控制包的各种控制程序。设备通过在连接上传输 LL Control PDU 实现通道参数重构，以应对外部环境的实时剧烈改变 [5.1, 5.2]。

5.1 典型链路层控制程序 (LL Control Procedures)

  • 连接更新程序 (Connection Update Procedure)：协商修改连接间隔（Connection Interval）、从机延迟（Peripheral Latency）和超时参数，以平衡功耗与时延。
  • 信道地图更新程序 (Channel Map Update Procedure)：排除掉由于同频干扰导致的“坏信道”，更新连接时采用的跳频信道图。
  • 加密使能程序 (Encryption Start Procedure)：交换随机数，在空中建立起 AES-CCM 机密传输会话。
  • 物理层切换程序 (PHY Update Procedure)：在空中信号条件好时切换为 LE 2M 提速，信号差时切换为 LE Coded 增加覆盖深度。

5.2 Bluetooth 6.0 升级带来的全新链路控制

  1. Channel Sounding 探测协商 Core 6.0：

    用于在两个低功耗蓝牙连接设备间，执行双向的高精度、高安全性的距离测量。包含基于相位测距（MCPD）以及飞行时间（RTT）原理的测试子事件触发。

  2. 链路层扩展特性集 (LL Extended Feature Set) Core 6.0：

    随着新版规格的演进，由于增加的新特性超出了老旧机制中位掩码特征图谱的大小，6.0 版本使用新增链路控制信令（LL_FEATURE_EXT_REQ / LL_FEATURE_EXT_RSP）来交互极其广阔的扩展功能组合表。

  3. 帧空间协商 (Frame Space Update Procedure) Core 6.0：

    设备通过传输控制包 LL_FRAME_SPACE_REQ 宣告各自支持的最小帧过渡时间，实现 T_IFS 动态调优。

Section 66. 隐私机制 (Privacy)

本节作用：在不中断连接的前提下提供防追踪手段。本节详细描述了解析列表（Resolving List）和身份解析密钥（IRK）的生成、管理以及由链路层硬件进行自动解析匹配的实现 [6.1, 6.2]。

6.1 地址解析列表 (Resolving List)

低功耗蓝牙控制器在物理芯片的存储器中存储了一张地址映射表，称为“解析列表”，其中保存了已配对设备的 Identity Address（身份物理地址）和相应的 IRK（Identity Resolving Key, 身份解析密钥）。

6.2 链路层级 RPA 生成与解析

  • 地址动态更新：启用隐私保护的设备定期在广播或包头中使用随机私有可解析地址（RPA）。其前 24 位为随机数（prand），后 24 位是用私有密钥（IRK）对 prand 进行单向哈希得到的 Hash 值。
  • 全硬件无缝筛选：对端接收设备收到该 RPA 包后，无需唤醒其 Host（主机应用处理器），直接在控制器链路层（Link Layer）内部通过查找解析列表中的所有 IRK 密钥，并对其进行哈希比对。若匹配，则还原出真实地址，从而避免唤醒主机，极大节省能耗。

Section 77. ISO 测试模式 (ISO Test Mode)

本节作用：规范无线电射频实验室对等时传输通道（等时流，主要用于 LE Audio 高清音频）的测量标准。定义了测试端与被测设备（DUT）之间的数据测试序列生成机制 [7.1, 7.2]。

7.1 ISO 发送测试 (ISO Transmit Test)

处于等时发送测试状态下的设备，其链路层需要以设定的等时传输时间间隔、指定的负载序列（例如：全“1”、全“0”、PRBS9 伪随机序列）持续不断生成数据流，用以检验射频前端的频偏漂移、相位噪声精度。

7.2 ISO 接收测试与指标计算 (ISO Receive Test)

接收设备接收空中发送的测试帧，并进行解码统计，通过内置计数器提供以下几个指标的记录：

  • Received_SDU_Count：正确接收并成功重构的等时服务数据单元（SDU）个数。
  • Missed_SDU_Count：由于信噪比差、多径反射导致包丢失或 MIC 错误导致未能解出来的 SDU 丢失数。
  • Failed_SDU_Count：收到了包，但由于 CRC 异常或序列包计数器对不齐而被标记为接收损毁的个数。

Section 88. 参考文献 (References)

本节作用：指向实现链路层研究所不可或缺的外部标准引用。这构成了整个协议可复现性的法律及工程基础框架。

  • Core Specification Supplement (CSS)：核心规范补充。它规定了广播数据段的常用 AD Type（如服务UUID、本地名称、发射功率等级）的字节排布结构。
  • FIPS PUB 197 (Advanced Encryption Standard)：美国国家标准技术研究所的 128 位对称 AES 安全架构规范，支持链路层硬件保密加密机制。
  • IEEE 802.15.4：作为蓝牙在短距离无线个人局域网发展历程上的工业频段规划和物理层天线共存参考标准。
