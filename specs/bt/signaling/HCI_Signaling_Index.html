主机控制器接口 (HCI) 传输帧与编址结构

主机控制器接口 (HCI) 介于主机层 (Host) 与射频控制器 (Controller) 之间。本页为单文件离线版（数据已内嵌，无需加载外部 JS）。内容依据 Core 6.0 HCI 规范第 7 章 与第 3 章总表，覆盖链路控制、链路策略、控制器基带、信息/状态、测试、LE 命令及标准/LE Meta 事件。左侧索引为两级分组：一级按 OGF / 事件类型，二级按 §7 业务场景（如链路控制下的「连接建立与管理」「认证与加密」等）。更新数据可运行 python3 docs/gen_hci_signaling.py 后重新内联。

HCI 命令包结构 (Command Packet)

包含两字节操作码 Opcode (16-bit) 与一字节参数总长度。Opcode 由两部分组成：低 10 bits 为 OCF (操作码命令字段)，高 6 bits 为 OGF (操作码组字段)。公式为：Opcode = (OGF << 10) | OCF。

HCI 事件包结构 (Event Packet)

包含一字节事件码 Event Code (8-bit) 与一字节参数总长度。低功耗蓝牙 (LE) 的事件被统一归纳在 Event Code 为 0x3E 的 LE Meta Event 中，通过首字节的 Subevent Code 区分各个具体低功耗状态事件。
