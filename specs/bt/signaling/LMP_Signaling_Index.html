蓝牙 Core 6.0 规范 - LMP PDU 传输机制简述

LMP 消息（协议数据单元，PDU）通常使用 DM1 物理层分组包进行传输，以保证可靠的数据传输控制，当物理链路存在语音业务（如 HV1）且载荷小于 9 字节时，也可以使用 DV 数据语音包传输。所有的 LMP 操作以事务（Transactions）形式进行，其信令帧格式由第一个字节决定。本索引与 Bluetooth Core 6.0 — Link Manager Protocol Specification 中 Table 5.1（PDU 汇总）及 §5.2（参数定义）对齐，共 88 条 LMP PDU。

7-bit 基础操作码 (Standard Opcode)

首字节低 1 位为事务 ID（TID，Central 发起为 0，Peripheral 发起为 1），高 7 位为操作码（值 0 ~ 123）。参数排布从 payload 的第 2 字节开始。

15-bit 扩展操作码 (Extended Opcode)

当首字节高 7 位包含转义码（Escape Opcode，固定为 127）时，第二字节将用于表示扩展操作码（值 1 ~ 37）。参数排布自第 3 字节开始。
