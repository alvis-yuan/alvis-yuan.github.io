第 0 章

AWK 简介与历史

AWK 是一种专为文本处理设计的编程语言与命令行工具，名字来源于三位创始人的姓氏首字母： Alfred Aho、Peter Weinberger 和 Brian Kernighan， 他们于 1977 年在贝尔实验室（Bell Laboratories）共同设计了这一语言。

GNU 项目的实现称为 gawk（GNU awk），它遵循 POSIX 1003.1 标准，并提供大量 GNU 扩展功能。 在大多数 Linux 发行版中，awk 命令默认指向 gawk。

📌 权威来源

本文内容严格依据 GNU AWK 用户手册（gawk manual） 与 gawk(1) Linux man page（March 2026） 编写。

AWK 的核心设计哲学

引用 GNU AWK 官方文档原文：

官方原文

"Programs in awk are different from programs in most other languages, because awk programs are data driven (i.e., you describe the data you want to work with and then what to do when you find it). Most other languages are procedural; you have to describe, in great detail, every step the program should take."
— GNU AWK User's Guide: Getting Started

AWK 能做什么？

逐行扫描文件 按分隔符切分字段 正则模式匹配 字段提取与重组 数学运算与统计 格式化输出报表 关联数组处理 调用外部命令

📌 章节小结

  • AWK 诞生于 1977 年贝尔实验室，是经过近 50 年检验的成熟工具
  • GNU gawk 是最广泛使用的实现，遵循 POSIX 标准并有扩展
  • 核心设计哲学：数据驱动（data-driven），描述"对什么数据做什么"，而非"每一步怎么做"
🔬 第一性原理

AWK 的本质是：模式匹配驱动的流式文本处理器。它将"规则化文本扫描"与"轻量编程语言"合二为一， 使得结构化文本的提取、变换与聚合可以在一行命令内完成，无需编写完整程序。

第 1 章

执行模型与工作原理

理解 AWK 的执行模型是掌握所有使用场景的基础。官方 man page 描述了完整的执行顺序：

官方描述（gawk man page）

"Gawk executes AWK programs in the following order. First, all variable assignments specified via the -v option are performed. Next, gawk compiles the program into an internal form. Then, gawk executes the code in the BEGIN rule(s) (if any), and then proceeds to read each file named in the ARGV array... For each record in the input, gawk tests to see if it matches any pattern in the AWK program. For each pattern that the record matches, gawk executes the associated action."
— gawk(1) Linux man page

AWK 程序的基本结构

引用 gawk man page 原文：

# AWK 程序由若干规则和可选的函数定义组成
# 每条规则形如：  pattern { action }

pattern { action statements }
pattern { action statements }
function name(parameter list) { statements }
关键规则

• 缺少 pattern：对每一条记录执行 action
• 缺少 action：等价于 { print }，打印整条记录
• pattern 与 action 不能同时缺失

基本语法

awk [options] 'program' file ...
awk [options] -f program-file file ...

📌 章节小结

  • AWK 的执行顺序固定：-v 赋值 → 编译 → BEGIN → 逐行处理（匹配+动作）→ END
  • "记录（record）"默认是一行，"字段（field）"默认以空白分隔
  • 程序结构为 pattern { action } 对的序列，模式控制"何时"，动作控制"做什么"
🔬 第一性原理

AWK 的执行模型本质是一个无限循环的规则引擎：读取 → 分割 → 匹配测试 → 触发动作 → 重复。 这个"输入驱动的事件循环"使程序员只需声明规则，免去手工管理循环迭代的复杂性。

第 2 章

记录与字段分割

引用 gawk man page：

官方原文

"Normally, records are separated by newline characters. You can control how records are separated by assigning values to the built-in variable RS."
"As each input record is read, gawk splits the record into fields, using the value of the FS variable as the field separator."
— gawk(1) man page

字段引用规则

引用方式含义示例
$0整条记录（含所有字段）print $0
$1, $2, ...第 N 个字段print $1, $3
$NF最后一个字段print $NF
$(NF-1)倒数第二个字段print $(NF-1)

场景示例

场景 1：提取指定列

# 打印 /etc/passwd 第1、7字段（用户名和登录shell）
awk -F: '{ print $1, $7 }' /etc/passwd

# 打印每行最后一个字段
awk '{ print $NF }' file.txt

# 打印倒数第二个字段
awk '{ print $(NF-1) }' file.txt

场景 2：自定义记录分隔符（RS）

# 将分号作为记录分隔符（处理单行多记录数据）
awk 'BEGIN { RS=";" } { print NR, $0 }' data.txt

# 空行作为记录分隔符（处理段落）
awk 'BEGIN { RS="" } { print NR, $0 }' paragraphs.txt

📌 章节小结

  • AWK 以 RS（默认换行）分割输入为记录，以 FS（默认空白）分割记录为字段
  • 字段通过 $1～$NF 引用，$0 代表整行
  • 修改字段值会触发 $0 的重新构建，字段间以 OFS 拼接
第 3 章

模式匹配

引用 gawk man page：

官方原文

"AWK patterns may be one of the following: BEGIN / END / BEGINFILE / ENDFILE / regular expression / relational expression / pattern && pattern / pattern || pattern / pattern ? pattern : pattern / ( pattern ) / ! pattern / pattern1, pattern2"
— gawk(1) man page

3.1 正则表达式模式（/regex/）

# 打印包含 "Apple" 的所有行
awk '/Apple/' sample.txt

# 打印以数字开头的行
awk '/^[0-9]/' sample.txt

# 不含 "error" 的行（否定匹配）
awk '!/error/' logfile.txt

3.2 关系表达式模式

# 第3字段大于50的行
awk '$3 > 50 { print $1, $3 }' teams.txt

# 第2字段包含 "ia" 的行（字段级正则）
awk '$2 ~ /ia/ { print $1 }' teams.txt

# 第5字段不匹配 "bash" 的行
awk -F: '$7 !~ /bash/ { print $1 }' /etc/passwd

3.3 范围模式（pattern1, pattern2）

官方说明

"The pattern1, pattern2 form of an expression is called a range pattern. It matches all input records starting with a record that matches pattern1, and continuing until a record that matches pattern2, inclusive."
— gawk(1) man page

# 打印 "Raptors" 行到 "Celtics" 行之间的所有记录
awk '/Raptors/,/Celtics/ { print $1 }' teams.txt

# 打印第5行到第10行
awk 'NR==5, NR==10 { print NR, $0 }' file.txt

# 提取 XML/HTML 标签间内容
awk '//,/<\/end>/' data.xml

3.4 复合模式（&& || !）

# 第3字段>50 且 第4字段<30
awk '$3 > 50 && $4 < 30 { print $1 }' teams.txt

# 匹配 "Apple" 或 "Google" 的行
awk '/Apple/ || /Google/' sample.txt

# 多条件组合
awk 'NR > 1 && $3 != "" && $5 > 0.5 { print $1 }' teams.txt

📌 章节小结

  • AWK 支持六大类模式：BEGIN/END、正则、关系表达式、范围、复合、无模式
  • ~ 和 !~ 用于字段级正则匹配，是 AWK 独有的运算符
  • 范围模式会保持"激活状态"直到第二个模式匹配，实现连续行提取
  • 无模式规则等价于"对每一行都执行"
第 4 章

BEGIN 与 END 特殊模式

官方原文（gawk man page）

"BEGIN and END are two special kinds of patterns which are not tested against the input. The action parts of all BEGIN patterns are merged as if all the statements had been written in a single BEGIN rule. They are executed before any of the input is read. Similarly, all the END rules are merged, and executed when all the input is exhausted (or when an exit statement is executed)."
— gawk(1) man page

4.1 BEGIN 的典型用途

场景：初始化变量、设置分隔符、打印表头

# 官方示例：打印并排序 /etc/passwd 中的登录名
# 来源：gawk(1) man page EXAMPLES 章节
awk 'BEGIN { FS = ":" }
     { print $1 | "sort" }' /etc/passwd

# 打印 CSV 报表，设置表头
awk -F, 'BEGIN {
    OFS = "\t"
    print "姓名\t得分\t等级"
}
NR > 1 {
    if ($2 >= 90) grade = "A"
    else if ($2 >= 60) grade = "B"
    else grade = "C"
    print $1, $2, grade
}
END { print "共处理", NR-1, "条记录" }' scores.csv

4.2 END 的典型用途

场景：汇总统计、计算最终结果、生成报告

# 官方示例：统计文件行数
# 来源：gawk(1) man page EXAMPLES 章节
awk '{ nlines++ }
END { print nlines }' file.txt

# 统计第3列的总和、平均值、最大值
awk 'BEGIN { max = -999999 }
{
    sum += $3
    count++
    if ($3 > max) max = $3
}
END {
    printf "总和: %.2f\n", sum
    printf "平均: %.2f\n", sum/count
    printf "最大: %.2f\n", max
}' data.txt

4.3 gawk 扩展：BEGINFILE 与 ENDFILE

gawk 扩展

"BEGINFILE and ENDFILE are additional special patterns whose actions are executed before reading the first record of each command-line input file and after reading the last record of each file."
— gawk(1) man page

# 处理多个文件，每个文件分别统计行数
gawk 'BEGINFILE { count = 0 }
{ count++ }
ENDFILE { printf "%s: %d 行\n", FILENAME, count }' *.log

📌 章节小结

  • BEGIN：在读取任何输入前执行，用于初始化变量、设置分隔符、打印表头
  • END：在所有输入耗尽后执行，用于汇总、计算、打印最终结果
  • 多个 BEGIN/END 块会被合并执行，顺序与源码一致
  • gawk 扩展的 BEGINFILE/ENDFILE 可针对每个文件执行初始化和收尾操作
第 5 章

内置变量

AWK 提供了一组内置变量，在程序运行过程中自动维护。以下来自 gawk man page 与官方手册：

变量说明示例
NR当前已读入的总记录数（行号，跨文件累计）awk '{print NR, $0}'
FNR当前文件中的记录号（每个新文件重置为1）awk '{print FNR, $0}' a.txt b.txt
NF当前记录的字段总数awk '{print NF}'
FS输入字段分隔符（默认空白）awk 'BEGIN{FS=":"}'
RS输入记录分隔符（默认换行）awk 'BEGIN{RS=";"}'
OFS输出字段分隔符（默认空格）awk 'BEGIN{OFS=","}{print $1,$2}'
ORS输出记录分隔符（默认换行）awk 'BEGIN{ORS=";"}{print}'
FILENAME当前处理的文件名awk '{print FILENAME, NR}'

实战示例

# 官方示例：用行号拼接多文件
# 来源：gawk(1) man page EXAMPLES
awk '{ print NR, $0 }' file1.txt file2.txt   # NR 全局累计
awk '{ print FNR, $0 }' file1.txt file2.txt  # FNR 每文件重置

# OFS 控制输出格式：将空格分隔转为 CSV
awk 'BEGIN { OFS="," } { $1=$1; print }' data.txt

# 用 ENVIRON 读取环境变量
awk 'BEGIN { print ENVIRON["HOME"] }'

# NF 过滤：只打印有5个字段的行
awk 'NF == 5' data.txt

📌 章节小结

  • 内置变量是 AWK 与处理引擎之间的"通信接口"，无需声明即可使用
  • NR vs FNR：前者全局累计，后者文件内重置，处理多文件时要注意区分
  • 修改 $1=$1（貌似无意义的自赋值）会触发 $0 用 OFS 重建，常用于格式转换
第 6 章

print 与 printf 输出

6.1 print 语句

官方说明（linuxize.com AWK Guide）

"When printing multiple items, you need to separate them with commas. The printed items are separated by single spaces. If you do not use commas, there will be no space between the items. When print is used without an argument, it defaults to print $0, printing the current record."
— linuxize.com AWK Command Guide

# 逗号分隔 → 输出用 OFS（默认空格）连接
awk '{ print $1, $2 }' file.txt     # 输出: John Milwaukee

# 无逗号 → 直接拼接（无空格）
awk '{ print $1 $2 }' file.txt     # 输出: JohnMilwaukee

# 打印字面文本
awk '{ print "用户:", $1, "| 行号:", NR }' file.txt

# 无参数 print 等同于 print $0
awk '/error/ { print }' logfile.txt

6.2 printf 语句

说明

"The printf statement gives you more control over the output format. Unlike print, printf does not automatically add a newline after each output."
— linuxize.com AWK Command Guide

格式符含义示例
%s字符串printf "%s\n", $1
%d整数printf "%d\n", $3
%f浮点数printf "%.2f\n", $5
%x十六进制printf "%x\n", 255
%-10s左对齐，宽度10printf "%-10s %5d\n", $1,$3
# 格式化对齐输出表格
awk 'BEGIN {
    printf "%-12s %-12s %5s %5s\n", "队名", "城市", "胜", "负"
    printf "%s\n", "------------------------------------"
}
{ printf "%-12s %-12s %5d %5d\n", $1, $2, $3, $4 }' teams.txt

# 输出 CSV 格式
awk -F: 'BEGIN { printf "用户名,UID,主目录\n" }
{ printf "%s,%s,%s\n", $1, $3, $6 }' /etc/passwd

6.3 输出重定向

# 重定向到文件
awk '{ print $1 > "output.txt" }' input.txt

# 追加到文件
awk '{ print $0 >> "log.txt" }' input.txt

# 管道给外部命令
awk '{ print $1 | "sort -u" }' input.txt

# 官方示例：打印排序后的登录名
awk 'BEGIN { FS=":" } { print $1 | "sort" }' /etc/passwd

📌 章节小结

  • print 自动换行，逗号分隔的项用 OFS 连接；printf 不自动换行，提供格式化控制
  • AWK 的输出可以重定向到文件（>、>>）或通过管道（|）传给命令
  • printf 格式符与 C 语言 printf 基本一致
第 7 章

字段分隔符控制

引用 GNU AWK 官方手册的字段分割总结表：

官方原文（GNU AWK User's Guide 4.14 Summary）

FS 值的不同分割行为：
FS == " "：按空白序列分割（默认行为）
FS == 任意单字符：按该字符分割
FS == 正则表达式：按正则匹配分割
FIELDWIDTHS == 列宽列表：按固定宽度分割（gawk 扩展）
FPAT == 正则：按字段内容（而非分隔符）来定义字段（gawk 扩展）
— GNU AWK User's Guide

场景示例

# -F 选项等同于 BEGIN{FS=...}
awk -F: '{ print $1, $6 }' /etc/passwd     # 冒号分隔
awk -F, '{ print $1, $3 }' data.csv         # 逗号分隔
awk -F'\t' '{ print $2 }' data.tsv          # Tab 分隔

# 多字符/正则分隔符
awk -F'[,;|]' '{ print $1 }' data.txt        # 逗号或分号或管道符
awk -F'::' '{ print $1 }' data.txt           # 双冒号

# gawk 扩展：FPAT 处理带引号的 CSV
# 来源：linuxize.com AWK Command Guide
gawk 'BEGIN { FPAT = "([^,]*)|(\"[^\"]*\")" }
{ print $1, $3 }' quoted.csv

# OFS 控制输出分隔
awk -F: 'BEGIN { OFS="\t" } { print $1, $3, $6 }' /etc/passwd

📌 章节小结

  • -F 选项与 BEGIN{FS=...} 等效，但 -F 更简洁
  • FS 为单个空格（默认）时有特殊行为：忽略首尾空白、多空格视为一个
  • FS 设为正则表达式时可实现复杂分割，例如 FS="[,;|]"
  • gawk 的 FPAT 变量通过"字段内容的正则"而非"分隔符"来定义字段，专为复杂 CSV 设计
第 8 章

控制流语句

官方原文（gawk man page）

"The control statements are as follows: if ( condition ) statement [ else statement ] / while ( condition ) statement / do statement while ( condition ) / for ( init; condition; incr ) statement / for ( var in array ) statement / break / continue / delete / exit / next / nextfile / return expr / switch"
— gawk(1) man page

8.1 if / else

# 根据成绩分类
awk '{ 
    if ($4 >= 6000) category = "高级"
    else if ($4 >= 5000) category = "中级"
    else category = "初级"
    print $1, category
}' employees.txt

# 带 else 的字段过滤
awk '$3 > 50 { print $1, "达标" }
       $3 <= 50 { print $1, "未达标" }' teams.txt

8.2 for 循环

# C 风格 for：打印每行的所有字段
awk '{ 
    for (i = 1; i <= NF; i++)
        printf "[%s] ", $i
    print ""
}' data.txt

# for...in 遍历数组（后详述）
awk '{ freq[$1]++ }
END {
    for (word in freq)
        print word, freq[word]
}' text.txt

8.3 while / do-while

# while 循环：逐字段打印
awk '{ 
    i = 1
    while (i <= NF) {
        print i, $i
        i++
    }
}' data.txt

8.4 next —— 跳过当前记录

# 跳过空行和注释行（#开头）
awk '/^$/ || /^#/ { next }
{ print NR, $0 }' config.txt

# 跳过表头（NR==1）再处理数据
awk 'NR == 1 { next }
{ sum += $3 }
END { print "总和:", sum }' data.csv

8.5 exit —— 提前退出

# 找到第一条匹配就退出
awk '/ERROR/ { print "首个错误:", $0; exit }' logfile.txt

# exit 后仍会执行 END 块
awk '/CRITICAL/ { found=1; exit }
END { 
    if (found) print "发现严重错误"
    else print "无严重错误"
}' logfile.txt

📌 章节小结

  • AWK 控制流与 C 语言高度相似：if/else、for、while、do-while
  • next：跳过当前记录，进入下一记录处理（类似 continue 但针对记录级别）
  • exit：立即停止读取输入，但仍会执行 END 块
第 9 章

字符串函数

官方来源

以下函数均来自 gawk(1) man page — String Functions 章节 及 linuxize.com AWK Guide。

函数说明示例
length(s)返回字符串长度；无参则为 $0 长度length($1)
substr(s,i[,n])从位置 i 提取 n 个字符（位置从1开始）substr($1,1,3)
index(s,t)返回 t 在 s 中的首次出现位置，0表示未找到index($0,"error")
match(s,r)在 s 中匹配正则 r，返回位置，0为未找到；设置 RSTART 和 RLENGTHmatch($0,/[0-9]+/)
sub(r,s[,t])将 t 中第一个匹配 r 的子串替换为 s（默认 $0）sub(/foo/,"bar",$1)
gsub(r,s[,t])将 t 中所有匹配 r 的子串替换为 sgsub(/[aeiou]/,"")
split(s,a[,r])按分隔符 r 拆分字符串 s 到数组 a，返回元素数split($0,arr,":")
sprintf(fmt,...)格式化字符串并返回（不打印）sprintf("%.2f",$3)
tolower(s)转为小写tolower($1)
toupper(s)转为大写toupper($1)

场景示例

# gsub：全局替换（处理日志中的IP脱敏）
awk '{ gsub(/[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+/, "x.x.x.x"); print }' access.log

# sub：替换每行第一个 ERROR 为 [ERROR]
awk '{ sub(/ERROR/, "[ERROR]"); print }' logfile.txt

# substr：提取日期字段中的年份
awk '{ year = substr($1, 1, 4); print year, $2 }' dates.txt

# split：拆分冒号分隔字段
awk '{ 
    n = split($0, parts, ":")
    for (i=1; i<=n; i++) print i, parts[i]
}' data.txt

# 去除首尾空格（AWK 中的 trim 技巧）
awk '{ gsub(/^[ \t]+|[ \t]+$/, ""); print }' data.txt

# 统计行内单词数量
awk '{ 
    n = split($0, words, /[[:space:]]+/)
    print NR, "共", n, "个单词"
}' text.txt

📌 章节小结

  • sub 替换首个匹配，gsub 替换所有匹配，替换目标默认为 $0
  • AWK 字符串位置从 1 开始（与 C/Python 不同），substr($0,1,4) 取前4字符
  • gawk 中字符串操作按字符数计算（多字节感知），而非字节数
第 10 章

关联数组

官方原文（GNU AWK User's Guide）

"AWK's one-and-only data structure: the associative array. Deleting array elements and whole arrays is described, as well as sorting arrays in gawk. The chapter also describes how gawk provides arrays of arrays."
— GNU AWK User's Guide: This Manual

AWK 数组是关联数组（Associative Array），即以字符串为下标的哈希表。 无需声明，直接赋值即可创建。

10.1 基本操作

# 赋值与访问
awk 'BEGIN {
    arr["apple"] = 5
    arr["banana"] = 12
    print arr["apple"]    # 输出: 5
}'

# 检查键是否存在（用 in 操作符）
awk 'BEGIN {
    arr["x"] = 1
    if ("x" in arr) print "存在"
    if (!("y" in arr)) print "不存在"
}'

# 删除元素
awk 'BEGIN {
    arr[1] = "a"; arr[2] = "b"
    delete arr[1]       # 删除单个元素
    delete arr          # 删除整个数组
}'

10.2 经典场景：词频统计

# 统计每个单词出现次数
# 来源：linuxize.com AWK Command Guide
awk '{ for (i=1; i<=NF; i++) freq[$i]++ }
END { for (word in freq) print word, freq[word] }' text.txt | sort -rn -k2

10.3 去重（利用数组键唯一性）

# 去除重复行，保持原顺序
# 来源：linuxize.com AWK Command Guide
# "This works by using the current line as an array key."
awk '!seen[$0]++' file.txt

# 原理：第一次见到某行时，seen[$0]为0(假)，!0=真，打印；
# 再次见到时，seen[$0]已为非零(假)，!非零=假，不打印

10.4 多维数组（模拟）

# AWK 用 SUBSEP 模拟多维数组
awk '{ matrix[$1][$2] += $3 }
END {
    for (r in matrix)
        for (c in matrix[r])
            print r, c, matrix[r][c]
}' data.txt   # gawk 支持真正的多维数组

# 统计各产品各月份销量
awk -F, 'NR>1 { sales[$2][$1] += $3 }
END {
    for (product in sales)
        for (month in sales[product])
            printf "%-10s %s: %d\n", product, month, sales[product][month]
}' sales.csv

📌 章节小结

  • AWK 只有一种数据结构：关联数组（哈希表），键为字符串，值为字符串或数字
  • 用 key in array 检查键是否存在，避免访问不存在键时创建空元素
  • !seen[$0]++ 是 AWK 去重的惯用写法，利用了数组键唯一性和自增副作用
  • gawk 支持真正的多维数组（arrays of arrays）
🔬 第一性原理

关联数组是 AWK 强大统计能力的核心。它将"记录维度"（行）与"统计维度"（键）解耦，使得 分组聚合（group by）、去重（distinct）、频率统计（count） 等操作仅需一行代码即可实现。

第 11 章

自定义函数

官方原文（gawk man page）

"Functions in AWK are defined as follows: function name(parameter list) { statements }. Functions execute when they are called from within expressions in either patterns or actions. Arrays are passed by reference, other variables are passed by value."
— gawk(1) man page

函数定义语法

function name(param1, param2,    # 局部变量用额外空格隔开
             local_var1, local_var2) {
    # 函数体
    return value
}
局部变量约定

AWK 没有专用的局部变量声明关键字。惯例是在参数列表末尾用多个空格分隔，将局部变量作为"额外参数"声明：
function f(p, q, a, b) # a 和 b 是局部变量
— gawk(1) man page

场景示例

# 定义最大值函数
function max(a, b) {
    return (a > b) ? a : b
}

# 定义 trim 函数（去首尾空格）
function trim(s) {
    gsub(/^[ \t]+|[ \t]+$/, "", s)
    return s
}

# 递归函数：计算阶乘
function factorial(n) {
    if (n <= 1) return 1
    return n * factorial(n - 1)
}

# 使用函数
awk '
function max(a, b) { return (a > b) ? a : b }
function trim(s)   { gsub(/^[ \t]+|[ \t]+$/, "", s); return s }
{
    print trim($1), max($3, $4)
}' data.txt

gawk 扩展：间接函数调用

# 通过字符串变量间接调用函数（gawk 扩展）
# 来源：gawk(1) man page
awk '
function myfunc() { print "myfunc called" }
{
    the_func = "myfunc"
    @the_func()    # 通过变量 the_func 调用 myfunc
}'

📌 章节小结

  • AWK 函数支持递归，数组按引用传递，其他变量按值传递
  • 局部变量通过"额外参数+空格分隔"的惯例实现
  • gawk 支持通过 @变量名() 的间接函数调用
  • 函数定义可出现在程序任何位置（不需要先定义再使用）
第 12 章

实战：日志分析

日志分析是 AWK 最高频的使用场景之一。以下示例基于 linuxblog.io 实践指南 整理。

12.1 统计 HTTP 状态码

# Nginx 访问日志：统计各 HTTP 状态码出现次数
# 状态码通常在第9字段
awk '{ status[$9]++ }
END {
    printf "%-10s %s\n", "状态码", "次数"
    for (code in status)
        printf "%-10s %d\n", code, status[code]
}' /var/log/nginx/access.log | sort

# 统计 Top 10 最频繁请求的 URL（第7字段）
awk '{ urls[$7]++ }
END { for (u in urls) print urls[u], u }' access.log \
  | sort -rn | head -10

12.2 SSH 暴力破解检测

# 统计 SSH 登录失败 Top 20 IP
# 来源：linuxblog.io AWK Practical Guide
awk '/Failed password/ { print $(NF-3) }' /var/log/auth.log \
  | sort | uniq -c | sort -rn | head -20

# 同时统计失败时间分布（按小时）
awk '/Failed password/ {
    split($3, t, ":")
    hour[t[1]]++
}
END {
    for (h in hour) printf "%s 时: %d 次\n", h, hour[h]
}' /var/log/auth.log | sort

12.3 系统日志错误提取

# 提取 syslog 中 ERROR 和 WARN 行，带行号
awk '/ERROR|WARN/ { printf "[%d] %s\n", NR, $0 }' /var/log/syslog

# 提取特定时间段的日志（范围模式）
awk '/^Dec 10 08:00/,/^Dec 10 09:00/ { print }' /var/log/syslog

# 统计每个服务的错误数量
awk '/ERROR/ { service[$5]++ }
END { for (s in service) print s, service[s] }' /var/log/syslog \
  | sort -rn -k2

📌 章节小结

  • 日志分析中常用的模式：正则过滤 + 关联数组计数 + END输出排名
  • $(NF-3) 等动态字段引用是处理字段数不固定日志的关键技巧
  • AWK 配合 sort | uniq -c | sort -rn | head 管道是日志统计的经典组合
第 13 章

实战：系统管理

13.1 处理 /etc/passwd

# 官方示例：打印并排序所有登录名
# 来源：gawk(1) man page EXAMPLES 章节
awk 'BEGIN { FS = ":" } { print $1 | "sort" }' /etc/passwd

# 列出使用 /bin/bash 的用户
# 来源：linuxize.com AWK Command Guide
awk -F: '$7 == "/bin/bash" { print $1 }' /etc/passwd

# 格式化输出用户信息
awk -F: 'BEGIN { printf "%-15s %-5s %-5s %s\n","用户名","UID","GID","主目录" }
{ printf "%-15s %-5s %-5s %s\n", $1, $3, $4, $6 }' /etc/passwd

13.2 进程监控

# 显示内存占用最高的 Top 10 进程
# 来源：linuxblog.io AWK Practical Guide
# Field 4 from ps aux is %MEM, field 11 is the command
ps aux | awk 'NR>1 { print $4, $11 }' | sort -rn | head -10

# 统计各用户的进程数
ps aux | awk 'NR>1 { user[$1]++ }
END { for (u in user) print u, user[u] }' | sort -rn -k2

13.3 磁盘监控

# 打印使用率超过80%的文件系统
# 来源：linuxblog.io AWK Practical Guide
df -h | awk 'NR>1 {
    sub(/%/, "", $5)       # 去掉 % 符号
    if ($5+0 > 80) print $6, $5"%"
}'

# 统计目录下各类型文件的总大小
ls -la | awk 'NR>1 && $1 !~ /^d/ {
    ext = $9
    gsub(/.*\./, "", ext)   # 提取扩展名
    size[ext] += $5
}
END {
    for (e in size)
        printf "%-10s %d bytes\n", e, size[e]
}'

13.4 网络监控

# 统计各 IP 的 ESTABLISHED 连接数
# 来源：the.hosting AWK Guide
netstat -an | awk '/ESTABLISHED/ { count[$5]++ }
END { for (ip in count) print ip, count[ip] }' | sort -rn -k2

# 统计各端口监听情况
ss -tlnp | awk 'NR>1 { print $4 }' | sort -u

📌 章节小结

  • AWK 可直接处理 /etc/passwd、ps aux、df -h、netstat 等系统命令输出
  • 关键技巧：NR>1 跳过表头，sub(/%/,"",$5) 清理格式字符
  • 系统管理场景大量使用"命令 | awk 过滤/统计"的管道模式
第 14 章

实战：数据统计与报表

14.1 列求和、均值、最大/最小值

# 多统计量一次完成
awk 'BEGIN { min = 999999; max = -999999 }
{
    sum += $3
    count++
    if ($3 > max) max = $3
    if ($3 < min) min = $3
}
END {
    printf "记录数: %d\n", count
    printf "总  和: %.2f\n", sum
    printf "平  均: %.2f\n", sum / count
    printf "最  大: %.2f\n", max
    printf "最  小: %.2f\n", min
}' data.txt

14.2 分组聚合（GROUP BY）

# 按产品分组统计销售额
# 来源：the.hosting AWK Guide
awk -F, 'BEGIN { print "产品 | 总销售额" }
NR > 1 { total[$2] += $3 * $4 }
END {
    for (p in total)
        printf "%-10s | ¥%.2f\n", p, total[p]
}' sales.csv | sort

14.3 生成柱状图（ASCII）

# 用 AWK 生成简单的 ASCII 柱状图
awk '{
    bar = ""
    for (i = 0; i < $2; i++) bar = bar "#"
    printf "%-15s |%s (%d)\n", $1, bar, $2
}' counts.txt

14.4 数据透视（交叉统计）

# 统计每个部门的员工薪资分布（按薪资区间）
awk 'NR>1 {
    dept = $2
    salary = $4
    if (salary >= 10000) bracket = "高"
    else if (salary >= 6000) bracket = "中"
    else bracket = "低"
    matrix[dept][bracket]++
}
END {
    printf "%-10s %5s %5s %5s\n", "部门", "高", "中", "低"
    for (d in matrix)
        printf "%-10s %5d %5d %5d\n", d,
               matrix[d]["高"]+0, matrix[d]["中"]+0, matrix[d]["低"]+0
}' employees.csv

📌 章节小结

  • 统计场景的通用模式：主体循环累加 + END 块输出结果
  • 分组聚合使用"关联数组下标 = 分组键"的模式，等价于 SQL 的 GROUP BY
  • 多维关联数组可实现数据透视（pivot table）效果
第 15 章

实战：CSV 与结构化数据

15.1 基本 CSV 处理

# 简单 CSV（字段不含逗号）：设置 FS=","
# 来源：linuxize.com AWK Command Guide
awk -F, 'NR>1 { print $1, $3 }' data.csv

# 跳过表头并计算总金额
awk -F, 'NR>1 { total += $3 }
END { printf "总金额: ¥%.2f\n", total }' orders.csv

15.2 带引号的复杂 CSV（FPAT）

# 字段可能包含逗号时使用 gawk 的 FPAT
# 来源：linuxize.com AWK Command Guide
gawk 'BEGIN { FPAT = "([^,]*)|(\"[^\"]*\")" }
{ print $1, $3 }' complex.csv

# gawk 5.x 支持 --csv 原生 CSV 模式
gawk --csv '{ print $1, $2 }' data.csv

15.3 CSV 格式转换

# CSV → TSV（Tab 分隔）
awk -F, 'BEGIN { OFS="\t" } { $1=$1; print }' input.csv > output.tsv

# 冒号分隔 → CSV
awk -F: 'BEGIN { OFS="," } { $1=$1; print }' /etc/passwd > passwd.csv

# 选择特定列输出
awk -F, 'BEGIN { OFS="," }
NR==1 { print "姓名,得分" }
NR>1  { print $1, $3 }' data.csv

15.4 数据清洗

# 去除空行、注释行，归一化空格
awk '/^$/ || /^#/ { next }
{ gsub(/[ \t]+/, " ")      # 多空格缩为一个
  gsub(/^[ \t]+/, "")      # 去首部空格
  gsub(/[ \t]+$/, "")      # 去尾部空格
  print }' messy.txt

# 过滤掉第3字段为空或非数字的行
awk -F, '$3 ~ /^[0-9]+(\.[0-9]+)?$/ { print }' data.csv

📌 章节小结

  • 简单 CSV 使用 -F,；含引号的 CSV 应使用 gawk 的 FPAT 或 --csv 选项
  • 格式转换的核心技巧：设置 OFS 后执行 $1=$1 触发 $0 重建
  • 数据清洗场景中 gsub + next 的组合非常高效

传递 Shell 变量给 AWK

最佳实践

"The recommended way to use shell variables in awk programs is to assign the shell variable to an awk variable using the -v option."
— linuxize.com AWK Command Guide

# 推荐方式：-v 选项（BEGIN 块中也可用）
threshold=50
awk -v limit="$threshold" '$3 > limit { print $1 }' data.txt

# 传递多个变量
awk -v min="$min_val" -v max="$max_val" \
    '$3 >= min && $3 <= max { print }' data.txt

# 在文件间传递变量（命令行赋值，在 BEGIN 后生效）
awk '{ print var, $0 }' var=hello file1.txt var=world file2.txt

从文件运行 AWK 程序（Shebang）

#!/usr/bin/awk -f
# 保存为 analyze.awk，chmod +x analyze.awk
# 运行：./analyze.awk data.txt

BEGIN {
    FS = ","
    OFS = "\t"
    print "姓名\t得分"
}
NR > 1 { print $1, $3 }
END   { print "处理完成，共", NR-1, "行" }

第 17 章

总结与第一性原理

使用场景总览

📌 全书总结

  • AWK 是数据驱动的：你描述"对什么数据做什么"，而非"一步步怎么做"
  • 执行模型固定：-v赋值 → 编译 → BEGIN → 逐行(匹配+动作) → END
  • 记录与字段：RS 分割记录，FS 分割字段，通过 $N 引用
  • 模式驱动：六大模式类型控制"何时执行"，动作控制"做什么"
  • 关联数组是 AWK 唯一的数据结构，支撑了几乎所有统计和聚合需求
  • 管道友好：AWK 是 Unix 管道生态的优秀公民，与 sort、uniq、grep 等无缝协作
🔬 第一性原理 —— AWK 的本质

AWK 的本质是将计算机科学最基本的三个操作融合为一个优雅的工具：

1️⃣ 过滤（Filter）：模式选择满足条件的记录
2️⃣ 变换（Map）：动作将记录变换为新形式
3️⃣ 聚合（Reduce）：关联数组 + END 块将流式数据折叠为汇总结果

这正是现代大数据框架（MapReduce、Spark）的基础计算模型。 AWK 在 1977 年就以极简的命令行形式实现了这一范式， 这是它历经近 50 年仍不可替代的根本原因。
