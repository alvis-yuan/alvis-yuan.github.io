# 命令模式（Command）

将一个请求封装为一个对象，从而使你可用不同的请求对客户进行参数化；对请求排队或记录请求日志，以及支持可撤销的操作。 — GoF《设计模式》

### 📐 原理 & 使用场景

命令对象封装了"做什么"和"如何做"，可以像数据一样传递、存储、执行、撤销。

### 🎯 使用场景

- **操作历史/撤销重做**（文本编辑器）

- **事务日志**（数据库操作记录）

- **任务队列**（异步处理）

- **Redis AOF**（命令持久化日志）

- **Nginx 请求处理**（封装HTTP请求命令）

### Redis — AOF 持久化（命令日志）

Redis — AOF 持久化（命令日志）
          
        
        Redis 的 AOF（Append Only File）机制是命令模式的典型应用：每个写命令被封装并追加到日志文件，重启时重放这些命令对象恢复数据，支持"撤销"（回滚）操作。
        **示例：Redis AOF — 命令封装与回放**

```c
/* 命令封装：将 SET key val 写入AOF */
feedAppendOnlyFile(cmd, dbid, argv, argc);

/* 命令回放：重启时重新执行每条命令 */
loadAppendOnlyFile(server.aof_filename);
  → processCommand(fakeClient); /* 回放命令对象 */
```
