# 工厂方法模式（Factory Method）

定义一个用于创建对象的接口，让子类决定实例化哪一个类。工厂方法使一个类的实例化延迟到其子类。 — GoF《设计模式》

参考来源：
- [🔗 Redis Event Library 参考](https://redis.io/docs/latest/operate/oss_and_stack/reference/internals/internals-rediseventlib/)

### 📐 原理

工厂方法将对象的创建逻辑集中到一个工厂函数中，客户端无需知道具体类的名字，只需调用工厂函数传入类型标识符即可获得正确的对象实例。在C语言中通过函数指针（vtable）实现产品的多态行为。

**图2：工厂方法模式 — 由工厂函数决定创建哪种产品**

- createProduct(type)
- «Factory Function»
- → Product*
- ConcreteProductA
- operationA()
- ConcreteProductB
- operationB()
- type="A"
- type="B"
- Client 调用工厂

### 🎯 使用场景

- 当**不知道该创建哪个具体类**的对象时

- 需要**集中对象创建逻辑**，便于维护

- 希望通过**配置或参数**决定创建什么对象

- 需要在**运行时动态切换**产品类型

### ✅ 优势

- **解耦：**客户端与具体类解耦，只依赖抽象接口

- **开闭原则：**新增产品类型只需扩展工厂，无需修改客户端

- **集中创建：**对象创建逻辑集中管理，易于维护

### 💻 C语言实现示例

**示例：C — factory_method.c（基于 mohitmishra786.github.io）**

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

/* 产品接口（通过函数指针模拟多态） */
typedef struct {
    void (*operation)(void);
} Product;

/* 具体产品A */
void operationA(void) { printf("ProductA: 执行操作A\n"); }

/* 具体产品B */
void operationB(void) { printf("ProductB: 执行操作B\n"); }

/* 工厂函数 — 核心 */
Product* createProduct(const char* type) {
    Product* product = (Product*)malloc(sizeof(Product));
    if (strcmp(type, "A") == 0) {
        product->operation = operationA;
    } else if (strcmp(type, "B") == 0) {
        product->operation = operationB;
    } else {
        free(product); return NULL;
    }
    return product;
}

int main() {
    Product* pa = createProduct("A");
    Product* pb = createProduct("B");
    pa->operation();  /* 输出：ProductA: 执行操作A */
    pb->operation();  /* 输出：ProductB: 执行操作B */
    free(pa); free(pb);
    return 0;
}
```

### libevent — 事件后端工厂
*evsel.c / event.c*

libevent — 事件后端工厂
            evsel.c / event.c
          
        
        libevent 使用工厂方法根据操作系统自动选择最优的 I/O 多路复用后端（epoll/kqueue/select），对外提供统一的 `event_base` 接口，客户端无需关心底层实现。
        **示例：libevent 后端工厂（简化）**

```c
/* 每种后端实现的"产品"接口（函数指针表） */
const struct eventop epollops = {
    "epoll", epoll_init, epoll_add,
    epoll_del, epoll_dispatch, epoll_dealloc
};

/* 工厂：按平台选择最优后端 */
static const struct eventop *eventops[] = {
    &epollops, &kqueueops, &selectops, NULL
};
```
