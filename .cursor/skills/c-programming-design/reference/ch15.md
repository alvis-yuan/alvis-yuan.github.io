# 抽象工厂模式（Abstract Factory）

提供一个接口，用于创建相关或相互依赖的对象族，而无需指定其具体类。 — GoF《设计模式》

### 📐 原理

抽象工厂是"工厂的工厂"。它提供创建**一系列相关对象**的接口，确保这些对象相互兼容。客户端使用工厂接口，不依赖任何具体产品实现。

**图3：抽象工厂模式 — 创建相关产品族**

- «AbstractFactory»
- + createProductA(): ProductA*
- + createProductB(): ProductB*
- ConcreteFactory1
- createProductA() → A1
- createProductB() → B1
- ConcreteFactory2
- createProductA() → A2
- createProductB() → B2
