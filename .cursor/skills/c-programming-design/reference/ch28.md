# 责任链模式（Chain of Responsibility）

使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。将这些对象连成一条链，并沿着这条链传递该请求，直到有一个对象处理它为止。 — GoF《设计模式》

参考来源：
- [🔗 Design Patterns for C 参考](https://dev.to/khozaei/design-patterns-for-c-32an)

### 📐 原理

请求沿着处理链传递，每个处理者决定：① 自己处理，② 传递给下一个。发送者无需知道谁最终处理了请求。

**图8：责任链模式 — 请求沿链传递直到被处理**

- Client
- Handler1
- handleRequest()
- ✓ 处理 or →
- Handler2
- Handler3
- ✓ 最终处理
- 传递

### 🔍 Nginx — HTTP请求处理过滤链

### 

Nginx 的 HTTP 请求处理是责任链的典型实现：请求依次经过认证→限流→日志→内容处理等处理器，每个处理器决定是否处理或传递给下一个。
        **示例：Nginx HTTP 处理链（简化）**

```c
/* Nginx 将各模块处理函数链接成链 */
ngx_int_t ngx_http_handler(ngx_http_request_t *r) {
    r->phase_handler = 0;
    return ngx_http_core_run_phases(r);
}

/* 每个阶段的处理函数决定：处理 or 传递 */
/* NGX_OK: 处理完毕 | NGX_DECLINED: 传递给下一个 */
```
