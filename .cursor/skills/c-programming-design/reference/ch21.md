# 装饰器模式（Decorator）

动态地给一个对象添加一些额外的职责。就增加功能来说，Decorator 模式相比生成子类更为灵活。 — GoF《设计模式》

### 📐 原理 & 使用场景

装饰器通过包装对象，在调用被包装对象的方法前后增加额外行为，形成装饰链。



### Nginx — HTTP响应过滤器链

Nginx — HTTP响应过滤器链
          
        
        Nginx 的 HTTP 模块使用装饰器链处理响应：gzip压缩过滤器→内容替换过滤器→header过滤器，每个过滤器包装上一个，依次增强输出。
        **示例：Nginx 过滤链（ngx_http_filter.c 简化）**

```c
/* 每个过滤模块装饰（包装）上一个 */
static ngx_http_output_body_filter_pt ngx_http_next_body_filter;

ngx_int_t ngx_http_gzip_body_filter(ngx_http_request_t *r, ngx_chain_t *in) {
    /* 先执行自己的处理（压缩）*/
    do_gzip(in);
    /* 再调用下一个过滤器（装饰链） */
    return ngx_http_next_body_filter(r, in);
}
```
