### 事件驱动架构与单线程模型

Redis 用 C 实现了**单线程事件循环**架构（ae 事件引擎），这是 C 语言高性能系统设计的经典范式。单一主线程处理所有命令执行，通过 I/O 多路复用（epoll/kqueue/select）驱动网络事件，避免了多线程锁竞争的复杂性，同时保持了极高的吞吐量。这与 nginx、Node.js 事件循环同属一个设计哲学。

> "One main thread runs an event loop and uses OS I/O multiplexing (epoll/kqueue/select) to detect readable/writable sockets and timers. Single process (single-threaded for command execution) written in C. Keeps command execution lock-free."
> — — Redis Internal Architecture

**✅ 设计优势**
- 无锁设计：单线程串行执行，操作天然原子
- I/O 多路复用让单线程处理数万并发连接
- 代码简单：无死锁、无竞争条件
- 可预测延迟：不受线程调度影响
- 便于添加定时器和周期性任务
- 缓存服务器、键值数据库（Redis）
- 高并发网络服务器（nginx, lighttpd）
- 实时消息中间件
- 游戏服务器的事件处理循环
- 嵌入式系统的主事件循环

### 代码示例对比

**反例 — 每连接一线程（高并发时）**

```c
/* ❌ 反例：每连接一线程
   高并发时线程开销巨大，
   锁竞争严重 */
void server_loop(int srv_fd) {
    while (1) {
        int cli = accept(srv_fd,
            NULL, NULL);
        /* 每个连接创建一个线程，
           10000 连接 = 10000 线程
           内存: ~80MB，切换成本高 */
        pthread_t t;
        pthread_create(&t, NULL,
            handle_client,
            (void*)(intptr_t)cli);
        pthread_detach(t);
        /* 共享状态需要加锁 */
        pthread_mutex_lock(&g_lock);
        g_conn_count++;
        pthread_mutex_unlock(&g_lock);
    }
}
```

**正例 — Redis 事件循环风格**

```c
/* ✅ 正例：类 Redis ae 事件循环 */
struct ae_loop {
    int epfd;
    struct ae_event events[MAX_EV];
    int n_events;
};

void ae_run(struct ae_loop *el) {
    while (!el->stop) {
        struct epoll_event fired[128];
        int n = epoll_wait(
            el->epfd, fired, 128, 100);
        /* 单线程处理所有就绪事件
           无锁，无竞争，高吞吐 */
        for (int i=0; i<n; i++) {
            struct ae_event *e =
                fired[i].data.ptr;
            e->handler(el, e->fd,
                       e->data);
        }
        ae_process_timers(el);
    }
}
```
