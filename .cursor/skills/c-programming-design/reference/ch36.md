# 设计模式决策树

从需求出发，通过系列判断问题，逐步收窄范围，最终确定最优设计模式。每个节点代表一个关键判断条件。

> **使用方法：**从"需求类型"开始，根据每个问题的答案沿箭头走向下一步，直到到达推荐的设计模式节点。多数情况下最终答案不唯一，可组合使用。

### ⚠️ 选择原则与注意事项

> **最佳实践**
>             
>               
> - 先识别**核心问题类型**（创建/结构/行为）
>               
> - 明确**主要约束**（性能/内存/扩展性）
>               
> - 模式可**组合使用**（如Factory+Singleton）
>               
> - 优先使用**简单方案**，避免过度设计

> **常见误区**
>             
>               
> - 强行套用模式，**过度工程化**
>               
> - 把 Singleton 用于所有全局变量
>               
> - 混淆 Decorator 和 Proxy（目的不同）
>               
> - 忽略**C语言特性**（无GC，需手动管理内存）
