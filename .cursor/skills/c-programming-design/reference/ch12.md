# 23 种设计模式总览

GoF定义的全部23种模式，按类别分组展示，包含核心意图与C语言开源项目中的典型应用。

### 🏗️ 创建型模式（5种）

| 模式 | 核心意图 | C开源项目应用 |
| --- | --- | --- |
| **单例 Singleton** | 确保一个类只有一个实例，并提供全局访问点 | Redis 全局服务器状态 server、Linux内核设备管理 |
| **工厂方法 Factory Method** | 定义创建对象的接口，让子类决定实例化哪个类 | libevent 事件后端工厂、Linux VFS inode_operations |
| **抽象工厂 Abstract Factory** | 提供创建相关对象族的接口，无需指定具体类 | Nginx 模块化配置解析族、GLib 类型系统 |
| **建造者 Builder** | 将复杂对象的构建与表示分离，同一构建可创建不同表示 | Redis 协议响应构建、HTTP请求报文构建 |
| **原型 Prototype** | 通过克隆现有对象来创建新对象 | Linux进程fork()、Nginx配置复制 |

### 🔗 结构型模式（7种）

| 模式 | 核心意图 | C开源项目应用 |
| --- | --- | --- |
| **适配器 Adapter** | 将不兼容接口转换为客户期望的接口 | libevent 多路复用后端适配（select/epoll/kqueue） |
| **桥接 Bridge** | 将抽象与实现解耦，使二者可独立变化 | Linux DRM图形驱动、libevent 事件驱动后端 |
| **组合 Composite** | 将对象组织成树形结构，客户统一处理单个对象和组合 | Linux VFS 文件系统树、Nginx 配置节点树 |
| **装饰器 Decorator** | 动态地给对象添加职责，比继承更灵活 | Linux I/O层叠加（buffering/crypto）、Nginx 过滤器链 |
| **外观 Facade** | 为复杂子系统提供统一的简化接口 | Redis ae事件库 API、libevent event_base接口 |
| **享元 Flyweight** | 通过共享细粒度对象来减少内存使用 | Linux页缓存、Redis 小整数/短字符串对象共享 |
| **代理 Proxy** | 为另一个对象提供代理/占位符以控制访问 | Linux内核系统调用代理、mmap虚拟内存代理 |

### 🎭 行为型模式（11种）

| 模式 | 核心意图 | C开源项目应用 |
| --- | --- | --- |
| **观察者 Observer** | 定义一对多依赖，状态改变时自动通知所有依赖者 | Redis pub/sub机制、libevent 事件回调、GLib信号机制 |
| **策略 Strategy** | 定义算法族，封装每种算法，使其可互换 | Redis hash/set 编码策略、Nginx 负载均衡策略 |
| **命令 Command** | 将请求封装为对象，支持排队、日志、撤销 | Redis AOF命令日志、Nginx 请求处理命令 |
| **责任链 Chain of Responsibility** | 请求沿处理链传递，直到被处理 | Nginx HTTP请求处理过滤链、Linux网络协议栈 |
| **状态 State** | 对象内部状态改变时改变其行为 | Redis 客户端连接状态机、libevent 事件状态 |
| **模板方法 Template Method** | 定义算法骨架，将某些步骤延迟到子类实现 | Linux file_operations、Nginx 模块处理框架 |
| **迭代器 Iterator** | 提供顺序访问聚合对象元素的方法，不暴露内部结构 | Linux list_for_each宏、Redis dictIterator |
| **中介者 Mediator** | 定义对象间交互的中介对象，减少直接依赖 | Redis 服务器作为客户端中介、libevent event_base |
| **访问者 Visitor** | 在不修改元素类的前提下定义新操作 | Linux AST遍历、编译器前端语法树访问 |
| **备忘录 Memento** | 在不破坏封装的前提下捕获并恢复对象状态 | Redis RDB快照、游戏存档系统 |
| **解释器 Interpreter** | 给语言定义文法和解释器，解释该语言的语句 | Redis命令解析器、Nginx配置语言解析 |

> **C语言实现的关键**
>           GoF模式设计于面向对象环境，但其核心思想与语言无关。C语言通过**函数指针**实现多态，通过**结构体**实现封装，通过**不透明指针**实现抽象，完全可以实现全部23种模式。
