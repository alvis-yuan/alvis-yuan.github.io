# 观察者模式（Observer）

定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都得到通知并被自动更新。 — GoF《设计模式》

参考来源：
- [🔗 Redis Event Library 官方文档](https://redis.io/docs/latest/operate/oss_and_stack/reference/internals/internals-rediseventlib/)

### 📐 原理

Subject（主题）维护一个观察者列表，状态改变时自动通知所有注册的 Observer（观察者）。也称为"发布-订阅"模式。

### 🔍 C开源项目中的应用

### Redis — 发布/订阅（Pub/Sub）机制

Redis — 发布/订阅（Pub/Sub）机制
          
        
        Redis 的 Pub/Sub 是观察者模式的直接实现：发布者（Publisher）发布消息到频道，Redis服务器作为主题（Subject），所有订阅该频道的客户端（Observer）自动收到通知。
        **示例：Redis pub/sub（简化）**

```c
/* 订阅 = 注册观察者 */
subscribeCommand: client → pubsubSubscribeChannel(channel, client)

/* 发布 = 通知所有观察者 */
publishCommand: pubsubPublishMessage(channel, message)
  → for each subscriber: addReply(subscriber, message)
```

### libevent — 事件回调机制

libevent — 事件回调机制
          
        
        libevent 的事件注册和回调本质上是观察者模式：I/O事件（Subject）发生时，自动调用所有注册的回调函数（Observer）。
        **示例：libevent 事件注册**

```c
/* 注册观察者（回调函数）*/
event_set(&ev, fd, EV_READ|EV_PERSIST, read_callback, arg);
event_add(&ev, NULL);

/* 当fd可读时（Subject状态变化），自动调用 read_callback（Observer更新）*/
```
