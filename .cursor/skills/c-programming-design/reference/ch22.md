# 外观模式（Facade）

为子系统中的一组接口提供一个一致的界面，Facade 模式定义了一个高层接口，使得这一子系统更加容易使用。 — GoF《设计模式》

参考来源：
- [🔗 Redis Event Library 官方文档](https://redis.io/docs/latest/operate/oss_and_stack/reference/internals/internals-rediseventlib/)

### 📐 原理

外观对象拦截所有客户端调用，并将其委托给内部能处理的子系统。客户端只与外观交互，不直接接触复杂子系统。

### 🔍 Redis — ae事件库（外观模式）

### 

Redis 的 `ae`（Another Event Loop）是外观模式：提供统一的 `aeCreateFileEvent` / `aeMain` 等简单API，内部根据平台选择epoll/kqueue/select，完全屏蔽底层复杂性。
        **示例：Redis ae.c — 外观接口**

```c
/* 外观API：客户端只需知道这些 */
aeEventLoop* aeCreateEventLoop(int setsize);
int  aeCreateFileEvent(aeEventLoop*, int, int, aeFileProc*, void*);
void aeMain(aeEventLoop*);  /* 启动事件循环 */

/* 内部根据平台选择后端（epoll/kqueue/select）*/
/* 对外完全透明 */
```
