# 单例模式（Singleton）

确保一个类只有一个实例，并提供一个全局访问点。 — GoF《设计模式》

参考来源：
- [🔗 Redis Event Library 内部文档](https://redis.io/docs/latest/operate/oss_and_stack/reference/internals/internals-rediseventlib/)

### 📐 原理

单例模式通过将构造函数私有化（在C中通过静态变量+访问函数实现），确保类只能被实例化一次，并提供一个全局访问该实例的方法。

**图1：单例模式结构图 — 确保全局唯一实例**

- Singleton
- - instance: Singleton*
- + getInstance(): Singleton*
- + operation(): void
- self-ref
- Client
- 调用方
- getInstance()

### 🎯 使用场景

- 管理**共享资源**（如数据库连接池、线程池）

- **全局配置管理器**（应用程序配置只需一份）

- **日志系统**（统一写入日志）

- 操作系统的**设备驱动**（硬件资源唯一）

### ✅ 优势

> **控制全局状态**全局只有一个实例，避免资源的多重占用和状态不一致

> **全局访问点**无需传递对象引用，任何地方都可访问该实例

> **注意：**单例模式引入全局状态，多线程环境下需加锁保护。过度使用会导致强耦合，应谨慎使用。

### 💻 C语言实现示例（含线程安全）

以下为线程安全的单例模式C实现，使用 `pthread_mutex` 进行双重检查锁定（DCL）：

**示例：C — singleton.c**

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

typedef struct {
    int value;
} Singleton;

static Singleton* instance = NULL;
static pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;

/* 双重检查锁定（DCL）— 线程安全的懒汉式单例 */
Singleton* getInstance() {
    if (instance == NULL) {
        pthread_mutex_lock(&mutex);
        if (instance == NULL) {
            instance = (Singleton*)malloc(sizeof(Singleton));
            instance->value = 0;
        }
        pthread_mutex_unlock(&mutex);
    }
    return instance;
}

int main() {
    Singleton* s1 = getInstance();
    Singleton* s2 = getInstance();
    s1->value = 42;
    printf("s1==s2: %s, value=%d\n",
           s1 == s2 ? "true" : "false", s2->value);
    /* 输出：s1==s2: true, value=42 */
    free(instance);
    return 0;
}
```
