# 策略模式（Strategy）

定义一系列的算法，把它们一个个封装起来，并且使它们可互换。本模式使得算法可独立于使用它的客户而变化。 — GoF《设计模式》

### 📐 原理 & 使用场景

Context 持有策略接口的引用，在运行时可动态切换不同的算法实现。通过函数指针在C语言中实现。

### 💻 C语言实现

**示例：C — strategy.c（基于 mohitmishra786.github.io）**

```c
typedef struct { int (*execute)(int, int); } Strategy;
typedef struct { Strategy* strategy; } Context;

int add(int a, int b)      { return a + b; }
int multiply(int a, int b)  { return a * b; }

/* 运行时切换策略 */
Context ctx;
Strategy add_strat = { add };
ctx.strategy = &add_strat;
printf("%d\n", ctx.strategy->execute(10, 5)); /* 15 */

Strategy mul_strat = { multiply };
ctx.strategy = &mul_strat;
printf("%d\n", ctx.strategy->execute(10, 5)); /* 50 */
```

### 🔍 C开源项目应用

### Redis — 编码策略（Encoding Strategy）

Redis — 编码策略（Encoding Strategy）
          
        
        Redis 对同一数据结构（如Hash、Set、List）根据数据量和元素大小动态选择编码策略：小数据用 ziplist（节省内存），大数据用 hashtable/skiplist（高性能）。
        **示例：Redis 编码策略切换**

```c
/* Hash 对象编码策略 */
if (hashTypeLength(o) > OBJ_HASH_MAX_ZIPLIST_ENTRIES ||
    value_len > OBJ_HASH_MAX_ZIPLIST_VALUE) {
    /* 策略切换：ziplist → hashtable */
    hashTypeConvert(o, OBJ_ENCODING_HT);
}
```
