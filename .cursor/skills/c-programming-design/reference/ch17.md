# 原型模式（Prototype）

用原型实例指定创建对象的种类，并通过拷贝这些原型创建新对象。 — GoF《设计模式》

### 📐 原理 & 使用场景

原型模式通过克隆已有对象来创建新对象，适用于对象初始化代价高昂或对象结构复杂时。



### Linux — fork() 写时复制（Copy-on-Write）

Linux — fork() 写时复制（Copy-on-Write）
          
        
        Linux 的 `fork()` 系统调用是原型模式的经典应用：子进程以父进程为原型创建，通过写时复制（COW）机制高效共享内存页，只在写入时才真正复制。
        **示例：Unix — fork原型克隆示意**

```c
pid_t pid = fork();  /* 克隆当前进程为原型 */
if (pid == 0) {
    /* 子进程：从父进程原型克隆 */
} else {
    /* 父进程：原型保持不变 */
}
```
