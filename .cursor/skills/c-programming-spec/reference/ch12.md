# 头文件与包含守卫

*12预处理与宏*

参考来源：[SEI CERT PRE06-C — Enclose header files in an inclusion guard](https://wiki.sei.cmu.edu/confluence/display/c/PRE06-C)、
    ISO/IEC 9899:1999 §6.10

## 12.1 包含守卫（Include Guard）

**反面示例（不规范）**

```c
/* 无守卫 —— 重复包含导致错误 */
typedef struct { int x; } Point;

/* 使用 #pragma once（非标准）*/
#pragma once
/* 虽然广泛支持但不在C99标准内 */

/* 守卫名过于简单，可能冲突 */
#ifndef UTILS_H
#define UTILS_H
```

**正面示例（规范（C99 标准方式））**

```c
/* 守卫宏: 项目名_目录_文件名_H */
#ifndef MYPROJECT_NET_SOCKET_H
#define MYPROJECT_NET_SOCKET_H

/* 头文件内容 */

#endif /* MYPROJECT_NET_SOCKET_H */
/* ^^ 注释标明对应的守卫 */
```


## 12.2 #include 顺序规范

**示例：推荐的 #include 排列顺序（Google/CERT 建议）**

```c
/* 1. 本文件对应的头文件（优先，暴露自包含问题）*/
#include "net_socket.h"

/* 2. C 标准库头文件（字母序）*/
#include <errno.h>
#include <stdint.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

/* 3. POSIX/系统头文件 */
#include <sys/socket.h>
#include <unistd.h>

/* 4. 第三方库头文件 */
#include "third_party/openssl/ssl.h"

/* 5. 本项目其他模块头文件 */
#include "common/logger.h"
#include "common/types.h"
```

### [必须] 头文件必须自包含（Self-Contained）

每个头文件必须能独立被包含，不依赖调用者先包含其他文件。
      即头文件内部应包含自身所需的所有依赖。通过将自身头文件放在 `#include` 最前面来验证。
