# 并发与线程安全

*18高级主题*

参考来源：[SEI CERT CON00-C to CON41-C](https://wiki.sei.cmu.edu/confluence/display/c/CON00-C)（并发规则）、
    ISO/IEC 9899:2011 §7.26（C11 线程库）

## 18.1 共享数据保护

**示例：线程安全模式示例**

```c
#include <stdint.h>
#include <stdbool.h>

/* ① volatile 仅用于防止编译器优化（不保证原子性）*/
volatile static bool g_irq_flag = false;

/* ② 共享数据结构：数据与锁封装在一起（CERT CON02-C）*/
typedef struct {
    pthread_mutex_t lock;    /* 锁与数据同级 */
    uint32_t        counter;
    bool            is_valid;
} shared_state_t;

/* ③ 所有公共访问路径必须持锁 */
int32_t shared_state_increment(shared_state_t *s)
{
    if (s == NULL) { return -EINVAL; }

    pthread_mutex_lock(&s->lock);
    s->counter++;
    pthread_mutex_unlock(&s->lock);
    return NET_OK;
}
```


## 18.2 并发安全规则

| 规则 | 说明 |
| --- | --- |
| volatile ≠ 原子操作 | `volatile` 防止编译器缓存，但不防止 CPU 乱序；需使用内存屏障或原子操作 |
| 锁定顺序一致 | 多锁时始终按固定顺序获取，防止死锁（CERT CON35-C） |
| 禁止数据竞争 | 多线程访问同一非原子对象时必须同步（C11 §5.1.2.4） |
| 信号处理函数限制 | 信号处理函数中只能使用 async-signal-safe 函数（CERT SIG30-C） |
| static 变量非线程安全 | 函数内 static 变量在多线程中需显式同步或改用 TLS（线程局部存储） |
| 单整型共享计数/标志 | 多线程下优先 `atomic_t` 与 `atomic_*`（见 [第22章](ch22.md)），勿用普通 `int` 或仅靠 `volatile` |

> 共享 **单个整型**（引用计数、连接数、就绪标志）且操作可用一次原子 RMW 完成时，**优先**使用 [第22章 GCC 内置原子操作](ch22.md)；多字段或复杂结构仍用本节互斥锁方案。
