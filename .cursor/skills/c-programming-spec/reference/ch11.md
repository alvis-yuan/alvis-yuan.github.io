# 宏定义规范

*11预处理与宏*

参考来源：[Linux Kernel §12 Macros, Enums and RTL](https://www.kernel.org/doc/Documentation/process/coding-style.rst)、
    [SEI CERT PRE00-C to PRE12-C](https://wiki.sei.cmu.edu/confluence/display/c/PRE00-C)

## 11.1 常量宏规范

**反面示例（不规范）**

```c
/* 无类型信息 */
#define SIZE 256
#define PI 3.14159

/* 表达式未加括号 */
#define DOUBLE(x) x * 2
/* DOUBLE(1+2) → 1+2*2 = 5 ❌ */

/* 多语句宏没有 do-while */
#define SWAP(a,b) \
    { int t = a; a = b; b = t; }
/* if(cond) SWAP(x,y); else ... → 语法错 */
```

**正面示例（规范）**

```c
/* 常量: 优先 const 或含类型后缀 */
static const uint32_t BUF_SIZE = 256U;
#define BUF_SIZE_MAX  (512U)   /* 括号+后缀 */
#define PI_F          (3.14159f)

/* 表达式宏：所有参数加括号 */
#define DOUBLE(x)     ((x) * 2U)
#define MAX(a, b)     (((a) > (b)) ? (a) : (b))

/* 多语句宏：使用 do-while(0) */
#define SWAP(a, b)          \
    do {                    \
        typeof(a) _t = (a); \
        (a) = (b);          \
        (b) = _t;           \
    } while (0)
```


## 11.2 函数宏 vs. inline 函数

### [必须] 复杂逻辑优先使用 static inline 而非函数宏

**Linux Kernel §12：**"Generally, inline functions are preferable to macros resembling functions."

      `static inline` 函数具备：类型检查、不会重复求值参数副作用、可调试、可被编译器内联优化。

**反面示例**

```c
/* ❌ 函数宏：参数副作用问题 */
#define ABS(x)   ((x) < 0 ? -(x) : (x))
/* ABS(i++) → i 被求值两次！*/
```

**正面示例**

```c
/* ✅ static inline：类型安全，无副作用 */
static inline int32_t abs_i32(int32_t x)
{
    return (x < 0) ? -x : x;
}
```


## 11.3 宏的禁忌模式

| 禁忌模式 | 问题说明 | 替代方案 |
| --- | --- | --- |
| 宏中使用 `return` | 调用者不知道函数被提前退出 | 改为 inline 函数 |
| 宏依赖特定局部变量名 | `#define FOO(v) bar(index, v)` —— 隐式依赖 `index` | 显式传参 |
| 宏参数作为左值 | `FOO(x) = y` 转为 inline 后语义改变 | 避免此模式 |
| 多次求值参数 | `MAX(f(), g())` 会调用两次 | 使用 inline 或临时变量 |
| 无括号表达式 | `#define N 2+3` → `N*4=11` | 始终加括号 |
