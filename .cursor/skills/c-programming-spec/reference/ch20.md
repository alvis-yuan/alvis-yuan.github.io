# 注释与文档

*20高级主题*

参考来源：[Linux Kernel §8 Commenting](https://www.kernel.org/doc/Documentation/process/coding-style.rst)、
    GNU Coding Standards §6.3 Commenting Your Work

## 20.1 注释的原则

### [必须] 注释说明"为什么"，代码说明"怎么做"

**Linux Kernel §8：**"Comments are good, but there is also a danger of over-commenting. NEVER try to explain HOW your code works... Generally, you want your comments to tell WHAT your code does, not HOW."

      不要写显而易见的注释，要写设计意图、约束条件、注意事项。

**反面示例（无意义注释）**

```c
/* 将 i 加 1 */
i++;

/* 检查是否为空 */
if (ptr == NULL) { }

/* 返回 0 */
return 0;
```

**正面示例（有价值的注释）**

```c
/* 跳过 BOM（U+FEFF），UTF-8 文件可选 */
if (buf[0] == 0xEFU) { offset += 3; }

/* 此处不释放 ctx：调用者拥有所有权 */
return NET_OK;

/* FIXME: 此算法在 n>65535 时精度下降
 * 参见 https://example.com/issue/42 */
```


## 20.2 Doxygen 文档注释规范

**示例：完整 Doxygen 注释模板**

```c
/**
 * @defgroup net_socket 网络套接字模块
 * @{
 */

/**
 * @brief   连接到远程主机
 *
 * @details 创建 TCP 套接字并尝试连接，连接超时后自动重试最多
 *          @p retry_count 次。调用成功后通过 @p sock 输出句柄。
 *
 * @param[out] sock         输出套接字句柄（不可为 NULL）
 * @param[in]  host         目标主机 IP 或域名
 * @param[in]  port         目标端口（1-65535）
 * @param[in]  timeout_ms   连接超时毫秒数（0 = 使用系统默认）
 * @param[in]  retry_count  最大重试次数（0 = 不重试）
 *
 * @return NET_OK           连接成功
 * @return NET_ERR_PARAM    参数无效
 * @return NET_ERR_TIMEOUT  连接超时
 * @return NET_ERR_REFUSED  目标拒绝连接
 *
 * @pre  网络子系统已初始化（net_init() 已调用）
 * @post 成功时 sock 有效，必须调用 net_close() 释放
 *
 * @note 此函数是线程安全的
 * @warning 不可在中断上下文中调用
 *
 * @see net_close(), net_send()
 */
net_err_e net_connect(net_socket_t **sock,
                       const char   *host,
                       uint16_t      port,
                       uint32_t      timeout_ms,
                       uint32_t      retry_count);
```


## 20.3 特殊注释标记

| 标记 | 用途 | 格式示例 |
| --- | --- | --- |
| `TODO` | 待实现功能 | `/* TODO: 增加 IPv6 支持 */` |
| `FIXME` | 已知缺陷，待修复 | `/* FIXME: 竞争条件，见 issue#42 */` |
| `HACK` | 临时解决方案 | `/* HACK: 规避芯片 bug，Rev B 后移除 */` |
| `NOTE` | 重要提示 | `/* NOTE: 调用前必须持有 ctx->lock */` |
| `WARNING` | 危险操作警告 | `/* WARNING: 此处不检查边界，调用者负责 */` |


## 20.4 注释与日志语言

### [必须] 注释中文，运行期日志英文

| 输出 | 语言 | 示例 |
| --- | --- | --- |
| 源码注释、Doxygen | **中文** | `/* 跳过 BOM，UTF-8 文件可选 */` · `@brief 连接到远程主机` |
| `LogError` / `LogInfo` / `LOG_*` | **英文** | `LogError("open(%s) failed: %s", path, strerror(saved));` |

**原因：** 注释服务本地维护者；日志常进入集中采集与跨环境排障，英文便于 `grep`，并与 `errno`/`strerror` 等系统输出语言一致。

**反面示例**

```c
if (fd < 0) {
    LogError("打开文件失败: %s", path);   /* ❌ 日志中文 */
    return -errno;
}

/* Open file and check fd */              /* ❌ 应用中文说明设计意图 */
if (fd < 0) { ... }
```

**正面示例**

```c
/* 失败路径立即保存 errno，避免后续系统调用覆盖 */
if (fd < 0) {
    int saved = errno;
    LogError("open(%s) failed: %s (errno=%d)", path, strerror(saved), saved);
    return -saved;
}
```
