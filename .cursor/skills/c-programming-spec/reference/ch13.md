# 条件编译

*13预处理与宏*

参考来源：[Linux Kernel §21 Conditional Compilation](https://www.kernel.org/doc/Documentation/process/coding-style.rst)、
    [SEI CERT PRE10-C](https://wiki.sei.cmu.edu/confluence/display/c/PRE10-C)

## 13.1 条件编译规范

**示例：条件编译最佳实践**

```c
/* ✅ 在头文件中定义存根，.c 中无条件调用 */
#ifdef CONFIG_FEATURE_LOG
  void log_debug(const char *fmt, ...);
#else
  static inline void log_debug(const char *fmt, ...) { (void)fmt; }
#endif

/* ✅ 非平凡的 #if 块结尾必须注释 */
#ifdef CONFIG_PLATFORM_ARM
  /* ARM 平台特定实现 ... */
#elif defined(CONFIG_PLATFORM_X86)
  /* x86 平台实现 ... */
#else
  #error "Unsupported platform"
#endif /* CONFIG_PLATFORM_ARM */

/* ✅ 使用 defined() 进行多宏检查 */
#if defined(PLATFORM_LINUX) && defined(ARCH_64BIT)
  typedef uint64_t native_word_t;
#endif

/* ❌ 避免在函数体中用 #ifdef（拆函数更好）*/
```


## 13.2 调试宏模式

**示例：可控调试输出宏**

```c
/* 带文件/行号的调试宏 */
#ifdef DEBUG
  #define LOG_DEBUG(fmt, ...) \
      fprintf(stderr, "[DEBUG] %s:%d: " fmt "\n", \
              __FILE__, __LINE__, ##__VA_ARGS__)
#else
  #define LOG_DEBUG(fmt, ...)  do { } while (0)
#endif

#define LOG_ERROR(fmt, ...) \
    fprintf(stderr, "[ERROR] %s:%d: " fmt "\n", \
            __FILE__, __LINE__, ##__VA_ARGS__)
```
