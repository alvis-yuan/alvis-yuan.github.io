# 内存管理

*14内存与安全*

参考来源：[SEI CERT MEM00-C to MEM12-C](https://wiki.sei.cmu.edu/confluence/display/c/MEM00-C)、
    [Linux Kernel §14 Allocating memory](https://www.kernel.org/doc/Documentation/process/coding-style.rst)

## 14.1 动态内存分配规范

**示例：安全的内存分配模式**

```c
/* ✅ 使用 sizeof(*ptr) 而非 sizeof(type) —— 防止类型改变时遗漏修改 */
net_socket_t *sock = (net_socket_t *)malloc(sizeof(*sock));
if (sock == NULL) {
    return NET_ERR_NOMEM;
}
/* 分配后立即清零（防止未初始化读取）*/
memset(sock, 0, sizeof(*sock));

/* ✅ 数组分配：使用 calloc（带溢出检查且自动清零）*/
uint8_t *buf = (uint8_t *)calloc(count, sizeof(uint8_t));
if (buf == NULL) { return NET_ERR_NOMEM; }

/* ✅ 释放后立即置 NULL，防止 use-after-free */
free(sock);
sock = NULL;

/* 安全释放宏 */
#define SAFE_FREE(p)  do { free(p); (p) = NULL; } while (0)
```


## 14.2 内存管理禁忌

| 错误模式 | 后果 | 规范做法 |
| --- | --- | --- |
| double-free | 堆损坏、崩溃、安全漏洞 | 释放后立即置 NULL |
| use-after-free | UB，可被利用为漏洞 | 置 NULL + 所有权转移设计 |
| 内存泄漏 | 长期运行耗尽内存 | 每个 malloc 配对 free，goto 清理模式 |
| malloc 返回值未检查 | NULL 解引用崩溃 | 始终检查返回值 |
| realloc 赋值原指针 | 失败时原指针丢失（泄漏） | 使用临时指针接收 realloc 结果 |
| VLA（可变长数组）栈分配 | 大数组导致栈溢出 | 改用 malloc；MISRA 禁止 VLA |

**示例：realloc 安全模式**

```c
/* ❌ 危险: realloc 失败时原指针丢失 */
buf = (uint8_t *)realloc(buf, new_size);

/* ✅ 安全: 使用临时指针 */
uint8_t *tmp = (uint8_t *)realloc(buf, new_size);
if (tmp == NULL) {
    /* buf 仍然有效，可以处理错误 */
    free(buf);
    buf = NULL;
    return NET_ERR_NOMEM;
}
buf = tmp;
```

> **多处 return** 下的堆内存释放可配合 [raii.h](raii.h) 的 `autofree`（见 [第24章](ch24.md)）；单 return 路径仍可直接 `free`。
