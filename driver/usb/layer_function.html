Function 驱动层

function/f_*.c · struct usb_function · USB-IF 类协议实现

1. 层职责

Function 驱动实现 USB 规范中的「接口/功能」：向 Host 暴露 Printer、ACM、MSC 等类设备。 每个 Function 是独立可加载模块（usb_f_*.ko），通过 Composite 层组合成复合设备。 对上可暴露字符设备、网络接口或 ALSA；对下通过 usb_ep_queue() 使用 UDC 端点。

2. usb_function 与注册

核心结构定义于 include/linux/usb/composite.h：

struct usb_function {
    const char *name;
    struct usb_descriptor_header **fs_descriptors;
    struct usb_descriptor_header **hs_descriptors;
    struct usb_descriptor_header **ss_descriptors;
    struct usb_configuration *config;
    int  (*bind)(struct usb_configuration *, struct usb_function *);
    void (*unbind)(struct usb_configuration *, struct usb_function *);
    int  (*set_alt)(struct usb_function *, unsigned intf, unsigned alt);
    void (*disable)(struct usb_function *);
    int  (*setup)(struct usb_function *, const struct usb_ctrlrequest *);
    /* suspend/resume/get_status/… */
    struct list_head list;
    const struct usb_function_instance *fi;
};

模块注册宏（functions.c + 各 f_*.c）：

DECLARE_USB_FUNCTION_INIT(printer, printer_alloc_instance, printer_alloc_func);
/* 模块加载时 usb_function_register()，别名 usbfunc:printer */

ConfigFS / Composite 通过 usb_get_function_instance("printer") 查找并实例化。

3. 生命周期回调

回调典型实现（以 f_printer / f_iap2 为例）
bindusb_interface_id() · usb_ep_autoconfig() · 注册 /dev/g_printer*
set_altconfig_ep_by_speed() · usb_ep_enable() · 预 queue 空闲 request
setup类请求：Printer GET_DEVICE_ID / GET_PORT_STATUS / SOFT_RESET
disableusb_ep_disable() · 回收 request · 唤醒阻塞的 read/write
unbind删除 cdev · 释放端点与私有结构

4. 描述符与端点

  • 每个 Function 维护 fs_descriptors[] / hs_descriptors[] / ss_descriptors[] 以支持多速率
  • bind() 中调用 usb_ep_autoconfig(gadget, &ep_desc)（epautoconf.c）匹配硬件端点
  • set_alt() 中 config_ep_by_speed() 更新 ep->desc 后 usb_ep_enable()
