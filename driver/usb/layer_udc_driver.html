UDC 硬件驱动层

udc/*.c · SoC USB Device Controller · 寄存器/DMA/中断

1. 层职责

UDC（USB Device Controller）驱动直接操作 SoC 上的 USB 外设控制器 IP： 初始化硬件、实现 usb_ep_ops（enable/queue/dequeue）、 在中断里处理 setup 包与 transfer complete，并调用 usb_gadget_giveback_request() 归还请求。 每个 UDC 驱动向 Core 注册一个 usb_gadget 实例。

2. 驱动实现要点

任务典型实现
注册usb_initialize_gadget() · 填充 ep_list · usb_add_gadget_udc()
端点每个硬件 endpoint 对应 struct usb_ep + usb_ep_ops
SetupEP0 收到 SETUP → 调用 gadget->setup(gadget, ctrl)
Pull-upusb_gadget_connect/disconnect → ops->pullup
速率报告 gadget->max_speed，处理 USB 2.0/3.0 差异
Traceudc/trace.h — tracepoints 辅助调试

新 UDC 开发应优先用 dummy_hcd + usbtest 验证 EP0 与 bulk 路径， 见 gadget-testing.rst。

4. dummy_hcd 调试

dummy_hcd.c 文件头说明其用途：

/* 暴露 device side "USB gadget" API，由 Linux-USB host 驱动驱动。
 * USB 流量被模拟，无需真实硬件。可与 gadget 驱动 + host 驱动联调。
 * 注意：模拟不包含 isochronous 传输！ */
modprobe dummy_hcd
# 出现 /sys/class/udc/dummy_udc.N
echo dummy_udc.0 > /sys/kernel/config/usb_gadget/g1/UDC
