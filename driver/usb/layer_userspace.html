用户态配置与交互层

ConfigFS · 字符设备 · FunctionFS · 与内核 Gadget 框架的边界

1. 层职责

用户态不直接操作 UDC 寄存器，而是通过两类接口驱动 Gadget 行为： ConfigFS 决定「设备长什么样」（VID/PID、配置、Function 组合）； 字符设备 / 网络 / ALSA 等在枚举完成后承载业务数据。

2. ConfigFS 动态组装 Gadget

官方文档 gadget_configfs.rst 与源码 configfs.c 一致：用户在 configfs 中描述 configurations 与 functions， 写入 UDC 属性时触发绑定（见 gadget_dev_desc_UDC_store()）。 

# 内核文档推荐流程（Documentation/usb/gadget_configfs.rst）
modprobe libcomposite
mount -t configfs none /sys/kernel/config

mkdir /sys/kernel/config/usb_gadget/g1
cd /sys/kernel/config/usb_gadget/g1
echo 0x1d6b > idVendor    # Linux Foundation
echo 0x0104 > idProduct
mkdir -p strings/0x409
echo "0123456789" > strings/0x409/serialnumber
echo "Manufacturer" > strings/0x409/manufacturer
echo "Product" > strings/0x409/product
mkdir configs/c.1
mkdir functions/printer.0    # 自动 request_module usbfunc:printer
ln -s functions/printer.0 configs/c.1/
echo "$(cat /sys/class/udc/*/device/../.. 2>/dev/null || ls /sys/class/udc)" > UDC

内核侧关键逻辑（configfs.c）：

  • gadget_dev_desc_UDC_store() — 写 UDC 名 → usb_gadget_probe_driver()
  • 写空或 none → usb_gadget_unregister_driver() 解绑
  • Function 实例通过 usb_get_function_instance() 创建，触发 modprobe usbfunc:<name>
ConfigFS 通用机制： Configfs — Userspace-driven Kernel Object Configuration 描述 config_group / config_item 模型；Gadget 是其上的一个应用实例。

3. 字符设备接口

多数 Function 在 bind() 中注册 cdev，用户态通过标准 I/O 交换 USB 载荷：

设备节点Function 模块典型用户态
/dev/g_printerNusb_f_printer打印服务、CUPS backend
/dev/iap2Nusb_f_iap2iOS IAP2 打印引擎
/dev/ttyGSNusb_f_acm / usb_f_serial串口终端、Modem
/dev/gadget/Nlegacy g_*与 Function 版 ABI 类似

打印机类用户态 API 详见 gadget_printer.rst； 串口类见 gadget_serial.rst。

4. FunctionFS 用户态协议

function/f_fs.c 实现 usb_f_fs：内核只提供 EP 与 EP0 代理， 描述符与协议逻辑可在用户态完成（类似旧版 GadgetFS 的 Composite 版）。 挂载点通常为 /dev/usb-ffs/<name>/，配合 ConfigFS 中 functions/ffs.<name> 使用。

头文件 linux/usb/functionfs.h 定义 EP 写入/事件读取协议； 适合 ADB、MTP 等需频繁变更协议栈的场景。
