Composite 层（libcomposite）

composite.c · configfs.c · 多 Function 复合设备基础设施

1. 层职责

Composite 层把多个 usb_function「粘合」为一个 USB 设备：统一设备描述符、 管理 configuration、在 EP0 上响应标准 Chapter 9 请求，并在 SET_CONFIGURATION 时激活各 Function 的端点。 源码主体为 libcomposite.ko（见顶层 Makefile）。

2. 核心对象

结构体文件作用
usb_composite_drivercomposite.h驱动模板：dev 描述符、strings、bind/unbind
usb_composite_devcomposite.h / composite.c运行时设备：gadget*、configs 链表、ep0 req
usb_configurationcomposite.h单一配置：functions 链表、MaxPower、interface[]
gadget_infoconfigfs.cConfigFS 私有：嵌入 composite + cdev

usb_composite_probe() 将 composite_driver_template 填入 gadget_driver 并注册到 UDC Core。

3. libcomposite 模块组成

libcomposite-y := usbstring.o config.o epautoconf.o
libcomposite-y += composite.o functions.o configfs.o u_f.o
源文件职责
composite.cprobe/bind、EP0、set_config/reset_config、字符串 ID
configfs.c用户态 Gadget 配置、UDC 绑定
functions.cusb_function_register 全局表、modprobe 加载
config.cusb_gadget_config_buf 组装配置描述符
epautoconf.cusb_ep_autoconfig 端点匹配
usbstring.cUTF-16LE 字符串描述符

4. EP0 与 set_config

composite_setup() 处理标准请求；set_config()（composite.c）在 Host 选择配置后：

  1. 若已有配置则 reset_config() → 各 f->disable()
  2. 遍历 c->interface[]，记录 endpoint 位图，调用 f->set_alt(f, tmp, 0)
  3. 设置 usb_gadget_vbus_draw、USB_STATE_CONFIGURED

5. configfs.c 集成

gadget_info 内嵌 struct usb_composite_dev cdev 与 struct usb_composite_driver composite。 写 UDC 属性时（gadget_dev_desc_UDC_store）调用 usb_gadget_probe_driver(&gi->composite.gadget_driver)， 最终进入 composite_bind() → ConfigFS 预先挂好的 configuration/function 被 bind。

官方文档明确说明绑定逻辑在 configfs.c 中迭代 configurations 与 functions： gadget_configfs.html。 更完整的 Composite 架构、使用与调试说明见专题 Composite 框架。
