USB Gadget Function 驱动

基于 function/f_*.c · functions.c · include/linux/usb/composite.h · USB-IF 类规范

1. 概述与原理

在 USB Gadget 栈中，Function 对应 Host 看到的接口（Interface）或 一组相关接口（IAD）：例如 Printer Class、CDC ACM、Mass Storage、HID 等。 每个 Function 由独立内核模块 usb_f_*.ko 实现，通过 Composite 层 （usb_add_function）组合进复合设备。

官方文档将 Function 描述为 Composite 框架中的「协议单元」： 负责类/厂商 EP0 请求、端点 I/O，以及（可选）向用户态暴露字符设备、网络或 ALSA 接口。 详见 USB Gadget API — Composite Device Functions。

Function 层在整体栈中的位置：

  • 上：用户态 read/write、ioctl、FunctionFS、网络协议栈
  • 下：Composite（配置/EP0 路由）→ Gadget API → UDC 驱动
  • 旁：functions.c 全局注册表；ConfigFS 通过 usb_get_function_instance 按需加载模块

2. 架构与目录

drivers/usb/gadget/
├── functions.c              ← 全局 Function 注册表、modprobe 加载
├── u_f.c / u_f.h            ← alloc_ep_req 等共用工具
├── epautoconf.c             ← usb_ep_autoconfig（bind 阶段调用）
└── function/
    ├── Makefile             ← CONFIG_USB_F_* → usb_f_*.ko
    ├── f_printer.c          Printer 类
    ├── f_iap2.c             本树 iAP2 打印
    ├── f_acm.c + u_serial.c CDC ACM
    ├── f_ecm.c + u_ether.c    USB 以太网
    ├── f_fs.c               FunctionFS
    └── …                    HID / MSC / UVC / MIDI 等

各 Function 模块编译为独立 .ko，仅在 ConfigFS mkdir functions/xxx.N 或 Legacy composite 驱动引用时加载。共享辅助代码（如 u_serial.c、u_ether.c） 可单独成模块供多个 Function 链接。

3. 核心数据结构

结构体作用
struct usb_function运行时功能对象：描述符表、回调、所属 configuration
struct usb_function_instance可配置实例（ConfigFS 目录）；嵌入 config_group；含 set_inst_name
struct usb_function_driver模块级驱动：name、alloc_inst、alloc_func
 
/* composite.h — usb_function 关键字段 */
struct usb_function {
    const char *name;
    struct usb_descriptor_header **fs_descriptors;
    struct usb_descriptor_header **hs_descriptors;
    struct usb_descriptor_header **ss_descriptors;
    struct usb_configuration *config;
    int  (*bind)(struct usb_configuration *, struct usb_function *);
    void (*unbind)(struct usb_configuration *, struct usb_function *);
    int  (*set_alt)(struct usb_function *, unsigned intf, unsigned alt);
    void (*disable)(struct usb_function *);
    int  (*setup)(struct usb_function *, const struct usb_ctrlrequest *);
    int  (*req_match)(struct usb_function *, const struct usb_ctrlrequest *, bool);
    /* suspend/resume/get_status/func_suspend … */
    struct list_head list;
    const struct usb_function_instance *fi;
    DECLARE_BITMAP(endpoints, 32);  /* Composite set_config 用于 EP0 路由 */
};

驱动私有数据通常用 container_of 模式：私有结构体末尾嵌入 struct usb_function function;，例如 struct printer_dev（f_printer.c）。

4. 注册与实例化

标准注册宏（composite.h）：

DECLARE_USB_FUNCTION_INIT(printer, printer_alloc_instance, printer_alloc_func);
/* 展开为：
 *   usb_function_driver { .name = "printer", .alloc_inst, .alloc_func }
 *   MODULE_ALIAS("usbfunc:printer")
 *   module_init → usb_function_register()
 */

ConfigFS 创建实例时的调用链（configfs.c + functions.c）：

  1. mkdir functions/printer.0 → function_make()
  2. usb_get_function_instance("printer") — 查表或 request_module("usbfunc:printer")
  3. alloc_inst() — 创建 f_printer_opts 等，注册 ConfigFS 属性（pnp_string、q_len）
  4. symlink 到 configs/c.1/ → config_usb_cfg_link() → usb_get_function(fi) → alloc_func()
  5. 写 UDC 后 usb_add_function(c, f) → f->bind()

5. 生命周期回调

回调调用时机典型工作（f_printer 为例）
bindusb_add_function 期间usb_interface_id() · usb_ep_autoconfig() · usb_assign_descriptors() · 注册 /dev/g_printerN
set_altHost SET_CONFIGURATION 或 SET_INTERFACEconfig_ep_by_speed() · usb_ep_enable() · queue 初始 OUT request
setupEP0 类/厂商请求路由到本 interfacePrinter：GET_DEVICE_ID / GET_PORT_STATUS / SOFT_RESET
disable配置切换、disconnect、resetusb_ep_disable() · 唤醒阻塞 read/write · 回收 in-flight request
unbind从 configuration 移除或 gadget 卸载cdev_del · device_destroy · usb_free_all_descriptors
Composite 要求 必须实现 set_alt 与 disable，否则 usb_add_function() 直接返回 -EINVAL（见 composite.c）。

可选机制：

  • usb_function_deactivate/activate — 延迟 gadget 枚举（如 FunctionFS 等待用户态就绪）
  • USB_GADGET_DELAYED_STATUS — set_alt/setup 延迟 EP0 完成
  • req_match — 在未配置阶段匹配类请求（Host 在 SET_CONFIGURATION 前发送）

6. 描述符与端点

多速率描述符

每个 Function 维护 fs_descriptors[]、hs_descriptors[]、 ss_descriptors[]（及可选 ssp_descriptors[]）。 Composite 在 GET_DESCRIPTOR 与 set_config 时按当前 link speed 选择。 双速率设备 FS/HS 端点地址通常在 bind 里从 autoconfig 结果复制（见 f_printer.c）。

端点自动配置

in_ep = usb_ep_autoconfig(gadget, &fs_ep_in_desc);
/* epautoconf.c：按 descriptor 的 type/direction 在 gadget->ep_list 中找空闲 ep */

常用辅助 API

  • usb_assign_descriptors(f, fs, hs, ss, ssp) — 一次性挂载各速率描述符表
  • config_ep_by_speed(gadget, f, ep) — set_alt 前更新 ep->desc
  • usb_string_id(cdev) — bind 阶段分配字符串 ID
  • alloc_ep_req(ep, len)（u_f.c）— 分配对齐 buffer 的 request

7. 数据通路与用户态接口

模式代表 Function用户态接口
字符设备f_printer, f_acm (via u_serial)/dev/g_printer*、/dev/ttyGS*
网络f_ecm, f_ncm, f_rndisusb0 等（u_ether.c 注册 net_device）
FunctionFSf_fs用户态完全自定义描述符与 EP0/EP 协议
直连 VFSf_mass_storageLUN 映射到文件或 block 设备
ALSAf_uac1/uac2, f_midi音频子系统

典型 Bulk 传输模式（Printer）：

  1. set_alt 后 pre-queue 多个 OUT usb_request
  2. Host 发送数据 → complete 回调 → 放入 rx 队列 → 唤醒用户态 read
  3. 用户态 write → 从 tx 池取 request → usb_ep_queue(in_ep, req)

8. 使用方式

8.1 ConfigFS 加载 Function

modprobe libcomposite
modprobe usbfunc:printer    # 或首次 mkdir 时自动加载

G=/sys/kernel/config/usb_gadget/g1
mkdir -p $G/functions/printer.0
echo "MDL:Generic Printer;" > $G/functions/printer.0/pnp_string
echo 10 > $G/functions/printer.0/q_len
ln -sf $G/functions/printer.0 $G/configs/c.1/
echo $UDC > $G/UDC

# 用户态
cat /dev/g_printer0

各 Function 的 ConfigFS 属性见 Documentation/ABI/testing/configfs-usb-gadget-* 及 ConfigFS 专题。

8.2 编写新 Function 检查清单

  1. 定义 FS/HS/SS 描述符数组（interface + endpoint + 类特定描述符）
  2. 实现 alloc_inst（ConfigFS group/属性）与 alloc_func
  3. 填充 usb_function 回调：bind/set_alt/disable/setup
  4. DECLARE_USB_FUNCTION_INIT + Kconfig/Makefile 条目
  5. 在 bind 中 usb_ep_autoconfig，勿硬编码 endpoint 号
  6. disable 必须可安全重入；disconnect 时唤醒所有阻塞 IO

9. 调试

9.1 排查清单

现象检查点
mkdir functions/xxx 失败名称格式 func.inst；modprobe usbfunc:xxx；dmesg 注册冲突
usb_add_function / bind 失败端点 autoconfig 失败（UDC 资源不足）；缺少 set_alt/disable
Host 枚举 OK 但无 /devbind 中 cdev 创建失败；查看 function 驱动 ERROR 日志
open /dev 阻塞尚未 set_alt（未 CONFIGURED）；connect_status 类变量
读写零字节 / EIOdisable 后未清理 request；端点 stall；检查 complete 回调 status
类请求失败setup() 返回值；是否实现 req_match（未配置阶段）
修改 ConfigFS 属性 EBUSYinstance 已被 symlink（refcnt）；需先 unlink 或清空 UDC
HS 正常 FS 失败HS 描述符 endpoint 地址与 FS 不一致；maxpacket 不匹配
