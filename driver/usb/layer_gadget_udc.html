Gadget API 与 UDC Core

include/linux/usb/gadget.h · udc/core.c · 控制器与驱动的统一抽象

1. 层职责

本层是硬件无关的 Gadget 抽象：向上为 Composite / Legacy 驱动提供 usb_gadget_driver 注册接口；向下通过 usb_gadget_ops 调用具体 UDC 驱动。 udc/core.c 作为 UDC 类驱动，管理 /sys/class/udc/ 与驱动匹配。

2. usb_gadget 与端点 API

struct usb_gadget（gadget.h）只读字段供 Function 使用：

  • ep0 — 控制端点；Composite 在此处理 setup
  • ep_list — 非零端点链表，gadget_for_each_ep() 遍历
  • speed / max_speed — 当前与最大速率
  • state — USB_STATE_NOTATTACHED … CONFIGURED
  • ops — usb_gadget_ops：wakeup、set_selfpowered 等

端点操作（实现在 udc/core.c，最终调用 ep->ops）：

API说明
usb_ep_enable(ep)按 ep->desc 配置硬件端点
usb_ep_disable(ep)禁用并可能 flush 未完成请求
usb_ep_alloc_request / usb_ep_queue分配并提交 I/O
usb_ep_set_halt / usb_ep_clear_haltSTALL 处理
usb_gadget_ep_match_desc()判断硬件 ep 是否匹配描述符

3. usb_gadget_driver 绑定

struct usb_gadget_driver {
    char *function;
    enum usb_device_speed max_speed;
    int  (*bind)(struct usb_gadget *, struct usb_gadget_driver *);
    void (*unbind)(struct usb_gadget *);
    int  (*setup)(struct usb_gadget *, const struct usb_ctrlrequest *);
    void (*disconnect)(struct usb_gadget *);
    void (*suspend/resume/reset)(...);
    char *udc_name;   /* 可选：绑定指定 UDC */
};

usb_gadget_probe_driver()（udc/core.c）逻辑：

  1. 在 udc_list 中按 udc_name 或取第一个空闲 UDC
  2. udc_bind_to_driver() → driver->bind(gadget) → usb_gadget_udc_start()
  3. 若无 UDC，驱动进入 gadget_driver_pending_list 等待

4. usb_udc 与 udc/core.c

struct usb_udc {
    struct usb_gadget_driver *driver;
    struct usb_gadget        *gadget;
    struct device             dev;   /* /sys/class/udc/ */
    struct list_head          list;
    bool vbus, started;
};

UDC 驱动在 probe 中填充 usb_gadget 并调用 usb_add_gadget_udc()。 调试时可读 /sys/kernel/debug/usb//（需 CONFIG_USB_GADGET_DEBUG_FS） 或 /proc/driver/udc（CONFIG_USB_GADGET_DEBUG_FILES）。

5. 请求队列与 DMA

usb_gadget_map_request() 映射 DMA 缓冲区； 传输完成后 UDC 驱动必须调用 usb_gadget_giveback_request(ep, req)， 进而触发 Function 的 req->complete()。
