从第一性原理到 Linux 内核：UART (TTY) 架构全解

深度剖析 LDM 演进、核心对象与数据流转

一、 UART 的第一性原理：解决异步通信的矛盾

总线与点对点通信的本质区别在于“时钟与同步”。UART（通用异步收发传输器）没有时钟线，仅靠两根独立的数据线（TX 发送、RX 接收）进行点对点全双工通信。

  • 同步（无钟之钟）： 双方必须在软件层面约定相同的“波特率（Baud Rate）”。时间成为了隐形的同步介质。
  • 帧定界： 线路空闲时保持高电平。用从高到低的跳变作为起始位唤醒接收方；用固定宽度的拉高作为停止位结束一帧。
  • 硬件解耦： CPU 速度是纳秒级，UART 是微秒级。硬件引入 移位寄存器 和 FIFO 缓存，抹平极端的速率差。
第一性原理总结：时间的契约与空间换时间的缓冲哲学。CPU 绝不参与机械的移位等待，硬件控制器成为了极速 CPU 与慢速物理世界之间的巨型减震器。

二、 架构哲学分歧：为什么 UART 没有像 I2C 那样的总线？

在 Linux 架构史中，I2C 拥有标准的 LDM（Linux 设备模型）总线，而早期的 UART 却没有。这是由于两者诞生时的“第一性原理”不同：

属性 I2C 的灵魂 (集成电路总线) 传统 UART 的灵魂 (终端节点)
设计初衷 连接 PCB 板子上的多种芯片，需要拓扑管理 连接人机交互设备（键盘、显示器、TTY终端）
内核认知 挂载外设的物理总线 提供纯粹字符流的逻辑端点 (Endpoint)
暴露接口 LDM 自动加载驱动提供业务接口 (如 /dev/input) 直接暴露字符设备 /dev/ttyS0 给用户态

随着嵌入式发展，蓝牙、GPS 等智能芯片挂载到 UART 上，传统 TTY 模式暴露出严重的设备匹配和电源管理缺失问题。Linux 4.11 引入了 serdev (Serial Device Bus) 完美解决了这一痛点。

破局法则：当现实硬件已经变成了总线形态，软件就必须跟进。serdev 并没有重写 TTY，而是在 Serial Core 向上汇报时加了一个分流阀（MUX），让现代设备完美回归 LDM 的怀抱。

三、 核心抽象概念：6大对象与指针级联

如果走传统的 TTY 路线，内核不使用 LDM 总线，它是如何将千差万别的控制器与用户空间关联起来的？答案是面向对象的抽象与指针级联。系统启动并执行 open() 时，会建立以下关联：

  • uart_driver: 总厂。管理同类芯片（如 8250）的全部逻辑。
  • uart_port: 具象化物理抽象。每一个 uart_add_one_port() 代表系统中增加了一个可用的 TX/RX 通道。
  • uart_state: 黏合剂。上方挂接软状态（tty_port），下方挂接硬实体（uart_port），内部维护 TX 环形缓冲区。
  • tty_ldisc: 协议过滤器。决定收发的数据是纯透传，还是被当作回车换行指令解析。

四、 核心本质：数据的收发流转路由 (MSC)

串行通信的本质冲突是“突发的硬件中断”与“不确定的软件调度”之间的矛盾。内核连建三道“水库”（TX Circular Buffer，RX Flip Buffer，TTY Read Buffer），用空间的冗余换取了系统时序的优雅。

五、 终极总结

UART 在 Linux 中的形态是一部活生生的计算机架构编年史：

  1. 底层机制： 积极拥抱现代，依靠 uart_port 和 platform_bus 解决 M 种芯片的硬件差异。
  2. 上层策略： 坚守历史，用自己发明的 线路规程 (Line Discipline) 和 tty_struct 维护着 UNIX 古老的终端语义。
  3. 扩展破局： 当传统的“万物皆文件（/dev/ttyS*）+ 用户态 udev”无法满足现代复杂外设的电源与拓扑管理时，内核果断架起 serdev 总线，实现了通信接口与业务设备的优雅解耦。

“指针即路由，缓冲即解耦。” 这就是 UART 在 Linux 内核中生存的终极哲学。
